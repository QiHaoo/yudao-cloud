# 02-代码生成器实现

本章实现代码生成器的核心功能：从数据库读取表元数据、通过 Velocity 模板引擎生成代码、预览和同步。读者完成后将拥有一个可工作的代码生成器，能为电商系统的商品表、订单表自动生成 CRUD 代码。

> **前置阅读**：本章对应功能设计中的「03-代码生成器设计」。如果对代码生成器的设计目标和模板体系还不清楚，建议先阅读功能设计部分。

## 整体架构

## 表元数据读取

### 从数据库导入表结构

当用户在代码生成页面点击「导入」按钮时，首先需要从数据库读取表结构。这个过程通过 `DatabaseTableService` 完成。

源码位置：`yudao-module-infra-server/src/main/java/cn/iocoder/yudao/module/infra/service/codegen/CodegenServiceImpl.java`

```java
private Long createCodegen(String author, Long dataSourceConfigId, String tableName) {
    // 从数据库中，获得数据库表结构
    TableInfo tableInfo = databaseTableService.getTable(dataSourceConfigId, tableName);
    // 导入
    return createCodegen0(author, dataSourceConfigId, tableInfo);
}
```

`DatabaseTableService` 内部使用 MyBatis-Plus 的 `TableInfo` 和 `TableField` 来描述表结构。这是 MyBatis-Plus 代码生成器模块提供的标准模型，包含了表名、注释、字段列表等信息。

### 构建 CodegenTableDO 和 CodegenColumnDO

读取到 `TableInfo` 后，需要转换为框架自己的 `CodegenTableDO` 和 `CodegenColumnDO`，存储到数据库中供后续使用。这个转换工作由 `CodegenBuilder` 完成。

源码位置：`yudao-module-infra-server/src/main/java/cn/iocoder/yudao/module/infra/service/codegen/inner/CodegenBuilder.java`

```java
@Component
public class CodegenBuilder {

    public CodegenTableDO buildTable(TableInfo tableInfo) {
        CodegenTableDO table = CodegenConvert.INSTANCE.convert(tableInfo);
        initTableDefault(table);
        return table;
    }

    private void initTableDefault(CodegenTableDO table) {
        String tableName = table.getTableName().toLowerCase();
        // 第一步，_ 前缀的前面，作为 module 名字
        table.setModuleName(subBefore(tableName, '_', false).toLowerCase());
        // 第一步，第一个 _ 前缀的后面，作为 business 名字
        table.setBusinessName(toCamelCase(subAfter(tableName, '_', false)).toLowerCase());
        // 驼峰 + 首字母大写
        table.setClassName(upperFirst(toCamelCase(subAfter(tableName, '_', false))));
        // 去除结尾的表，作为类描述
        table.setClassComment(StrUtil.removeSuffixIgnoreCase(
            sanitizeComment(table.getTableComment()), "表"));
        table.setTemplateType(CodegenTemplateTypeEnum.ONE.getType());
    }
}
```

**设计要点**：

以电商系统的 `product_spu` 表为例，`initTableDefault` 方法会自动推导：

| 原始表名 | moduleName | businessName | className | classComment |
|----------|------------|--------------|-----------|--------------|
| product_spu | product | spu | Spu | 商品 SPU |

这种基于命名约定的自动推导，减少了用户手动配置的工作量。用户只需在导入后微调即可。

### 字段默认值处理

`CodegenBuilder` 还负责为每个字段设置 CRUD 操作的默认值：

```java
public List<CodegenColumnDO> buildColumns(Long tableId, List<TableField> tableFields) {
    List<CodegenColumnDO> columns = CodegenConvert.INSTANCE.convertList(tableFields);
    int index = 1;
    for (CodegenColumnDO column : columns) {
        column.setTableId(tableId);
        column.setOrdinalPosition(index++);
        // 特殊处理：Byte => Integer
        if (Byte.class.getSimpleName().equals(column.getJavaType())) {
            column.setJavaType(Integer.class.getSimpleName());
        }
        // 初始化 Column 列的默认字段
        processColumnOperation(column); // 处理 CRUD 相关的字段的默认值
        processColumnUI(column);        // 处理 UI 相关的字段的默认值
        processColumnExample(column);   // 处理字段的 swagger example 示例
    }
    return columns;
}
```

**智能推导逻辑**：

1. **操作类型推导**：根据字段名后缀自动设置查询条件
   - `*name` 后缀 → LIKE 查询
   - `*time`、`*date` 后缀 → BETWEEN 查询
   - 其他 → EQ 精确匹配

2. **UI 组件推导**：根据字段名后缀自动选择表单组件
   - `*status`、`*sex` → Radio 单选框
   - `*type` → Select 下拉框
   - `*image` → 图片上传组件
   - `*content`、`*description` → 富文本编辑器
   - `*time`、`*date` → 日期时间选择器

3. **示例值推导**：为 Swagger 文档生成示例值
   - `*id`、`*price` → 随机整数
   - `*name` → 随机中文姓名
   - `*url` → `https://www.iocoder.cn`

## CodegenServiceImpl 核心方法

### 代码预览（preview）

代码预览是用户最常用的功能，点击「预览」按钮即可看到生成的代码。核心逻辑在 `generationCodes` 方法中：

```java
@Override
public Map<String, String> generationCodes(Long tableId) {
    // 校验是否已经存在
    CodegenTableDO table = codegenTableMapper.selectById(tableId);
    if (table == null) {
        throw exception(CODEGEN_TABLE_NOT_EXISTS);
    }
    List<CodegenColumnDO> columns = codegenColumnMapper.selectListByTableId(tableId);
    if (CollUtil.isEmpty(columns)) {
        throw exception(CODEGEN_COLUMN_NOT_EXISTS);
    }

    // 如果是主子表，则加载对应的子表信息
    List<CodegenTableDO> subTables = null;
    List<List<CodegenColumnDO>> subColumnsList = null;
    if (CodegenTemplateTypeEnum.isMaster(table.getTemplateType())) {
        subTables = codegenTableMapper.selectListByTemplateTypeAndMasterTableId(
                CodegenTemplateTypeEnum.SUB.getType(), tableId);
        // ... 校验子表逻辑
    }

    // 获取数据源对应的数据库类型
    DataSourceConfigDO dataSourceConfig = dataSourceConfigService
        .getDataSourceConfig(table.getDataSourceConfigId());
    DbType dbType = JdbcUtils.getDbType(dataSourceConfig.getUrl());
    // 执行生成
    return codegenEngine.execute(dbType, table, columns, subTables, subColumnsList);
}
```

**返回值说明**：`Map<String, String>` 的 key 是生成文件的路径，value 是文件内容。例如：

```json
{
  "yudao-module-product/yudao-module-product-server/src/main/java/.../SpuController.java": "...",
  "yudao-module-product/yudao-module-product-server/src/main/java/.../SpuDO.java": "...",
  "yudao-module-product/yudao-module-product-server/src/main/java/.../SpuMapper.java": "..."
}
```

### 同步表结构（sync）

当数据库表结构发生变化（新增字段、修改字段类型）时，可以通过「同步」功能更新代码生成配置：

```java
@Override
@Transactional(rollbackFor = Exception.class)
public void syncCodegenFromDB(Long tableId) {
    // 校验是否已经存在
    CodegenTableDO table = codegenTableMapper.selectById(tableId);
    if (table == null) {
        throw exception(CODEGEN_TABLE_NOT_EXISTS);
    }
    // 从数据库中，获得数据库表结构
    TableInfo tableInfo = databaseTableService.getTable(
        table.getDataSourceConfigId(), table.getTableName());
    // 执行同步
    syncCodegen0(tableId, tableInfo);
}
```

同步的核心逻辑是**三向对比**：

```
┌─────────────────────────────────────────────────────────────────┐
│                    同步逻辑示意图                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  数据库表结构          CodegenColumnDO          操作              │
│  ─────────────         ──────────────          ────              │
│  name (存在)           name (存在)             保留              │
│  age (新增)            --                      插入新字段         │
│  --                   old_field (已删除)       删除旧字段         │
│  price (类型变化)      price (类型不同)         更新字段信息       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

源码中的实现：

```java
private void syncCodegen0(Long tableId, TableInfo tableInfo) {
    List<TableField> tableFields = tableInfo.getFields();
    List<CodegenColumnDO> codegenColumns = codegenColumnMapper.selectListByTableId(tableId);
    Set<String> codegenColumnNames = convertSet(codegenColumns, CodegenColumnDO::getColumnName);

    // 计算需要【修改】的字段
    BiPredicate<TableField, CodegenColumnDO> primaryKeyPredicate =
            (tableField, codegenColumn) -> 
                tableField.getMetaInfo().getJdbcType().name().equals(codegenColumn.getDataType())
                && tableField.getMetaInfo().isNullable() == codegenColumn.getNullable()
                && tableField.isKeyFlag() == codegenColumn.getPrimaryKey()
                && tableField.getComment().equals(codegenColumn.getColumnComment());
    
    // 计算需要【删除】的字段
    Set<String> tableFieldNames = convertSet(tableFields, TableField::getName);
    Set<Long> deleteColumnIds = codegenColumns.stream()
            .filter(column -> !tableFieldNames.contains(column.getColumnName()) 
                || modifyFieldNames.contains(column.getColumnName()))
            .map(CodegenColumnDO::getId).collect(Collectors.toSet());

    // 插入新增的字段 + 删除不存在的字段
    List<CodegenColumnDO> columns = codegenBuilder.buildColumns(tableId, tableFields);
    codegenColumnMapper.insertBatch(columns);
    if (CollUtil.isNotEmpty(deleteColumnIds)) {
        codegenColumnMapper.deleteByIds(deleteColumnIds);
    }
}
```

**设计要点**：
- 同步只处理字段的增删改，不会覆盖用户在 CodegenColumnDO 上的自定义配置（如 htmlType、listOperationCondition 等）
- 对于已存在的字段，只有当数据库结构发生变化时才会更新

## Velocity 模板引擎集成

### 引擎初始化

代码生成器使用 Hutool 封装的 Velocity 模板引擎，通过 `CodegenEngine` 类实现。

源码位置：`yudao-module-infra-server/src/main/java/cn/iocoder/yudao/module/infra/service/codegen/inner/CodegenEngine.java`

```java
@Component
public class CodegenEngine {

    private final TemplateEngine templateEngine;
    private final Map<String, Object> globalBindingMap = new HashMap<>();

    public CodegenEngine() {
        // 初始化 TemplateEngine 属性
        TemplateConfig config = new TemplateConfig();
        config.setResourceMode(TemplateConfig.ResourceMode.CLASSPATH);
        this.templateEngine = new VelocityEngine(config);
        // 设置 jakartaEnable，按照是否使用 JDK17 来判断
        this.jakartaEnable = SystemUtil.getJavaInfo().isJavaVersionAtLeast(1700)
                && ClassUtils.isPresent("jakarta.annotation.Resource", 
                    ClassUtils.getDefaultClassLoader());
        // 设置 cloudEnable，按照是否使用 Spring Cloud 来判断
        this.cloudEnable = ClassUtils.isPresent(
            "cn.iocoder.yudao.module.infra.framework.rpc.config.RpcConfiguration",
            ClassUtils.getDefaultClassLoader());
    }
}
```

**设计要点**：
- 使用 `TemplateConfig.ResourceMode.CLASSPATH` 表示模板文件从 classpath 加载
- 自动检测 JDK 版本和是否为 Cloud 项目，生成兼容的代码

### 执行代码生成

核心的 `execute` 方法负责遍历所有模板，逐个渲染生成代码：

```java
public Map<String, String> execute(DbType dbType, CodegenTableDO table, 
        List<CodegenColumnDO> columns,
        List<CodegenTableDO> subTables, List<List<CodegenColumnDO>> subColumnsList) {
    // 1.1 初始化 bindMap 上下文
    Map<String, Object> bindingMap = initBindingMap(dbType, table, columns, 
        subTables, subColumnsList);
    // 1.2 获得模版
    Map<String, String> templates = getTemplates(table.getFrontType());

    // 2. 执行生成
    Map<String, String> result = Maps.newLinkedHashMapWithExpectedSize(templates.size());
    templates.forEach((vmPath, filePath) -> {
        // 特殊：主子表专属逻辑
        if (isSubTemplate(vmPath)) {
            generateSubCode(table, subTables, result, vmPath, filePath, bindingMap);
            return;
        }
        // 默认生成
        generateCode(result, vmPath, filePath, bindingMap);
    });
    return result;
}

private void generateCode(Map<String, String> result, String vmPath,
                          String filePath, Map<String, Object> bindingMap) {
    filePath = formatFilePath(filePath, bindingMap);
    String content = templateEngine.getTemplate(vmPath).render(bindingMap);
    // 格式化代码
    content = prettyCode(content, vmPath);
    result.put(filePath, content);
}
```

### 代码格式化

生成的代码需要经过格式化处理，避免前端 Pretty 检查报错：

```java
private String prettyCode(String content, String vmPath) {
    // Vue 界面：去除字段后面多余的 , 逗号
    if (!StrUtil.containsAny(vmPath, "vben5", "vue3_admin_uniapp")) {
        content = content.replaceAll(",\n}", "\n}").replaceAll(",\n  }", "\n  }");
    }
    // Vue 界面：去除多的 dateFormatter
    if (StrUtil.count(content, "dateFormatter") == 1) {
        content = StrUtils.removeLineContains(content, "dateFormatter");
    }
    // Vue2 界面：修正 $refs
    if (StrUtil.count(content, "this.refs") >= 1) {
        content = content.replace("this.refs", "this.$refs");
    }
    return content;
}
```

## 上下文变量构建

### initBindingMap 方法

这个方法是代码生成的核心，它将表和列的信息转换为模板可用的变量：

```java
private Map<String, Object> initBindingMap(DbType dbType, CodegenTableDO table, 
        List<CodegenColumnDO> columns,
        List<CodegenTableDO> subTables, List<List<CodegenColumnDO>> subColumnsList) {
    // 创建 bindingMap
    Map<String, Object> bindingMap = new HashMap<>(globalBindingMap);
    bindingMap.put("dbType", dbType);
    bindingMap.put("table", table);
    bindingMap.put("columns", columns);
    bindingMap.put("primaryColumn", CollectionUtils.findFirst(columns, 
        CodegenColumnDO::getPrimaryKey));
    bindingMap.put("sceneEnum", CodegenSceneEnum.valueOf(table.getScene()));
    
    // className 相关
    String className = table.getClassName();
    String simpleClassName = equalsAnyIgnoreCase(table.getClassName(), table.getModuleName()) 
        ? table.getClassName()
        : removePrefix(table.getClassName(), upperFirst(table.getModuleName()));
    String classNameVar = lowerFirst(simpleClassName);
    bindingMap.put("simpleClassName", simpleClassName);
    bindingMap.put("classNameVar", classNameVar);
    
    // permission 前缀
    bindingMap.put("permissionPrefix", table.getModuleName() + ":" 
        + toSymbolCase(simpleClassName, '-'));

    // 特殊：主子表专属逻辑
    if (CollUtil.isNotEmpty(subTables)) {
        bindingMap.put("subTables", subTables);
        bindingMap.put("subColumnsList", subColumnsList);
        // ... 子表相关变量
    }

    // 多个 vm 公用的 VO 变量
    if (ObjectUtil.equal(codegenProperties.getVoType(), CodegenVOTypeEnum.VO.getType())) {
        String prefixClass = CodegenSceneEnum.valueOf(table.getScene()).getPrefixClass();
        bindingMap.put("saveReqVOClass", prefixClass + className + "SaveReqVO");
        bindingMap.put("respVOClass", prefixClass + className + "RespVO");
    }
    return bindingMap;
}
```

**变量说明**（以 `product_spu` 表为例）：

| 变量名 | 值 | 用途 |
|--------|-----|------|
| table | CodegenTableDO 对象 | 表的完整信息 |
| columns | List<CodegenColumnDO> | 字段列表 |
| primaryColumn | CodegenColumnDO | 主键字段 |
| simpleClassName | Spu | 简单类名 |
| classNameVar | spu | 变量名（首字母小写） |
| permissionPrefix | product:spu | 权限前缀 |
| saveReqVOClass | AdminSpuSaveReqVO | 保存请求 VO 类名 |
| respVOClass | AdminSpuRespVO | 响应 VO 类名 |

## Controller 层实现

### CodegenController

源码位置：`yudao-module-infra-server/src/main/java/cn/iocoder/yudao/module/infra/controller/admin/codegen/CodegenController.java`

```java
@Tag(name = "管理后台 - 代码生成")
@RestController
@RequestMapping("/infra/codegen")
public class CodegenController {

    @Resource
    private CodegenService codegenService;

    @PostMapping("/create")
    @Operation(summary = "创建代码生成")
    public CommonResult<List<Long>> createCodegen(@Valid @RequestBody CodegenCreateListReqVO reqVO) {
        return success(codegenService.createCodegenList(
            SecurityFrameworkUtils.getLoginUsername(), reqVO));
    }

    @GetMapping("/preview")
    @Operation(summary = "预览代码")
    public CommonResult<Map<String, String>> previewCodegen(@RequestParam("tableId") Long tableId) {
        return success(codegenService.generationCodes(tableId));
    }

    @PutMapping("/sync")
    @Operation(summary = "同步数据库表结构")
    public CommonResult<Boolean> syncCodegen(@RequestParam("tableId") Long tableId) {
        codegenService.syncCodegenFromDB(tableId);
        return success(true);
    }

    @DeleteMapping("/delete")
    @Operation(summary = "删除代码生成")
    public CommonResult<Boolean> deleteCodegen(@RequestParam("tableId") Long tableId) {
        codegenService.deleteCodegen(tableId);
        return success(true);
    }
}
```

本章实现了代码生成器的核心功能。关键要点：

1. **元数据读取**：通过 MyBatis-Plus 的 `TableInfo` 读取数据库表结构
2. **智能推导**：`CodegenBuilder` 根据命名约定自动推导模块名、类名、UI 组件类型
3. **模板引擎**：使用 Hutool 封装的 Velocity 引擎，通过变量绑定渲染模板
4. **同步机制**：支持增量同步数据库表结构变化，不丢失用户自定义配置
