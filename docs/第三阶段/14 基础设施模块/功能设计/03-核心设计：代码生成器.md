# 03-核心设计：代码生成器

## 场景引入

假设你正在开发一个电商系统，需要为以下数据库表生成 CRUD 代码：

- `product` - 商品表
- `product_sku` - 商品 SKU 表（与商品表是主子关系）
- `order` - 订单表
- `order_item` - 订单项表（与订单表是主子关系）

手动编写这些表的 Controller、Service、Mapper、VO、DO 代码，至少需要 2-3 天。而使用代码生成器，只需要几分钟。

## 核心实体设计

### CodegenTableDO：表定义

表定义实体存储了数据库表的元数据和代码生成的配置信息。

```java
// 源码位置：yudao-module-infra-server/.../dal/dataobject/codegen/CodegenTableDO.java
@TableName(value = "infra_codegen_table", autoResultMap = true)
@Data
@TenantIgnore  // 代码生成配置全局共享，不按租户隔离
public class CodegenTableDO extends BaseDO {

    @TableId
    private Long id;

    // ========== 数据源相关 ==========
    /**
     * 数据源编号
     * 关联 DataSourceConfigDO#getId()
     */
    private Long dataSourceConfigId;

    /**
     * 生成场景
     * 枚举 CodegenSceneEnum
     */
    private Integer scene;

    // ========== 表相关字段 ==========
    /**
     * 表名称
     * 关联 TableInfo#getName()
     */
    private String tableName;

    /**
     * 表描述
     * 关联 TableInfo#getComment()
     */
    private String tableComment;

    // ========== 类相关字段 ==========
    /**
     * 模块名，即一级目录
     * 例如：system、infra、product
     */
    private String moduleName;

    /**
     * 业务名，即二级目录
     * 例如：user、permission、dict
     */
    private String businessName;

    /**
     * 类名称（首字母大写）
     * 例如：SysUser、Product、Order
     */
    private String className;

    /**
     * 类描述
     */
    private String classComment;

    /**
     * 作者
     */
    private String author;

    // ========== 生成相关字段 ==========
    /**
     * 模板类型
     * 枚举 CodegenTemplateTypeEnum
     */
    private Integer templateType;

    /**
     * 代码生成的前端类型
     * 枚举 CodegenFrontTypeEnum
     */
    private Integer frontType;

    // ========== 菜单相关字段 ==========
    /**
     * 父菜单编号
     * 关联 MenuDO 的 id 属性
     */
    private Long parentMenuId;

    // ========== 主子表相关字段 ==========
    /**
     * 主表的编号
     * 关联 CodegenTableDO#getId()
     */
    private Long masterTableId;

    /**
     * 子表关联主表的字段编号
     * 关联 CodegenColumnDO#getId()
     */
    private Long subJoinColumnId;

    /**
     * 主表与子表是否一对多
     * true：一对多（如：订单 -> 订单项）
     * false：一对一
     */
    private Boolean subJoinMany;

    // ========== 树表相关字段 ==========
    /**
     * 树表的父字段编号
     */
    private Long treeParentColumnId;

    /**
     * 树表的名字字段编号
     */
    private Long treeNameColumnId;
}
```

**设计要点**：

1. **`@TenantIgnore`**：代码生成配置是全局共享的，不按租户隔离。因为代码生成是开发阶段的工作，与运行时的多租户无关。

2. **主子表支持**：通过 `masterTableId`、`subJoinColumnId`、`subJoinMany` 三个字段，支持主子表的代码生成。例如：
   - 订单表（主表） -> 订单项表（子表），一对多关系
   - 商品表（主表） -> 商品详情表（子表），一对一关系

3. **树表支持**：通过 `treeParentColumnId`、`treeNameColumnId` 两个字段，支持树形结构表的代码生成。例如：
   - 商品分类表：parent_id 指向父分类
   - 地区表：parent_id 指向父地区

### CodegenColumnDO：列定义

列定义实体存储了数据库列的元数据和代码生成的配置信息。

```java
// 源码位置：yudao-module-infra-server/.../dal/dataobject/codegen/CodegenColumnDO.java
@TableName(value = "infra_codegen_column", autoResultMap = true)
@Data
@TenantIgnore
public class CodegenColumnDO extends BaseDO {

    @TableId
    private Long id;

    /**
     * 表编号
     * 关联 CodegenTableDO#getId()
     */
    private Long tableId;

    // ========== 表相关字段 ==========
    /**
     * 字段名
     * 例如：user_name, create_time
     */
    private String columnName;

    /**
     * 数据库字段类型
     * 例如：varchar, bigint, datetime
     */
    private String dataType;

    /**
     * 字段描述
     */
    private String columnComment;

    /**
     * 是否允许为空
     */
    private Boolean nullable;

    /**
     * 是否主键
     */
    private Boolean primaryKey;

    /**
     * 排序
     */
    private Integer ordinalPosition;

    // ========== Java 相关字段 ==========
    /**
     * Java 属性类型
     * 例如：String、Long、LocalDateTime
     */
    private String javaType;

    /**
     * Java 属性名
     * 例如：userName, createTime
     */
    private String javaField;

    /**
     * 字典类型
     * 关联 DictTypeDO 的 type 属性
     */
    private String dictType;

    /**
     * 数据示例
     * 主要用于生成 Swagger 注解的 example 字段
     */
    private String example;

    // ========== CRUD 相关字段 ==========
    /**
     * 是否为 Create 创建操作的字段
     */
    private Boolean createOperation;

    /**
     * 是否为 Update 更新操作的字段
     */
    private Boolean updateOperation;

    /**
     * 是否为 List 查询操作的字段
     */
    private Boolean listOperation;

    /**
     * List 查询操作的条件类型
     * 枚举 CodegenColumnListConditionEnum
     */
    private String listOperationCondition;

    /**
     * 是否为 List 查询操作的返回字段
     */
    private Boolean listOperationResult;

    // ========== UI 相关字段 ==========
    /**
     * 显示类型
     * 枚举 CodegenColumnHtmlTypeEnum
     */
    private String htmlType;
}
```

**设计要点**：

1. **CRUD 操作控制**：通过 `createOperation`、`updateOperation`、`listOperation` 等字段，可以精细控制每个字段参与哪些 CRUD 操作。例如：
   - `create_time` 字段：只在查询时返回，创建和更新时不传入
   - `status` 字段：创建时传入，更新时传入，查询时作为条件

2. **查询条件控制**：`listOperationCondition` 字段定义了查询条件的类型，支持：
   - `=`：等于
   - `!=`：不等于
   - `>`：大于
   - `<`：小于
   - `LIKE`：模糊匹配
   - `BETWEEN`：范围查询

3. **UI 组件控制**：`htmlType` 字段定义了前端表单使用的组件类型，支持：
   - `input`：输入框
   - `select`：下拉选择
   - `datetime`：日期时间选择器
   - `textarea`：文本域
   - `radio`：单选框
   - `checkbox`：多选框

## 表元数据获取

代码生成器通过 JDBC 的 `DatabaseMetaData` 接口读取数据库表结构。

```java
// 核心逻辑伪代码
public List<DatabaseTableRespVO> getDatabaseTableList(Long dataSourceConfigId, 
                                                       String name, String comment) {
    // 1. 获取数据源连接
    DataSource dataSource = getDataSource(dataSourceConfigId);
    
    // 2. 通过 JDBC DatabaseMetaData 读取表信息
    DatabaseMetaData metaData = dataSource.getConnection().getMetaData();
    ResultSet tables = metaData.getTables(null, null, "%", new String[]{"TABLE"});
    
    // 3. 遍历表信息，过滤已导入的表
    List<DatabaseTableRespVO> result = new ArrayList<>();
    while (tables.next()) {
        String tableName = tables.getString("TABLE_NAME");
        String tableComment = tables.getString("REMARKS");
        
        // 过滤条件
        if (name != null && !tableName.contains(name)) continue;
        if (comment != null && !tableComment.contains(comment)) continue;
        
        // 检查是否已导入
        if (codegenTableMapper.selectByTableName(tableName) != null) continue;
        
        result.add(new DatabaseTableRespVO(tableName, tableComment));
    }
    
    return result;
}
```

**获取列信息**：

```java
// 核心逻辑伪代码
public List<CodegenColumnDO> getColumnList(String tableName) {
    DatabaseMetaData metaData = connection.getMetaData();
    ResultSet columns = metaData.getColumns(null, null, tableName, "%");
    
    List<CodegenColumnDO> result = new ArrayList<>();
    while (columns.next()) {
        CodegenColumnDO column = new CodegenColumnDO();
        column.setColumnName(columns.getString("COLUMN_NAME"));
        column.setDataType(columns.getString("TYPE_NAME"));
        column.setColumnComment(columns.getString("REMARKS"));
        column.setNullable("YES".equals(columns.getString("IS_NULLABLE")));
        
        // 数据库类型 -> Java 类型映射
        column.setJavaType(convertJavaType(column.getDataType()));
        column.setJavaField(convertToCamelCase(column.getColumnName()));
        
        result.add(column);
    }
    
    return result;
}
```

**类型映射规则**：

| 数据库类型 | Java 类型 |
|-----------|----------|
| varchar, char, text | String |
| bigint | Long |
| int, integer | Integer |
| tinyint | Boolean / Integer |
| decimal, numeric | BigDecimal |
| datetime, timestamp | LocalDateTime |
| date | LocalDate |
| time | LocalTime |
| blob, binary | byte[] |

## 模板引擎：Velocity

代码生成器使用 Apache Velocity 作为模板引擎，将表元数据和列元数据填充到模板中，生成最终的代码。

### Velocity 模板示例

**Controller 模板**：

```velocity
## 源码位置：resources/codegen/java/controller.java.vm
package ${basePackage}.controller.admin.${moduleName};

import cn.iocoder.yudao.framework.common.pojo.CommonResult;
import cn.iocoder.yudao.framework.common.pojo.PageResult;
import org.springframework.security.access.prepost.PreAuthorize;
import org.springframework.validation.annotation.Validated;
import org.springframework.web.bind.annotation.*;
import javax.annotation.Resource;
import javax.validation.Valid;

@Tag(name = "管理后台 - ${tableComment}")
@RestController
@RequestMapping("/${moduleName}/${businessName}")
@Validated
public class ${className}Controller {

    @Resource
    private ${className}Service ${classNameLower}Service;

    @PostMapping("/create")
    @Operation(summary = "创建${tableComment}")
    @PreAuthorize("@ss.hasPermission('${moduleName}:${businessName}:create')")
    public CommonResult<Long> create${className}(@Valid @RequestBody ${className}SaveReqVO createReqVO) {
        return success(${classNameLower}Service.create${className}(createReqVO));
    }

    @PutMapping("/update")
    @Operation(summary = "更新${tableComment}")
    @PreAuthorize("@ss.hasPermission('${moduleName}:${businessName}:update')")
    public CommonResult<Boolean> update${className}(@Valid @RequestBody ${className}SaveReqVO updateReqVO) {
        ${classNameLower}Service.update${className}(updateReqVO);
        return success(true);
    }

    @DeleteMapping("/delete")
    @Operation(summary = "删除${tableComment}")
    @Parameter(name = "id", description = "编号", required = true)
    @PreAuthorize("@ss.hasPermission('${moduleName}:${businessName}:delete')")
    public CommonResult<Boolean> delete${className}(@RequestParam("id") Long id) {
        ${classNameLower}Service.delete${className}(id);
        return success(true);
    }

    @GetMapping("/get")
    @Operation(summary = "获得${tableComment}")
    @Parameter(name = "id", description = "编号", required = true, example = "1024")
    @PreAuthorize("@ss.hasPermission('${moduleName}:${businessName}:query')")
    public CommonResult<${className}RespVO> get${className}(@RequestParam("id") Long id) {
        return success(BeanUtils.toBean(${classNameLower}Service.get${className}(id), ${className}RespVO.class));
    }

    @GetMapping("/page")
    @Operation(summary = "获得${tableComment}分页")
    @PreAuthorize("@ss.hasPermission('${moduleName}:${businessName}:query')")
    public CommonResult<PageResult<${className}RespVO>> get${className}Page(@Valid ${className}PageReqVO pageReqVO) {
        return success(${classNameLower}Service.get${className}Page(pageReqVO));
    }
}
```

## 单表 vs 主子表

### 单表生成

单表是最简单的情况，只需要一个表定义即可生成完整的 CRUD 代码。

```
┌─────────────────┐
│    product      │
├─────────────────┤
│ id              │ ← 主键
│ name            │
│ price           │
│ status          │
│ create_time     │
└─────────────────┘
        │
        ▼ 代码生成
┌─────────────────────────────────────────┐
│  Controller + Service + Mapper + VO/DO  │
└─────────────────────────────────────────┘
```

### 主子表生成

主子表需要定义两个表，并建立关联关系。

```
┌─────────────────┐         ┌─────────────────────┐
│    order        │         │    order_item        │
│   (主表)        │  1 ──► N│   (子表)             │
├─────────────────┤         ├─────────────────────┤
│ id              │         │ id                   │
│ order_no        │         │ order_id  ◄── 关联字段│
│ user_id         │         │ product_id           │
│ total_amount    │         │ price                │
│ status          │         │ quantity             │
│ create_time     │         │ create_time          │
└─────────────────┘         └─────────────────────┘
```

**配置方式**：
1. 导入主表（order）和子表（order_item）
2. 在子表配置中设置：
   - `masterTableId` = 主表 ID
   - `subJoinColumnId` = 关联字段 ID（order_id）
   - `subJoinMany` = true（一对多）

**生成的代码特点**：
- 主表 Controller 包含子表的 CRUD 操作
- Service 层使用事务保证主子表操作的原子性
- VO 中包含子表的列表字段

## 代码生成 API

代码生成器提供了完整的 REST API：

```java
// 源码位置：yudao-module-infra-server/.../controller/admin/codegen/CodegenController.java
@Tag(name = "管理后台 - 代码生成器")
@RestController
@RequestMapping("/infra/codegen")
public class CodegenController {

    @GetMapping("/db/table/list")
    @Operation(summary = "获得数据库自带的表定义列表")
    public CommonResult<List<DatabaseTableRespVO>> getDatabaseTableList(
            @RequestParam("dataSourceConfigId") Long dataSourceConfigId,
            @RequestParam(value = "name", required = false) String name,
            @RequestParam(value = "comment", required = false) String comment) {
        return success(codegenService.getDatabaseTableList(dataSourceConfigId, name, comment));
    }

    @PostMapping("/create-list")
    @Operation(summary = "基于数据库的表结构，创建代码生成器的表和字段定义")
    public CommonResult<List<Long>> createCodegenList(@Valid @RequestBody CodegenCreateListReqVO reqVO) {
        return success(codegenService.createCodegenList(getLoginUserNickname(), reqVO));
    }

    @GetMapping("/preview")
    @Operation(summary = "预览生成代码")
    public CommonResult<List<CodegenPreviewRespVO>> previewCodegen(@RequestParam("tableId") Long tableId) {
        Map<String, String> codes = codegenService.generationCodes(tableId);
        return success(CodegenConvert.INSTANCE.convert(codes));
    }

    @GetMapping("/download")
    @Operation(summary = "下载生成代码")
    public void downloadCodegen(@RequestParam("tableId") Long tableId,
                                HttpServletResponse response) throws IOException {
        Map<String, String> codes = codegenService.generationCodes(tableId);
        // 构建 zip 包
        String[] paths = codes.keySet().toArray(new String[0]);
        ByteArrayInputStream[] ins = codes.values().stream()
                .map(IoUtil::toUtf8Stream).toArray(ByteArrayInputStream[]::new);
        ByteArrayOutputStream outputStream = new ByteArrayOutputStream();
        ZipUtil.zip(outputStream, paths, ins);
        // 输出
        writeAttachment(response, "codegen.zip", outputStream.toByteArray());
    }
}
```

## 本章小结

代码生成器是 yudao-module-infra 的核心功能之一，它通过以下设计实现了代码生成的自动化：

1. **元数据驱动**：通过 JDBC DatabaseMetaData 读取表结构，确保障码与数据库一致
2. **模板引擎**：使用 Velocity 模板引擎，支持自定义模板，生成符合项目规范的代码
3. **精细控制**：支持配置每个字段参与哪些 CRUD 操作，满足复杂的业务需求
4. **主子表支持**：支持一对多、一对一的主子表关系，生成完整的关联查询代码
5. **多端支持**：支持生成 Vue2/Vue3 前端代码，以及多种后端框架代码
