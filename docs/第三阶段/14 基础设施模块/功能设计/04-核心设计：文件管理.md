# 04-核心设计：文件管理

## 场景引入

假设你正在开发一个电商系统，需要处理以下文件上传场景：

- **商品图片**：用户上传商品主图、详情图，需要支持 CDN 加速访问
- **用户头像**：用户上传个人头像，需要支持裁剪和压缩
- **发票 PDF**：系统生成电子发票，需要长期存储
- **导入导出**：Excel 导入导出，需要临时存储

不同的场景对存储的要求不同：

| 场景 | 存储要求 | 推荐方案 |
|------|---------|---------|
| 商品图片 | 高并发读取、CDN 加速 | 阿里云 OSS / 腾讯云 COS |
| 用户头像 | 中等并发、需要裁剪 | 阿里云 OSS + 图片处理 |
| 发票 PDF | 低并发、长期存储 | 本地存储 / 数据库存储 |
| 导入导出 | 临时存储、自动清理 | 本地存储 |

如何设计一个统一的文件管理方案，既能满足不同场景的需求，又能让业务代码简单易用？

## FileDO：文件元数据

文件元数据实体记录了每个文件的基本信息：

```java
// 源码位置：yudao-module-infra-server/.../dal/dataobject/file/FileDO.java
@TableName("infra_file")
@Data
@Builder
@NoArgsConstructor
@AllArgsConstructor
@TenantIgnore  // 文件数据全局共享，不按租户隔离
public class FileDO extends BaseDO {

    /**
     * 编号，数据库自增
     */
    private Long id;

    /**
     * 配置编号
     * 关联 FileConfigDO#getId()
     */
    private Long configId;

    /**
     * 原文件名
     * 例如：商品图片.jpg
     */
    private String name;

    /**
     * 路径，即文件名
     * 例如：2024/01/01/abc123.jpg
     */
    private String path;

    /**
     * 访问地址
     * 例如：https://oss.example.com/2024/01/01/abc123.jpg
     */
    private String url;

    /**
     * 文件的 MIME 类型
     * 例如：image/jpeg, application/pdf
     */
    private String type;

    /**
     * 文件大小（字节）
     */
    private Long size;
}
```

**设计要点**：

1. **`configId` 关联存储配置**：每个文件关联一个存储配置，支持多存储后端并存。例如：
   - 商品图片使用 OSS 配置（configId = 1）
   - 发票 PDF 使用本地配置（configId = 2）

2. **`path` 与 `url` 分离**：
   - `path` 是文件在存储系统中的相对路径，用于文件操作（删除、读取）
   - `url` 是文件的完整访问地址，用于前端展示

3. **`@TenantIgnore`**：文件数据全局共享，因为文件存储通常是跨租户的公共资源。

## FileService 接口：统一的文件 API

文件服务接口提供了统一的文件操作 API，业务代码只依赖接口，不依赖具体实现：

```java
// 源码位置：yudao-module-infra-server/.../service/file/FileService.java
public interface FileService {

    /**
     * 保存文件，并返回文件的访问路径
     *
     * @param content   文件内容
     * @param name      文件名称，允许空
     * @param directory 目录，允许空
     * @param type      文件的 MIME 类型，允许空
     * @return 文件路径
     */
    String createFile(@NotEmpty(message = "文件内容不能为空") byte[] content,
                      String name, String directory, String type);

    /**
     * 生成文件预签名地址信息，用于上传
     *
     * @param name      文件名
     * @param directory 目录
     * @return 预签名地址信息
     */
    FilePresignedUrlRespVO presignPutUrl(@NotEmpty(message = "文件名不能为空") String name,
                                         String directory);

    /**
     * 生成文件预签名地址信息，用于读取
     *
     * @param url 完整的文件访问地址
     * @param expirationSeconds 访问有效期，单位秒
     * @return 文件预签名地址
     */
    String presignGetUrl(String url, Integer expirationSeconds);

    /**
     * 获得文件分页
     */
    PageResult<FileDO> getFilePage(FilePageReqVO pageReqVO);

    /**
     * 创建文件
     */
    Long createFile(FileCreateReqVO createReqVO);

    /**
     * 获得文件
     */
    FileDO getFile(Long id);

    /**
     * 删除文件
     */
    void deleteFile(Long id) throws Exception;

    /**
     * 批量删除文件
     */
    void deleteFileList(List<Long> ids) throws Exception;

    /**
     * 获得文件内容
     */
    byte[] getFileContent(Long configId, String path) throws Exception;
}
```

**使用示例**：

```java
// 业务代码中使用文件服务
@Service
public class ProductService {

    @Resource
    private FileService fileService;

    public Long uploadProductImage(MultipartFile file) throws Exception {
        // 1. 读取文件内容
        byte[] content = file.getBytes();

        // 2. 上传文件，返回访问 URL
        String url = fileService.createFile(
            content,
            file.getOriginalFilename(),
            "product/images",  // 目录
            file.getContentType()
        );

        // 3. 保存图片 URL 到商品表
        // ...
        return productId;
    }
}
```

## 存储抽象：FileClient 接口

文件客户端接口是存储层的核心抽象，所有存储后端都实现这个接口：

```java
// 源码位置：yudao-module-infra-server/.../framework/file/core/client/FileClient.java
public interface FileClient {

    /**
     * 获得客户端编号
     */
    Long getId();

    /**
     * 上传文件
     *
     * @param content 文件流
     * @param path    相对路径
     * @param type    文件类型
     * @return 完整路径，即 HTTP 访问地址
     */
    String upload(byte[] content, String path, String type) throws Exception;

    /**
     * 删除文件
     *
     * @param path 相对路径
     */
    void delete(String path) throws Exception;

    /**
     * 获得文件的内容
     *
     * @param path 相对路径
     * @return 文件的内容
     */
    byte[] getContent(String path) throws Exception;

    /**
     * 获得文件预签名地址，用于上传
     */
    default String presignPutUrl(String path) {
        throw new UnsupportedOperationException("不支持的操作");
    }

    /**
     * 生成文件预签名地址，用于读取
     */
    default String presignGetUrl(String url, Integer expirationSeconds) {
        throw new UnsupportedOperationException("不支持的操作");
    }
}
```

## 存储实现

### 本地存储（LocalFileClient）

本地存储将文件保存在服务器的本地磁盘上，适用于开发环境和小型项目。

```java
// 源码位置：yudao-module-infra-server/.../framework/file/core/client/local/LocalFileClient.java
public class LocalFileClient extends AbstractFileClient<LocalFileClientConfig> {

    public LocalFileClient(Long id, LocalFileClientConfig config) {
        super(id, config);
    }

    @Override
    protected void doInit() {
        // 本地存储无需初始化
    }

    @Override
    public String upload(byte[] content, String path, String type) {
        // 执行写入
        String filePath = getFilePath(path);
        FileUtil.writeBytes(content, filePath);
        // 拼接返回路径
        return super.formatFileUrl(config.getDomain(), path);
    }

    @Override
    public void delete(String path) {
        String filePath = getFilePath(path);
        FileUtil.del(filePath);
    }

    @Override
    public byte[] getContent(String path) {
        String filePath = getFilePath(path);
        try {
            return FileUtil.readBytes(filePath);
        } catch (IORuntimeException ex) {
            if (ex.getMessage().startsWith("File not exist:")) {
                return null;
            }
            throw ex;
        }
    }

    private String getFilePath(String path) {
        return config.getBasePath() + File.separator + path;
    }
}
```

**配置示例**：

```json
{
    "domain": "http://localhost:48080",
    "basePath": "/data/upload"
}
```

**访问方式**：通过 `FileController` 的 `/get/{configId}/{path}` 接口访问文件。

### S3 兼容存储（S3FileClient）

S3 协议是云存储的事实标准，支持阿里云 OSS、腾讯云 COS、MinIO、AWS S3 等多种云服务。

```java
// 源码位置：yudao-module-infra-server/.../framework/file/core/client/s3/S3FileClient.java
public class S3FileClient extends AbstractFileClient<S3FileClientConfig> {

    private static final Duration EXPIRATION_DEFAULT = Duration.ofHours(24);

    private S3Client client;
    private S3Presigner presigner;

    public S3FileClient(Long id, S3FileClientConfig config) {
        super(id, config);
    }

    @Override
    protected void doInit() {
        // 补全 domain
        if (StrUtil.isEmpty(config.getDomain())) {
            config.setDomain(buildDomain());
        }

        // 初始化 S3 客户端
        String regionStr = resolveRegion();
        Region region = Region.of(regionStr);
        AwsCredentialsProvider credentialsProvider = StaticCredentialsProvider.create(
                AwsBasicCredentials.create(config.getAccessKey(), config.getAccessSecret()));

        URI endpoint = URI.create(buildEndpoint());
        S3Configuration serviceConfiguration = S3Configuration.builder()
                .pathStyleAccessEnabled(Boolean.TRUE.equals(config.getEnablePathStyleAccess()))
                .chunkedEncodingEnabled(false)
                .build();

        client = S3Client.builder()
                .credentialsProvider(credentialsProvider)
                .region(region)
                .endpointOverride(endpoint)
                .serviceConfiguration(serviceConfiguration)
                .build();

        presigner = S3Presigner.builder()
                .credentialsProvider(credentialsProvider)
                .region(region)
                .endpointOverride(URI.create(buildPresignerEndpoint()))
                .serviceConfiguration(serviceConfiguration)
                .build();
    }

    @Override
    public String upload(byte[] content, String path, String type) {
        PutObjectRequest putRequest = PutObjectRequest.builder()
                .bucket(config.getBucket())
                .key(path)
                .contentType(type)
                .contentLength((long) content.length)
                .build();
        client.putObject(putRequest, RequestBody.fromBytes(content));
        return presignGetUrl(path, null);
    }

    @Override
    public void delete(String path) {
        DeleteObjectRequest deleteRequest = DeleteObjectRequest.builder()
                .bucket(config.getBucket())
                .key(path)
                .build();
        client.deleteObject(deleteRequest);
    }

    @Override
    public byte[] getContent(String path) {
        GetObjectRequest getRequest = GetObjectRequest.builder()
                .bucket(config.getBucket())
                .key(path)
                .build();
        return IoUtil.readBytes(client.getObject(getRequest));
    }

    @Override
    public String presignPutUrl(String path) {
        return presigner.presignPutObject(PutObjectPresignRequest.builder()
                .signatureDuration(EXPIRATION_DEFAULT)
                .putObjectRequest(b -> b.bucket(config.getBucket()).key(path))
                .build())
                .url().toString();
    }

    @Override
    public String presignGetUrl(String url, Integer expirationSeconds) {
        // 1. 将 url 转换为 path
        String path = StrUtil.removePrefix(url, config.getDomain() + "/");
        path = HttpUtils.decodeUrlPath(HttpUtils.removeUrlQuery(path));

        // 2.1 情况一：公开访问：无需签名
        if (!BooleanUtil.isFalse(config.getEnablePublicAccess())) {
            return config.getDomain() + "/" + path;
        }

        // 2.2 情况二：私有访问：生成 GET 预签名 URL
        Duration expiration = expirationSeconds != null ?
                Duration.ofSeconds(expirationSeconds) : EXPIRATION_DEFAULT;
        URL signedUrl = presigner.presignGetObject(GetObjectPresignRequest.builder()
                .signatureDuration(expiration)
                .getObjectRequest(b -> b.bucket(config.getBucket()).key(path))
                .build())
                .url();
        return signedUrl.toString();
    }
}
```

**配置示例**（阿里云 OSS）：

    "accessKey": "your-access-key",
    "accessSecret": "your-access-secret",
    "bucket": "your-bucket-name",
    "enablePublicAccess": true
}
```

**配置示例**（MinIO）：

```

## 存储枚举：FileStorageEnum

存储枚举定义了所有支持的存储类型：

```java
// 源码位置：yudao-module-infra-server/.../framework/file/core/enums/FileStorageEnum.java
@AllArgsConstructor
@Getter
public enum FileStorageEnum {

    DB(1, DBFileClientConfig.class, DBFileClient.class),

    LOCAL(10, LocalFileClientConfig.class, LocalFileClient.class),
    FTP(11, FtpFileClientConfig.class, FtpFileClient.class),
    SFTP(12, SftpFileClientConfig.class, SftpFileClient.class),

    S3(20, S3FileClientConfig.class, S3FileClient.class),
    ;

    /**
     * 存储器
     */
    private final Integer storage;

    /**
     * 配置类
     */
    private final Class<? extends FileClientConfig> configClass;

    /**
     * 客户端类
     */
    private final Class<? extends FileClient> clientClass;
}
```

**设计要点**：

1. **S3 统一协议**：阿里云 OSS、腾讯云 COS、MinIO、AWS S3 都支持 S3 协议，使用 `S3FileClient` 统一处理
2. **配置与实现分离**：每个存储类型关联一个配置类和一个客户端类，便于扩展
3. **工厂模式**：通过 `FileClientFactory` 根据配置创建对应的客户端实例

## 文件访问

文件访问方式取决于存储类型：

| 存储类型 | 访问方式 | 示例 |
|---------|---------|------|
| 本地存储 | 通过 FileController 代理 | `/admin-api/infra/file/10/get/2024/01/01/abc.jpg` |
| S3（公开） | 直接访问 OSS URL | `https://bucket.oss-cn-hangzhou.aliyuncs.com/2024/01/01/abc.jpg` |
| S3（私有） | 预签名 URL | `https://bucket.oss-cn-hangzhou.aliyuncs.com/...?Signature=xxx` |

**预签名 URL 的作用**：

```
┌─────────────────────────────────────────────────────────────────┐
│                    预签名 URL 流程                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 前端请求预签名 URL                                           │
│     GET /admin-api/infra/file/presigned-url?url=xxx             │
│                                                                 │
│  2. 后端生成带签名的 URL                                         │
│     https://bucket.oss-cn-hangzhou.aliyuncs.com/...?Signature=xxx│
│                                                                 │
│  3. 前端使用预签名 URL 直接访问文件                               │
│     （绕过后端，减轻服务器压力）                                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 本章小结

文件管理模块通过以下设计实现了可插拔的文件存储方案：

1. **接口抽象**：`FileClient` 接口统一了文件操作 API，业务代码只依赖接口
2. **工厂模式**：`FileClientFactory` 根据配置创建对应的客户端实例
3. **配置驱动**：通过数据库配置切换存储后端，无需修改代码
4. **多存储支持**：支持本地、S3、FTP、SFTP、数据库等多种存储后端
5. **预签名 URL**：支持生成预签名 URL，实现客户端直传，减轻服务器压力
