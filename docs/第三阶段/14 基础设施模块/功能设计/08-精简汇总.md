## 核心设计决策列表

| 决策 | 选择 | 理由 |
|------|------|------|
| 文件存储 | 接口抽象 + 多实现 | 支持多种存储后端，业务代码无感知 |
| 存储配置 | 数据库存储 | 支持运行时切换，无需重启 |
| 日志记录 | Filter + 注解 | 业务代码零侵入 |
| 日志写入 | 异步写入 | 不影响主业务性能 |
| 代码生成 | Velocity 模板 | 成熟稳定，易于自定义 |
| 敏感脱敏 | 内置 + 自定义 | 内置常见敏感字段，支持扩展 |
| 日志保留 | 定时任务清理 | 避免数据无限增长 |

## 文件存储选型速查

| 存储类型 | 适用场景 | 优点 | 缺点 |
|---------|---------|------|------|
| **本地存储** | 开发环境、小型项目 | 简单快速、无外部依赖 | 不支持分布式、服务器迁移困难 |
| **数据库存储** | 小文件、开发测试 | 无需额外服务 | 性能差、占用数据库空间 |
| **阿里云 OSS** | 中大型项目、阿里云用户 | 稳定可靠、CDN 加速 | 需要付费 |
| **腾讯云 COS** | 中大型项目、腾讯云用户 | 稳定可靠、CDN 加速 | 需要付费 |
| **MinIO** | 自建对象存储 | 开源免费、S3 兼容 | 需要自行维护 |
| **AWS S3** | 国际化项目 | 全球覆盖、功能丰富 | 国内访问慢 |
| **FTP** | 传统系统对接 | 兼容性好 | 安全性差、性能一般 |
| **SFTP** | 安全要求高的场景 | 加密传输 | 配置复杂 |

**选型建议**：

```
┌─────────────────────────────────────────────────────────────────┐
│                    文件存储选型决策树                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  是否需要分布式部署？                                             │
│  ├── 否 → 本地存储                                              │
│  └── 是                                                         │
│      │                                                          │
│      ├── 是否使用云服务？                                        │
│      │   ├── 阿里云 → 阿里云 OSS                                │
│      │   ├── 腾讯云 → 腾讯云 COS                                │
│      │   └── AWS → AWS S3                                       │
│      │                                                          │
│      └── 是否需要自建？                                          │
│          ├── 是 → MinIO                                         │
│          └── 否 → 选择合适的云服务                               │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 配置速查

### 代码生成配置

```yaml
yudao:
  codegen:
    base-package: cn.iocoder.yudao.module.product  # 基础包名
    db-schemas: [yudao-cloud]                      # 数据库名
    front-type: 2                                   # 1=Vue2, 2=Vue3
    vo-type: 1                                      # VO 结构类型
    delete-batch-enable: true                       # 批量删除
    unit-test-enable: true                          # 单元测试
    import-enable: false                            # Excel 导入

## 核心流程图

### 文件上传流程

```
业务代码调用 FileService.createFile()
       │
       ▼
FileServiceImpl 获取主配置的 FileClient
       │
       ▼
FileClient.upload(content, path, type)
       │
       ├── LocalFileClient: 写入本地磁盘
       ├── S3FileClient: 调用 S3 API
       ├── FtpFileClient: 上传到 FTP
       └── DBFileClient: 存入数据库
       │
       ▼
返回文件访问 URL
       │
       ▼
保存 FileDO 到数据库
```

### 日志记录流程

```
HTTP 请求进入
       │
       ▼
ApiAccessLogFilter 拦截
       │
       ├── 记录开始时间
       ├── 获取请求参数
       │
       ▼
执行 Controller 逻辑
       │
       ├── 正常返回 ──► 构建访问日志
       │
       └── 异常抛出 ──► 构建访问日志 + 错误日志
       │
       ▼
敏感参数脱敏
       │
       ▼
异步写入数据库
```

```

       │
       ▼

       │
       ▼

## 常见问题

### Q1: 如何切换文件存储后端？

在数据库中修改 `infra_file_config` 表：
1. 新增一条配置记录，设置 `master = 1`
2. 将原配置的 `master` 设置为 `0`
3. 无需重启，新上传的文件自动使用新配置

### Q2: 如何自定义代码生成模板？

1. 在 `resources/codegen/java/` 目录下创建模板文件
2. 使用 Velocity 语法编写模板
3. 在 `CodegenTemplateTypeEnum` 中注册模板类型
4. 在代码生成配置中选择自定义模板

### Q3: 如何排除特定接口的日志记录？

在 Controller 方法上添加注解：

```java
@GetMapping("/list")
@ApiAccessLog(enable = false)  // 关闭日志记录
public CommonResult<List<DataRespVO>> list() {
    // ...
}
```

### Q4: 日志数据量太大怎么办？

1. 调整日志保留天数（修改定时任务配置）
2. 对高频接口关闭日志记录（`@ApiAccessLog(enable = false)`）
3. 关闭请求参数记录（`@ApiAccessLog(requestEnable = false)`）
4. 定期归档历史日志到其他存储

### Q5: 如何查看某个用户的操作记录？

```sql
SELECT * FROM infra_api_access_log
WHERE user_id = 12345
ORDER BY create_time DESC
LIMIT 100;
```

## 总结

yudao-module-infra 作为技术基础设施模块，提供了四大核心能力：

1. **代码生成**：从数据库表自动生成 CRUD 代码，提升开发效率
2. **文件管理**：统一的文件上传/下载 API，支持多种存储后端
3. **API 访问日志**：自动记录所有 API 请求，用于性能监控和错误追踪
4. **配置管理**：通用的配置存储和读取能力

这些能力通过接口抽象、注解驱动、异步处理等设计模式，实现了业务代码零侵入、存储后端可插拔、日志记录自动化的技术基础设施。
