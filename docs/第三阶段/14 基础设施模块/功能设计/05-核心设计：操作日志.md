# 05-核心设计：操作日志

## 场景引入

假设你正在开发一个电商系统，管理员需要执行以下关键操作：

- 修改商品价格：从 99 元改为 199 元
- 删除订单：删除异常订单
- 调整库存：手动调整商品库存
- 审批退款：审批用户的退款申请

这些操作都需要记录日志，用于审计和问题追溯。传统的手动记录方式存在以下问题：

```java
// 传统方式：手动记录日志
public void updateProductPrice(Long productId, BigDecimal newPrice) {
    // 1. 获取原价格
    ProductDO product = productMapper.selectById(productId);
    BigDecimal oldPrice = product.getPrice();

    // 2. 更新价格
    product.setPrice(newPrice);
    productMapper.updateById(product);

    // 3. 手动记录日志（侵入业务代码）
    OperateLogDO log = new OperateLogDO();
    log.setUserId(SecurityFrameworkUtils.getLoginUserId());
    log.setUserName(SecurityFrameworkUtils.getLoginUserNickname());
    log.setOperateType(OperateTypeEnum.UPDATE.getType());
    log.setOperateName("修改商品价格");
    log.setRequestParam("{\"productId\":" + productId + ",\"newPrice\":" + newPrice + "}");
    log.setTimestamp(LocalDateTime.now());
    operateLogMapper.insert(log);
}
```

**问题**：
- 日志代码与业务逻辑混在一起，违反单一职责原则
- 容易遗漏：开发者可能忘记记录日志
- 维护困难：日志格式变更时，需要修改所有业务代码

如何实现声明式的日志记录，让业务代码零侵入？

## @ApiAccessLog 注解

`@ApiAccessLog` 注解用于标记需要记录日志的方法：

```java
// 源码位置：yudao-framework/.../apilog/core/annotation/ApiAccessLog.java
@Target({ElementType.METHOD})
@Retention(RetentionPolicy.RUNTIME)
public @interface ApiAccessLog {

    // ========== 开关字段 ==========

    /**
     * 是否记录访问日志
     * 默认 true，设置为 false 可以关闭日志记录
     */
    boolean enable() default true;

    /**
     * 是否记录请求参数
     * 默认记录，主要考虑请求数据一般不大
     * 可手动设置为 false 进行关闭
     */
    boolean requestEnable() default true;

    /**
     * 是否记录响应结果
     * 默认不记录，主要考虑响应数据可能比较大
     * 可手动设置为 true 进行打开
     */
    boolean responseEnable() default false;

    /**
     * 敏感参数数组
     * 添加后，请求参数、响应结果不会记录该参数
     */
    String[] sanitizeKeys() default {};

    // ========== 模块字段 ==========

    /**
     * 操作模块
     * 为空时，会尝试读取 @Tag.name() 属性
     */
    String operateModule() default "";

    /**
     * 操作名
     * 为空时，会尝试读取 @Operation.summary() 属性
     */
    String operateName() default "";

    /**
     * 操作分类
     * 实际并不是数组，因为枚举不能设置 null 作为默认值
     */
    OperateTypeEnum[] operateType() default {};
}
```

**使用示例**：

```java
// 方式一：使用 Swagger 注解自动填充
@Tag(name = "管理后台 - 商品")
@RestController
@RequestMapping("/product/goods")
public class ProductController {

    @PostMapping("/update")
    @Operation(summary = "更新商品")
    @ApiAccessLog  // 自动从 @Tag 和 @Operation 读取模块名和操作名
    public CommonResult<Boolean> updateProduct(@Valid @RequestBody ProductSaveReqVO updateReqVO) {
        productService.updateProduct(updateReqVO);
        return success(true);
    }
}

// 方式二：手动指定模块名和操作名
@PostMapping("/update-price")
@Operation(summary = "更新商品价格")
@ApiAccessLog(
    operateModule = "商品管理",
    operateName = "修改商品价格",
    operateType = OperateTypeEnum.UPDATE
)
public CommonResult<Boolean> updateProductPrice(@Valid @RequestBody UpdatePriceReqVO reqVO) {
    productService.updateProductPrice(reqVO);
    return success(true);
}

// 方式三：记录响应结果
@GetMapping("/get")
@Operation(summary = "获得商品详情")
@ApiAccessLog(responseEnable = true)  // 开启响应结果记录
public CommonResult<ProductRespVO> getProduct(@RequestParam("id") Long id) {
    return success(productService.getProduct(id));
}

// 方式四：敏感参数脱敏
@PostMapping("/login")
@Operation(summary = "用户登录")
@ApiAccessLog(sanitizeKeys = {"password", "captcha"})  // 脱敏密码和验证码
public CommonResult<LoginRespVO> login(@Valid @RequestBody LoginReqVO reqVO) {
    return success(authService.login(reqVO));
}
```

## ApiAccessLogFilter：日志拦截器

`ApiAccessLogFilter` 是一个 Servlet Filter，负责拦截所有 API 请求并记录日志：

```java
// 源码位置：yudao-framework/.../apilog/core/filter/ApiAccessLogFilter.java
@Slf4j
public class ApiAccessLogFilter extends ApiRequestFilter {

    private static final String[] SANITIZE_KEYS = new String[]{
        "password", "token", "accessToken", "refreshToken"
    };

    private final String applicationName;
    private final ApiAccessLogCommonApi apiAccessLogApi;

    public ApiAccessLogFilter(WebProperties webProperties, String applicationName,
                              ApiAccessLogCommonApi apiAccessLogApi) {
        super(webProperties);
        this.applicationName = applicationName;
        this.apiAccessLogApi = apiAccessLogApi;
    }

    @Override
    protected void doFilterInternal(HttpServletRequest request, HttpServletResponse response,
                                    FilterChain filterChain) throws ServletException, IOException {
        // 获得开始时间
        LocalDateTime beginTime = LocalDateTime.now();
        // 提前获得参数，避免 XssFilter 过滤处理
        Map<String, String> queryString = ServletUtils.getParamMap(request);
        String requestBody = ServletUtils.getBody(request);

        try {
            // 继续过滤器
            filterChain.doFilter(request, response);
            // 正常执行，记录日志
            createApiAccessLog(request, beginTime, queryString, requestBody, null);
        } catch (Exception ex) {
            // 异常执行，记录日志
            createApiAccessLog(request, beginTime, queryString, requestBody, ex);
            throw ex;
        }
    }

    private void createApiAccessLog(HttpServletRequest request, LocalDateTime beginTime,
                                    Map<String, String> queryString, String requestBody,
                                    Exception ex) {
        ApiAccessLogCreateReqDTO accessLog = new ApiAccessLogCreateReqDTO();
        try {
            boolean enable = buildApiAccessLog(accessLog, request, beginTime,
                                               queryString, requestBody, ex);
            if (!enable) {
                return;
            }
            // 异步写入日志
            apiAccessLogApi.createApiAccessLogAsync(accessLog);
        } catch (Throwable th) {
            log.error("[createApiAccessLog][url({}) log({}) 发生异常]",
                      request.getRequestURI(), toJsonString(accessLog), th);
        }
    }
}
```

```

│                    ApiAccessLogFilter 工作流程                   │
```

## 敏感参数脱敏

### 内置脱敏字段

`ApiAccessLogFilter` 内置了以下敏感字段的脱敏：

```java
private static final String[] SANITIZE_KEYS = new String[]{
    "password",      // 密码
    "token",         // Token
    "accessToken",   // 访问令牌
    "refreshToken"   // 刷新令牌
};
```

### 自定义脱敏

通过 `@ApiAccessLog` 注解的 `sanitizeKeys` 属性，可以指定额外的脱敏字段：

```java
@PostMapping("/login")
@Operation(summary = "用户登录")
@ApiAccessLog(sanitizeKeys = {"password", "captcha", "smsCode"})
public CommonResult<LoginRespVO> login(@Valid @RequestBody LoginReqVO reqVO) {
    return success(authService.login(reqVO));
}
```

### 脱敏实现

```java
// 请求参数脱敏
private static String sanitizeMap(Map<String, ?> map, String[] sanitizeKeys) {
    if (CollUtil.isEmpty(map)) {
        return null;
    }
    // 移除自定义脱敏字段
    if (sanitizeKeys != null) {
        MapUtil.removeAny(map, sanitizeKeys);
    }
    // 移除内置脱敏字段
    MapUtil.removeAny(map, SANITIZE_KEYS);
    return JsonUtils.toJsonString(map);
}

// JSON 脱敏
private static void sanitizeJson(JsonNode node, String[] sanitizeKeys) {
    // 数组：遍历处理
    if (node.isArray()) {
        for (JsonNode childNode : node) {
            sanitizeJson(childNode, sanitizeKeys);
        }
        return;
    }
    // 非 Object：直接返回
    if (!node.isObject()) {
        return;
    }
    // Object：遍历处理
    Iterator<Map.Entry<String, JsonNode>> iterator = node.fields();
    while (iterator.hasNext()) {
        Map.Entry<String, JsonNode> entry = iterator.next();
        // 移除敏感字段
        if (ArrayUtil.contains(sanitizeKeys, entry.getKey())
                || ArrayUtil.contains(SANITIZE_KEYS, entry.getKey())) {
            iterator.remove();
            continue;
        }
        // 递归处理嵌套对象
        sanitizeJson(entry.getValue(), sanitizeKeys);
    }
}
```

## 日志的异步写入

为了避免日志写入影响主业务性能，采用异步写入方式：

```
┌─────────────────────────────────────────────────────────────────┐
│                    异步日志写入流程                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  请求处理 ──► 构建日志对象                                       │
│       │                                                         │
│       │ 异步调用                                                 │
│       ▼                                                         │
│  ApiAccessLogCommonApi.createApiAccessLogAsync()                │
│       │                                                         │
│       │ 放入线程池                                               │
│       ▼                                                         │
│  线程池执行 ──► 写入数据库                                       │
│       │                                                         │
│       │ 立即返回                                                 │
│       ▼                                                         │
│  主业务继续执行                                                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**实现方式**：

```java
// RPC 接口实现
@Service
public class ApiAccessLogCommonApiImpl implements ApiAccessLogCommonApi {

    @Resource
    private ApiAccessLogService apiAccessLogService;

    @Override
    @Async  // 异步执行
    public void createApiAccessLogAsync(ApiAccessLogCreateReqDTO reqDTO) {
        apiAccessLogService.createApiAccessLog(reqDTO);
    }
}
```

## 与 Spring AOP 的集成

虽然 `ApiAccessLogFilter` 是基于 Servlet Filter 实现的，但它与 Spring AOP 有良好的集成：

1. **注解解析**：通过 `HandlerMethod` 获取方法上的 `@ApiAccessLog` 注解
2. **Swagger 注解读取**：自动读取 `@Tag` 和 `@Operation` 注解的信息
3. **拦截器配合**：`ApiAccessLogInterceptor` 提前将 `HandlerMethod` 存入请求属性

```java
// 源码位置：yudao-framework/.../apilog/core/interceptor/ApiAccessLogInterceptor.java
public class ApiAccessLogInterceptor implements HandlerInterceptor {

    public static final String ATTRIBUTE_HANDLER_METHOD = "ATTRIBUTE_HANDLER_METHOD";

    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response,
                             Object handler) {
        if (handler instanceof HandlerMethod) {
            // 将 HandlerMethod 存入请求属性，供 ApiAccessLogFilter 使用
            request.setAttribute(ATTRIBUTE_HANDLER_METHOD, handler);
        }
        return true;
    }
}
```

## 本章小结

操作日志模块通过以下设计实现了声明式的日志记录：

1. **注解驱动**：通过 `@ApiAccessLog` 注解标记需要记录日志的方法
2. **Filter 拦截**：`ApiAccessLogFilter` 自动拦截所有 API 请求
3. **自动填充**：自动读取 Swagger 注解，填充模块名和操作名
4. **敏感脱敏**：内置敏感字段脱敏，支持自定义脱敏字段
5. **异步写入**：日志异步写入数据库，不影响主业务性能
