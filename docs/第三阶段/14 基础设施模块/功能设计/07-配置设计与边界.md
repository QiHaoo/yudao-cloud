# 07-配置设计与边界

## 场景引入

在使用 yudao-module-infra 时，你可能会遇到以下问题：

- 代码生成器应该配置哪些参数？
- 文件存储如何从本地切换到阿里云 OSS？
- 访问日志保留多少天合适？
- infra 模块与 system 模块的职责边界在哪里？

本章将详细解答这些问题。

## 代码生成配置

### 配置属性类

```java
// 源码位置：yudao-module-infra-server/.../framework/codegen/config/CodegenProperties.java
@ConfigurationProperties(prefix = "yudao.codegen")
@Data
public class CodegenProperties {

    /**
     * 生成的 Java 代码的基础包
     * 例如：cn.iocoder.yudao.module.product
     */
    @NotNull(message = "Java 代码的基础包不能为空")
    private String basePackage;

    /**
     * 数据库名数组
     * 用于限制只读取指定数据库的表结构
     */
    @NotEmpty(message = "数据库不能为空")
    private Collection<String> dbSchemas;

    /**
     * 代码生成的前端类型（默认）
     * 枚举 CodegenFrontTypeEnum
     * 1 = Vue2 + Element UI
     * 2 = Vue3 + Element Plus
     */
    @NotNull(message = "代码生成的前端类型不能为空")
    private Integer frontType;

    /**
     * 代码生成的 VO 类型
     * 枚举 CodegenVOTypeEnum
     * 1 = SaveReqVO + PageReqVO + RespVO
     * 2 = SaveReqVO + PageReqVO + RespVO + SimpleRespVO
     */
    @NotNull(message = "代码生成的 VO 类型不能为空")
    private Integer voType;

    /**
     * 是否生成批量删除接口
     */
    @NotNull(message = "是否生成批量删除接口不能为空")
    private Boolean deleteBatchEnable;

    /**
     * 是否生成单元测试
     */
    @NotNull(message = "是否生成单元测试不能为空")
    private Boolean unitTestEnable;

    /**
     * 是否生成 Excel 导入接口
     */
    @NotNull(message = "是否生成 Excel 导入接口不能为空")
    private Boolean importEnable;
}
```

### 配置示例

```yaml
# application.yml
yudao:
  codegen:
    # Java 代码的基础包
    base-package: cn.iocoder.yudao.module.product

    # 数据库名数组
    db-schemas:
      - yudao-cloud

    # 前端类型：2 = Vue3 + Element Plus
    front-type: 2

    # VO 类型：1 = 标准 VO 结构
    vo-type: 1

    # 是否生成批量删除接口
    delete-batch-enable: true

    # 是否生成单元测试
    unit-test-enable: true

    # 是否生成 Excel 导入接口
    import-enable: false
```

### 配置说明

| 配置项 | 说明 | 推荐值 |
|--------|------|--------|
| `base-package` | 生成代码的基础包名 | 根据项目实际情况设置 |
| `db-schemas` | 要读取的数据库名 | 设置为项目使用的数据库名 |
| `front-type` | 前端框架类型 | 新项目建议用 2（Vue3） |
| `vo-type` | VO 结构类型 | 一般用 1 即可 |
| `delete-batch-enable` | 是否生成批量删除 | 建议开启 |
| `unit-test-enable` | 是否生成单元测试 | 建议开启 |
| `import-enable` | 是否生成导入接口 | 按需开启 |

## 文件存储配置

### 存储配置表

文件存储配置存储在数据库表 `infra_file_config` 中：

```sql
CREATE TABLE `infra_file_config` (
  `id` bigint NOT NULL AUTO_INCREMENT COMMENT '编号',
  `name` varchar(63) NOT NULL COMMENT '配置名',
  `storage` int NOT NULL COMMENT '存储器',
  `remark` varchar(255) DEFAULT NULL COMMENT '备注',
  `master` bit(1) NOT NULL DEFAULT b'0' COMMENT '是否为主配置',
  `config` varchar(4096) NOT NULL COMMENT '存储配置',
  `creator` varchar(64) DEFAULT '' COMMENT '创建者',
  `create_time` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updater` varchar(64) DEFAULT '' COMMENT '更新者',
  `update_time` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `deleted` bit(1) NOT NULL DEFAULT b'0' COMMENT '是否删除',
  `tenant_id` bigint NOT NULL DEFAULT 0 COMMENT '租户编号',
  PRIMARY KEY (`id`)
) ENGINE=InnoDB COMMENT='文件配置表';
```


### 各存储类型的配置

**本地存储配置**（storage=10）：

```json
{
    "domain": "http://localhost:48080",
    "basePath": "/data/upload"
}
```

| 配置项 | 说明 | 示例 |
|--------|------|------|
| `domain` | 访问域名 | `http://localhost:48080` |
| `basePath` | 存储基础路径 | `/data/upload` |

**S3 协议配置**（storage=20）：

```json
{
    "endpoint": "oss-cn-hangzhou.aliyuncs.com",
    "accessKey": "your-access-key",
    "accessSecret": "your-access-secret",
    "bucket": "your-bucket-name",
    "region": "cn-hangzhou",
    "enablePublicAccess": true,
    "enablePathStyleAccess": false
}
```

| 配置项 | 说明 | 阿里云 OSS | 腾讯云 COS | MinIO |
|--------|------|-----------|-----------|-------|
| `endpoint` | 服务端点 | `oss-cn-hangzhou.aliyuncs.com` | `cos.ap-guangzhou.myqcloud.com` | `http://localhost:9000` |
| `accessKey` | 访问密钥 | AccessKey ID | SecretId | minioadmin |
| `accessSecret` | 访问密钥 | AccessKey Secret | SecretKey | minioadmin |
| `bucket` | 存储桶 | Bucket 名称 | Bucket 名称 | Bucket 名称 |
| `region` | 区域 | cn-hangzhou | ap-guangzhou | us-east-1 |
| `enablePublicAccess` | 是否公开访问 | true/false | true/false | true |
| `enablePathStyleAccess` | 是否路径风格 | false | false | true |

### 切换存储后端

**场景**：从本地存储切换到阿里云 OSS

```
步骤 1：在数据库中新增 OSS 配置
┌─────────────────────────────────────────────────────────────────┐
│ INSERT INTO infra_file_config (name, storage, master, config)   │
│ VALUES ('阿里云 OSS', 20, 1, '{...}');                          │
└─────────────────────────────────────────────────────────────────┘

步骤 2：将原配置的 master 设置为 0
┌─────────────────────────────────────────────────────────────────┐
│ UPDATE infra_file_config SET master = 0 WHERE id = 1;           │
└─────────────────────────────────────────────────────────────────┘

步骤 3：无需修改代码，新上传的文件自动使用 OSS
```

## 日志配置

### 访问日志配置

访问日志的配置主要通过 `@ApiAccessLog` 注解控制：

```java
// 全局默认配置（在 ApiAccessLogFilter 中）
private static final String[] SANITIZE_KEYS = new String[]{
    "password", "token", "accessToken", "refreshToken"
};
```

**注解配置**：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `enable` | true | 是否记录日志 |
| `requestEnable` | true | 是否记录请求参数 |
| `responseEnable` | false | 是否记录响应结果 |
| `sanitizeKeys` | {} | 自定义脱敏字段 |

### 日志保留策略

通过定时任务配置日志保留天数：

```java
// 访问日志：保留 7 天
@Scheduled(cron = "0 0 2 * * ?")
public void cleanAccessLog() {
    apiAccessLogService.cleanAccessLog(7);
}

// 错误日志：保留 30 天
@Scheduled(cron = "0 0 3 * * ?")
public void cleanErrorLog() {
    apiErrorLogService.cleanErrorLog(30);
}
```

**推荐配置**：

| 日志类型 | 保留天数 | 说明 |
|---------|---------|------|
| 访问日志 | 7-15 天 | 数据量大，建议短期保留 |
| 错误日志 | 30-90 天 | 用于问题追溯，建议长期保留 |

## 与其他模块的边界

### infra 与 system 的分工

```
┌─────────────────────────────────────────────────────────────────┐
│                    infra 与 system 的分工                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  yudao-module-system（业务实体管理）：                            │
│  ├── 用户管理：SysUserDO, SysUserService                        │
│  ├── 角色管理：SysRoleDO, SysRoleService                        │
│  ├── 菜单管理：SysMenuDO, SysMenuService                        │
│  ├── 字典管理：SysDictTypeDO, SysDictDataDO                     │
│  ├── 部门管理：SysDeptDO, SysDeptService                        │
│  ├── 岗位管理：SysPostDO, SysPostService                        │
│  └── 租户管理：TenantDO, TenantService                          │
│                                                                 │
│  yudao-module-infra（技术基础设施）：                             │
│  ├── 代码生成：CodegenTableDO, CodegenColumnDO                  │
│  ├── 文件管理：FileDO, FileConfigDO                             │
│  ├── 日志记录：ApiAccessLogDO, ApiErrorLogDO                    │
│  ├── 配置管理：ConfigDO                                         │
│  ├── 数据源管理：DataSourceConfigDO                             │
│  └── Redis 监控                                                 │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 边界划分原则

| 原则 | 说明 | 示例 |
|------|------|------|
| **业务 vs 技术** | system 管业务实体，infra 管技术设施 | 用户是业务实体 → system；文件存储是技术设施 → infra |
| **数据归属** | 数据属于哪个业务域 | 用户数据属于系统域 → system；日志数据属于基础设施域 → infra |
| **使用关系** | 谁使用谁的能力 | 业务模块使用文件服务 → infra 提供；业务模块查询用户信息 → system 提供 |

### 跨模块引用方式

**方式一：RPC 接口（推荐）**

```java

    @PostMapping(PREFIX + "/create")
    CommonResult<String> createFile(@Valid @RequestBody FileCreateReqDTO createReqDTO);
}

// 其他模块使用
@Service
public class ProductService {
    @Resource
    private FileApi fileApi;

    public String uploadImage(byte[] content, String name) {
        return fileApi.createFile(content, name, "product", "image/jpeg");
    }
}
```

## 跨模块调用示例

### 示例一：商品模块上传图片

```java
// yudao-module-product-server
@Service
public class ProductImageService {

    @Resource
    private FileApi fileApi;  // 注入 infra 模块的文件服务

    public Long uploadProductImage(MultipartFile file, Long productId) throws Exception {
        // 1. 上传文件到 infra 模块
        String url = fileApi.createFile(
            file.getBytes(),
            file.getOriginalFilename(),
            "product/" + productId,
            file.getContentType()
        );

        // 2. 保存图片记录到本地表
        ProductImageDO image = new ProductImageDO();
        image.setProductId(productId);
        image.setUrl(url);
        productImageMapper.insert(image);

        return image.getId();
    }
}
```

### 示例二：系统模块查询文件信息

```java
// yudao-module-system-server
@Service
public class UserAvatarService {

    @Resource
    private FileApi fileApi;  // 注入 infra 模块的文件服务

    public String uploadAvatar(MultipartFile file, Long userId) throws Exception {
        // 上传头像
        String url = fileApi.createFile(
            file.getBytes(),
            file.getOriginalFilename(),
            "avatar/" + userId,
            file.getContentType()
        );

        // 更新用户头像 URL
        SysUserDO user = new SysUserDO();
        user.setId(userId);
        user.setAvatar(url);
        sysUserMapper.updateById(user);

        return url;
    }
}
```

## 配置管理 API

infra 模块还提供了通用的配置管理功能：

```java
// 源码位置：yudao-module-infra-server/.../controller/admin/config/ConfigController.java
@Tag(name = "管理后台 - 配置管理")
@RestController
@RequestMapping("/infra/config")
public class ConfigController {

    @Resource
    private ConfigService configService;

    @GetMapping("/get")
    @Operation(summary = "获得配置")
    public CommonResult<ConfigRespVO> getConfig(@RequestParam("id") Long id) {
        return success(BeanUtils.toBean(configService.getConfig(id), ConfigRespVO.class));
    }

    @GetMapping("/get-by-key")
    @Operation(summary = "获得配置值")
    public CommonResult<String> getConfigByKey(
            @RequestParam("category") String category,
            @RequestParam("key") String key) {
        return success(configService.getConfigValueByKey(category, key));
    }

    @PostMapping("/create")
    @Operation(summary = "创建配置")
    @PreAuthorize("@ss.hasPermission('infra:config:create')")
    public CommonResult<Long> createConfig(@Valid @RequestBody ConfigSaveReqVO createReqVO) {
        return success(configService.createConfig(createReqVO));
    }
}
```

**使用场景**：

```java

        // 创建订单
        OrderDO order = new OrderDO();
        order.setTimeoutTime(LocalDateTime.now().plusMinutes(timeoutMinutes));

    }
}
```

## 本章小结

本章详细介绍了 yudao-module-infra 的配置设计和模块边界：

**配置设计**：
- 代码生成配置通过 `CodegenProperties` 管理，支持自定义包名、前端类型等
- 文件存储配置存储在数据库中，支持运行时切换存储后端
- 日志配置通过注解和定时任务控制，支持灵活的保留策略

**模块边界**：
- system 管理业务实体（用户、角色、菜单等）
- infra 提供技术基础设施（代码生成、文件、日志等）
- 跨模块通过 RPC 接口调用，保持松耦合
