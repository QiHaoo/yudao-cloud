# 03-测试支持与源码索引

## 第一部分：测试支持

### 盘点现状

在没有专门测试工具的情况下，测试涉及多租户的业务代码时，开发人员通常需要手动设置 `TenantContextHolder`：

```java
@Test
public void testCreateProduct() {
    // 手动设置租户上下文
    TenantContextHolder.setTenantId(100L);
    try {
        productService.create(productVO);
        ProductDO product = productMapper.selectById(productId);
        assertEquals(100L, product.getTenantId());
    } finally {
        TenantContextHolder.clear(); // 别忘了清理！
    }
}
```

这种方式有几个痛点：

1. **容易忘记清理**：如果测试中途抛异常，`finally` 之前的代码不会执行清理（虽然这里用了 finally，但很多人会忘记）
2. **重复样板代码**：每个测试方法都要写设置/清理的逻辑
3. **无法测试越权场景**：模拟"用户 A 试图访问租户 B 的数据"需要同时设置登录用户和租户上下文，比较繁琐

### 推荐的测试方式

#### 方式一：使用 TenantUtils 包裹

最简单的方式，利用已有的 `TenantUtils.execute()` 方法：

```java
@Test
public void testCreateProduct() {
    TenantUtils.execute(100L, () -> {
        productService.create(productVO);
        ProductDO product = productMapper.selectById(productId);
        assertEquals(100L, product.getTenantId());
    });
}
```

`TenantUtils.execute()` 会自动保存/恢复上下文，不需要手动 try-finally。

#### 方式二：JUnit 5 扩展（推荐）

对于频繁使用多租户测试的项目，可以创建一个自定义注解和 JUnit 5 扩展：

```java
/**
 * 测试用注解：指定当前测试方法的租户编号
 */
@Target({ElementType.METHOD, ElementType.TYPE})
@Retention(RetentionPolicy.RUNTIME)
@ExtendWith(TenantTestExtension.class)
public @interface TenantTest {
    long value() default 100L; // 默认租户编号
}
```

```java
/**
 * JUnit 5 扩展：自动设置和清理租户上下文
 */
public class TenantTestExtension implements BeforeEachCallback, AfterEachCallback {

    @Override
    public void beforeEach(ExtensionContext context) {
        TenantTest annotation = AnnotationUtils.findAnnotation(
            context.getRequiredTestMethod(), TenantTest.class);
        if (annotation == null) {
            annotation = AnnotationUtils.findAnnotation(
                context.getRequiredTestClass(), TenantTest.class);
        }
        if (annotation != null) {
            TenantContextHolder.setTenantId(annotation.value());
        }
    }

    @Override
    public void afterEach(ExtensionContext context) {
        TenantContextHolder.clear();
    }
}
```

使用方式：

```java
@SpringBootTest
public class ProductServiceTest {

    @Test
    @TenantTest(100L)  // 自动设置 tenantId=100，测试结束后自动清理
    public void testCreateProduct() {
        productService.create(productVO);
        ProductDO product = productMapper.selectById(productId);
        assertEquals(100L, product.getTenantId());
    }

    @Test
    @TenantTest(100L)
    public void testCrossTenantAccess() {
        // 尝试操作租户 200 的商品，应该失败
        assertThrows(NotFoundException.class, () -> {
            TenantUtils.execute(200L, () -> {
                productService.update(productId, updateVO);
            });
        });
    }
}
```

#### 方式三：H2 内存数据库测试

项目使用 H2 作为测试数据库，多租户拦截器在 H2 上同样生效：

```java
@SpringBootTest
@Sql(scripts = "classpath:sql/product.sql") // 初始化表结构和数据
public class ProductMapperTest {

    @Resource
    private ProductMapper productMapper;

    @Test
    @TenantTest(100L)
    public void testSelectList() {
        // 插入测试数据
        ProductDO product = new ProductDO();
        product.setName("测试手机壳");
        product.setTenantId(100L);
        productMapper.insert(product);

        // 查询时自动带上 WHERE tenant_id = 100
        List<ProductDO> list = productMapper.selectList(null);
        assertTrue(list.stream().allMatch(p -> p.getTenantId().equals(100L)));
    }

    @Test
    public void testTenantIsolation() {
        // 租户 100 插入的数据
        TenantUtils.execute(100L, () -> {
            ProductDO product = new ProductDO();
            product.setName("商家A的商品");
            productMapper.insert(product);
        });

        // 租户 200 查询，应该看不到租户 100 的数据
        TenantUtils.execute(200L, () -> {
            List<ProductDO> list = productMapper.selectList(null);
            assertTrue(list.isEmpty());
        });
    }
}
```

#### 测试 @TenantIgnore 场景

```java
@SpringBootTest
public class SystemConfigServiceTest {

    @Resource
    private SystemConfigService systemConfigService;

    @Test
    public void testGlobalConfig() {
        // 在租户 100 下创建配置
        TenantUtils.execute(100L, () -> {
            systemConfigService.saveConfig("site.name", "商家A的店铺");
        });

        // 使用 TenantUtils.executeIgnore 查询，应该能看到所有租户的配置
        List<SystemConfigDO> configs = TenantUtils.executeIgnore(() -> {
            return systemConfigService.listAllConfigs();
        });
        assertFalse(configs.isEmpty());
    }
}
```

### 测试清单

| 测试场景 | 验证点 |
|---------|--------|
| 正常查询 | SQL 自动带上 `WHERE tenant_id = ?` |
| 租户隔离 | 租户 A 看不到租户 B 的数据 |
| 自动填充 | INSERT 时 `tenant_id` 字段自动填充 |
| 跨租户越权 | 请求头的 tenantId 与登录用户的 tenantId 不一致时返回 403 |
| @TenantIgnore | 标注的方法/类跳过租户过滤 |
| 忽略 URL | 配置的 ignoreUrls 不要求携带 tenant-id |
| MQ 传播 | 消息消费后 TenantContextHolder 中有正确的 tenantId |
| 定时任务 | @TenantJob 遍历所有租户执行 |

### 业务层使用示例

#### 示例一：电商商品管理

实体类继承 `TenantBaseDO`：

```java
@TableName("product")
public class ProductDO extends TenantBaseDO {
    private Long id;
    private String name;
    private Long price;
    private Integer status;
    // tenantId 继承自 TenantBaseDO
}
```

Service 层无需关心租户，框架自动处理：

```java
@Service
public class ProductServiceImpl implements ProductService {

    @Resource
    private ProductMapper productMapper;

    public void createProduct(ProductCreateReqVO reqVO) {
        ProductDO product = BeanUtils.toBean(reqVO, ProductDO.class);
        // 不需要手动设置 tenantId，MetaObjectHandler 会自动填充
        productMapper.insert(product);
    }

    public PageResult<ProductDO> getProductPage(ProductPageReqVO reqVO) {
        // SQL 自动追加 WHERE tenant_id = ?，只查当前租户的商品
        return productMapper.selectPage(reqVO);
    }
}
```

#### 示例二：全局共享的字典数据

配置忽略表：

```yaml
yudao:
  tenant:
    ignore-tables:
      - system_dict_data
```

或者在实体类上标注 `@TenantIgnore`：

```java
@TableName("system_dict_data")
@TenantIgnore
public class SystemDictDataDO extends BaseDO {
    private Long id;
    private String dictType;
    private String label;
    private String value;
    // 没有 tenantId 字段，所有租户共享
}
```

#### 示例三：平台管理员代操作

平台管理员需要为某个商家创建商品：

```java
@Service
public class AdminProductService {

    @Resource
    private ProductService productService;

    public void createProductForTenant(Long tenantId, ProductCreateReqVO reqVO) {
        TenantUtils.execute(tenantId, () -> {
            productService.createProduct(reqVO);
        });
    }
}
```

#### 示例四：忽略租户的全局统计

```java
@Service
public class StatisticsService {

    @Resource
    private OrderMapper orderMapper;

    @TenantIgnore
    public Long getTotalOrderCount() {
        // 查询所有租户的订单总数
        return orderMapper.selectCount(null);
    }
}
```

### 配置文件

#### application.yml 多租户配置

```yaml
yudao:
  tenant:
    enable: true                           # 是否开启多租户
    ignore-urls:                           # 忽略租户的请求 URL
      - /admin-api/infra/file/callback     # 文件回调
      - /admin-api/pay/notify/**           # 支付回调
      - /admin-api/sms/callback            # 短信回调
    ignore-visit-urls:                     # 忽略跨租户访问的 URL
      - /admin-api/system/user/profile     # 个人信息接口
    ignore-tables:                         # 忽略租户的表
      - system_tenant                      # 租户表本身
      - system_tenant_package              # 租户套餐表
      - system_dict_data                   # 字典数据（全局共享）
      - system_dict_type                   # 字典类型（全局共享）
      - system_config                      # 系统配置（全局共享）
    ignore-caches:                         # 忽略租户的缓存
      - system_dict                        # 字典缓存
```

#### 前端请求头

前端在每个 API 请求中需要携带 `tenant-id` 请求头：

```javascript
// axios 请求拦截器
axios.interceptors.request.use(config => {
    const tenantId = getTenantId(); // 从登录信息或 URL 中获取
    if (tenantId) {
        config.headers['tenant-id'] = tenantId;
    }
    return config;
});
```
