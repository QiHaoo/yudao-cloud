# 02-安全防护与上下文传播

## 本章目标

上一章我们搭建了租户上下文和 DB 层拦截器，但还缺少几块关键的拼图：

1. **安全防护**：用户登录后，如何防止他篡改 `tenant-id` 请求头去访问其他租户的数据？
2. **Web 层上下文提取**：HTTP 请求中的 `tenant-id` 怎么自动提取到 `TenantContextHolder`？
3. **异步线程传播**：线程池中执行的任务怎么继承父线程的租户编号？
4. **MQ 消息传播**：消息发送时租户编号怎么带过去，消费时怎么恢复？
5. **注解式豁免**：某些接口或方法需要跳过租户过滤，怎么用注解优雅实现？

## 第一步：TenantContextWebFilter —— 从请求头提取租户编号

源码位置：`yudao-framework/yudao-spring-boot-starter-biz-tenant/src/main/java/cn/iocoder/yudao/framework/tenant/core/web/TenantContextWebFilter.java`

```java
package cn.iocoder.yudao.framework.tenant.core.web;

import cn.iocoder.yudao.framework.tenant.core.context.TenantContextHolder;
import cn.iocoder.yudao.framework.web.core.util.WebFrameworkUtils;
import org.springframework.web.filter.OncePerRequestFilter;

import javax.servlet.FilterChain;
import javax.servlet.ServletException;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import java.io.IOException;

/**
 * 多租户 Context Web 过滤器
 * 将请求 Header 中的 tenant-id 解析出来，添加到 TenantContextHolder 中
 */
public class TenantContextWebFilter extends OncePerRequestFilter {

    @Override
    protected void doFilterInternal(HttpServletRequest request, HttpServletResponse response, FilterChain chain)
            throws ServletException, IOException {
        // 设置
        Long tenantId = WebFrameworkUtils.getTenantId(request);
        if (tenantId != null) {
            TenantContextHolder.setTenantId(tenantId);
        }
        try {
            chain.doFilter(request, response);
        } finally {
            // 清理
            TenantContextHolder.clear();
        }
    }

}
```

### 执行顺序

这个 Filter 是整个多租户链路的**起点**，它在请求进入时从 `tenant-id` 请求头提取租户编号，写入 `TenantContextHolder`；在请求结束后清理上下文，避免线程复用导致数据串租户。

在自动配置中，它的顺序通过 `WebFilterOrderEnum.TENANT_CONTEXT_FILTER` 控制，确保在其他需要租户信息的 Filter 之前执行。

### 在自动配置中注册

```java
@Bean
public FilterRegistrationBean<TenantContextWebFilter> tenantContextWebFilter() {
    FilterRegistrationBean<TenantContextWebFilter> registrationBean = new FilterRegistrationBean<>();
    registrationBean.setFilter(new TenantContextWebFilter());
    registrationBean.setOrder(WebFilterOrderEnum.TENANT_CONTEXT_FILTER);
    return registrationBean;
}
```

## 第二步：TenantSecurityWebFilter —— 防租户越权

源码位置：`yudao-framework/yudao-spring-boot-starter-biz-tenant/src/main/java/cn/iocoder/yudao/framework/tenant/core/security/TenantSecurityWebFilter.java`

```java
package cn.iocoder.yudao.framework.tenant.core.security;

import cn.hutool.core.collection.CollUtil;
import cn.iocoder.yudao.framework.common.exception.enums.GlobalErrorCodeConstants;
import cn.iocoder.yudao.framework.common.pojo.CommonResult;
import cn.iocoder.yudao.framework.common.util.servlet.ServletUtils;
import cn.iocoder.yudao.framework.security.core.LoginUser;
import cn.iocoder.yudao.framework.security.core.util.SecurityFrameworkUtils;
import cn.iocoder.yudao.framework.tenant.config.TenantProperties;
import cn.iocoder.yudao.framework.tenant.core.context.TenantContextHolder;
import cn.iocoder.yudao.framework.tenant.core.service.TenantFrameworkService;
import cn.iocoder.yudao.framework.web.config.WebProperties;
import cn.iocoder.yudao.framework.web.core.filter.ApiRequestFilter;
import cn.iocoder.yudao.framework.web.core.handler.GlobalExceptionHandler;
import lombok.extern.slf4j.Slf4j;
import org.springframework.util.AntPathMatcher;

import javax.servlet.FilterChain;
import javax.servlet.ServletException;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import java.io.IOException;
import java.util.Objects;
import java.util.Set;

/**
 * 多租户 Security Web 过滤器
 * 1. 如果是登陆的用户，校验是否有权限访问该租户，避免越权问题。
 * 2. 如果请求未带租户的编号，检查是否是忽略的 URL，否则也不允许访问。
 * 3. 校验租户是合法，例如说被禁用、到期
 */
@Slf4j
public class TenantSecurityWebFilter extends ApiRequestFilter {

    private final TenantProperties tenantProperties;

    /**
     * 允许忽略租户的 URL 列表
     */
    private final Set<String> ignoreUrls;

    private final AntPathMatcher pathMatcher;

    private final GlobalExceptionHandler globalExceptionHandler;
    private final TenantFrameworkService tenantFrameworkService;

    public TenantSecurityWebFilter(WebProperties webProperties,
                                   TenantProperties tenantProperties,
                                   Set<String> ignoreUrls,
                                   GlobalExceptionHandler globalExceptionHandler,
                                   TenantFrameworkService tenantFrameworkService) {
        super(webProperties);
        this.tenantProperties = tenantProperties;
        this.ignoreUrls = ignoreUrls;
        this.pathMatcher = new AntPathMatcher();
        this.globalExceptionHandler = globalExceptionHandler;
        this.tenantFrameworkService = tenantFrameworkService;
    }

    @Override
    protected void doFilterInternal(HttpServletRequest request, HttpServletResponse response, FilterChain chain)
            throws ServletException, IOException {
        Long tenantId = TenantContextHolder.getTenantId();
        // 1. 登陆的用户，校验是否有权限访问该租户，避免越权问题。
        LoginUser user = SecurityFrameworkUtils.getLoginUser();
        if (user != null) {
            // 如果获取不到租户编号，则尝试使用登陆用户的租户编号
            if (tenantId == null) {
                tenantId = user.getTenantId();
                TenantContextHolder.setTenantId(tenantId);
                // 如果传递了租户编号，则进行比对租户编号，避免越权问题
            } else if (!Objects.equals(user.getTenantId(), TenantContextHolder.getTenantId())) {
                log.error("[doFilterInternal][租户({}) User({}/{}) 越权访问租户({}) URL({}/{})]",
                        user.getTenantId(), user.getId(), user.getUserType(),
                        TenantContextHolder.getTenantId(), request.getRequestURI(), request.getMethod());
                ServletUtils.writeJSON(response, CommonResult.error(GlobalErrorCodeConstants.FORBIDDEN.getCode(),
                        "您无权访问该租户的数据"));
                return;
            }
        }

        // 如果非允许忽略租户的 URL，则校验租户是否合法
        if (!isIgnoreUrl(request)) {
            // 2. 如果请求未带租户的编号，不允许访问。
            if (tenantId == null) {
                log.error("[doFilterInternal][URL({}/{}) 未传递租户编号]", request.getRequestURI(), request.getMethod());
                ServletUtils.writeJSON(response, CommonResult.error(GlobalErrorCodeConstants.BAD_REQUEST.getCode(),
                        "请求的租户标识未传递，请进行排查"));
                return;
            }
            // 3. 校验租户是合法，例如说被禁用、到期
            try {
                tenantFrameworkService.validTenant(tenantId);
            } catch (Throwable ex) {
                CommonResult<?> result = globalExceptionHandler.allExceptionHandler(request, ex);
                ServletUtils.writeJSON(response, result);
                return;
            }
        } else { // 如果是允许忽略租户的 URL，若未传递租户编号，则默认忽略租户编号，避免报错
            if (tenantId == null) {
                TenantContextHolder.setIgnore(true);
            }
        }

        // 继续过滤
        chain.doFilter(request, response);
    }

    private boolean isIgnoreUrl(HttpServletRequest request) {
        String apiUri = request.getRequestURI().substring(request.getContextPath().length());
        // 快速匹配，保证性能
        if (CollUtil.contains(tenantProperties.getIgnoreUrls(), apiUri)
                || CollUtil.contains(ignoreUrls, apiUri)) {
            return true;
        }
        // 逐个 Ant 路径匹配
        for (String url : tenantProperties.getIgnoreUrls()) {
            if (pathMatcher.match(url, apiUri)) {
                return true;
            }
        }
        for (String url : ignoreUrls) {
            if (pathMatcher.match(url, apiUri)) {
                return true;
            }
        }
        return false;
    }

}
```

### 安全检查流程

```
HTTP 请求进入
    │
    ▼
TenantContextWebFilter（已执行，tenantId 已设置）
    │
    ▼
TenantSecurityWebFilter
    │
    ├── 用户已登录？
    │     ├── tenantId == null ──→ 使用登录用户的 tenantId（兜底）
    │     └── tenantId != 用户的 tenantId ──→ 越权！返回 403
    │
    ├── 是忽略 URL？
    │     └── 是 ──→ 如果没传 tenantId，设置 ignore=true，放行
    │
    └── 非忽略 URL
          ├── tenantId == null ──→ 返回 400（缺少租户标识）
          └── tenantId 有效 ──→ 调用 validTenant() 校验租户状态
                ├── 租户被禁用/到期 ──→ 返回错误
                └── 租户合法 ──→ 放行
```

### 越权防护的核心逻辑

以电商系统为例：商家 A（tenantId=100）登录后台，如果他在请求头中伪造 `tenant-id: 200` 想查看商家 B 的商品数据，Filter 会发现：

```java
user.getTenantId() == 100  ≠  TenantContextHolder.getTenantId() == 200
```

直接返回 403 拒绝，商家 B 的数据完全不会被查询到。

### ignoreUrls 的两种来源

1. **配置文件**：`yudao.tenant.ignore-urls` 中配置的 URL，如支付回调、短信回调等外部接口
2. **注解扫描**：自动配置时扫描所有 Controller 方法上的 `@TenantIgnore` 注解，将其 URL 加入忽略集合

```java
// 自动配置中的扫描逻辑
private Set<String> getTenantIgnoreUrls() {
    Set<String> ignoreUrls = new HashSet<>();
    RequestMappingHandlerMapping requestMappingHandlerMapping = (RequestMappingHandlerMapping)
            applicationContext.getBean("requestMappingHandlerMapping");
    Map<RequestMappingInfo, HandlerMethod> handlerMethodMap = requestMappingHandlerMapping.getHandlerMethods();
    for (Map.Entry<RequestMappingInfo, HandlerMethod> entry : handlerMethodMap.entrySet()) {
        HandlerMethod handlerMethod = entry.getValue();
        if (!handlerMethod.hasMethodAnnotation(TenantIgnore.class)           // 方法级
            && !handlerMethod.getBeanType().isAnnotationPresent(TenantIgnore.class)) { // 类级
            continue;
        }
        // 添加到忽略的 URL 中
        if (entry.getKey().getPatternsCondition() != null) {
            ignoreUrls.addAll(entry.getKey().getPatternsCondition().getPatterns());
        }
        if (entry.getKey().getPathPatternsCondition() != null) {
            ignoreUrls.addAll(convertList(
                entry.getKey().getPathPatternsCondition().getPatterns(), PathPattern::getPatternString));
        }
    }
    return ignoreUrls;
}
```

## 第三步：异步线程中的上下文传播

### 为什么需要关注异步传播？

在电商系统中，很多操作是异步的：

- 用户下单后，异步发送订单确认邮件
- 定时任务遍历所有租户执行统计
- 使用 `@Async` 异步记录操作日志

如果用普通的 `ThreadLocal`，主线程设置了 `tenantId=100`，提交到线程池后，工作线程的 `TenantContextHolder.getTenantId()` 返回 `null`，SQL 查询就丢失了租户过滤条件。

### 解决方案：TransmittableThreadLocal

`TenantContextHolder` 已经使用了阿里的 `TransmittableThreadLocal`（TTL），它在线程池提交任务时会自动捕获父线程的上下文，在工作线程执行时自动回放。

使用 TTL 需要配合 `TtlExecutors` 包装线程池：

```java
// 普通线程池
ExecutorService executor = Executors.newFixedThreadPool(10);

// 包装后支持 TTL 上下文传播
ExecutorService ttlExecutor = TtlExecutors.getTtlExecutorService(executor);
```

### 手动包装 Runnable 的方式

如果不想依赖 TTL 的线程池包装，也可以手动在提交任务前捕获上下文：

```java
Long currentTenantId = TenantContextHolder.getTenantId();
executor.submit(() -> {
    TenantContextHolder.setTenantId(currentTenantId);
    try {
        // 执行业务逻辑
        orderService.sendConfirmEmail(orderId);
    } finally {
        TenantContextHolder.clear();
    }
});
```

但这种方式侵入性太强，每个异步调用都要手动处理。推荐使用 TTL 一劳永逸。

### 定时任务的多租户执行

对于需要遍历所有租户执行的定时任务，框架提供了 `@TenantJob` 注解 + `TenantJobAspect` 切面：

源码位置：`yudao-framework/yudao-spring-boot-starter-biz-tenant/src/main/java/cn/iocoder/yudao/framework/tenant/core/job/TenantJobAspect.java`

```java
@Aspect
@RequiredArgsConstructor
@Slf4j
public class TenantJobAspect {

    private final TenantFrameworkService tenantFrameworkService;

    @Around("@annotation(tenantJob)")
    public void around(ProceedingJoinPoint joinPoint, TenantJob tenantJob) {
        // 获得租户列表
        List<Long> tenantIds = tenantFrameworkService.getTenantIds();
        if (CollUtil.isEmpty(tenantIds)) {
            return;
        }

        // 逐个租户，执行 Job
        Map<Long, String> results = new ConcurrentHashMap<>();
        AtomicBoolean success = new AtomicBoolean(true);
        XxlJobContext xxlJobContext = XxlJobContext.getXxlJobContext();
        tenantIds.parallelStream().forEach(tenantId -> {
            TenantUtils.execute(tenantId, () -> {
                try {
                    XxlJobContext.setXxlJobContext(xxlJobContext);
                    Object result = joinPoint.proceed();
                    results.put(tenantId, StrUtil.toStringOrEmpty(result));
                } catch (Throwable e) {
                    results.put(tenantId, ExceptionUtil.getRootCauseMessage(e));
                    success.set(false);
                    XxlJobHelper.log(StrUtil.format("[多租户({}) 执行任务({})，发生异常：{}]",
                            tenantId, joinPoint.getSignature(), ExceptionUtils.getStackTrace(e)));
                }
            });
        });
        // 记录执行结果
        if (success.get()) {
            XxlJobHelper.handleSuccess(JsonUtils.toJsonString(results));
        } else {
            XxlJobHelper.handleFail(JsonUtils.toJsonString(results));
        }
    }

}
```

业务代码使用方式：

```java
@TenantJob
@XxlJob("orderTimeoutJob")
public void orderTimeoutJob() {
    // 查询当前租户下超时未支付的订单并关闭
    List<OrderDO> orders = orderService.getTimeoutOrders();
    orders.forEach(order -> orderService.closeOrder(order.getId()));
    // 不需要关心租户编号，TenantJobAspect 会自动遍历每个租户执行
}
```

## 第四步：MQ 消息中的租户传播

### 问题

在微服务架构中，服务 A 通过 MQ 发消息给服务 B。如果服务 A 当前的租户是 100，消息到达服务 B 消费时，`TenantContextHolder` 是空的（因为是不同的线程甚至不同的进程），SQL 查询就会丢失租户过滤。

### 解决方案

框架为四种 MQ 都提供了租户传播机制，原理相同：**发送时把 tenantId 写入消息 Header，消费时从 Header 中读取并设置到 TenantContextHolder**。

以 Redis 消息队列为例：

源码位置：`yudao-framework/yudao-spring-boot-starter-biz-tenant/src/main/java/cn/iocoder/yudao/framework/tenant/core/mq/redis/TenantRedisMessageInterceptor.java`

```java
package cn.iocoder.yudao.framework.tenant.core.mq.redis;

import cn.hutool.core.util.StrUtil;
import cn.iocoder.yudao.framework.mq.redis.core.interceptor.RedisMessageInterceptor;
import cn.iocoder.yudao.framework.mq.redis.core.message.AbstractRedisMessage;
import cn.iocoder.yudao.framework.tenant.core.context.TenantContextHolder;

import static cn.iocoder.yudao.framework.web.core.util.WebFrameworkUtils.HEADER_TENANT_ID;

/**
 * 多租户 Redis 消息拦截器
 * 1. Producer 发送消息时，将 tenantId 添加到消息的 Header 中
 * 2. Consumer 消费消息时，将消息 Header 的 tenantId 添加到 TenantContextHolder 中
 */
public class TenantRedisMessageInterceptor implements RedisMessageInterceptor {

    @Override
    public void sendMessageBefore(AbstractRedisMessage message) {
        Long tenantId = TenantContextHolder.getTenantId();
        if (tenantId != null) {
            message.addHeader(HEADER_TENANT_ID, tenantId.toString());
        }
    }

    @Override
    public void consumeMessageBefore(AbstractRedisMessage message) {
        String tenantIdStr = message.getHeader(HEADER_TENANT_ID);
        if (StrUtil.isNotEmpty(tenantIdStr)) {
            TenantContextHolder.setTenantId(Long.valueOf(tenantIdStr));
        }
    }

    @Override
    public void consumeMessageAfter(AbstractRedisMessage message) {
        // Consumer 是一个逻辑的入口，不考虑原本上下文就存在租户编号的情况
        TenantContextHolder.clear();
    }

}
```

## 第五步：@TenantIgnore 注解的 AOP 处理

源码位置：`yudao-framework/yudao-spring-boot-starter-biz-tenant/src/main/java/cn/iocoder/yudao/framework/tenant/core/aop/TenantIgnore.java`

```java
package cn.iocoder.yudao.framework.tenant.core.aop;

import java.lang.annotation.*;

/**
 * 忽略租户，标记指定方法不进行租户的自动过滤
 *
 * 特殊：
 * 1、如果添加到 Controller 类上，则该 URL 自动添加到 ignoreUrls 中
 * 2、如果添加到 DO 实体类上，则它对应的表名"相当于"自动添加到 ignoreTables 中
 */
@Target({ElementType.METHOD, ElementType.TYPE})
@Retention(RetentionPolicy.RUNTIME)
@Inherited
public @interface TenantIgnore {

    /**
     * 是否开启忽略租户，默认为 true 开启
     * 支持 Spring EL 表达式，如果返回 true 则满足条件，进行租户的忽略
     */
    String enable() default "true";

}
```

### AOP 切面实现

源码位置：`yudao-framework/yudao-spring-boot-starter-biz-tenant/src/main/java/cn/iocoder/yudao/framework/tenant/core/aop/TenantIgnoreAspect.java`

```java
package cn.iocoder.yudao.framework.tenant.core.aop;

import cn.iocoder.yudao.framework.common.util.spring.SpringExpressionUtils;
import cn.iocoder.yudao.framework.tenant.core.context.TenantContextHolder;
import cn.iocoder.yudao.framework.tenant.core.util.TenantUtils;
import lombok.extern.slf4j.Slf4j;
import org.aspectj.lang.ProceedingJoinPoint;
import org.aspectj.lang.annotation.Around;
import org.aspectj.lang.annotation.Aspect;

/**
 * 忽略多租户的 Aspect，基于 @TenantIgnore 注解实现
 */
@Aspect
@Slf4j
public class TenantIgnoreAspect {

    @Around("@annotation(tenantIgnore)")
    public Object around(ProceedingJoinPoint joinPoint, TenantIgnore tenantIgnore) throws Throwable {
        Boolean oldIgnore = TenantContextHolder.isIgnore();
        try {
            // 计算条件，满足的情况下，才进行忽略
            Object enable = SpringExpressionUtils.parseExpression(tenantIgnore.enable());
            if (Boolean.TRUE.equals(enable)) {
                TenantContextHolder.setIgnore(true);
            }
            // 执行逻辑
            return joinPoint.proceed();
        } finally {
            TenantContextHolder.setIgnore(oldIgnore);
        }
    }

}
```

### @TenantIgnore 的三种使用场景

**场景一：标记在 Service 方法上**

某个方法需要查询所有租户的数据，比如统计全平台的销售总额：

```java
@Service
public class OrderServiceImpl {

    @TenantIgnore
    public Long getTotalSales() {
        // SQL 不会追加 WHERE tenant_id，查询所有租户的订单
        return orderMapper.selectTotalSales();
    }
}
```

**场景二：标记在 Controller 类上**

某个 Controller 的所有接口都不需要租户过滤，比如支付回调接口：

```java
@RestController
@RequestMapping("/pay/notify")
@TenantIgnore
public class PayNotifyController {

    @PostMapping("/alipay")
    public String alipayNotify(@RequestBody AlipayNotifyDTO dto) {
        // 支付宝回调不带 tenant-id，这里不需要租户过滤
        return "success";
    }
}
```

被标记的 Controller 的 URL 会被自动扫描加入 `ignoreUrls` 集合（前面 `getTenantIgnoreUrls()` 方法的逻辑）。

**场景三：标记在 DO 实体类上**

某个表的数据是所有租户共享的，比如系统配置表：

```java
@TableName("system_config")
@TenantIgnore
public class SystemConfigDO extends BaseDO {
    private Long id;
    private String key;
    private String value;
    // 不继承 TenantBaseDO，没有 tenantId 字段
}
```

`TenantDatabaseInterceptor.computeIgnoreTable()` 会检测到 `@TenantIgnore` 注解，将该表加入忽略列表。

### 条件式忽略

`@TenantIgnore` 支持 Spring EL 表达式，可以实现条件式忽略：

```java
@TenantIgnore(enable = "#userId != null")
public OrderDO getOrder(Long userId) {
    // 如果传入了 userId，忽略租户过滤（查个人订单）；否则按租户过滤
    return orderMapper.selectByUserId(userId);
}
```

## 小结

本章实现了多租户的安全防护与上下文传播机制：

| 组件 | 职责 |
|------|------|
| `TenantContextWebFilter` | 从 HTTP 请求头提取 tenantId，写入 TenantContextHolder |
| `TenantSecurityWebFilter` | 防越权：校验登录用户的 tenantId 与请求的 tenantId 一致性 |
| `TenantRequestInterceptor` | Feign 调用时自动将 tenantId 放入请求头 |
| `TenantRedisMessageInterceptor` | Redis MQ 发送时写入 Header，消费时恢复到 TenantContextHolder |
| `TenantRabbitMQMessagePostProcessor` | RabbitMQ 的租户传播实现 |
| `TenantRocketMQSendMessageHook` | RocketMQ 的租户传播实现 |
| `TenantJobAspect` | 定时任务遍历所有租户执行 |
| `TenantIgnore` + `TenantIgnoreAspect` | 注解式忽略租户过滤，支持条件表达式 |
