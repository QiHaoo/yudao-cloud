# 08-精简汇总

本文是多租户组件功能设计的精简回顾，适合在完成全部章节后快速复习，或在实现阶段作为速查手册。

## 能力速查表

### 核心能力

| 能力 | 实现方式 | 关键类/配置 |
|------|---------|------------|
| SQL 自动追加 `tenant_id` | MyBatis-Plus TenantLineInnerInterceptor | `TenantDatabaseInterceptor` |
| 表级豁免 | 三级检查：全局忽略 → 配置忽略 → 反射检查 | `ignoreTable()` |
| 请求头读取 | Filter 从 `tenant-id` 头解析 | `TenantContextWebFilter` |
| 跨线程传播 | TransmittableThreadLocal | `TenantContextHolder` |
| 跨服务传播 | Feign RequestInterceptor | `TenantRequestInterceptor` |
| 越权防护 | 校验用户所属租户与请求租户是否一致 | `TenantSecurityWebFilter` |
| 租户有效性校验 | 检查禁用/过期，Guava 缓存 | `TenantFrameworkServiceImpl` |
| 缓存隔离 | Redis Cache key 自动拼接租户后缀 | `TenantRedisCacheManager` |
| 定时任务多租户 | 注解驱动，自动遍历所有租户 | `@TenantJob` + `TenantJobAspect` |
| 编程式切换 | 工具方法切换/忽略租户上下文 | `TenantUtils` |
| 管理员跨租户查看 | visit-tenant-id 请求头 + 权限校验 | `TenantVisitContextInterceptor` |

### 配置项速查

| 配置项 | 作用 | 默认值 |
|--------|------|--------|
| `yudao.tenant.enable` | 全局开关 | `true` |
| `yudao.tenant.ignore-urls` | 豁免租户的 URL 列表 | 空 |
| `yudao.tenant.ignore-visit-urls` | 不允许跨租户访问的 URL | 空 |
| `yudao.tenant.ignore-tables` | 豁免租户的表名列表 | 空 |
| `yudao.tenant.ignore-caches` | 豁免租户的缓存名列表 | 空 |

### 注解速查

| 注解 | 标注位置 | 作用 |
|------|---------|------|
| `@TenantIgnore` | Controller 方法 | 该接口不需要租户标识，自动加入 ignoreUrls |
| `@TenantIgnore` | Controller 类 | 该类所有接口不需要租户标识 |
| `@TenantIgnore` | DO 实体类 | 该表不需要 tenant_id 过滤 |
| `@TenantIgnore(enable = "SpEL")` | 方法 | 基于 SpEL 表达式动态决定是否忽略 |
| `@TenantJob` | XXL-Job 方法 | 自动为每个租户执行一次任务 |

### 请求头速查

| 请求头 | 方向 | 作用 |
|--------|------|------|
| `tenant-id` | 前端 → 后端 | 携带当前租户编号 |
| `tenant-id` | 服务 A → 服务 B（Feign） | 跨服务传播租户编号 |
| `visit-tenant-id` | 前端 → 后端 | 管理员切换到目标租户查看数据 |

## 设计概念精要

### 整体架构

```
请求进入
  │
  ├── TenantContextWebFilter ──→ 读取 tenant-id，写入上下文
  ├── TenantSecurityWebFilter ──→ 越权校验 + 租户有效性校验
  ├── TenantVisitContextInterceptor ──→ 可选：切换到目标租户
  │
  ├── Controller → Service → Mapper ──→ 业务代码（无感知）
  │
  ├── TenantDatabaseInterceptor ──→ SQL 自动追加 WHERE tenant_id = ?
  ├── TenantRedisCacheManager ──→ 缓存 key 自动拼接 :tenantId
  ├── TenantRequestInterceptor ──→ Feign 调用自动传播 tenant-id
  │
  └── TenantContextWebFilter (finally) ──→ 清理上下文
```

### 行级隔离的三级豁免决策树

```
需要忽略这张表吗？
  │
  ├── TenantContextHolder.isIgnore() == true？ ──→ 是，忽略
  │
  ├── 表名在 ignoreTables 配置中？ ──→ 是，忽略
  │
  └── 反射检查：
        │
        ├── 不是项目中的表？ ──→ 忽略
        ├── 继承了 TenantBaseDO？ ──→ 不忽略（需要隔离）
        └── 有 @TenantIgnore 注解？ ──→ 是，忽略
```

### 上下文传播的核心机制

| 层次 | 机制 | 存储位置 |
|------|------|---------|
| 跨请求 | HTTP 请求头 `tenant-id` | HTTP Header |
| 请求内 | TenantContextHolder (ThreadLocal) | 线程上下文 |
| 跨线程 | TransmittableThreadLocal | 自动捕获/回放 |
| 跨服务 | Feign RequestInterceptor | HTTP Header |

### 安全防线

```
防线 1：TenantContextWebFilter
  └── 从请求头读取 tenant-id，保证上下文有值

防线 2：TenantSecurityWebFilter
  ├── 用户所属租户 ≠ 请求租户 ──→ 拒绝（防越权）
  ├── 非豁免 URL + 无 tenant-id ──→ 拒绝（防遗漏）
  └── 租户被禁用/过期 ──→ 拒绝（防失效）

防线 3：TenantDatabaseInterceptor
  └── SQL 层面强制过滤，即使业务代码有 Bug 也只看到自己的数据
```

### 缓存策略

| 缓存对象 | 技术 | 过期时间 | 刷新方式 |
|---------|------|---------|---------|
| 租户列表 | Guava LoadingCache | 1 分钟 | 异步刷新 |
| 租户有效性 | Guava LoadingCache | 1 分钟 | 异步刷新 |
| Redis 业务缓存 | TenantRedisCacheManager | 取决于业务配置 | key 拼接租户后缀 |

### TenantUtils 工具方法

| 方法 | 作用 | 典型场景 |
|------|------|---------|
| `execute(tenantId, Runnable)` | 在指定租户上下文中执行 | 定时任务遍历租户 |
| `execute(tenantId, Callable)` | 同上，带返回值 | 跨租户数据查询 |
| `executeIgnore(Runnable)` | 忽略租户执行 | 全局统计、全局配置读取 |
| `executeIgnore(Callable)` | 同上，带返回值 | 全局统计返回结果 |

### 设计取舍速查

| 决策 | 选择 | 放弃 | 原因 |
|------|------|------|------|
| 隔离方案 | 共享 Schema 行级隔离 | 独立数据库/Schema | 成本最低，运维最简单 |
| 上下文存储 | TransmittableThreadLocal | 普通 ThreadLocal | 需要跨线程传播 |
| 缓存技术 | Guava LoadingCache | Redis | 无外部依赖，纳秒级延迟 |
| 定时任务 | 并行执行 | 串行执行 | 提高效率，低峰期执行 |
| "沉默的表" | 默认忽略 | 默认不忽略 | 保守策略，不误伤外部表 |
| 缓存过期 | 异步刷新 | 同步加载 | 避免请求阻塞 |

### 已知局限

| 局限 | 说明 | 应对方案 |
|------|------|---------|
| 跨租户查询 | 需显式使用 TenantUtils | 业务层自行处理 |
| 不支持分库 | 只支持行级隔离 | 需要扩展数据源路由 |
| 缓存一致性 | 禁用租户最多 1 分钟延迟 | 业务可接受，紧急情况重启 |
| 原生 SQL | @Select 等不自动改写 | 尽量用 Wrapper 查询 |
