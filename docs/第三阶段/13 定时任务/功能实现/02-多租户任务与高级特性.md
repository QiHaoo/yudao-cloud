# 02-多租户任务与高级特性

## 场景引入

假设你正在为一个 SaaS 电商平台开发定时任务，这个平台服务多个商家（租户）：

```
场景：每天凌晨 2 点执行"订单统计报表生成"任务

问题：平台有 100 个商家（租户），每个商家的数据需要隔离
      - 商家 A 的订单统计：只能统计商家 A 的数据
      - 商家 B 的订单统计：只能统计商家 B 的数据

期望：一个定时任务能自动遍历所有租户，分别执行各自的统计逻辑
```

本章我们实现 `@TenantJob` 注解和切面，让定时任务支持多租户自动遍历执行。

## 第一步：实现 @TenantJob 注解

注解非常简洁，只需要标记方法即可。

### TenantJob

> 源文件路径：`yudao-framework/yudao-spring-boot-starter-biz-tenant/src/main/java/cn/iocoder/yudao/framework/tenant/core/job/TenantJob.java`

```java
package cn.iocoder.yudao.framework.tenant.core.job;

import java.lang.annotation.ElementType;
import java.lang.annotation.Retention;
import java.lang.annotation.RetentionPolicy;
import java.lang.annotation.Target;

/**
 * 多租户 Job 注解
 */
@Target({ElementType.METHOD})
@Retention(RetentionPolicy.RUNTIME)
public @interface TenantJob {
}
```

**设计说明**：

- `@Target(ElementType.METHOD)`：只能标注在方法上
- `@Retention(RetentionPolicy.RUNTIME)`：运行时保留，供 AOP 切面识别
- **无需属性**：所有逻辑由切面统一处理，保持注解简洁

## 第二步：实现 TenantJobAspect 切面

切面是多租户任务的核心，它会在任务执行前遍历所有租户，为每个租户设置上下文后执行任务。

### TenantJobAspect

> 源文件路径：`yudao-framework/yudao-spring-boot-starter-biz-tenant/src/main/java/cn/iocoder/yudao/framework/tenant/core/job/TenantJobAspect.java`

```java
package cn.iocoder.yudao.framework.tenant.core.job;

import cn.hutool.core.collection.CollUtil;
import cn.hutool.core.exceptions.ExceptionUtil;
import cn.hutool.core.util.StrUtil;
import cn.iocoder.yudao.framework.common.util.json.JsonUtils;
import cn.iocoder.yudao.framework.tenant.core.service.TenantFrameworkService;
import cn.iocoder.yudao.framework.tenant.core.util.TenantUtils;
import com.xxl.job.core.context.XxlJobContext;
import com.xxl.job.core.context.XxlJobHelper;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.apache.commons.lang3.exception.ExceptionUtils;
import org.aspectj.lang.ProceedingJoinPoint;
import org.aspectj.lang.annotation.Around;
import org.aspectj.lang.annotation.Aspect;

import java.util.List;
import java.util.Map;
import java.util.concurrent.ConcurrentHashMap;
import java.util.concurrent.atomic.AtomicBoolean;

/**
 * 多租户 JobHandler AOP
 *
 * 任务执行时，会按照租户逐个执行 Job 的逻辑
 * 注意：需要保证 JobHandler 的幂等性，因为某个租户失败重试时，之前成功的租户也会再次执行
 *
 * @author 芋道源码
 */
@Aspect
@RequiredArgsConstructor
@Slf4j
public class TenantJobAspect {

    private final TenantFrameworkService tenantFrameworkService;

    @Around("@annotation(tenantJob)")
    public void around(ProceedingJoinPoint joinPoint, TenantJob tenantJob) {
        // 1. 获取所有租户 ID
        List<Long> tenantIds = tenantFrameworkService.getTenantIds();
        if (CollUtil.isEmpty(tenantIds)) {
            return;
        }

        // 2. 准备执行结果收集器
        Map<Long, String> results = new ConcurrentHashMap<>();
        AtomicBoolean success = new AtomicBoolean(true);
        XxlJobContext xxlJobContext = XxlJobContext.getXxlJobContext();

        // 3. 并行遍历租户执行任务
        tenantIds.parallelStream().forEach(tenantId -> {
            TenantUtils.execute(tenantId, () -> {
                try {
                    // 设置 XXL-Job 上下文（确保日志等正常工作）
                    XxlJobContext.setXxlJobContext(xxlJobContext);
                    // 执行实际的 Job 逻辑
                    Object result = joinPoint.proceed();
                    results.put(tenantId, StrUtil.toStringOrEmpty(result));
                } catch (Throwable e) {
                    // 记录失败信息
                    results.put(tenantId, ExceptionUtil.getRootCauseMessage(e));
                    success.set(false);
                    // 打印异常日志
                    XxlJobHelper.log(StrUtil.format(
                        "[多租户({}) 执行任务({})，发生异常：{}]",
                        tenantId, joinPoint.getSignature(),
                        ExceptionUtils.getStackTrace(e)));
                }
            });
        });

        // 4. 汇总执行结果
        if (success.get()) {
            XxlJobHelper.handleSuccess(JsonUtils.toJsonString(results));
        } else {
            XxlJobHelper.handleFail(JsonUtils.toJsonString(results));
        }
    }

}
```

**执行流程图解**：

```
@TenantJob 标注的 JobHandler 方法被调用
    │
    ├─ 1. 获取所有租户 ID
    │     └─ tenantFrameworkService.getTenantIds()
    │           → SELECT id FROM system_tenant WHERE deleted = 0
    │           → [1, 2, 3, ... 100]
    │
    ├─ 2. 并行遍历租户执行
    │     ┌─────────────────────────────────────────────────────┐
    │     │  tenantId=1        tenantId=2        tenantId=3     │
    │     │      │                 │                 │          │
    │     │      ▼                 ▼                 ▼          │
    │     │  TenantUtils       TenantUtils       TenantUtils    │
    │     │  .execute()        .execute()        .execute()     │
    │     │      │                 │                 │          │
    │     │      ▼                 ▼                 ▼          │
    │     │  设置租户上下文     设置租户上下文     设置租户上下文  │
    │     │  TenantContext     TenantContext     TenantContext   │
    │     │  .setTenantId(1)  .setTenantId(2)  .setTenantId(3) │
    │     │      │                 │                 │          │
    │     │      ▼                 ▼                 ▼          │
    │     │  执行业务逻辑       执行业务逻辑       执行业务逻辑   │
    │     │  (SQL自动带        (SQL自动带         (SQL自动带     │
    │     │   tenant_id=1)      tenant_id=2)       tenant_id=3) │
    │     │      │                 │                 │          │
    │     │      ▼                 ▼                 ▼          │
    │     │  收集结果           收集结果           收集结果       │
    │     │  results[1]=ok     results[2]=ok     results[3]=fail│
    │     └─────────────────────────────────────────────────────┘
    │
    └─ 5. 汇总结果
          ├─ 全部成功 → XxlJobHelper.handleSuccess(json)
          └─ 存在失败 → XxlJobHelper.handleFail(json)
```

**关键设计点**：

### 1. TenantUtils.execute() 的作用

```java
TenantUtils.execute(tenantId, () -> {
    // 这里的代码执行时，TenantContext 中已经设置了 tenantId
    // 所有数据库操作都会自动追加 WHERE tenant_id = ?
});
```

`TenantUtils.execute()` 内部会：
- 设置当前线程的租户上下文：`TenantContext.setTenantId(tenantId)`
- 执行传入的 Lambda
- 执行完毕后清除租户上下文：`TenantContext.clear()`

### 2. 并行执行与上下文传递

```java
// 获取当前 XXL-Job 上下文
XxlJobContext xxlJobContext = XxlJobContext.getXxlJobContext();

// 并行执行时，每个线程需要重新设置上下文
tenantIds.parallelStream().forEach(tenantId -> {
    TenantUtils.execute(tenantId, () -> {
        // 在子线程中重新设置 XXL-Job 上下文
        XxlJobContext.setXxlJobContext(xxlJobContext);
        // 执行任务...
    });
});
```

**为什么需要手动设置 XxlJobContext？**

- `parallelStream()` 使用 ForkJoinPool，子线程无法继承主线程的上下文
- XXL-Job 的日志记录依赖 `XxlJobContext`，不设置会导致日志丢失

### 3. 结果收集与错误处理

```java
// 使用 ConcurrentHashMap 保证线程安全
Map<Long, String> results = new ConcurrentHashMap<>();

// 使用 AtomicBoolean 标记整体成功/失败
AtomicBoolean success = new AtomicBoolean(true);

// 每个租户执行后记录结果
results.put(tenantId, StrUtil.toStringOrEmpty(result));  // 成功
results.put(tenantId, ExceptionUtil.getRootCauseMessage(e));  // 失败
success.set(false);  // 标记失败
```

**执行结果示例**：

```json
{
    "1": "ok",
    "2": "ok",
    "3": "NullPointerException: ...",
    "4": "ok"
}
```

## 第三步：在业务代码中使用

### 1. 定义多租户 JobHandler

```java
@Component
public class OrderStatisticsJob {

    @Resource
    private OrderStatisticsService orderStatisticsService;

    /**
     * 每天凌晨 2 点生成前一天的订单统计报表
     * cron: 0 0 2 * * ?
     */
    @XxlJob("orderStatisticsJob")
    @TenantJob  // 关键：标注 @TenantJob
    public void execute() {
        // 这里的代码会自动遍历所有租户执行
        // 每次执行时，TenantContext 中已有正确的 tenantId
        // 所有 SQL 查询都会自动追加 WHERE tenant_id = ?
        orderStatisticsService.generateYesterdayReport();
    }
}
```

### 2. 业务 Service 实现

```java

        );

```

### 3. 执行效果

假设平台有 3 个商家（租户）：

```
任务触发：orderStatisticsJob 执行
    │
    ├─ 租户 1（商家 A）
    │     SQL: SELECT * FROM trade_order WHERE tenant_id = 1 AND date = '2024-01-09'
    │     结果: 生成报表成功
```

## 高级特性：任务失败重试与告警

### 1. XXL-Job 内置重试机制

XXL-Job 支持任务失败自动重试，在 Admin 管理界面配置：

```
任务配置：
  - 失败重试次数：3
  - 重试间隔：60 秒
```

当任务执行失败时（调用 `XxlJobHelper.handleFail()`），XXL-Job 会自动重试。

### 2. 多租户任务的重试注意事项

**重要**：`@TenantJob` 标注的任务重试时，**所有租户都会重新执行**，不仅是失败的租户。

```
第一次执行：
  租户 1 → 成功
  租户 2 → 失败
  租户 3 → 成功
  结果: handleFail

重试执行（XXL-Job 自动触发）：
  租户 1 → 成功（再次执行）
  租户 2 → 成功（这次成功了）
  租户 3 → 成功（再次执行）
  结果: handleSuccess
```

**设计要求**：业务代码必须保证**幂等性**，即多次执行结果一致。

```java
// 幂等设计示例：先查询再插入
@Override
public void generateYesterdayReport() {
    // 1. 检查是否已生成
    ReportDO existing = reportMapper.selectOne(
        new LambdaQueryWrapperX<ReportDO>()
            .eq(ReportDO::getTenantId, TenantContext.getTenantId())
            .eq(ReportDO::getDate, LocalDate.now().minusDays(1))
    );
    if (existing != null) {
        return; // 已存在，跳过
    }

    // 2. 生成报表
    // ... 业务逻辑

    // 3. 保存报表
    reportMapper.insert(report);
}
```

### 3. 告警配置

XXL-Job 支持任务失败告警，可在 Admin 界面配置：

```
告警方式：
  - 邮件告警
  - 钉钉告警
  - 企业微信告警

告警配置：
  - 任务失败时发送告警
  - 告警接收人：运维团队、业务负责人
```

## 总结

本章我们实现了多租户定时任务的核心能力：

| 组件 | 文件 | 作用 |
|------|------|------|
| `@TenantJob` | `TenantJob.java` | 标记多租户任务方法 |
| `TenantJobAspect` | `TenantJobAspect.java` | AOP 切面，自动遍历租户执行 |

**关键设计决策**：

1. **并行执行**：使用 `parallelStream()` 并行处理多个租户，提高执行效率
2. **上下文传递**：手动设置 `XxlJobContext`，确保日志正常记录
3. **结果收集**：使用 `ConcurrentHashMap` + `AtomicBoolean` 收集并行执行结果
4. **幂等要求**：重试时所有租户重新执行，业务代码必须保证幂等

**使用示例**：

```java
@XxlJob("myJob")
@TenantJob  // 只需加这个注解
public void execute() {
    // 业务代码自动按租户隔离执行
    // SQL 自动追加 WHERE tenant_id = ?
}
```
