# 03-测试支持与源码索引

## 测试支持

在编写包含定时任务的业务逻辑时，我们面临几个测试挑战：

1. **XXL-Job 依赖**：JobHandler 依赖 `XxlJobHelper` 等 XXL-Job 组件
2. **多租户上下文**：`@TenantJob` 依赖租户服务和上下文
3. **定时触发**：任务由 XXL-Job Admin 调度，测试时无法自动触发

### 测试策略概览

| 测试类型 | 测试对象 | 测试方式 | 复杂度 |
|---------|---------|---------|-------|
| 单元测试 | 业务 Service | Mock 依赖，直接调用 | 低 |
| 集成测试 | JobHandler | Mock XxlJobHelper | 中 |
| 多租户测试 | @TenantJob | Mock TenantFrameworkService | 中 |
| 端到端测试 | 完整流程 | 启动 XXL-Job Admin | 高 |

### 1. 单元测试：业务 Service

业务 Service 是定时任务的核心逻辑所在，应该独立于 XXL-Job 进行测试。

```java
@SpringBootTest
class OrderStatisticsServiceTest {

    @Resource
    private OrderStatisticsService orderStatisticsService;

    @Resource
    private OrderMapper orderMapper;

    @Test
    void testGenerateYesterdayReport() {
        // Given: 准备测试数据
        // 使用 @Sql 或手动插入测试订单
        OrderDO testOrder = createTestOrder(LocalDate.now().minusDays(1));
        orderMapper.insert(testOrder);

        // When: 执行业务逻辑
        orderStatisticsService.generateYesterdayReport();

        // Then: 验证结果
        ReportDO report = reportMapper.selectOne(
            new LambdaQueryWrapperX<ReportDO>()
                .eq(ReportDO::getDate, LocalDate.now().minusDays(1))
        );
        assertThat(report).isNotNull();
        assertThat(report.getTotalOrders()).isEqualTo(1);
    }

    private OrderDO createTestOrder(LocalDate date) {
        OrderDO order = new OrderDO();
        order.setDate(date);
        order.setStatus(OrderStatus.PAID);
        order.setTotalAmount(100L);
        return order;
    }
}
```

### 2. 集成测试：JobHandler

测试 JobHandler 时，需要 Mock XXL-Job 的上下文。

```java
@SpringBootTest
class OrderStatisticsJobTest {

    @Resource
    private OrderStatisticsJob orderStatisticsJob;

    @Test
    void testExecute() {
        // Given: 准备测试数据和 Mock 上下文
        // 注意：XxlJobHelper 依赖 XxlJobContext，需要在测试前设置
        XxlJobContext context = new XxlJobContext();
        context.setJobParam("test");
        XxlJobContext.setXxlJobContext(context);

        // When: 执行 JobHandler
        orderStatisticsJob.execute();

        // Then: 验证执行结果
        // 检查数据库中的报表记录
        ReportDO report = reportMapper.selectOne(/* ... */);
        assertThat(report).isNotNull();
    }
}
```

### 3. 多租户任务测试

测试 `@TenantJob` 标注的任务时，需要 Mock 租户服务。

```java
@SpringBootTest
class TenantJobTest {

    @Resource
    private OrderStatisticsJob orderStatisticsJob;

    @MockBean
    private TenantFrameworkService tenantFrameworkService;

    @Test
    void testTenantJobExecute() {
        // Given: Mock 返回多个租户
        when(tenantFrameworkService.getTenantIds())
            .thenReturn(Arrays.asList(1L, 2L, 3L));

        // 准备各租户的测试数据
        insertTestOrder(1L, LocalDate.now().minusDays(1));
        insertTestOrder(2L, LocalDate.now().minusDays(1));
        // 租户 3 没有订单

        // 设置 XXL-Job 上下文
        XxlJobContext context = new XxlJobContext();
        XxlJobContext.setXxlJobContext(context);

        // When: 执行多租户任务
        orderStatisticsJob.execute();

        // Then: 验证各租户的执行结果
        // 租户 1 的报表
        ReportDO report1 = reportMapper.selectOne(
            new LambdaQueryWrapperX<ReportDO>()
                .eq(ReportDO::getTenantId, 1L)
        );
        assertThat(report1).isNotNull();

        // 租户 2 的报表
        ReportDO report2 = reportMapper.selectOne(
            new LambdaQueryWrapperX<ReportDO>()
                .eq(ReportDO::getTenantId, 2L)
        );
        assertThat(report2).isNotNull();
    }

    private void insertTestOrder(Long tenantId, LocalDate date) {
        // 设置租户上下文
        TenantContext.setTenantId(tenantId);
        try {
            OrderDO order = new OrderDO();
            order.setDate(date);
            order.setStatus(OrderStatus.PAID);
            order.setTotalAmount(100L);
            orderMapper.insert(order);
        } finally {
            TenantContext.clear();
        }
    }
}
```

### 4. 测试配置

在测试环境中，通常需要禁用 XXL-Job 的真实连接。

```yaml
# application-test.yaml
xxl:
  job:
    enabled: false  # 测试时禁用 XXL-Job

# 或者使用 Mock 的 Admin 地址
# xxl:
#   job:
#     admin:
#       addresses: http://mock-xxl-job-admin:9090
```

### 5. 测试工具类建议

可以封装一个测试工具类，简化 XXL-Job 上下文的设置：

```java
public class XxlJobTestUtils {

    /**
     * 设置测试用的 XXL-Job 上下文
     */
    public static XxlJobContext mockXxlJobContext() {
        XxlJobContext context = new XxlJobContext();
        // 设置必要的上下文信息
        XxlJobContext.setXxlJobContext(context);
        return context;
    }

    /**
     * 清除 XXL-Job 上下文
     */
    public static void clearXxlJobContext() {
        XxlJobContext.setXxlJobContext(null);
    }
}
```

使用示例：

```java
@BeforeEach
void setUp() {
    XxlJobTestUtils.mockXxlJobContext();
}

@AfterEach
void tearDown() {
    XxlJobTestUtils.clearXxlJobContext();
}
```

### 业务层使用示例

#### 示例 1：简单的定时任务

```java
@Component
public class TradeOrderAutoCancelJob {

    @Resource
    private TradeOrderUpdateService tradeOrderUpdateService;

    /**
     * 自动取消超时未支付的订单
     * 每小时执行一次：0 0 * * * ?
     */
    @XxlJob("tradeOrderAutoCancelJob")
    public void execute() {
        int count = tradeOrderUpdateService.cancelTimeoutOrders();
        XxlJobHelper.handleSuccess(String.format("取消超时订单 %d 个", count));
    }
}
```

#### 示例 2：多租户定时任务

```java
@Component
public class OrderStatisticsJob {

    @Resource
    private OrderStatisticsService orderStatisticsService;

    /**
     * 每天凌晨 2 点生成订单统计报表（多租户版本）
     * cron: 0 0 2 * * ?
     */
    @XxlJob("orderStatisticsJob")
    @TenantJob  // 关键：标注 @TenantJob 实现多租户遍历
    public void execute() {
        orderStatisticsService.generateYesterdayReport();
    }
}
```

#### 示例 3：带参数的定时任务

```java
@Component
public class SyncInventoryJob {

    @Resource
    private InventorySyncService inventorySyncService;

    /**
     * 同步第三方仓库库存
     * 每 30 分钟执行：0 0/30 * * * ?
     */
    @XxlJob("syncInventoryJob")
    public void execute() {
        // 获取任务参数（在 XXL-Job Admin 中配置）
        String warehouseId = XxlJobHelper.getJobParam();

        if (StrUtil.isNotBlank(warehouseId)) {
            // 同步指定仓库
            inventorySyncService.syncWarehouse(warehouseId);
        } else {
            // 同步所有仓库
            inventorySyncService.syncAllWarehouses();
        }

        XxlJobHelper.handleSuccess("库存同步完成");
    }
}
```

#### 示例 4：分片广播任务

```java
@Component
public class BatchProcessJob {

    @Resource
    private OrderService orderService;

    /**
     * 批量处理订单（分片广播）
     * 多个实例并行执行，每个实例处理一部分数据
     */
    @XxlJob("batchProcessJob")
    public void execute() {
        // 获取分片参数
        int shardIndex = XxlJobHelper.getShardIndex();  // 当前分片序号
        int shardTotal = XxlJobHelper.getShardTotal();  // 分片总数

        // 查询待处理的订单（按分片过滤）
        List<OrderDO> orders = orderService.getPendingOrders(shardIndex, shardTotal);

        // 处理订单
        for (OrderDO order : orders) {
            try {
                orderService.processOrder(order);
            } catch (Exception e) {
                XxlJobHelper.log("处理订单 {} 失败: {}", order.getId(), e.getMessage());
            }
        }

        XxlJobHelper.handleSuccess(String.format("处理订单 %d 个", orders.size()));
    }
}
```

### 配置文件

#### application.yaml 配置

```yaml
xxl:
  job:
    # 是否开启，默认 true
    enabled: true

    # 访问令牌（与 XXL-Job Admin 配置一致）
    access-token: default_token

    # Admin 调度器配置
    admin:
      addresses: http://127.0.0.1:9090/xxl-job-admin

    # 执行器配置
    executor:
      # 应用名（执行器名称，建议与 spring.application.name 一致）
      appname: ${spring.application.name}

      # 执行器 IP（可选，留空自动获取）
      ip: ""

      # 执行器端口（-1 表示随机）
      port: -1

      # 日志存储路径
      log-path: ${user.home}/logs/xxl-job/${spring.application.name}

      # 日志保留天数（-1 表示永久）
      log-retention-days: 30
```

#### 多环境配置示例

```yaml
# application-dev.yaml（开发环境）
xxl:
  job:
    admin:
      addresses: http://localhost:9090/xxl-job-admin

# application-prod.yaml（生产环境）
xxl:
  job:
    admin:
      addresses: http://xxl-job-admin.internal:9090/xxl-job-admin
    executor:
      log-retention-days: 90  # 生产环境保留更长时间
```

#### 禁用配置

```yaml
# application-test.yaml（测试环境）
xxl:
  job:
    enabled: false  # 测试时禁用
```

## 总结

本文档涵盖了定时任务组件的测试支持和源码索引：

**测试支持**：
- 单元测试：独立测试业务 Service，不依赖 XXL-Job
- 集成测试：Mock XxlJobContext 测试 JobHandler
- 多租户测试：Mock TenantFrameworkService 测试 @TenantJob
- 测试配置：通过 `xxl.job.enabled=false` 禁用真实连接

**源码索引**：
- 核心文件按模块分组，★ 标记关键文件
- 业务层使用示例覆盖常见场景
- 配置文件示例支持多环境
- 章节与源码对照表方便快速定位

通过本文档，开发者可以快速定位源码、理解实现细节，并编写高质量的测试用例。
