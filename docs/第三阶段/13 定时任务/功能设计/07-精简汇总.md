# 07-精简汇总

本章是功能设计部分的精简回顾。如果你想快速查阅某个知识点，不需要翻阅前面的长文，看这里就够了。

## 能力与使用速查

### 核心能力一览

| 能力 | 实现方式 | 说明 |
|------|---------|------|
| 分布式调度 | XXL-Job Admin 集中调度 | 防重复执行，只触发一个实例 |
| 自动装配 | `YudaoXxlJobAutoConfiguration` | 引入依赖 + 配置 YAML 即可使用 |
| 多租户遍历 | `@TenantJob` 注解 | 自动遍历所有租户执行任务 |
| 忽略租户 | `@TenantIgnore` 注解 | 明确标记不区分租户的任务 |
| 与 @Scheduled 共存 | `@EnableScheduling` | 简单任务用 @Scheduled，复杂任务用 @XxlJob |
| 异步任务 TTL | `YudaoAsyncAutoConfiguration` | 异步线程自动继承租户上下文 |

### 使用速查

**编写一个普通任务：**

```java
@Component
public class MyJob {

    @XxlJob("myJob")
    public void execute() {
        // 业务逻辑
    }
}
```

**编写一个多租户任务：**

```java
@Component
public class MyTenantJob {

    @XxlJob("myTenantJob")
    @TenantJob
    public String execute() {
        // 自动遍历所有租户执行
        return "执行结果";
    }
}
```

**编写一个忽略租户的任务：**

```java
@Component
public class MyGlobalJob {

    @XxlJob("myGlobalJob")
    @TenantIgnore
    public void execute() {
        // 不区分租户，全局操作
    }
}
```

**配置 YAML：**

```yaml
xxl:
  job:
    admin:
      addresses: http://127.0.0.1:9090/xxl-job-admin
    executor:
      appname: ${spring.application.name}
      logpath: ${user.home}/logs/xxl-job/${spring.application.name}
    accessToken: default_token
```

**本地开发禁用：**

```yaml
# application-local.yaml
xxl:
  job:
    enabled: false
```

### 任务模式选型

```
需要区分不同租户的数据吗？
  │
  ├── 不需要 → @TenantIgnore 或不加注解
  │
  └── 需要 → 业务逻辑幂等吗？
              │
              ├── 是 → @TenantJob
              │
              └── 否 → 自行实现带幂等保护的遍历逻辑
```

### @Scheduled vs @XxlJob 选型

| 特征 | @Scheduled | @XxlJob |
|------|-----------|---------|
| 每个实例都执行 | 是 | 否（只触发一个实例） |
| 需要管理界面 | 否 | 是 |
| 需要失败重试 | 否 | 是 |
| 需要执行日志 | 否 | 是 |
| 需要分片并行 | 否 | 是 |
| 需要动态修改调度时间 | 否 | 是（Admin 控制台） |

**一句话：能用 @Scheduled 解决的就用 @Scheduled，解决不了的用 @XxlJob。**

## 设计与概念精要

### 架构核心

```
调度中心（Admin）+ 执行器（Executor）+ 任务处理器（JobHandler）

Admin ─── 集中调度，HTTP 触发 ───→ 执行器 ─── 反射调用 ───→ JobHandler
  ↑                                  │
  └──────── 回调执行结果 ────────────┘
```

- Admin 是独立部署的 Spring Boot 应用 + MySQL
- 执行器嵌入在业务应用中，通过 xxl-job-core 实现
- 启动时自动向 Admin 注册，心跳维持

### 自动配置三层结构

```
1. AutoConfiguration.imports → 发现配置类
2. YudaoXxlJobAutoConfiguration → 条件装配，创建执行器 Bean
3. XxlJobProperties → YAML 映射 + 参数校验
```

关键条件注解：

| 注解 | 含义 | 意图 |
|------|------|------|
| `@ConditionalOnClass` | xxl-job-core 存在 | 不引入依赖则不生效 |
| `@ConditionalOnProperty` | xxl.job.enabled=true | 允许配置禁用 |
| `@ConditionalOnMissingBean` | 容器中无 XxlJobExecutor | 允许用户自定义覆盖 |

### @TenantJob 的工作原理

```
@TenantJob
    ↓
TenantJobAspect（AOP 切面）
    ↓
1. tenantFrameworkService.getTenantIds() → 获取所有租户
2. parallelStream 遍历
3. TenantUtils.execute(tenantId, () -> { 业务逻辑 })
   → 设置 TenantContextHolder
   → 执行业务逻辑（MyBatis 自动追加 WHERE tenant_id = ?）
   → finally 恢复上下文
4. 汇总结果，上报 Admin
```

关键要点：
- **幂等性要求**：重试时所有租户会重新执行，业务逻辑必须幂等
- **并行执行**：使用 `parallelStream` 加速，需要线程安全的结果收集
- **上下文传递**：手动传递 `XxlJobContext` 到子线程

### 模块依赖关系

```
yudao-spring-boot-starter-job
  ├── XxlJob 自动配置
  ├── XxlJobProperties
  └── @EnableScheduling（同时启用 @Scheduled）

yudao-spring-boot-starter-biz-tenant
  ├── @TenantJob 注解
  ├── TenantJobAspect 切面
  ├── 条件：xxl-job-core 存在时激活
  └── 条件：yudao.tenant.enable=true 时激活

两者正交组合，互不耦合
```

### 配置速查

| 配置项 | 默认值 | 必填 | 说明 |
|--------|-------|------|------|
| `xxl.job.enabled` | true | 否 | 开关，本地开发可设为 false |
| `xxl.job.accessToken` | null | 否 | 通信令牌，生产环境建议修改 |
| `xxl.job.admin.addresses` | - | **是** | Admin 地址 |
| `xxl.job.executor.appname` | - | **是** | 执行器名称，用 `${spring.application.name}` |
| `xxl.job.executor.logpath` | - | **是** | 日志路径 |
| `xxl.job.executor.port` | -1 | 否 | 端口，-1 为随机 |
| `xxl.job.executor.log-retention-days` | 30 | 否 | 日志保留天数 |

### 设计边界

**做：**
- 自动装配 XXL-Job 执行器
- 配置绑定 + 参数校验
- @TenantJob 多租户遍历
- @Scheduled 与 @XxlJob 共存

**不做：**
- 自研调度引擎
- 封装 XXL-Job 原生 API
- 租户级任务隔离（不同租户不同调度规则）
- 任务热部署
- 任务 DAG 编排
- 分布式锁保护
