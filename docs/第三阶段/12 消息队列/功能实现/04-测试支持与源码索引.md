# 04-测试支持与源码索引

## 第一部分：测试策略

### 测试现状分析

yudao-cloud 的消息队列框架目前没有单元测试覆盖。业务模块中的 MQ 消费者通常是手动测试——启动应用后通过 Swagger 或数据库操作触发消息发送，观察消费日志。

这种测试方式的痛点：

| 痛点 | 说明 |
|------|------|
| 反馈慢 | 每次修改都要启动整个应用 |
| 不可重复 | 依赖 Redis 运行环境，CI/CD 中难以保证 |
| 难以模拟异常 | 无法方便地测试消费失败、Pending 重发等边界场景 |
| 覆盖不全 | 只能测试正常路径，拦截器链、ACK 机制等难以验证 |

### 测试基础设施设计

针对 MQ 框架的测试，可以从三个层面入手：

#### 层面一：消息对象测试（最简单）

消息对象是纯 POJO，可以直接用单元测试验证：

```java
@Test
public void testChannelMessage() {
    RedisWebSocketMessage message = new RedisWebSocketMessage()
            .setUserId(1L).setMessageType("chat");
    // 验证 channel 默认使用类名
    assertEquals("RedisWebSocketMessage", message.getChannel());
    // 验证 headers 可用
    message.addHeader("tenantId", "1");
    assertEquals("1", message.getHeader("tenantId"));
}

@Test
public void testStreamMessage() {
    BpmProcessInstanceResultMessage message = new BpmProcessInstanceResultMessage()
            .setProcessInstanceId(1L).setStatus(2);
    // 验证 streamKey 默认使用类名
    assertEquals("BpmProcessInstanceResultMessage", message.getStreamKey());
}
```

#### 层面二：RedisMQTemplate 测试（需要嵌入式 Redis）

`RedisMQTemplate` 的测试需要一个 Redis 实例。可以使用嵌入式 Redis（如 `it.ozimov:embedded-redis`）或者 Testcontainers：

```java
@SpringBootTest
public class RedisMQTemplateTest {

    @Resource
    private RedisMQTemplate redisMQTemplate;

    @Test
    public void testSendChannelMessage() {
        RedisWebSocketMessage message = new RedisWebSocketMessage()
                .setUserId(1L).setMessageType("chat");
        // 发送不应抛异常
        redisMQTemplate.send(message);
    }

    @Test
    public void testSendStreamMessage() {
        BpmProcessInstanceResultMessage message = new BpmProcessInstanceResultMessage()
                .setProcessInstanceId(1L).setStatus(2);
        RecordId recordId = redisMQTemplate.send(message);
        assertNotNull(recordId);
    }
}
```

#### 层面三：消费者测试（Mock 或嵌入式 Redis）

消费者测试的核心是验证 `onMessage(T)` 方法的业务逻辑。最简单的方式是绕过 Redis，直接调用业务方法：

```java
@Test
public void testConsumer() {
    // 直接构造消息对象，不需要经过 Redis
    BpmProcessInstanceResultMessage message = new BpmProcessInstanceResultMessage()
            .setProcessInstanceId(1L).setStatus(2);

    // Mock 依赖
    BpmProcessInstanceApi processInstanceApi = mock(BpmProcessInstanceApi.class);
    BpmProcessInstanceResultConsumer consumer = new BpmProcessInstanceResultConsumer(processInstanceApi);

    // 直接调用业务方法
    consumer.onMessage(message);

    // 验证业务逻辑
    verify(processInstanceApi).updateProcessInstanceResult(1L, 2);
}
```

### 拦截器测试

拦截器是独立的组件，可以直接单元测试：

```java
@Test
public void testTenantInterceptor() {
    TenantRedisMessageInterceptor interceptor = new TenantRedisMessageInterceptor();

    // 测试发送前设置租户
    AbstractRedisMessage message = new BpmProcessInstanceResultMessage();
    TenantContextHolder.setTenantId(1L);
    interceptor.sendMessageBefore(message);
    assertEquals("1", message.getHeader("tenantId"));

    // 测试消费前读取租户
    AbstractRedisMessage consumeMessage = new BpmProcessInstanceResultMessage();
    consumeMessage.addHeader("tenantId", "2");
    interceptor.consumeMessageBefore(consumeMessage);
    assertEquals(2L, TenantContextHolder.getTenantId());
}
```

### 测试配置参考

在测试环境中使用 `application-test.yml` 配置 Redis：

```yaml
spring:
  redis:
    host: localhost
    port: 6379
    # 测试环境可以使用不同的 database 隔离
    database: 15
```

### 依赖关系

```
yudao-spring-boot-starter-mq
  ├── spring-boot-starter-data-redis   ← RedisTemplate、Stream 操作
  ├── redisson-spring-boot-starter     ← 分布式锁（后台任务）
  └── jackson-databind                 ← JSON 序列化

业务模块（如 yudao-module-system）引入 yudao-spring-boot-starter-mq 后，
自动获得 RedisMQTemplate Bean 和消费者容器。
```
