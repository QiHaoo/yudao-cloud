# 06-精简汇总

## 能力速查表

### RedisMQTemplate API

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `send(T extends AbstractRedisChannelMessage)` | Channel 消息 | void | Pub/Sub 广播发送 |
| `send(T extends AbstractRedisStreamMessage)` | Stream 消息 | RecordId | Stream 可靠发送 |
| `addInterceptor(RedisMessageInterceptor)` | 拦截器 | void | 注册拦截器 |
| `getRedisTemplate()` | - | RedisTemplate | 获取底层 Redis 模板 |
| `getInterceptors()` | - | List | 获取拦截器列表 |

### 消息类型

| 基类 | 消费模式 | 持久化 | 适用场景 |
|------|---------|--------|---------|
| `AbstractRedisChannelMessage` | 广播（所有订阅者收到） | 否 | WebSocket 广播、实时通知 |
| `AbstractRedisStreamMessage` | 集群（组内一个消费者） | 是 | 订单处理、库存扣减 |

### 拦截器 API

| 方法 | 执行时机 | 执行顺序 | 典型用途 |
|------|---------|---------|---------|
| `sendMessageBefore()` | 发送前 | 正序 | 写入租户 ID、记录日志 |
| `sendMessageAfter()` | 发送后 | 倒序 | 清理上下文、统计耗时 |
| `consumeMessageBefore()` | 消费前 | 正序 | 读取租户 ID、设置上下文 |
| `consumeMessageAfter()` | 消费后 | 倒序 | 清理上下文、记录消费日志 |

## 核心设计决策列表

| 决策 | 选择 | 原因 |
|------|------|------|
| 默认 MQ 实现 | Redis | 零额外部署，复用已有 Redis |
| 消息序列化 | JSON | 可读、跨语言、方便调试 |
| Channel 消息 channel 名 | 类名（`getClass().getSimpleName()`） | 自动推导，无需配置 |
| Stream 消费者组名 | 应用名（`spring.application.name`） | 同应用实例自动同组 |
| 消费者名 | `IP@PID` | 参考 RocketMQ clientId 设计 |
| 拦截器执行模型 | 洋葱模型（Before 正序、After 倒序） | 保证"谁先设置谁最后清理" |
| 发送/消费包裹方式 | try-finally | 确保 After 一定执行 |
| Stream 版本要求 | Redis >= 5.0 | Stream 是 5.0 新增特性 |
| 消息幂等性 | 业务层实现 | 不同业务判断逻辑不同 |
| 事务消息 | 未内置（业务层处理） | Redis 不支持事务消息 |

## 消息发送/消费流程图

### Channel 消息（Pub/Sub 广播）

```
Producer                          Redis                         Consumer
   │                                │                               │
   │ send(ChannelMessage)           │                               │
   │──→ sendMessageBefore()         │                               │
   │    [拦截器正序执行]            │                               │
   │──→ JsonUtils.toJsonString()    │                               │
   │──→ PUBLISH channel json        │                               │
   │                                │──→ onMessage(Message)         │
   │                                │    [解析 JSON]                │
   │                                │    [拦截器 before 正序]       │
   │                                │    [业务消费 onMessage(T)]    │
   │                                │    [拦截器 after 倒序]        │
   │──→ sendMessageAfter()          │                               │
   │    [拦截器倒序执行]            │                               │
   │                                │                               │
```

### Stream 消息（可靠投递）

```
Producer                          Redis                         Consumer
   │                                │                               │
   │ send(StreamMessage)            │                               │
   │──→ sendMessageBefore()         │                               │
   │──→ XADD streamKey json         │                               │
   │    (消息持久化)                │                               │
   │──→ sendMessageAfter()          │                               │
   │                                │                               │
   │                                │──→ XREADGROUP                 │
   │                                │    (消息进入 Pending 队列)    │
   │                                │                               │
   │                                │    [解析 JSON]                │
   │                                │    [拦截器 before]            │
   │                                │    [业务消费]                 │
   │                                │    [拦截器 after]             │
   │                                │                               │
   │                                │──→ XACK                       │
   │                                │    (从 Pending 队列移除)      │
   │                                │                               │
   │                    ┌───────────┴───────────┐                   │
   │                    │  Pending 消息超时？    │                   │
   │                    │  RedisPendingMessage  │                   │
   │                    │  ResendJob 重发       │                   │
   │                    └───────────────────────┘                   │
```

## MQ 选型速查表

| 场景 | 推荐方案 | 理由 |
|------|---------|------|
| 同服务内事件解耦 | Spring Event | 零依赖，最简单 |
| 跨服务广播通知 | Redis Pub/Sub | 无持久化，实时性好 |
| 跨服务可靠投递 | Redis Stream | 复用 Redis，有 ACK 机制 |
| 大规模生产环境 | RocketMQ | 高可靠，事务消息，延迟消息 |
| 企业级复杂路由 | RabbitMQ | 灵活的 Exchange/Queue 绑定 |
| 大数据管道 | Kafka | 超高吞吐，顺序写入 |
| 需要事务消息 | RocketMQ | 原生支持，两阶段提交 |
| 需要延迟消息 | RocketMQ / RabbitMQ | RocketMQ 原生 / RabbitMQ TTL+DLQ |
| 需要消息回溯 | Kafka / Redis Stream | 按 offset/时间重新消费 |
