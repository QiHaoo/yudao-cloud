## 测试策略

### 现状：单元测试

数据权限组件的测试主要集中在框架层，使用 JUnit 5 + Mockito 进行单元测试。

#### DataPermissionRuleHandlerTest

`DataPermissionRuleHandlerTest` 是数据权限处理器的单元测试，主要测试 SQL 改写功能：

```java
// 文件路径：yudao-framework/yudao-spring-boot-starter-biz-data-permission/src/test/java/cn/iocoder/yudao/framework/datapermission/core/db/DataPermissionRuleHandlerTest.java
public class DataPermissionRuleHandlerTest extends BaseMockitoUnitTest {

    @InjectMocks
    private DataPermissionRuleHandler handler;

    @Mock
    private DataPermissionRuleFactory ruleFactory;

    private DataPermissionInterceptor interceptor;

    @BeforeEach
    public void setUp() {
        interceptor = new DataPermissionInterceptor(handler);

        // 租户的数据权限规则
        DataPermissionRule tenantRule = new DataPermissionRule() {
            private static final String COLUMN = "tenant_id";

            @Override
            public Set<String> getTableNames() {
                return asSet("entity", "entity1", "entity2", "entity3", "t1", "t2", "sys_dict_item",
                        "t_user", "t_role");
            }

            @Override
            public Expression getExpression(String tableName, Alias tableAlias) {
                Column column = MyBatisUtils.buildColumn(tableName, tableAlias, COLUMN);
                LongValue value = new LongValue(1L);
                return new EqualsTo(column, value);
            }
        };

        // 部门的数据权限规则
        DataPermissionRule deptRule = new DataPermissionRule() {
            private static final String COLUMN = "dept_id";

            @Override
            public Set<String> getTableNames() {
                return asSet("t_user");
            }

            @Override
            public Expression getExpression(String tableName, Alias tableAlias) {
                Column column = MyBatisUtils.buildColumn(tableName, tableAlias, COLUMN);
                ExpressionList<LongValue> values = new ExpressionList<>(new LongValue(10L),
                        new LongValue(20L));
                return new InExpression(column, new ParenthesedExpressionList((values)));
            }
        };

        // 设置到上下文
        when(ruleFactory.getDataPermissionRule(any())).thenReturn(Arrays.asList(tenantRule, deptRule));
    }

    @Test
    void selectSingle() {
        // 单表
        assertSql("select * from entity where id = ?",
                "SELECT * FROM entity WHERE id = ? AND entity.tenant_id = 1");

        assertSql("select * from entity where id = ? or name = ?",
                "SELECT * FROM entity WHERE (id = ? OR name = ?) AND entity.tenant_id = 1");
    }

    @Test
    public void testSelectSingle() {
        // 单表
        assertSql("select * from t_user where id = ?",
                "SELECT * FROM t_user WHERE id = ? AND t_user.tenant_id = 1 AND t_user.dept_id IN (10, 20)");
    }

    // ... 更多测试用例

    private void assertSql(String sql, String targetSql) {
        assertEquals(targetSql, interceptor.parserSingle(sql, null));
    }
}
```

## 常见问题

### Q: 数据权限和多租户会冲突吗？

A: 不会。两者是叠加关系，先多租户过滤，再数据权限过滤。数据权限禁用不影响租户隔离。

### Q: 如何临时禁用数据权限？

A: 两种方式：
1. 注解方式：`@DataPermission(enable = false)`
2. 工具类方式：`DataPermissionUtils.executeIgnore(() -> { ... })`

### Q: 数据权限只对管理员生效吗？

A: 是的。`DeptDataPermissionRule.getExpression()` 中判断了 `UserTypeEnum.ADMIN`，非管理员类型用户不处理。

### Q: 用户没有角色时会怎样？

A: `PermissionServiceImpl.getDeptDataPermission()` 中，如果用户没有角色，设置 `self = true`，即只能查看自己创建的数据。

### Q: 多个角色的数据范围如何合并？

A: 取并集。遍历所有角色，`deptIds` 合并，`self` 取 OR，`all` 取 OR。

### Q: 数据权限的 SQL 改写在什么时候执行？

A: 在 MyBatis 执行 SQL 之前，由 `DataPermissionInterceptor` 拦截并改写。

### Q: 如何自定义数据权限规则？

A: 实现 `DataPermissionRule` 接口，定义 `getTableNames()` 和 `getExpression()` 方法，然后注册为 Spring Bean。

### Q: 数据权限的 enable 属性如何在微服务间传播？

A: 通过 HTTP Header 传播。`DataPermissionRequestInterceptor` 在 Feign 请求发送前设置 header，`DataPermissionRpcWebFilter` 在接收端读取 header 并禁用数据权限。
