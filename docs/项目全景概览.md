## 模块依赖关系

```
                    yudao-common
                         │
          ┌──────────────┼──────────────┐
          │              │              │
   starter-mybatis  starter-web   starter-security
          │              │              │
          └──────────────┼──────────────┘
                         │
        ┌────────────────┼────────────────┐
        │                │                │
  module-system-api  module-infra-api  module-*-api
        │                │                │
        │    ┌───────────┘                │
        │    │                            │
  module-system-server  module-infra-server  module-*-server
```

**规则**：
- 任何模块都可以依赖 `yudao-common` 和框架 Starter
- 业务模块只能依赖其他模块的 `-api`，不能依赖 `-server`
- 框架 Starter 不能依赖业务模块（单向依赖）
- `*CommonApi` 接口放在 `yudao-common` 中，打破框架→业务的反向依赖

## 电商系统示例：模块协作全景

以一个电商下单流程为例，展示各模块如何协作：

```
用户下单
  │
  ├─→ mall-trade（交易模块）
  │     ├─→ mall-product（商品模块）── 查询商品信息、库存
  │     ├─→ mall-promotion（促销模块）── 计算优惠、满减
  │     └─→ pay（支付模块）── 创建支付订单
  │           └─→ system（系统模块）── 查询用户信息
  │
  ├─→ bpm（工作流模块）── 发起订单审批流程（如果需要）
  │
  └─→ mp（公众号模块）── 发送订单通知到微信
```

每个箭头都是一次 Feign RPC 调用，通过 `-api` 子模块的接口定义。
