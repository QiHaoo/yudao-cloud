# 09-测试支持与基础设施

## 目标

盘点认证鉴权框架的测试现状，识别测试痛点，提供测试工具和方案。

## 现状盘点

当前项目的测试策略：
- **集成测试**：使用 Spring Boot Test + MockMvc，需要完整的 Spring 上下文
- **单元测试**：使用 Mockito mock 依赖，验证业务逻辑

认证鉴权框架的测试痛点：
1. `@PreAuthorize` 注解的方法需要模拟认证上下文才能测试
2. `SecurityFrameworkUtils.getLoginUser()` 依赖 SecurityContextHolder，单元测试中返回 null
3. Feign RPC 调用（`OAuth2TokenCommonApi`、`PermissionCommonApi`）需要 mock

## 测试工具方案

### 方案一：设置 SecurityContextHolder

最直接的方式——在测试方法中手动设置认证信息：

```java
@BeforeEach
void setUp() {
    // 构建测试用户
    LoginUser loginUser = new LoginUser()
        .setId(1L)
        .setUserType(UserTypeEnum.ADMIN.getValue())
        .setTenantId(1L);

    // 设置到 SecurityContextHolder
    UsernamePasswordAuthenticationToken authentication =
        new UsernamePasswordAuthenticationToken(loginUser, null, Collections.emptyList());
    SecurityContextHolder.getContext().setAuthentication(authentication);
}

@AfterEach
void tearDown() {
    SecurityContextHolder.clearContext();
}
```

### 方案二：@WithMockUser（Spring Security Test）

Spring Security Test 提供了 `@WithMockUser` 注解，但它使用的是 `UserDetails` 体系，与本项目的 `LoginUser` 不兼容。

### 方案三：自定义测试注解

可以创建自定义注解，简化测试中的用户设置：

```java
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.METHOD)
public @interface WithLoginUser {
    long userId() default 1L;
    int userType() default UserTypeEnum.ADMIN.getValue();
    long tenantId() default 1L;
}
```

配合 JUnit 5 Extension 或 `@BeforeEach` 使用。

## Mock Feign 调用

```java
@ExtendWith(MockitoExtension.class)
class SecurityFrameworkServiceImplTest {

    @Mock
    private PermissionCommonApi permissionApi;

    @InjectMocks
    private SecurityFrameworkServiceImpl securityService;

    @Test
    void testHasPermission() {
        // 设置当前用户
        LoginUser loginUser = new LoginUser().setId(1L);
        SecurityContextHolder.getContext().setAuthentication(
            new UsernamePasswordAuthenticationToken(loginUser, null, Collections.emptyList()));

        // mock RPC 调用
        when(permissionApi.hasAnyPermissions(1L, "system:user:create"))
            .thenReturn(CommonResult.success(true));

        // 执行测试
        assertTrue(securityService.hasPermission("system:user:create"));
    }
}
```

## 测试配置参考

```yaml
# application-test.yml
yudao:
  security:
    mock-enable: true
    mock-secret: "test"
    password-encoder-length: 4  # 测试环境用最低强度，加快速度
```

## 小结

认证鉴权框架的测试核心是解决"如何在测试中模拟用户身份"。最实用的方案是手动设置 `SecurityContextHolder`，配合 Mockito mock Feign 调用。
