# 01-起步：骨架搭建与自动配置

## 目标

从零搭建 `yudao-spring-boot-starter-security` 模块的骨架：一个空的 Spring Boot Starter，能被 Spring Boot 自动加载，包含配置属性类和自动配置类的空壳。

## 模块结构

```
yudao-spring-boot-starter-security/
├── pom.xml
└── src/main/java/cn/iocoder/yudao/framework/security/
    └── config/
        ├── SecurityProperties.java              # 配置属性
        ├── YudaoSecurityAutoConfiguration.java   # 自动配置（组件注册）
        └── YudaoWebSecurityConfigurerAdapter.java # SecurityFilterChain 配置
```

## 第一步：pom.xml 依赖

```xml
<project>
    <parent>
        <groupId>cn.iocoder.cloud</groupId>
        <artifactId>yudao-framework</artifactId>
        <version>${revision}</version>
    </parent>
    <artifactId>yudao-spring-boot-starter-security</artifactId>

    <dependencies>
        <!-- Spring Security -->
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-security</artifactId>
        </dependency>

        <!-- Web 层（依赖 yudao-spring-boot-starter-web） -->
        <dependency>
            <groupId>cn.iocoder.cloud</groupId>
            <artifactId>yudao-spring-boot-starter-web</artifactId>
        </dependency>

        <!-- yudao-common（提供 OAuth2TokenCommonApi、PermissionCommonApi） -->
        <dependency>
            <groupId>cn.iocoder.cloud</groupId>
            <artifactId>yudao-common</artifactId>
        </dependency>

        <!-- TransmittableThreadLocal -->
        <dependency>
            <groupId>com.alibaba</groupId>
            <artifactId>transmittable-thread-local</artifactId>
        </dependency>

        <!-- Guava（LoadingCache） -->
        <dependency>
            <groupId>com.google.guava</groupId>
            <artifactId>guava</artifactId>
        </dependency>
    </dependencies>
</project>
```

关键依赖说明：
- `spring-boot-starter-security`：Spring Security 核心
- `yudao-spring-boot-starter-web`：提供 `WebProperties`（URL 前缀配置）和 `GlobalExceptionHandler`
- `yudao-common`：提供 Feign RPC 接口定义（`OAuth2TokenCommonApi`、`PermissionCommonApi`）
- `transmittable-thread-local`：解决异步场景上下文丢失
- `guava`：权限校验的本地缓存

## 第二步：SecurityProperties 配置属性类

```java
// 文件路径：config/SecurityProperties.java
@ConfigurationProperties(prefix = "yudao.security")
@Validated
@Data
public class SecurityProperties {

    @NotEmpty(message = "Token Header 不能为空")
    private String tokenHeader = "Authorization";

    @NotEmpty(message = "Token Parameter 不能为空")
    private String tokenParameter = "token";

    @NotNull(message = "mock 模式的开关不能为空")
    private Boolean mockEnable = false;

    @NotEmpty(message = "mock 模式的密钥不能为空")
    private String mockSecret = "test";

    private List<String> permitAllUrls = Collections.emptyList();

    private Integer passwordEncoderLength = 4;
}
```

这个类定义了框架的所有可配置项，前缀 `yudao.security`。每个配置项都有默认值，零配置即可启动。

## 第三步：YudaoSecurityAutoConfiguration 自动配置类

```java
// 文件路径：config/YudaoSecurityAutoConfiguration.java
@AutoConfiguration
@AutoConfigureOrder(-1)  // 先于 Spring Security 自动配置
@EnableConfigurationProperties(SecurityProperties.class)
public class YudaoSecurityAutoConfiguration {

    @Resource
    private SecurityProperties securityProperties;

    // 暂时只注册空壳 Bean，后续逐步填充
}
```

`@AutoConfigureOrder(-1)` 确保本配置类在 Spring Security 自动配置之前执行，避免默认配置抢先生效。

## 第四步：注册自动配置

在 `src/main/resources/META-INF/spring/` 下创建文件：

```
org.springframework.boot.autoconfigure.AutoConfiguration.imports
```

内容：
```
cn.iocoder.yudao.framework.security.config.YudaoSecurityAutoConfiguration
cn.iocoder.yudao.framework.security.config.YudaoWebSecurityConfigurerAdapter
```

这是 Spring Boot 3.x 的自动配置注册方式（替代旧版的 `spring.factories`）。

## 验证

此时模块已可被 Spring Boot 自动加载。引入依赖后，`SecurityProperties` 会自动绑定配置，但还没有任何安全功能。

## 小结

本章搭建了 Starter 的骨架：
- **pom.xml**：声明依赖
- **SecurityProperties**：配置属性类
- **YudaoSecurityAutoConfiguration**：自动配置入口
- **AutoConfiguration.imports**：注册自动配置

后续章节将逐步向骨架中填充功能。
