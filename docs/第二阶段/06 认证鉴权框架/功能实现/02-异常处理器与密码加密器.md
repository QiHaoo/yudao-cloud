# 02-异常处理器与密码加密器

## 目标

实现认证鉴权框架的三个基础设施 Bean：认证失败处理器（401）、权限不足处理器（403）、密码加密器（BCrypt）。

## AuthenticationEntryPointImpl：未认证处理

当未登录用户访问需要认证的接口时，Spring Security 调用此处理器。

```java
// 文件路径：core/handler/AuthenticationEntryPointImpl.java
@Slf4j
public class AuthenticationEntryPointImpl implements AuthenticationEntryPoint {

    @Override
    public void commence(HttpServletRequest request, HttpServletResponse response,
                         AuthenticationException e) {
        log.debug("[commence][访问 URL({}) 时，没有登录]", request.getRequestURI(), e);
        ServletUtils.writeJSON(response, CommonResult.error(UNAUTHORIZED));
    }
}
```

关键设计：
- 使用 `log.debug` 而不是 `warn`——未登录访问是正常场景（如前端路由守卫触发的探测请求）
- 返回 `CommonResult.error(UNAUTHORIZED)` 而不是 HTML 页面——前后端分离架构中，前端依赖 JSON 格式的错误码做跳转

## AccessDeniedHandlerImpl：权限不足处理

当已登录用户访问没有权限的接口时，Spring Security 调用此处理器。

```java
// 文件路径：core/handler/AccessDeniedHandlerImpl.java
@Slf4j
public class AccessDeniedHandlerImpl implements AccessDeniedHandler {

    @Override
    public void handle(HttpServletRequest request, HttpServletResponse response,
                       AccessDeniedException e) throws IOException, ServletException {
        log.warn("[commence][访问 URL({}) 时，用户({}) 权限不够]",
                request.getRequestURI(), SecurityFrameworkUtils.getLoginUserId(), e);
        ServletUtils.writeJSON(response, CommonResult.error(FORBIDDEN));
    }
}
```

关键设计：
- 使用 `log.warn` 而不是 `debug`——已登录用户访问无权限接口，可能是恶意操作，需要关注
- 记录了 `userId`，便于事后审计

## PasswordEncoder：BCrypt 加密器

```java
// 在 YudaoSecurityAutoConfiguration 中注册
@Bean
public PasswordEncoder passwordEncoder() {
    return new BCryptPasswordEncoder(securityProperties.getPasswordEncoderLength());
}
```

BCrypt 是目前最安全的密码哈希算法之一：
- 内置盐值（salt），每次加密结果不同，防止彩虹表攻击
- 可配置计算强度（`passwordEncoderLength`），强度越高越安全但越慢
- 默认强度 4（约 16ms），生产环境建议 10（约 100ms）

## 注册到自动配置

在 `YudaoSecurityAutoConfiguration` 中添加：

```java
@Bean
public AuthenticationEntryPoint authenticationEntryPoint() {
    return new AuthenticationEntryPointImpl();
}

@Bean
public AccessDeniedHandler accessDeniedHandler() {
    return new AccessDeniedHandlerImpl();
}

@Bean
public PasswordEncoder passwordEncoder() {
    return new BCryptPasswordEncoder(securityProperties.getPasswordEncoderLength());
}
```

## 小结

本章实现了三个基础 Bean：
- `AuthenticationEntryPointImpl`：返回 401 JSON 响应
- `AccessDeniedHandlerImpl`：返回 403 JSON 响应
- `BCryptPasswordEncoder`：密码加密器

它们是后续认证鉴权流程的"兜底"——认证失败或权限不足时，返回统一格式的错误响应。
