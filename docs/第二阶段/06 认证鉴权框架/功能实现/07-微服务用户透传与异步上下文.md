# 07-微服务用户透传与异步上下文

## 目标

实现两个跨场景能力：Feign 调用时的用户信息透传，和 @Async 异步场景下的上下文传递。

## LoginUserRequestInterceptor：Feign 用户透传

```java
// 文件路径：core/rpc/LoginUserRequestInterceptor.java
@Slf4j
public class LoginUserRequestInterceptor implements RequestInterceptor {

    @Override
    @SneakyThrows
    public void apply(RequestTemplate requestTemplate) {
        LoginUser user = SecurityFrameworkUtils.getLoginUser();
        if (user == null) {
            return;
        }
        String userStr = JsonUtils.toJsonString(user);
        userStr = URLEncoder.encode(userStr, StandardCharsets.UTF_8.name());
        requestTemplate.header(SecurityFrameworkUtils.LOGIN_USER_HEADER, userStr);
    }
}
```

每次 Feign 调用时自动执行：获取当前 LoginUser → JSON 序列化 → URL 编码 → 放入 Header。

## YudaoSecurityRpcAutoConfiguration：RPC 配置

```java
// 文件路径：config/YudaoSecurityRpcAutoConfiguration.java
@AutoConfiguration
@AutoConfigureOrder(-1)
@EnableFeignClients(basePackages = {
    "cn.iocoder.yudao.framework.common.biz.system.oauth2",    // OAuth2TokenCommonApi
    "cn.iocoder.yudao.framework.common.biz.system.permission" // PermissionCommonApi
})
public class YudaoSecurityRpcAutoConfiguration {

    @Bean
    public LoginUserRequestInterceptor loginUserRequestInterceptor() {
        return new LoginUserRequestInterceptor();
    }
}
```

`@EnableFeignClients` 启用了 `OAuth2TokenCommonApi` 和 `PermissionCommonApi` 两个 Feign 客户端。`LoginUserRequestInterceptor` 作为全局 Feign 拦截器注册。

## TransmittableThreadLocalSecurityContextHolderStrategy

```java
// 文件路径：core/context/TransmittableThreadLocalSecurityContextHolderStrategy.java
public class TransmittableThreadLocalSecurityContextHolderStrategy
        implements SecurityContextHolderStrategy {

    private static final ThreadLocal<SecurityContext> CONTEXT_HOLDER =
        new TransmittableThreadLocal<>();

    @Override
    public SecurityContext getContext() {
        SecurityContext ctx = CONTEXT_HOLDER.get();
        if (ctx == null) {
            ctx = createEmptyContext();
            CONTEXT_HOLDER.set(ctx);
        }
        return ctx;
    }

    @Override
    public void setContext(SecurityContext context) {
        CONTEXT_HOLDER.set(context);
    }

    @Override
    public void clearContext() {
        CONTEXT_HOLDER.remove();
    }

    @Override
    public SecurityContext createEmptyContext() {
        return new SecurityContextImpl();
    }
}
```

用 `TransmittableThreadLocal` 替换 Spring Security 默认的 `ThreadLocal`，实现跨线程传递。

## 注册 TTL 策略

在 `YudaoSecurityAutoConfiguration` 中：

```java
@Bean
public MethodInvokingFactoryBean securityContextHolderMethodInvokingFactoryBean() {
    MethodInvokingFactoryBean bean = new MethodInvokingFactoryBean();
    bean.setTargetClass(SecurityContextHolder.class);
    bean.setTargetMethod("setStrategyName");
    bean.setArguments(TransmittableThreadLocalSecurityContextHolderStrategy.class.getName());
    return bean;
}
```

通过 `MethodInvokingFactoryBean` 在 Spring 容器初始化时调用 `SecurityContextHolder.setStrategyName()`，将策略替换为 TTL 版本。

## 小结

本章实现了两个跨场景能力：
- **Feign 透传**：`LoginUserRequestInterceptor` 自动将当前用户放入 `login-user` Header
- **异步上下文**：`TransmittableThreadLocal` 替换默认 `ThreadLocal`，@Async 时用户信息不丢失
