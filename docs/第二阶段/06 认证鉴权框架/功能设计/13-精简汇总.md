# 13-精简汇总

## 能力与使用速查

### 认证能力

| 能力 | 使用方式 | 说明 |
|------|---------|------|
| Token 认证 | 请求带 `Authorization: Bearer {token}` | 自动校验，无需业务代码 |
| 参数 Token | 请求带 `?token={token}` | WebSocket、文件下载场景 |
| Header 透传 | Gateway 注入 `login-user` Header | 微服务间自动解析 |
| Mock 登录 | `Authorization: {mockSecret}{userId}` | 仅开发环境，需开启 `mock-enable` |
| 免认证 URL | `@PermitAll` 注解 | 标记在 Controller 方法/类上 |
| 免认证 URL（配置） | `yudao.security.permit-all-urls` | application.yml 配置 |
| 免认证 URL（模块） | 继承 `AuthorizeRequestsCustomizer` | 每个 Maven 模块自定义 |

### 鉴权能力

| 能力 | 使用方式 | 说明 |
|------|---------|------|
| 权限校验 | `@PreAuthorize("@ss.hasPermission('xxx')")` | 方法级，精确到操作 |
| 多权限校验 | `@PreAuthorize("@ss.hasAnyPermissions('a','b')")` | 任一匹配即可 |
| 角色校验 | `@PreAuthorize("@ss.hasRole('admin')")` | 基于角色标识 |
| 多角色校验 | `@PreAuthorize("@ss.hasAnyRoles('a','b')")` | 任一匹配即可 |
| Scope 校验 | `@PreAuthorize("@ss.hasScope('user:read')")` | OAuth2 场景 |
| 跨租户跳过 | 自动（`visitTenantId != tenantId`） | 无需手动处理 |

### 工具方法

| 方法 | 位置 | 说明 |
|------|------|------|
| `SecurityFrameworkUtils.getLoginUser()` | 获取当前 LoginUser | 未登录返回 null |
| `SecurityFrameworkUtils.getLoginUserId()` | 获取当前用户 ID | 未登录返回 null |
| `SecurityFrameworkUtils.getLoginUserNickname()` | 获取当前用户昵称 | 从 info Map 读取 |
| `SecurityFrameworkUtils.getLoginUserDeptId()` | 获取当前用户部门 ID | 从 info Map 读取 |
| `SecurityFrameworkUtils.skipPermissionCheck()` | 是否跳过权限校验 | 跨租户时返回 true |

### 配置项速查

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `yudao.security.token-header` | `Authorization` | Token Header 名称 |
| `yudao.security.token-parameter` | `token` | Token 参数名称 |
| `yudao.security.mock-enable` | `false` | Mock 登录开关 |
| `yudao.security.mock-secret` | `test` | Mock 登录密钥 |
| `yudao.security.permit-all-urls` | `[]` | 免认证 URL 列表 |
| `yudao.security.password-encoder-length` | `4` | BCrypt 强度 |

## 设计与概念精要

### 核心设计决策

1. **自研 Token 替代 Spring Security OAuth2**
   - 用 UUID Token + Redis/MySQL 双写，替代 JWT + Spring Authorization Server
   - 原因：OAuth2 框架过重，自研更灵活可控

2. **框架层与业务层分离**
   - `yudao-spring-boot-starter-security`（框架层）：提供认证鉴权骨架
   - `yudao-module-system`（业务层）：实现 Token 管理、权限查询
   - 通过 Feign RPC 接口解耦

3. **@PreAuthorize + @ss Bean 替代 URL 级权限**
   - 方法级权限比 URL 级更精确、更直观
   - `@ss` Bean 通过 SpEL 表达式在运行时求值
   - Guava Cache 1 分钟缓存，平衡性能和实时性

4. **TransmittableThreadLocal 替代默认 ThreadLocal**
   - 解决 @Async 异步场景下用户上下文丢失
   - 阿里巴巴开源，线程池场景下也能正确传递

### 认证流程要点

```
请求 → TokenAuthenticationFilter
  ├── 策略一：login-user Header 解析（微服务透传）
  ├── 策略二：Token 校验（前端直连）
  ├── 策略三：Mock 模拟（开发环境）
  └── 设置 LoginUser 到 SecurityContextHolder
```

### 鉴权流程要点

```
@PreAuthorize("@ss.hasPermission('xxx')") 触发
  → SecurityFrameworkServiceImpl.hasAnyPermissions()
    → 跳过跨租户检查
    → Guava Cache 查询
    → 缓存未命中 → PermissionCommonApi.hasAnyPermissions()（Feign RPC）
    → system 服务查询：用户→角色→菜单→权限匹配
    → 返回 true/false
```

### RBAC 权限模型

```
用户(User) ──N:N──> 角色(Role) ──N:N──> 菜单(Menu/Permission)
                        │
                        └── DataScope（数据权限范围）
                              ├── ALL(1) 全部
                              ├── DEPT_CUSTOM(2) 指定部门
                              ├── DEPT_ONLY(3) 本部门
                              ├── DEPT_AND_CHILD(4) 本部门及下级
                              └── SELF(5) 仅本人
```
