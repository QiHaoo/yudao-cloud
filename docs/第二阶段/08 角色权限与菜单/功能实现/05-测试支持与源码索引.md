## 一、测试支持

### 1.1 权限校验测试

测试权限校验时，需要模拟两个核心依赖：`PermissionService` 和 Spring Security 的 `SecurityContextHolder`。

**模拟 PermissionService**：

```java
@MockBean
private PermissionService permissionService;

@Test
public void testCreateOrder_withPermission() {
    // 模拟用户有创建订单的权限
    when(permissionService.hasAnyPermissions(eq(1L), eq("system:order:create")))
        .thenReturn(true);

    // 调用创建订单接口
    // ... 验证正常创建
}

@Test
public void testCreateOrder_withoutPermission() {
    // 模拟用户没有创建订单的权限
    when(permissionService.hasAnyPermissions(eq(1L), eq("system:order:create")))
        .thenReturn(false);

    // 调用创建订单接口
    // ... 验证返回 403
}
```

**设置 SecurityContextHolder**：

```java
// 在测试方法中模拟已登录用户
private void setLoginUser(Long userId) {
    LoginUser loginUser = new LoginUser();
    loginUser.setId(userId);
    loginUser.setUserType(UserTypeEnum.ADMIN.getValue());

    UsernamePasswordAuthenticationToken authentication =
        new UsernamePasswordAuthenticationToken(loginUser, null, Collections.emptyList());
    SecurityContextHolder.getContext().setAuthentication(authentication);
}
```

### 1.2 角色分配测试

验证 diff-based 分配逻辑的正确性：

```java
@Test
public void testAssignRoleMenu_diffBased() {
    Long roleId = 1L;
    // 数据库已有菜单：101, 102, 103
    when(roleMenuMapper.selectListByRoleId(roleId)).thenReturn(
        Arrays.asList(
            createRoleMenuDO(roleId, 101L),
            createRoleMenuDO(roleId, 102L),
            createRoleMenuDO(roleId, 103L)
        )
    );

    // 目标菜单：102, 103, 104
    Set<Long> targetMenuIds = Sets.newHashSet(102L, 103L, 104L);

    permissionService.assignRoleMenu(roleId, targetMenuIds);

    // 验证只新增了 104，只删除了 101
    verify(roleMenuMapper).insertBatch(argThat(list ->
        list.size() == 1 && list.get(0).getMenuId().equals(104L)
    ));
    verify(roleMenuMapper).deleteListByRoleIdAndMenuIds(eq(roleId),
        argThat(ids -> ids.size() == 1 && ids.contains(101L)));
}
```

### 1.3 数据权限测试

```java
@Test
public void testGetDeptDataPermission_multipleRoles() {
    Long userId = 1L;
    // 用户有两个角色
    RoleDO roleA = createRoleDO(DataScopeEnum.DEPT_ONLY.getScope());
    RoleDO roleB = createRoleDO(DataScopeEnum.DEPT_CUSTOM.getScope());
    roleB.setDataScopeDeptIds(Sets.newHashSet(200L, 300L));

    when(permissionService.getEnableUserRoleListByUserIdFromCache(userId))
        .thenReturn(Arrays.asList(roleA, roleB));
    when(userService.getUser(userId)).thenReturn(createUserDO(100L));

    DeptDataPermissionRespDTO result = permissionService.getDeptDataPermission(userId);

    // 验证结果是两个角色的并集
    assertFalse(result.getAll());
    assertFalse(result.getSelf());
    assertTrue(result.getDeptIds().containsAll(Arrays.asList(100L, 200L, 300L)));
}
```

### 1.4 H2 内存数据库测试配置

角色和菜单的 Mapper 测试可以使用 H2 内存数据库，避免依赖外部 MySQL：

## 四、关键设计模式速查

| 设计模式 | 应用位置 | 说明 |
|---------|---------|------|
| Diff-based 分配 | `PermissionServiceImpl.assignRoleMenu/assignUserRole` | 计算集合差集，只执行增量操作 |
| 缓存 + 失效 | `@Cacheable` / `@CacheEvict` | 读时缓存，写时清除 |
| AOP 自调用 | `getSelf()` 方法 | 通过 `SpringUtil.getBean()` 获取代理对象 |
| 惰性求值 | `Suppliers.memoize()` | 部门 ID 只查一次数据库 |
| 超管兜底 | `hasAnyPermissions` 末尾 | 超管跳过所有权限校验 |
| 租户过滤 | `@TenantIgnore` / `TenantBaseDO` | 菜单全局共享，角色租户隔离 |
| 枚举校验 | `@InEnum` | 参数必须是合法枚举值 |
| 操作日志 | `@LogRecord` + `DiffParseFunction` | 自动记录字段变更 |
