# 07-精简汇总

## 能力与使用速查

### 角色管理

|------|------|---------|
| 创建角色 | POST /system/role/create | system:role:create |
| 修改角色 | PUT /system/role/update | system:role:update |
| 删除角色 | DELETE /system/role/delete | system:role:delete |

## 设计与概念精要

### 核心设计决策

1. **RBAC 模型**：用户 → 角色 → 菜单（权限），三层解耦
2. **菜单即权限**：DIR/MENU/BUTTON 三种类型，BUTTON 对应权限标识
3. **五级数据范围**：ALL/DEPT_CUSTOM/DEPT_ONLY/DEPT_AND_CHILD/SELF
4. **多角色取并集**：用户有多个角色时，权限和数据范围取并集
5. **超管跳过检查**：super_admin 角色自动拥有所有权限

### 核心实体

| 实体 | 说明 | 关键字段 |
|------|------|---------|
| RoleDO | 角色 | name, code, type, dataScope, dataScopeDeptIds |
| MenuDO | 菜单/权限 | name, permission, type, parentId |
| RoleMenuDO | 角色-菜单关联 | roleId, menuId |
| UserRoleDO | 用户-角色关联 | userId, roleId |

### 菜单类型与权限的关系

```
DIR(1)  → 目录，前端导航栏的文件夹
MENU(2) → 菜单，前端导航栏的可点击项
BUTTON(3) → 按钮，对应 @PreAuthorize 的权限标识
```

### 数据范围

```
ALL(1)           全部数据
DEPT_CUSTOM(2)   指定部门
DEPT_ONLY(3)     本部门
DEPT_AND_CHILD(4) 本部门及下级
SELF(5)          仅本人
```

### 关键类

| 类 | 职责 |
|----|------|
| RoleController | 角色 CRUD 端点 |
| MenuController | 菜单 CRUD 端点 |
| PermissionController | 权限分配端点 |
| PermissionServiceImpl | 权限校验 + 数据范围计算 |
| RoleServiceImpl | 角色 CRUD + 缓存 |
| MenuServiceImpl | 菜单 CRUD + 缓存 |
