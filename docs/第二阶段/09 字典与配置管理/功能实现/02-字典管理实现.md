# 02 - 字典管理实现

## 本章目标

实现字典类型和字典数据的完整 CRUD 功能，包括：

1. 字典类型的增删改查与导出
2. 字典数据的增删改查与导出
3. 字典数据按类型查询的能力
4. 删除字典类型时的子数据校验

## 一、字典类型控制器：DictTypeController

> 源码位置：`yudao-module-system/yudao-module-system-server/src/main/java/cn/iocoder/yudao/module/system/controller/admin/dict/DictTypeController.java`

### 1.1 API 总览

| 接口 | 方法 | URL | 权限 | 说明 |
|------|------|-----|------|------|
| 创建字典类型 | POST | `/system/dict-type/create` | `system:dict:create` | 新增字典类型 |
| 修改字典类型 | PUT | `/system/dict-type/update` | `system:dict:update` | 修改字典类型 |
| 删除字典类型 | DELETE | `/system/dict-type/delete` | `system:dict:delete` | 删除单个字典类型 |
| 批量删除 | DELETE | `/system/dict-type/delete-list` | `system:dict:delete` | 批量删除字典类型 |
| 分页查询 | GET | `/system/dict-type/page` | `system:dict:query` | 分页查询字典类型 |
| 查询详情 | GET | `/system/dict-type/get` | `system:dict:query` | 查询单个字典类型 |
| 获取全部（简单） | GET | `/system/dict-type/list-all-simple` | 无需权限 | 下拉框用，返回全部类型 |
| 导出 Excel | GET | `/system/dict-type/export-excel` | `system:dict:query` | 导出字典类型数据 |

### 1.2 核心实现

**创建字典类型：**

```java
@PostMapping("/create")
@PreAuthorize("@ss.hasPermission('system:dict:create')")
public CommonResult<Long> createDictType(@Valid @RequestBody DictTypeSaveReqVO createReqVO) {
    Long dictTypeId = dictTypeService.createDictType(createReqVO);
    return success(dictTypeId);
}
```

请求体 `DictTypeSaveReqVO` 包含 `name`（字典名称）、`type`（字典类型编码）、`status`（状态）、`remark`（备注）。其中 `name` 和 `type` 必填，且有长度限制（不超过 100 字符）。

**list-all-simple 接口的特殊设计：**

```java
@GetMapping(value = {"/list-all-simple", "simple-list"})
// 无需添加权限认证，因为前端全局都需要
public CommonResult<List<DictTypeSimpleRespVO>> getSimpleDictTypeList() {
    List<DictTypeDO> list = dictTypeService.getDictTypeList();
    return success(BeanUtils.toBean(list, DictTypeSimpleRespVO.class));
}
```

这个接口有两个关键特点：

1. **无需权限认证**——前端几乎所有页面都可能需要字典下拉框，如果加权限会增加配置负担
2. **返回 `DictTypeSimpleRespVO`**——只包含 `id`、`name`、`type`、`status` 等必要字段，不返回 `remark` 等冗余信息
3. **双 URL 映射**——`/list-all-simple` 和 `/simple-list` 都能访问，兼容不同前端版本

### 1.3 请求/响应 VO

**DictTypeSaveReqVO（创建/修改请求）：**

```java
@Data
public class DictTypeSaveReqVO {
    private Long id;        // 编号（修改时必填）
    @NotBlank private String name;   // 字典名称，如"性别"
    @NotNull  private String type;   // 字典类型编码，如"sys_common_sex"
    @NotNull  private Integer status; // 状态
    private String remark;           // 备注
}
```

**DictTypeRespVO（完整响应）：** 包含所有字段加上 `createTime`。

**DictTypeSimpleRespVO（简单响应）：** 仅包含 `id`、`name`、`type`、`status`、`remark`，用于下拉选择框。

## 二、字典类型服务层：DictTypeServiceImpl

> 源码位置：`yudao-module-system/yudao-module-system-server/src/main/java/cn/iocoder/yudao/module/system/service/dict/DictTypeServiceImpl.java`

### 2.1 创建字典类型

```java
@Override
public Long createDictType(DictTypeSaveReqVO createReqVO) {
    // 1. 校验字典名称唯一
    validateDictTypeNameUnique(null, createReqVO.getName());
    // 2. 校验字典类型编码唯一
    validateDictTypeUnique(null, createReqVO.getType());

    // 3. 插入字典类型
    DictTypeDO dictType = BeanUtils.toBean(createReqVO, DictTypeDO.class);
    dictType.setDeletedTime(LocalDateTimeUtils.EMPTY); // 唯一索引，避免 null 值
    dictTypeMapper.insert(dictType);
    return dictType.getId();
}
```

注意 `deletedTime` 被设置为 `LocalDateTimeUtils.EMPTY`（一个特殊的时间值，如 `1970-01-01 00:00:00`），而不是 null。这是因为数据库的唯一索引在遇到 null 值时行为不一致——MySQL 中多个 null 值不会违反唯一约束，但 PostgreSQL 中会。使用空值代替 null 可以保证跨数据库的唯一索引正确工作。

### 2.2 删除字典类型

```java
@Override
public void deleteDictType(Long id) {
    // 1. 校验是否存在
    DictTypeDO dictType = validateDictTypeExists(id);
    // 2. 校验是否有字典数据
    if (dictDataService.getDictDataCountByDictType(dictType.getType()) > 0) {
        throw exception(DICT_TYPE_HAS_CHILDREN);
    }
    // 3. 软删除
    dictTypeMapper.updateToDelete(id, LocalDateTime.now());
}
```

删除流程有三个关键步骤：

1. **存在性校验**——防止删除不存在的记录
2. **子数据校验**——如果该字典类型下还有字典数据，拒绝删除，抛出 `DICT_TYPE_HAS_CHILDREN` 错误
3. **软删除**——通过 `updateToDelete()` 设置 `deleted = 1` 和 `deleted_time`

### 2.3 唯一性校验

```java
@VisibleForTesting
void validateDictTypeNameUnique(Long id, String name) {
    DictTypeDO dictType = dictTypeMapper.selectByName(name);
    if (dictType == null) {
        return; // 不存在同名，通过
    }
    if (id == null) {
        throw exception(DICT_TYPE_NAME_DUPLICATE); // 新增时发现重名
    }
    if (!dictType.getId().equals(id)) {
        throw exception(DICT_TYPE_NAME_DUPLICATE); // 修改时发现重名（且不是自己）
    }
}
```

这个校验模式在项目中非常常见，值得记住：

- `id == null` 表示新增操作，只要找到同名记录就报错
- `id != null` 表示修改操作，只有当同名记录不是自己时才报错（允许不修改名称）

`validateDictTypeUnique()` 对 `type` 编码的校验逻辑完全相同。

### 2.4 Mapper 层实现

> 源码位置：`yudao-module-system/yudao-module-system-server/src/main/java/cn/iocoder/yudao/module/system/dal/mysql/dict/DictTypeMapper.java`

```java
@Mapper
public interface DictTypeMapper extends BaseMapperX<DictTypeDO> {

    default PageResult<DictTypeDO> selectPage(DictTypePageReqVO reqVO) {
        return selectPage(reqVO, new LambdaQueryWrapperX<DictTypeDO>()
                .likeIfPresent(DictTypeDO::getName, reqVO.getName())
                .likeIfPresent(DictTypeDO::getType, reqVO.getType())
                .eqIfPresent(DictTypeDO::getStatus, reqVO.getStatus())
                .betweenIfPresent(DictTypeDO::getCreateTime, reqVO.getCreateTime())
                .orderByDesc(DictTypeDO::getId));
    }

    default DictTypeDO selectByType(String type) {
        return selectOne(DictTypeDO::getType, type);
    }

    @Update("UPDATE system_dict_type SET deleted = 1, deleted_time = #{deletedTime} WHERE id = #{id}")
    void updateToDelete(@Param("id") Long id, @Param("deletedTime") LocalDateTime deletedTime);
}
```

`selectPage()` 使用了 `LambdaQueryWrapperX` 的 `likeIfPresent` 和 `eqIfPresent` 方法——当请求参数为空时自动跳过该条件，无需手动写 `if` 判断。这是框架提供的便捷查询能力。

## 四、字典数据服务层：DictDataServiceImpl

> 源码位置：`yudao-module-system/yudao-module-system-server/src/main/java/cn/iocoder/yudao/module/system/service/dict/DictDataServiceImpl.java`

### 4.1 创建字典数据

```java
@Override
public Long createDictData(DictDataSaveReqVO createReqVO) {
    // 1. 校验字典类型存在且启用
    validateDictTypeExists(createReqVO.getDictType());
    // 2. 校验字典值在同一类型下唯一
    validateDictDataValueUnique(null, createReqVO.getDictType(), createReqVO.getValue());

    // 3. 插入字典数据
    DictDataDO dictData = BeanUtils.toBean(createReqVO, DictDataDO.class);
    dictDataMapper.insert(dictData);
    return dictData.getId();
}
```

创建字典数据时有两个关键校验：

1. **字典类型有效性**——通过 `validateDictTypeExists()` 检查字典类型是否存在且状态为启用
2. **值唯一性**——同一字典类型下，`value` 不能重复（比如"订单状态"下不能有两个 `value = "0"` 的数据）

### 4.2 字典类型有效性校验

```java
@VisibleForTesting
public void validateDictTypeExists(String type) {
    DictTypeDO dictType = dictTypeService.getDictType(type);
    if (dictType == null) {
        throw exception(DICT_TYPE_NOT_EXISTS);
    }
    if (!CommonStatusEnum.ENABLE.getStatus().equals(dictType.getStatus())) {
        throw exception(DICT_TYPE_NOT_ENABLE);
    }
}
```

注意这里不仅检查字典类型是否存在，还检查其状态是否为"启用"。禁用的字典类型不允许新增数据，这是一种业务约束。

### 4.3 值唯一性校验

```java
@VisibleForTesting
public void validateDictDataValueUnique(Long id, String dictType, String value) {
    DictDataDO dictData = dictDataMapper.selectByDictTypeAndValue(dictType, value);
    if (dictData == null) {
        return;
    }
    if (id == null) {
        throw exception(DICT_DATA_VALUE_DUPLICATE);
    }
    if (!dictData.getId().equals(id)) {
        throw exception(DICT_DATA_VALUE_DUPLICATE);
    }
}
```

这与字典类型的唯一性校验模式完全一致：新增时只要存在就报错，修改时只有当重复记录不是自己时才报错。

### 4.4 批量校验字典数据

```java
@Override
public void validateDictDataList(String dictType, Collection<String> values) {
    if (CollUtil.isEmpty(values)) {
        return;
    }
    Map<String, DictDataDO> dictDataMap = CollectionUtils.convertMap(
            dictDataMapper.selectByDictTypeAndValues(dictType, values), DictDataDO::getValue);
    values.forEach(value -> {
        DictDataDO dictData = dictDataMap.get(value);
        if (dictData == null) {
            throw exception(DICT_DATA_NOT_EXISTS);
        }
        if (!CommonStatusEnum.ENABLE.getStatus().equals(dictData.getStatus())) {
            throw exception(DICT_DATA_NOT_ENABLE, dictData.getLabel());
        }
    });
}
```

这个方法供其他业务模块调用，用于批量校验字典值是否合法。比如电商系统创建订单时，需要校验传入的"订单状态"值是否在字典中存在且启用。

## 五、数据访问层：DictDataMapper

> 源码位置：`yudao-module-system/yudao-module-system-server/src/main/java/cn/iocoder/yudao/module/system/dal/mysql/dict/DictDataMapper.java`

```java
@Mapper
public interface DictDataMapper extends BaseMapperX<DictDataDO> {

    default DictDataDO selectByDictTypeAndValue(String dictType, String value) {
        return selectOne(DictDataDO::getDictType, dictType, DictDataDO::getValue, value);
    }

    default long selectCountByDictType(String dictType) {
        return selectCount(DictDataDO::getDictType, dictType);
    }

    default PageResult<DictDataDO> selectPage(DictDataPageReqVO reqVO) {
        return selectPage(reqVO, new LambdaQueryWrapperX<DictDataDO>()
                .likeIfPresent(DictDataDO::getLabel, reqVO.getLabel())
                .eqIfPresent(DictDataDO::getDictType, reqVO.getDictType())
                .eqIfPresent(DictDataDO::getStatus, reqVO.getStatus())
                .orderByDesc(Arrays.asList(DictDataDO::getDictType, DictDataDO::getSort)));
    }
}
```

`selectPage()` 的排序逻辑是先按 `dictType` 降序，再按 `sort` 升序——同一个字典类型下的数据按排序值排列，不同字典类型之间按类型编码分组。

## 六、电商场景实战示例

### 6.1 定义订单状态字典

**第一步：创建字典类型**

```
POST /system/dict-type/create
{
    "name": "订单状态",
    "type": "trade_order_status",
    "status": 0,
    "remark": "电商交易订单的状态枚举"
```

```
POST /system/dict-data/create

    "sort": 1,

    "colorType": "warning",
    "remark": "用户下单后等待支付"

{
    "sort": 2,
    "label": "已付款",
```

## 八、小结

| 要点 | 说明 |
|------|------|
| 权限设计 | 字典类型和字典数据共用 `system:dict:*` 权限前缀 |
| list-all-simple | 无需权限，前端全局缓存用 |
| 删除保护 | 删除字典类型前检查是否有子数据，有则拒绝 |
| 唯一性校验 | 字典类型名称/编码唯一，字典值在同一类型下唯一 |
| 排序 | 字典数据按 `sort` 升序，前端展示时自动有序 |
