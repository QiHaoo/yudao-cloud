## 设计与概念精要

### 核心设计决策

1. **字典全局共享**：`@TenantIgnore` 让所有租户共享字典，避免重复配置
2. **字典类型软删除**：保留数据完整性，防止孤儿字典数据
3. **configKey 命名**：使用 `configKey` 而非 `key`，避免 SQL 保留字冲突
4. **配置类型区分**：SYSTEM 配置不可删除，CUSTOM 配置自由管理
5. **visible 控制**：敏感配置（密码、Key）不返回前端

### 核心实体

| 实体 | 模块 | 说明 | 关键字段 |
|------|------|------|---------|
| DictTypeDO | system | 字典类型 | name, type, status |
| DictDataDO | system | 字典数据 | label, value, dictType, colorType, cssClass |
| ConfigDO | infra | 系统配置 | category, name, configKey, value, type, visible |

### 字典 vs 配置

| 维度 | 字典 | 配置 |
|------|------|------|
| 用途 | 代码→标签映射 | 系统参数键值 |
| 结构 | 一对多 | 一对一 |
| 示例 | 性别：0=男, 1=女 | sys.user.init-password=123456 |
| 模块 | system | infra |
