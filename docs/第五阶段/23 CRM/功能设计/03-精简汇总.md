# 03 - 精简汇总

## CRM 流程速查

### 销售漏斗全流程

```
线索 ──转化──> 客户 ──跟进──> 商机 ──签约──> 合同 ──回款──> 完成
 │              │              │              │
 └── 公海/私海  └── 公海/私海  └── 阶段流转  └── 审批
```

### 客户池运作流程

```
公海池
  │
  │  领取（校验数量限制）
  ▼
销售私海 ──超时未跟进──> 回收到公海
  │
  │  转移（更新 ownerUserId + PermissionDO）
  ▼
另一个销售私海
```

### 商机阶段流转

```
初步接触(10%) → 需求确认(20%) → 方案报价(40%) → 商务谈判(60%) → 合同签订(80%) → 成交/丢失(100%)
```

### 合同审批流程

```
创建合同 → 提交审批 → 主管审批 → (总监审批) → 审核通过/驳回
                          │
                          └── 关联工作流引擎 (processInstanceId)
```

### 回款流程

```
创建回款记录 → 提交审批 → 财务确认 → 审核通过 → 更新合同已回款金额
```

## 核心设计决策

| 决策 | 选择 | 放弃的方案 | 原因 |
|------|------|-----------|------|
| 公海/私海判断 | ownerUserId 是否为 null | 显式 poolType 字段 | 语义更自然，减少冗余字段 |
| 跟进记录 | 独立表 CrmFollowUpRecordDO | 客户表的字段 | 支持完整历史，支持时间线展示 |
| 跟进关联 | bizType + bizId 多态 | 每种业务独立跟进表 | 通用设计，减少表数量 |
| CRM 权限 | CrmPermissionDO 专属表 | 复用系统数据权限 | 支持动态转移、多人协作 |
| 销售阶段 | 配置表 CrmBusinessStatusDO | 枚举硬编码 | 不同行业可自定义阶段 |
| 产品信息 | 合同/商机产品表冗余存储 | 只存 productId | 防止产品变更影响历史记录 |
| 合同审批 | 集成 BPM 工作流 | 自建审批逻辑 | 复用通用审批引擎，支持流程配置 |
| 客户回收 | 定时任务 + 配置化规则 | 手动回收 | 自动化运营，减少人为干预 |

## 模块集成关系

```
CRM
  │
  ├── ◄── System（用户、角色、部门、数据权限）
  │
  ├── ◄── Pay（合同收款、在线支付）
  │
  ├── ◄── BPM（合同/回款审批流程）
  │
  ├── ──> ERP（客户同步、订单关联）
  │
  └── ──> 消息通知（客户状态变更通知）
```

## 权限模型速查

```
Level 1: 菜单权限 (System 模块)
  ├── crm:customer:query    查看客户
  ├── crm:customer:create   新建客户
  ├── crm:customer:update   编辑客户
  └── crm:customer:delete   删除客户

Level 2: 数据权限 (System 模块)
  ├── 全部数据权限  ────  销售总监
  ├── 部门数据权限  ────  销售经理
  └── 个人数据权限  ────  普通销售

Level 3: CRM 专属权限 (CrmPermissionDO)
  ├── 负责人 (1)   ──── 完全控制
  ├── 协作者 (2)   ──── 可编辑
  └── 只读   (3)   ──── 只能查看
```
