# 01-起步：客户与商机实体

本章实现 CRM 模块的核心实体和客户池机制，让读者理解公海/私海模型的代码实现。

> **前置阅读**：本章对应功能设计中的「02-核心设计：客户与商机管理」中客户池部分。如果对公海/私海的设计动机还不清楚，建议先阅读功能设计。

## 核心实体：CrmCustomerDO

源码位置：`yudao-module-crm-server/.../dal/dataobject/customer/CrmCustomerDO.java`

```java
@TableName(value = "crm_customer")
@KeySequence("crm_customer_seq")
@Data
@EqualsAndHashCode(callSuper = true)
@Builder
@NoArgsConstructor
@AllArgsConstructor
public class CrmCustomerDO extends BaseDO {

    @TableId
    private Long id;
    private String name;              // 客户名称

    // ========== 跟进状态 ==========
    private Boolean followUpStatus;   // 跟进状态
    private LocalDateTime contactLastTime;   // 最后跟进时间
    private String contactLastContent;// 最后跟进内容
    private LocalDateTime contactNextTime;   // 下次联系时间

    // ========== 负责人 ==========
    private Long ownerUserId;         // 负责人的用户编号（null = 公海客户）
    private LocalDateTime ownerTime;  // 成为负责人的时间

    // ========== 状态标记 ==========
    private Boolean lockStatus;       // 锁定状态（锁定后不回收）
    private Boolean dealStatus;       // 成交状态

    // ========== 联系方式 ==========
    private String mobile;            // 手机
    private String telephone;         // 电话
    private String qq;                // QQ
    private String wechat;            // 微信
    private String email;             // 邮箱

    // ========== 基础信息 ==========
    private Integer areaId;           // 所在地
    private String detailAddress;     // 详细地址
    private Integer industryId;       // 所属行业（字典）
    private Integer level;            // 客户等级（字典）
    private Integer source;           // 客户来源（字典）
    private String remark;            // 备注
}
```

**关键设计：`ownerUserId` 是公海/私海的判断依据。**

- `ownerUserId = null`：客户在公海池，任何人都可以领取
- `ownerUserId = 某销售ID`：客户在该销售的私海中

这种设计比用一个显式的 `poolType` 字段更自然——负责人本身就是"归属"的语义。

## 客户创建流程

源码位置：`yudao-module-crm-server/.../service/customer/CrmCustomerServiceImpl.java`

```java
@Override
@Transactional(rollbackFor = Exception.class)
@LogRecord(type = CRM_CUSTOMER_TYPE, subType = CRM_CUSTOMER_CREATE_SUB_TYPE,
        bizNo = "{{#customer.id}}", success = CRM_CUSTOMER_CREATE_SUCCESS)
public Long createCustomer(CrmCustomerSaveReqVO createReqVO, Long userId) {
    createReqVO.setId(null);
    // 1. 校验拥有客户是否到达上限
    validateCustomerExceedOwnerLimit(createReqVO.getOwnerUserId(), 1);

    // 2. 插入客户
    CrmCustomerDO customer = initCustomer(createReqVO, userId);
    customerMapper.insert(customer);

    // 3. 创建数据权限
    permissionService.createPermission(new CrmPermissionCreateReqBO()
            .setBizType(CrmBizTypeEnum.CRM_CUSTOMER.getType())
            .setBizId(customer.getId())
            .setUserId(userId)
            .setLevel(CrmPermissionLevelEnum.OWNER.getLevel()));

    // 4. 记录操作日志上下文
    LogRecordContext.putVariable("customer", customer);
    return customer.getId();
}
```

**设计要点**：

- 创建客户前先校验负责人是否达到客户数量上限
- 创建客户的同时自动创建 CRM 数据权限记录，将当前操作人设为负责人
- 使用 `@LogRecord` 注解记录操作日志，支持审计追溯

## 客户转移流程

客户转移是 CRM 最核心的操作之一，涉及客户、联系人、商机、合同的联动转移：

```java
@Override
@Transactional(rollbackFor = Exception.class)
@LogRecord(type = CRM_CUSTOMER_TYPE, subType = CRM_CUSTOMER_TRANSFER_SUB_TYPE,
        bizNo = "{{#reqVO.id}}", success = CRM_CUSTOMER_TRANSFER_SUCCESS)
@CrmPermission(bizType = CrmBizTypeEnum.CRM_CUSTOMER, bizId = "#reqVO.id",
        level = CrmPermissionLevelEnum.OWNER)
public void transferCustomer(CrmCustomerTransferReqVO reqVO, Long userId) {
    // 1. 校验客户是否存在
    CrmCustomerDO customer = validateCustomerExists(reqVO.getId());
    // 2. 校验新负责人是否达到上限
    validateCustomerExceedOwnerLimit(reqVO.getNewOwnerUserId(), 1);

    // 3. 数据权限转移
    permissionService.transferPermission(new CrmPermissionTransferReqBO(
            userId, CrmBizTypeEnum.CRM_CUSTOMER.getType(),
            reqVO.getId(), reqVO.getNewOwnerUserId(),
            reqVO.getOldOwnerPermissionLevel()));

    // 4. 更新客户负责人
    customerMapper.updateById(new CrmCustomerDO().setId(reqVO.getId())
            .setOwnerUserId(reqVO.getNewOwnerUserId())
            .setOwnerTime(LocalDateTime.now()));

    // 5. 同时转移关联数据
    if (CollUtil.isNotEmpty(reqVO.getToBizTypes())) {
        transfer(reqVO, userId);
    }
}
```

关联数据的联动转移：

```java
private void transfer(CrmCustomerTransferReqVO reqVO, Long userId) {
    // 转移联系人
    if (reqVO.getToBizTypes().contains(CrmBizTypeEnum.CRM_CONTACT.getType())) {
        List<CrmContactDO> contactList = contactService
                .getContactListByCustomerIdOwnerUserId(reqVO.getId(), userId);
        contactList.forEach(item -> contactService.transferContact(
                new CrmContactTransferReqVO(item.getId(),
                        reqVO.getNewOwnerUserId(),
                        reqVO.getOldOwnerPermissionLevel()), userId));
    }
    // 转移商机
    if (reqVO.getToBizTypes().contains(CrmBizTypeEnum.CRM_BUSINESS.getType())) {
        List<CrmBusinessDO> businessList = businessService
                .getBusinessListByCustomerIdOwnerUserId(reqVO.getId(), userId);
        businessList.forEach(item -> businessService.transferBusiness(
                new CrmBusinessTransferReqVO(item.getId(),
                        reqVO.getNewOwnerUserId(),
                        reqVO.getOldOwnerPermissionLevel()), userId));
    }
    // 转移合同
    if (reqVO.getToBizTypes().contains(CrmBizTypeEnum.CRM_CONTRACT.getType())) {
        List<CrmContractDO> contractList = contractService
                .getContractListByCustomerIdOwnerUserId(reqVO.getId(), userId);
        contractList.forEach(item -> contractService.transferContract(
                new CrmContractTransferReqVO(item.getId(),
                        reqVO.getNewOwnerUserId(),
                        reqVO.getOldOwnerPermissionLevel()), userId));
    }
}
```

## 客户锁定机制

锁定客户可以防止被自动回收：

```java
@Override
@LogRecord(type = CRM_CUSTOMER_TYPE, subType = CRM_CUSTOMER_LOCK_SUB_TYPE,
        bizNo = "{{#lockReqVO.id}}", success = CRM_CUSTOMER_LOCK_SUCCESS)
@CrmPermission(bizType = CrmBizTypeEnum.CRM_CUSTOMER, bizId = "#lockReqVO.id",
        level = CrmPermissionLevelEnum.OWNER)
public void lockCustomer(CrmCustomerLockReqVO lockReqVO, Long userId) {
    // 1. 校验客户是否存在
    CrmCustomerDO customer = validateCustomerExists(lockReqVO.getId());
    // 2. 校验是否重复操作
    if (customer.getLockStatus().equals(lockReqVO.getLockStatus())) {
        throw exception(customer.getLockStatus()
                ? CUSTOMER_LOCK_FAIL_IS_LOCK : CUSTOMER_UNLOCK_FAIL_IS_UNLOCK);
    }
    // 3. 校验锁定上限
    if (lockReqVO.getLockStatus()) {
        validateCustomerExceedLockLimit(userId);
    }
    // 4. 更新锁定状态
    customerMapper.updateById(BeanUtils.toBean(lockReqVO, CrmCustomerDO.class));
}
```

源码位置：`yudao-module-crm-server/.../dal/dataobject/business/CrmBusinessDO.java`

```java
@TableName("crm_business")
@KeySequence("crm_business_seq")
@Data
@EqualsAndHashCode(callSuper = true)
@Builder
@NoArgsConstructor
@AllArgsConstructor
public class CrmBusinessDO extends BaseDO {

    @TableId
    private Long id;
    private String name;              // 商机名称
    private Long customerId;          // 客户编号，关联 CrmCustomerDO

    // ========== 跟进信息 ==========
    private Boolean followUpStatus;   // 跟进状态
    private LocalDateTime contactLastTime;   // 最后跟进时间
    private LocalDateTime contactNextTime;   // 下次联系时间
    private Long ownerUserId;         // 负责人

    // ========== 商机阶段 ==========
    private Long statusTypeId;        // 商机状态组编号，关联 CrmBusinessStatusTypeDO
    private Long statusId;            // 商机状态编号，关联 CrmBusinessStatusDO
    private Integer endStatus;        // 结束状态（赢单/输单）
    private String endRemark;         // 结束备注

    // ========== 金额信息 ==========
    private LocalDateTime dealTime;   // 预计成交日期
    private BigDecimal totalProductPrice; // 产品总金额
    private BigDecimal discountPercent;   // 整单折扣
    private BigDecimal totalPrice;    // 商机总金额
    private String remark;            // 备注
}
```

**设计要点**：

- 商机通过 `statusTypeId` 和 `statusId` 关联销售阶段，支持自定义阶段配置
- `endStatus` 记录商机的最终结果（赢单或输单），用于销售漏斗统计
- 金额字段使用 `BigDecimal` 而非 `Integer`（分），因为 CRM 场景金额通常以"元"为单位

## CRM 数据权限：CrmPermissionDO

源码位置：`yudao-module-crm-server/.../dal/dataobject/permission/CrmPermissionDO.java`

```java
@TableName("crm_permission")
@KeySequence("crm_permission_seq")
@Data
@EqualsAndHashCode(callSuper = true)
@Builder
@NoArgsConstructor
@AllArgsConstructor
public class CrmPermissionDO extends BaseDO {

    @TableId
    private Long id;
    private Integer bizType;    // 业务类型（客户/商机/合同/回款），枚举 CrmBizTypeEnum
    private Long bizId;         // 业务 ID
    private Long userId;        // 用户编号
    private Integer level;      // 权限级别，枚举 CrmPermissionLevelEnum
}
```

权限级别：

| 级别 | 枚举值 | 说明 |
|------|--------|------|
| 负责人 | OWNER(1) | 完全控制（编辑、删除、转移） |
| 协作者 | WRITE(2) | 可查看、可编辑 |
| 只读 | READ(3) | 只能查看 |

**为什么 CRM 需要自己的权限表？**

系统模块的数据权限基于组织架构（部门/角色），但 CRM 的权限基于业务关系（负责人/协作者）。两者独立运作：
- 销售 A 把客户转移给销售 B，系统权限不变，但 CRM 权限需要更新
- 某个大客户需要多人协作，每个人有不同的权限级别

## @CrmPermission 注解

CRM 模块实现了自定义的权限注解，用于方法级别的权限控制：

```java
@CrmPermission(bizType = CrmBizTypeEnum.CRM_CUSTOMER, bizId = "#id",
        level = CrmPermissionLevelEnum.WRITE)
public void updateCustomer(CrmCustomerSaveReqVO updateReqVO) {
    // 只有对该客户有 WRITE 权限的用户才能执行
}
```

`bizId = "#id"` 使用 SpEL 表达式从方法参数中提取业务 ID，运行时由 AOP 切面校验当前用户是否有指定级别的权限。

## 小结

| 组件 | 职责 | 关键设计 |
|------|------|---------|
| CrmCustomerDO | 客户实体 | ownerUserId 为 null 表示公海客户 |
| CrmCustomerPoolConfigDO | 客户池配置 | 联系过期天数、成交过期天数 |
| CrmCustomerLimitConfigDO | 客户数量限制 | 全局/部门/个人三级限制 |
| CrmBusinessDO | 商机实体 | 可配置的销售阶段 |
| CrmPermissionDO | CRM 数据权限 | bizType + bizId + userId + level |
| @CrmPermission | 权限注解 | SpEL 表达式提取业务 ID |
