# 02-业务流程实现

本章实现 CRM 的核心业务流程：商机跟进、合同管理、回款管理。

> **前置阅读**：本章对应功能设计中的「02-核心设计：客户与商机管理」中商机、合同、回款部分。

## 商机管理

### 商机与产品关联

商机涉及哪些产品，通过 `CrmBusinessProductDO` 关联：

源码位置：`yudao-module-crm-server/.../dal/dataobject/business/CrmBusinessProductDO.java`

```
CrmBusinessDO (商机)
  └── 1:N ── CrmBusinessProductDO (商机产品)
                ├── productId      - 产品编号
                ├── productName    - 产品名称（冗余）
                ├── productPrice   - 产品单价
                ├── quantity       - 数量
                └── totalPrice     - 小计金额
```

**为什么冗余存储 productName 和 productPrice？**

因为产品信息可能会变（改名、调价），但商机创建时的价格应该固定。冗余存储保证了历史商机的产品信息不会因为产品变更而"失真"。这是与订单快照相同的设计思想。

### 商机创建

商机创建时需要同时关联产品明细：

```java
// CrmBusinessServiceImpl.createBusiness() 核心逻辑
// 1. 校验客户存在
// 2. 插入商机主表
// 3. 插入商机产品明细
// 4. 创建 CRM 数据权限
```

### 商机阶段流转

商机阶段通过 `statusId` 字段控制，流转时需要更新状态：

```java
// CrmBusinessServiceImpl.updateBusinessStatus() 核心逻辑
// 1. 校验商机存在
// 2. 校验目标状态是否合法（必须在同一个状态组内）
// 3. 更新 statusId
// 4. 如果是最终状态，更新 endStatus（赢单/输单）
```

商机阶段配置通过 `CrmBusinessStatusDO` 管理，支持自定义：

```java
@TableName("crm_business_status")
@Data
public class CrmBusinessStatusDO extends BaseDO {
    private Long id;
    private Long typeId;        // 状态组编号
    private String name;        // 阶段名称
    private Integer sort;       // 排序
    private Integer probability;// 赢单概率（百分比）
}
```

不同行业可以配置不同的销售阶段，如：
- 标准 B2B：初步接触 → 需求确认 → 方案报价 → 商务谈判 → 合同签订
- 电商招商：意向沟通 → 资质审核 → 样品确认 → 合同签订 → 入驻完成

## 跟进记录

### 实体设计

源码位置：`yudao-module-crm-server/.../dal/dataobject/followup/CrmFollowUpRecordDO.java`

跟进记录是 CRM 的"日志系统"——记录每一次与客户的沟通。它不是客户表的一个字段，而是独立的实体。

**为什么不放在客户表里？**

```
方案 A：客户表加一个 lastContactContent 字段
  ├── 优点：简单
  └── 缺点：只能存最后一次，历史记录丢失

方案 B：独立的跟进记录表（CrmFollowUpRecordDO）
  ├── 优点：完整的历史记录，支持时间线展示
  └── 缺点：多一张表，查询需要 JOIN
```

项目采用方案 B。跟进是一个持续的过程，不是一次性记录。

### 多态关联设计

跟进记录通过 `bizType` + `bizId` 实现多态关联，可以关联客户、商机、合同等多种业务实体：

```java
@TableName("crm_follow_up_record")
@Data
public class CrmFollowUpRecordDO extends BaseDO {
    private Long id;
    private Integer bizType;          // 业务类型（客户/商机/合同），枚举 CrmBizTypeEnum
    private Long bizId;               // 业务 ID
    private Integer type;             // 跟进方式（电话/拜访/邮件/微信）
    private String content;           // 跟进内容
    private LocalDateTime nextTime;   // 下次跟进时间
    private List<String> fileUrls;    // 附件（录音、照片等）
}
```

### 跟进与客户状态联动

创建跟进记录时，自动更新客户的跟进状态：

```java
// CrmFollowUpRecordServiceImpl.createFollowUpRecord() 核心逻辑
// 1. 创建 CrmFollowUpRecordDO
// 2. 更新关联客户的跟进状态
//    ├── CrmCustomerDO.contactLastTime = 当前时间
//    ├── CrmCustomerDO.contactLastContent = 跟进内容
//    └── CrmCustomerDO.contactNextTime = 下次跟进时间
// 3. 重置回收计时器（因为 contactLastTime 更新了）
```

这个联动机制保证了：只要销售持续跟进客户，客户就不会被自动回收到公海。

## 合同管理

### 合同实体

源码位置：`yudao-module-crm-server/.../dal/dataobject/contract/CrmContractDO.java`

```java
@TableName("crm_contract")
@KeySequence("crm_contract_seq")
@Data
@EqualsAndHashCode(callSuper = true)
@Builder
@NoArgsConstructor
@AllArgsConstructor
public class CrmContractDO extends BaseDO {

    @TableId
    private Long id;
    private String name;              // 合同名称
    private String no;                // 合同编号（业务编号）
    private Long customerId;          // 客户编号，关联 CrmCustomerDO
    private Long businessId;          // 商机编号（可选），关联 CrmBusinessDO

    // ========== 跟进信息 ==========
    private LocalDateTime contactLastTime;   // 最后跟进时间
    private Long ownerUserId;         // 负责人

    // ========== 审批信息 ==========
    private String processInstanceId; // 工作流实例编号（关联 BPM 模块）
    private Integer auditStatus;      // 审批状态，枚举 CrmAuditStatusEnum

    // ========== 合同信息 ==========
    private LocalDateTime orderDate;  // 下单日期
    private LocalDateTime startTime;  // 开始时间
    private LocalDateTime endTime;    // 结束时间
    private BigDecimal totalProductPrice; // 产品总金额
    private BigDecimal discountPercent;   // 整单折扣
    private BigDecimal totalPrice;    // 合同总金额
    private Long signContactId;       // 客户签约人，关联 CrmContactDO
    private Long signUserId;          // 公司签约人
    private String remark;            // 备注
}
```

### 合同产品明细

通过 `CrmContractProductDO` 关联合同中的产品：

源码位置：`yudao-module-crm-server/.../dal/dataobject/contract/CrmContractProductDO.java`

```
CrmContractDO (合同)
  └── 1:N ── CrmContractProductDO (合同产品)
                ├── productId      - 产品编号
                ├── productName    - 产品名称（冗余）
                ├── productPrice   - 产品单价
                ├── quantity       - 数量
                ├── totalPrice     - 小计金额
                └── discountPercent - 单品折扣
```

### 合同金额计算

```
合同总金额计算逻辑：
  totalProductPrice = SUM(每个产品的小计金额)
  discountPrice = totalProductPrice * discountPercent / 100
  totalPrice = totalProductPrice - discountPrice
```

### 合同审批流程

合同通常需要审批。CRM 模块与工作流模块（BPM）集成：

```
销售创建合同
  │
  ▼
提交审批
  │
  ├── 创建 ProcessInstance（工作流实例）
  ├── 设置 CrmContractDO.processInstanceId
  └── 设置 CrmContractDO.auditStatus = 审核中

审批流程（由 BPM 模块驱动）
  │
  ├── 销售主管审批
  │     ├── 通过 → 继续
  │     └── 驳回 → auditStatus = 驳回，销售可修改后重新提交
  │
  └── 总监审批（金额超过阈值时）
        ├── 通过 → auditStatus = 审核通过
        └── 驳回 → auditStatus = 驳回
```

审批状态通过 `CrmAuditStatusEnum` 枚举定义：

| 状态值 | 含义 |
|--------|------|
| 0 | 未审批 |
| 1 | 审批中 |
| 2 | 审批通过 |
| 3 | 审批驳回 |

### 合同配置

通过 `CrmContractConfigDO` 管理合同的审批配置：

源码位置：`yudao-module-crm-server/.../dal/dataobject/contract/CrmContractConfigDO.java`

## 回款管理

### 回款实体

源码位置：`yudao-module-crm-server/.../dal/dataobject/receivable/CrmReceivableDO.java`

```java
@TableName("crm_receivable")
@KeySequence("crm_receivable_seq")
@Data
@EqualsAndHashCode(callSuper = true)
@Builder
@NoArgsConstructor
@AllArgsConstructor
public class CrmReceivableDO extends BaseDO {

    @TableId
    private Long id;
    private String no;                // 回款编号
    private Long planId;              // 回款计划编号（可选），关联 CrmReceivablePlanDO
    private Long customerId;          // 客户编号，关联 CrmCustomerDO
    private Long contractId;          // 合同编号，关联 CrmContractDO
    private Long ownerUserId;         // 负责人

    private LocalDateTime returnTime; // 回款日期
    private Integer returnType;       // 回款方式（现金/转账/支票）
    private BigDecimal price;         // 回款金额，单位：元
    private String remark;            // 备注

    private String processInstanceId; // 工作流实例编号
    private Integer auditStatus;      // 审批状态
}
```

### 回款与合同的关系

```
CrmContractDO (合同：总金额 100,000 元)
  │
  ├── 1:N ── CrmReceivableDO (回款记录)
  │           ├── 回款 1：30,000 元（定金）
  │           ├── 回款 2：40,000 元（发货后）
  │           └── 回款 3：30,000 元（验收后）
  │
  └── 已回款总额 = SUM(receivable.price)
      剩余待回款 = totalPrice - 已回款总额
```

### 回款计划

通过 `CrmReceivablePlanDO` 管理回款计划：

源码位置：`yudao-module-crm-server/.../dal/dataobject/receivable/CrmReceivablePlanDO.java`

回款计划定义了合同约定的分期付款安排，实际回款时关联计划编号。

### 回款审批

回款也需要审批，防止虚假回款：

```
销售录入回款记录
  │
  ▼
提交审批
  │
  ├── 财务确认到账
  │     ├── 确认 → auditStatus = 审核通过
  │     └── 未到账 → auditStatus = 驳回
  │
  └── 更新合同的已回款金额
```

## 线索管理

### 线索实体

源码位置：`yudao-module-crm-server/.../dal/dataobject/clue/CrmClueDO.java`

```java
@TableName("crm_clue")
@KeySequence("crm_clue_seq")
@Data
@EqualsAndHashCode(callSuper = true)
@Builder
@NoArgsConstructor
@AllArgsConstructor
public class CrmClueDO extends BaseDO {

    @TableId
    private Long id;
    private String name;              // 线索名称（客户名称）
    private String contactName;       // 联系人名称
    private String contactPhone;      // 联系电话
    private String contactEmail;      // 联系邮箱
    private Integer source;           // 线索来源
    private Long ownerUserId;         // 负责人
    private Integer status;           // 状态（待跟进/已转化/已关闭）
    private String remark;            // 备注
}
```

### 线索转化流程

```
线索（CrmClueDO）
  │
  ├── 确认有合作意向
  │
  ▼
转化操作
  │
  ├── 1. 创建 CrmCustomerDO（从线索信息复制）
  ├── 2. 创建 CrmContactDO（联系人）
  ├── 3. 创建 CrmPermissionDO（分配权限）
  ├── 4. 更新线索状态为"已转化"
  └── 5. 可选：自动创建商机
```

线索和客户分离的好处：
- 避免销售随意创建客户，污染客户库
- 线索可以批量导入，经过筛选后再转化为客户
- 线索转化过程本身就是一次质量筛选

## 统计分析

CRM 模块提供了丰富的统计分析能力：

| Service | 说明 |
|---------|------|
| CrmStatisticsCustomerServiceImpl | 客户统计（新增、转化、流失） |
| CrmStatisticsFunnelServiceImpl | 销售漏斗（各阶段转化率） |
| CrmStatisticsPerformanceServiceImpl | 业绩统计（签约金额、回款金额） |
| CrmStatisticsPortraitServiceImpl | 客户画像（行业、等级、来源分布） |
| CrmStatisticsRankServiceImpl | 排行榜（签约排行、回款排行） |

## 小结

| 组件 | 职责 | 关键设计 |
|------|------|---------|
| CrmBusinessDO | 商机 | 可配置的销售阶段 + 赢单概率 |
| CrmBusinessProductDO | 商机产品 | 冗余产品信息（快照模式） |
| CrmFollowUpRecordDO | 跟进记录 | bizType + bizId 多态关联 |
| CrmContractDO | 合同 | BPM 集成审批流程 |
| CrmContractProductDO | 合同产品 | 冗余产品信息 + 单品折扣 |
| CrmReceivableDO | 回款 | 关联回款计划 + 审批流程 |
| CrmClueDO | 线索 | 转化后生成客户+联系人+权限 |
