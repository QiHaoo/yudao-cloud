# 03-测试支持与源码索引

## 测试支持

### 现状

CRM 模块（yudao-module-crm）目前没有独立的单元测试目录。业务逻辑主要通过集成测试和手动测试验证。

### 测试策略建议

CRM 模块的测试挑战在于**权限校验**和**数据联动**。建议按以下层次组织测试：

**1. 权限层测试**：
- 验证 `@CrmPermission` 注解的 SpEL 表达式解析
- 验证 CrmPermissionDO 的 CRUD 操作
- 验证客户转移时权限的联动更新

**2. Service 层测试**：
- 客户创建：验证客户数量上限校验
- 客户转移：验证联系人、商机、合同的联动转移
- 商机阶段流转：验证状态变更的合法性
- 跟进记录：验证客户跟进状态的自动更新

**3. 客户池测试**：
- 客户回收：Mock 定时任务，验证超时未跟进的客户被回收
- 客户领取：验证数量上限校验
- 锁定机制：验证锁定客户不被回收

### Mock 建议

```java
// 示例：测试客户转移时 Mock 权限服务
@MockBean
private CrmPermissionService permissionService;

@Test
public void testTransferCustomer() {
    // given
    CrmCustomerTransferReqVO reqVO = new CrmCustomerTransferReqVO()
            .setId(1L).setNewOwnerUserId(2L);
    // when
    customerService.transferCustomer(reqVO, 1L);
    // then
    verify(permissionService).transferPermission(any());
}
```

### 测试数据准备

- 客户数据：准备公海客户和私海客户的测试数据
- 商机数据：准备不同阶段的商机测试数据
- 合同数据：准备不同审批状态的合同测试数据
- 权限数据：准备不同级别的权限测试数据
