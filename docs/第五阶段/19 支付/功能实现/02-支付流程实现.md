# 02 - 支付流程实现

## 本章目标

实现支付模块的三大核心流程：支付下单、回调通知、退款。完成后，你将理解：

1. "创建订单"与"提交支付"为什么要分成两步
2. 支付回调的幂等处理机制
3. 退款的状态流转与对账补偿

---

## 一、支付下单流程

支付下单分为两步：创建支付单 + 提交支付。

### 1.1 创建支付单 -- PayOrderService.createOrder()

业务系统（如电商模块）通过 Feign 调用 `PayOrderApi.createOrder()` 创建支付单。

```java
// 来源：yudao-module-pay-server/.../service/order/PayOrderServiceImpl.java
@Override
public Long createOrder(PayOrderCreateReqDTO reqDTO) {
    // 1. 校验 App
    PayAppDO app = appService.validPayApp(reqDTO.getAppKey());

    // 2. 查询对应的支付交易单是否已经存在（幂等）
    PayOrderDO order = orderMapper.selectByAppIdAndMerchantOrderId(
            app.getId(), reqDTO.getMerchantOrderId());
    if (order != null) {
        log.warn("[createOrder][appId({}) merchantOrderId({}) 已经存在对应的支付单({})]",
                order.getAppId(), order.getMerchantOrderId(), toJsonString(order));
        return order.getId();
    }

    // 3. 创建支付交易单
    order = PayOrderConvert.INSTANCE.convert(reqDTO).setAppId(app.getId())
            .setNotifyUrl(app.getOrderNotifyUrl())
            .setStatus(PayOrderStatusEnum.WAITING.getStatus())
            .setRefundPrice(0);
    orderMapper.insert(order);
    return order.getId();
}
```

设计要点：
- **幂等性**：通过 `appId + merchantOrderId` 查询是否已存在，避免重复创建。
- **状态初始化**：新创建的订单状态为 `WAITING`（待支付）。
- **回调地址**：从 `PayAppDO` 中获取，而非由业务系统传入，保证回调地址的统一管理。

### 1.2 提交支付 -- PayOrderService.submitOrder()

用户选择支付渠道后，调用 `submitOrder()` 发起实际的渠道调用。

```java
// 来源：yudao-module-pay-server/.../service/order/PayOrderServiceImpl.java
@Override
public PayOrderSubmitRespVO submitOrder(PayOrderSubmitReqVO reqVO, String userIp) {
    // 1. 校验支付单和渠道
    PayOrderDO order = validateOrderCanSubmit(reqVO.getId());
    PayChannelDO channel = validateChannelCanSubmit(order.getAppId(), reqVO.getChannelCode());
    PayClient<?> client = channelService.getPayClient(channel.getId());

    // 2. 插入 PayOrderExtensionDO（扩展记录）
    String no = noRedisDAO.generate(payProperties.getOrderNoPrefix());
    PayOrderExtensionDO orderExtension = PayOrderConvert.INSTANCE.convert(reqVO, userIp)
            .setOrderId(order.getId()).setNo(no)
            .setChannelId(channel.getId()).setChannelCode(channel.getCode())
            .setStatus(PayOrderStatusEnum.WAITING.getStatus());
    orderExtensionMapper.insert(orderExtension);

    // 3. 调用三方接口
    PayOrderUnifiedReqDTO unifiedOrderReqDTO = PayOrderConvert.INSTANCE.convert2(reqVO, userIp)
            .setOutTradeNo(orderExtension.getNo())  // 使用扩展单的 no
            .setSubject(order.getSubject()).setBody(order.getBody())
            .setNotifyUrl(genChannelOrderNotifyUrl(channel))
            .setReturnUrl(reqVO.getReturnUrl())
            .setPrice(order.getPrice()).setExpireTime(order.getExpireTime());
    PayOrderRespDTO unifiedOrderResp = client.unifiedOrder(unifiedOrderReqDTO);

    // 4. 如果直接支付成功（如付款码支付），更新状态
    if (unifiedOrderResp != null) {
        try {
            getSelf().notifyOrder(channel, unifiedOrderResp);
        } catch (Exception e) {
            log.warn("[submitOrder][order({}) channel({}) 支付结果({}) 通知时发生异常]",
                    order, channel, unifiedOrderResp, e);
        }
        if (StrUtil.isNotEmpty(unifiedOrderResp.getChannelErrorCode())) {
            throw exception(PAY_ORDER_SUBMIT_CHANNEL_ERROR,
                    unifiedOrderResp.getChannelErrorCode(),
                    unifiedOrderResp.getChannelErrorMsg());
        }
        order = orderMapper.selectById(order.getId());
    }
    return PayOrderConvert.INSTANCE.convert(order, unifiedOrderResp);
}
```

设计要点：
- **不加事务注解**：`submitOrder()` 方法没有 `@Transactional`，因为如果渠道调用失败，扩展记录不应该回滚（需要保留调用记录）。
- **扩展单编号**：使用 `PayOrderExtensionDO.no` 作为外部订单号调用渠道，而非 `PayOrderDO.id`。这样同一个支付单可以多次调用不同渠道。
- **直接支付成功**：付款码支付等场景会同步返回支付结果，需要立即处理。

### 1.3 校验逻辑

```java
private PayOrderDO validateOrderCanSubmit(Long id) {
    PayOrderDO order = orderMapper.selectById(id);
    if (order == null) {
        throw exception(PAY_ORDER_NOT_FOUND);
    }
    if (PayOrderStatusEnum.isSuccess(order.getStatus())) {
        throw exception(PAY_ORDER_STATUS_IS_SUCCESS);  // 已支付
    }
    if (!PayOrderStatusEnum.WAITING.getStatus().equals(order.getStatus())) {
        throw exception(PAY_ORDER_STATUS_IS_NOT_WAITING); // 非待支付
    }
    if (LocalDateTimeUtils.beforeNow(order.getExpireTime())) {
        throw exception(PAY_ORDER_IS_EXPIRED);  // 已过期
    }

    // 重要：校验扩展单是否已支付（防止回调延迟导致的重复支付）
    validateOrderActuallyPaid(id);
    return order;
}
```

`validateOrderActuallyPaid()` 会检查所有扩展单：
1. 检查数据库中的扩展单状态是否已支付
2. 调用三方接口查询扩展单是否已支付

这是一个防御性设计，防止因为回调延迟导致用户重复支付。

## 二、支付回调流程

### 2.1 回调入口

支付渠道（微信/支付宝）在用户支付成功后，会异步通知支付模块。

```java
// 来源：yudao-module-pay-server/.../controller/admin/notify/PayNotifyController.java
// 回调地址格式：/pay/notify/order/{channelId}
@PostMapping("/pay/notify/order/{channelId}")
public String notifyOrder(@PathVariable(" channelId") Long channelId,
                          HttpServletRequest request) {
    // 1. 解析回调参数
    Map<String, String> params = ServletUtils.getParamMap(request);
    String body = ServletUtils.getBody(request);

    // 2. 通过渠道客户端验签并解析
    PayClient<?> client = channelService.getPayClient(channelId);
    PayOrderRespDTO notify = client.parseOrderNotify(params, body,
            ServletUtils.getHeaderMap(request));

    // 3. 处理支付结果
    orderService.notifyOrder(channelId, notify);
    return "success"; // 返回 success 表示接收成功
}
```

### 2.2 处理支付结果

```java
// 来源：yudao-module-pay-server/.../service/order/PayOrderServiceImpl.java
@Override
public void notifyOrder(Long channelId, PayOrderRespDTO notify) {
    PayChannelDO channel = channelService.validPayChannel(channelId);
    // 在租户上下文中执行
    TenantUtils.execute(channel.getTenantId(),
            () -> getSelf().notifyOrder(channel, notify));
}

@Transactional(rollbackFor = Exception.class)
public void notifyOrder(PayChannelDO channel, PayOrderRespDTO notify) {
    if (PayOrderStatusEnum.isSuccess(notify.getStatus())) {
        notifyOrderSuccess(channel, notify);  // 支付成功
        return;
    }
    if (PayOrderStatusEnum.isClosed(notify.getStatus())) {
        notifyOrderClosed(channel, notify);   // 支付关闭
    }
}
```

### 2.3 支付成功处理

```java
private void notifyOrderSuccess(PayChannelDO channel, PayOrderRespDTO notify) {
    // 1. 更新 PayOrderExtensionDO 支付成功
    PayOrderExtensionDO orderExtension = updateOrderSuccess(notify);

    // 2. 更新 PayOrderDO 支付成功
    Boolean paid = updateOrderSuccess(channel, orderExtension, notify);
    if (paid) {
        return; // 已经成功回调过，不重复记录
    }

    // 3. 插入支付通知记录（异步通知业务系统）
    notifyService.createPayNotifyTask(PayNotifyTypeEnum.ORDER.getType(),
            orderExtension.getOrderId());
}

private PayOrderExtensionDO updateOrderSuccess(PayOrderRespDTO notify) {
    // 1. 查询扩展单
    PayOrderExtensionDO orderExtension = orderExtensionMapper.selectByNo(notify.getOutTradeNo());
    if (orderExtension == null) {
        throw exception(PAY_ORDER_EXTENSION_NOT_FOUND);
    }
    if (PayOrderStatusEnum.isSuccess(orderExtension.getStatus())) {
        return orderExtension; // 已经是成功状态，直接返回
    }
    if (ObjectUtil.notEqual(orderExtension.getStatus(), PayOrderStatusEnum.WAITING.getStatus())) {
        throw exception(PAY_ORDER_EXTENSION_STATUS_IS_NOT_WAITING);
    }

    // 2. 更新扩展单（CAS 乐观锁）
    int updateCounts = orderExtensionMapper.updateByIdAndStatus(
            orderExtension.getId(), orderExtension.getStatus(),
            PayOrderExtensionDO.builder()
                    .status(PayOrderStatusEnum.SUCCESS.getStatus())
                    .channelNotifyData(toJsonString(notify))
                    .build());
    if (updateCounts == 0) {
        throw exception(PAY_ORDER_EXTENSION_STATUS_IS_NOT_WAITING);
    }
    return orderExtension;
}

private Boolean updateOrderSuccess(PayChannelDO channel,
                                   PayOrderExtensionDO orderExtension,
                                   PayOrderRespDTO notify) {
    PayOrderDO order = orderMapper.selectById(orderExtension.getOrderId());
    if (order == null) {
        throw exception(PAY_ORDER_NOT_FOUND);
    }
    // 已经成功且是同一个扩展单，返回 true 表示重复回调
    if (PayOrderStatusEnum.isSuccess(order.getStatus())
            && Objects.equals(order.getExtensionId(), orderExtension.getId())) {
        return true;
    }
    if (!PayOrderStatusEnum.WAITING.getStatus().equals(order.getStatus())) {
        throw exception(PAY_ORDER_STATUS_IS_NOT_WAITING);
    }

    // 更新支付单（CAS 乐观锁）
    int updateCounts = orderMapper.updateByIdAndStatus(
            order.getId(), PayOrderStatusEnum.WAITING.getStatus(),
            PayOrderDO.builder()
                    .status(PayOrderStatusEnum.SUCCESS.getStatus())
                    .channelId(channel.getId()).channelCode(channel.getCode())
                    .successTime(notify.getSuccessTime())
                    .extensionId(orderExtension.getId()).no(orderExtension.getNo())
                    .channelOrderNo(notify.getChannelOrderNo())
                    .channelUserId(notify.getChannelUserId())
                    .channelFeeRate(channel.getFeeRate())
                    .channelFeePrice(MoneyUtils.calculateRatePrice(order.getPrice(), channel.getFeeRate()))
                    .build());
    if (updateCounts == 0) {
        throw exception(PAY_ORDER_STATUS_IS_NOT_WAITING);
    }
    return false;
}
```

设计要点：
- **幂等处理**：通过检查扩展单和支付单的当前状态，避免重复处理。
- **CAS 乐观锁**：使用 `updateByIdAndStatus()` 方法，只有状态匹配时才更新，防止并发问题。
- **事务保证**：`notifyOrder()` 方法有 `@Transactional` 注解，确保扩展单和支付单的状态一致性。

## 三、通知业务系统

### 3.1 创建通知任务

支付成功后，需要通知业务系统（如电商模块）更新订单状态。

```java
// 来源：yudao-module-pay-server/.../service/notify/PayNotifyServiceImpl.java
@Override
@Transactional(rollbackFor = Exception.class)
public void createPayNotifyTask(Integer type, Long dataId) {
    PayNotifyTaskDO task = new PayNotifyTaskDO()
            .setType(type).setDataId(dataId);
    task.setStatus(PayNotifyStatusEnum.WAITING.getStatus())
            .setNextNotifyTime(LocalDateTime.now())
            .setNotifyTimes(0)
            .setMaxNotifyTimes(PayNotifyTaskDO.NOTIFY_FREQUENCY.length + 1);

    // 补充 appId + notifyUrl + merchant* 字段
    if (Objects.equals(task.getType(), PayNotifyTypeEnum.ORDER.getType())) {
        PayOrderDO order = orderService.getOrder(task.getDataId());
        task.setAppId(order.getAppId())
                .setNotifyUrl(order.getNotifyUrl())
                .setMerchantOrderId(order.getMerchantOrderId());
    }
    // ... 退款和转账类似

    notifyTaskMapper.insert(task);

    // 事务提交后异步执行通知
    TransactionSynchronizationManager.registerSynchronization(new TransactionSynchronization() {
        @Override
        public void afterCommit() {
            getSelf().executeNotifyAsync(task);
        }
    });
}
```

设计要点：
- **事务后执行**：使用 `TransactionSynchronization` 确保通知任务在事务提交后才执行，避免通知到达时业务系统查询不到数据。
- **异步执行**：使用 `@Async` 注解异步执行，不阻塞当前线程。

### 3.2 执行通知

```java
@Async
public void executeNotifyAsync(PayNotifyTaskDO task) {
    executeNotify(task);
}

public void executeNotify(PayNotifyTaskDO task) {
    // 分布式锁，避免并发问题
    notifyLockCoreRedisDAO.lock(task.getId(), NOTIFY_TIMEOUT_MILLIS, () -> {
        // 校验任务是否已被处理
        PayNotifyTaskDO dbTask = notifyTaskMapper.selectById(task.getId());
        if (ObjectUtil.notEqual(task.getNotifyTimes(), dbTask.getNotifyTimes())) {
            return; // 已被其他线程处理
        }
        getSelf().executeNotify0(dbTask);
    });
}

@Transactional(rollbackFor = Exception.class)
public void executeNotify0(PayNotifyTaskDO task) {
    // 1. 发起 HTTP 回调
    CommonResult<?> invokeResult = null;
    Throwable invokeException = null;
    try {
        invokeResult = executeNotifyInvoke(task);
    } catch (Throwable e) {
        invokeException = e;
    }

    // 2. 处理结果
    Integer newStatus = processNotifyResult(task, invokeResult, invokeException);

    // 3. 记录日志
    String response = invokeException != null
            ? ExceptionUtil.getRootCauseMessage(invokeException)
            : JsonUtils.toJsonString(invokeResult);
    notifyLogMapper.insert(PayNotifyLogDO.builder()
            .taskId(task.getId())
            .notifyTimes(task.getNotifyTimes() + 1)
            .status(newStatus)
            .response(response)
            .build());
}
```

### 3.3 指数退避重试

```java
Integer processNotifyResult(PayNotifyTaskDO task, CommonResult<?> invokeResult, Throwable invokeException) {
    PayNotifyTaskDO updateTask = new PayNotifyTaskDO()
            .setId(task.getId())
            .setLastExecuteTime(LocalDateTime.now())
            .setNotifyTimes(task.getNotifyTimes() + 1);

    // 情况一：调用成功
    if (invokeResult != null && invokeResult.isSuccess()) {
        updateTask.setStatus(PayNotifyStatusEnum.SUCCESS.getStatus());
        notifyTaskMapper.updateById(updateTask);
        return updateTask.getStatus();
    }

    // 情况二：超过最大回调次数
    if (updateTask.getNotifyTimes() >= PayNotifyTaskDO.NOTIFY_FREQUENCY.length) {
        updateTask.setStatus(PayNotifyStatusEnum.FAILURE.getStatus());
        notifyTaskMapper.updateById(updateTask);
        return updateTask.getStatus();
    }

    // 情况三：未超过最大次数，设置下次通知时间
    updateTask.setNextNotifyTime(addTime(Duration.ofSeconds(
            PayNotifyTaskDO.NOTIFY_FREQUENCY[updateTask.getNotifyTimes()])));
    updateTask.setStatus(invokeException != null
            ? PayNotifyStatusEnum.REQUEST_FAILURE.getStatus()
            : PayNotifyStatusEnum.REQUEST_SUCCESS.getStatus());
    notifyTaskMapper.updateById(updateTask);
    return updateTask.getStatus();
}
```

通知频率：首次通知后，按 15s、15s、30s、3min、30min、30min、30min、1h 的间隔重试，最多 9 次。

## 四、退款流程

### 4.1 创建退款

```java
// 来源：yudao-module-pay-server/.../service/refund/PayRefundServiceImpl.java
@Override
public Long createRefund(PayRefundCreateReqDTO reqDTO) {
    // 1. 校验
    PayAppDO app = appService.validPayApp(reqDTO.getAppKey());
    PayOrderDO order = validatePayOrderCanRefund(reqDTO, app.getId());
    PayChannelDO channel = channelService.validPayChannel(order.getChannelId());
    PayClient<?> client = channelService.getPayClient(channel.getId());
    if (client == null) {
        throw exception(CHANNEL_NOT_FOUND);
    }

    // 校验退款单是否已存在（幂等）
    PayRefundDO refund = refundMapper.selectByAppIdAndMerchantRefundId(
            app.getId(), reqDTO.getMerchantRefundId());
    if (refund != null) {
        throw exception(REFUND_EXISTS);
    }

    // 2. 插入退款单
    String no = noRedisDAO.generate(payProperties.getRefundNoPrefix());
    refund = PayRefundConvert.INSTANCE.convert(reqDTO)
            .setNo(no).setAppId(app.getId())
            .setOrderId(order.getId()).setOrderNo(order.getNo())
            .setChannelId(order.getChannelId()).setChannelCode(order.getChannelCode())
            .setNotifyUrl(app.getRefundNotifyUrl())
            .setChannelOrderNo(order.getChannelOrderNo())
            .setStatus(PayRefundStatusEnum.WAITING.getStatus())
            .setPayPrice(order.getPrice()).setRefundPrice(reqDTO.getPrice());
    refundMapper.insert(refund);

    try {
        // 3. 调用渠道退款接口
        PayRefundUnifiedReqDTO unifiedReqDTO = new PayRefundUnifiedReqDTO()
                .setPayPrice(order.getPrice())
                .setRefundPrice(reqDTO.getPrice())
                .setOutTradeNo(order.getNo())
                .setOutRefundNo(refund.getNo())
                .setNotifyUrl(genChannelRefundNotifyUrl(channel))
                .setReason(reqDTO.getReason());
        PayRefundRespDTO refundRespDTO = client.unifiedRefund(unifiedReqDTO);

        // 4. 处理退款结果
        getSelf().notifyRefund(channel, refundRespDTO);
    } catch (Throwable e) {
        // 注意：这里仅打印异常，不抛出。后续通过退款回调或轮询补偿处理。
        log.error("[createPayRefund][退款 id({}) requestDTO({}) 发生异常]",
                refund.getId(), reqDTO, e);
    }

    return refund.getId();
}
```

设计要点：
- **异常不抛出**：调用渠道退款接口时的异常只记录日志，不抛出。因为即使网络超时，退款可能已经成功，后续通过回调或轮询补偿获取结果。
- **幂等校验**：通过 `appId + merchantRefundId` 检查退款单是否已存在。

### 4.2 退款校验

```java
private PayOrderDO validatePayOrderCanRefund(PayRefundCreateReqDTO reqDTO, Long appId) {
    PayOrderDO order = orderService.getOrder(appId, reqDTO.getMerchantOrderId());
    if (order == null) {
        throw exception(PAY_ORDER_NOT_FOUND);
    }
    // 校验状态：必须是已支付或已退款
    if (!PayOrderStatusEnum.isSuccessOrRefund(order.getStatus())) {
        throw exception(PAY_ORDER_REFUND_FAIL_STATUS_ERROR);
    }
    // 校验金额：退款金额不能超过原订单金额
    if (reqDTO.getPrice() + order.getRefundPrice() > order.getPrice()) {
        throw exception(REFUND_PRICE_EXCEED);
    }
    // 校验是否有退款中的订单
    if (refundMapper.selectCountByAppIdAndOrderId(appId, order.getId(),
            PayRefundStatusEnum.WAITING.getStatus()) > 0) {
        throw exception(REFUND_HAS_REFUNDING);
    }
    return order;
}
```

### 4.3 退款回调处理

```java
@Override
public void notifyRefund(Long channelId, PayRefundRespDTO notify) {
    PayChannelDO channel = channelService.validPayChannel(channelId);
    TenantUtils.execute(channel.getTenantId(),
            () -> getSelf().notifyRefund(channel, notify));
}

@Transactional(rollbackFor = Exception.class)
public void notifyRefund(PayChannelDO channel, PayRefundRespDTO notify) {
    if (PayRefundStatusEnum.isSuccess(notify.getStatus())) {
        notifyRefundSuccess(channel, notify);
        return;
    }
    if (PayRefundStatusEnum.isFailure(notify.getStatus())) {
        notifyRefundFailure(channel, notify);
    }
}

private void notifyRefundSuccess(PayChannelDO channel, PayRefundRespDTO notify) {
    // 1. 更新退款单状态
    PayRefundDO refund = refundMapper.selectByAppIdAndNo(channel.getAppId(), notify.getOutRefundNo());
    if (refund == null) {
        throw exception(REFUND_NOT_FOUND);
    }
    if (PayRefundStatusEnum.isSuccess(refund.getStatus())) {
        return; // 已经是成功状态
    }

    PayRefundDO updateRefundObj = new PayRefundDO()
            .setSuccessTime(notify.getSuccessTime())
            .setChannelRefundNo(notify.getChannelRefundNo())
            .setStatus(PayRefundStatusEnum.SUCCESS.getStatus())
            .setChannelNotifyData(toJsonString(notify));
    int updateCounts = refundMapper.updateByIdAndStatus(
            refund.getId(), refund.getStatus(), updateRefundObj);
    if (updateCounts == 0) {
        throw exception(REFUND_STATUS_IS_NOT_WAITING);
    }

    // 2. 更新支付单的退款金额
    orderService.updateOrderRefundPrice(refund.getOrderId(), refund.getRefundPrice());

    // 3. 创建通知任务，通知业务系统
    notifyService.createPayNotifyTask(PayNotifyTypeEnum.REFUND.getType(), refund.getId());
}
```

### 4.4 更新支付单退款金额

```java
// 来源：yudao-module-pay-server/.../service/order/PayOrderServiceImpl.java
@Override
public void updateOrderRefundPrice(Long id, Integer incrRefundPrice) {
    PayOrderDO order = orderMapper.selectById(id);
    if (order == null) {
        throw exception(PAY_ORDER_NOT_FOUND);
    }
    if (!PayOrderStatusEnum.isSuccessOrRefund(order.getStatus())) {
        throw exception(PAY_ORDER_REFUND_FAIL_STATUS_ERROR);
    }
    if (order.getRefundPrice() + incrRefundPrice > order.getPrice()) {
        throw exception(REFUND_PRICE_EXCEED);
    }

    // 更新订单
    PayOrderDO updateObj = new PayOrderDO()
            .setRefundPrice(order.getRefundPrice() + incrRefundPrice)
            .setStatus(PayOrderStatusEnum.REFUND.getStatus());
    int updateCount = orderMapper.updateByIdAndStatus(id, order.getStatus(), updateObj);
    if (updateCount == 0) {
        throw exception(PAY_ORDER_REFUND_FAIL_STATUS_ERROR);
    }
}
```

## 五、对账补偿

### 5.1 同步订单状态

定时任务定期同步渠道的订单状态，处理回调丢失的情况。

```java
@Override
public int syncOrder(LocalDateTime minCreateTime) {
    // 1. 查询待支付的扩展单
    List<PayOrderExtensionDO> orderExtensions = orderExtensionMapper
            .selectListByStatusAndCreateTimeGe(PayOrderStatusEnum.WAITING.getStatus(), minCreateTime);
    if (CollUtil.isEmpty(orderExtensions)) {
        return 0;
    }

    // 2. 遍历同步
    int count = 0;
    for (PayOrderExtensionDO orderExtension : orderExtensions) {
        count += syncOrder(orderExtension) ? 1 : 0;
    }
    return count;
}

private boolean syncOrder(PayOrderExtensionDO orderExtension) {
    try {
        // 1. 查询渠道订单状态
        PayClient<?> payClient = channelService.getPayClient(orderExtension.getChannelId());
        if (payClient == null) {
            return false;
        }
        PayOrderRespDTO respDTO = payClient.getOrder(orderExtension.getNo());

        // 2. 如果是关闭状态，不处理（以回调为准）
        if (PayOrderStatusEnum.isClosed(respDTO.getStatus())) {
            return false;
        }

        // 3. 回调处理
        notifyOrder(orderExtension.getChannelId(), respDTO);
        return PayOrderStatusEnum.isSuccess(respDTO.getStatus());
    } catch (Throwable e) {
        log.error("[syncOrder][orderExtension({}) 同步支付状态异常]", orderExtension.getId(), e);
        return false;
    }
}
```

### 5.2 过期订单处理

```java
@Override
public int expireOrder() {
    // 1. 查询过期的待支付订单
    List<PayOrderDO> orders = orderMapper.selectListByStatusAndExpireTimeLt(
            PayOrderStatusEnum.WAITING.getStatus(), LocalDateTime.now());
    if (CollUtil.isEmpty(orders)) {
        return 0;
    }

    // 2. 遍历处理
    int count = 0;
    for (PayOrderDO order : orders) {
        count += expireOrder(order) ? 1 : 0;
    }
    return count;
}

private boolean expireOrder(PayOrderDO order) {
    try {
        // 1. 先处理关联的扩展单
        List<PayOrderExtensionDO> orderExtensions = orderExtensionMapper
                .selectListByOrderId(order.getId());
        for (PayOrderExtensionDO orderExtension : orderExtensions) {
            if (PayOrderStatusEnum.isClosed(orderExtension.getStatus())) {
                continue;
            }
            // 校验扩展单是否已支付
            if (PayOrderStatusEnum.isSuccess(orderExtension.getStatus())) {
                return false; // 已支付，不能关闭
            }
            // 调用三方接口查询状态
            PayClient<?> payClient = channelService.getPayClient(orderExtension.getChannelId());
            if (payClient == null) {
                return false;
            }
            PayOrderRespDTO respDTO = payClient.getOrder(orderExtension.getNo());
            if (PayOrderStatusEnum.isRefund(respDTO.getStatus())) {
                return false; // 已退款，需要人工介入
            }
            if (PayOrderStatusEnum.isSuccess(respDTO.getStatus())) {
                notifyOrder(orderExtension.getChannelId(), respDTO);
                return false; // 已支付，触发回调处理
            }
            // 关闭扩展单
            PayOrderExtensionDO updateObj = new PayOrderExtensionDO()
                    .setStatus(PayOrderStatusEnum.CLOSED.getStatus())
                    .setChannelNotifyData(toJsonString(respDTO));
            orderExtensionMapper.updateByIdAndStatus(
                    orderExtension.getId(), PayOrderStatusEnum.WAITING.getStatus(), updateObj);
        }

        // 2. 关闭支付单
        PayOrderDO updateObj = new PayOrderDO().setStatus(PayOrderStatusEnum.CLOSED.getStatus());
        return orderMapper.updateByIdAndStatus(order.getId(), order.getStatus(), updateObj) > 0;
    } catch (Throwable e) {
        log.error("[expireOrder][order({}) 过期订单异常]", order.getId(), e);
        return false;
    }
}
```

## 六、AOP 代理问题

在 `PayOrderServiceImpl` 中，`notifyOrder()` 方法有 `@Transactional` 注解，但在 `submitOrder()` 中需要调用它。如果直接调用 `this.notifyOrder()`，事务注解不会生效（因为 Spring AOP 代理问题）。

解决方案是通过 `getSelf()` 获取代理对象：

```java
private PayOrderServiceImpl getSelf() {
    return SpringUtil.getBean(getClass());
}
```

在需要事务生效的地方，使用 `getSelf().notifyOrder()` 而非 `this.notifyOrder()`。
