### 1.2 测试基础设施

支付模块的测试依赖以下基础设施：

1. **MockPayClient**：模拟支付客户端，用于开发和测试环境。不调用真实支付渠道，直接返回预设的支付结果。

2. **H2 内存数据库**：单元测试使用 H2 内存数据库，无需依赖外部 MySQL。

3. **Spring Boot Test**：使用 `@SpringBootTest` 注解加载完整的 Spring 上下文。

### 1.3 常见测试场景

#### 场景一：支付下单流程测试

```java
@SpringBootTest
class PayOrderServiceTest {

    @Resource
    private PayOrderService orderService;

    @Test
    public void testCreateAndSubmitOrder() {
        // 1. 创建支付单
        PayOrderCreateReqDTO createReqDTO = new PayOrderCreateReqDTO()
                .setAppKey("test-app")
                .setMerchantOrderId("ORDER_001")
                .setSubject("测试商品")
                .setPrice(100) // 1 元
                .setExpireTime(LocalDateTime.now().plusHours(2));
        Long orderId = orderService.createOrder(createReqDTO);

        // 2. 提交支付（使用 Mock 渠道）
        PayOrderSubmitReqVO submitReqVO = new PayOrderSubmitReqVO()
                .setId(orderId)
                .setChannelCode("mock");
        PayOrderSubmitRespVO respVO = orderService.submitOrder(submitReqVO, "127.0.0.1");

        // 3. 验证结果
        PayOrderDO order = orderService.getOrder(orderId);
        assertEquals(PayOrderStatusEnum.SUCCESS.getStatus(), order.getStatus());
    }
}
```

#### 场景二：退款流程测试

```java
@SpringBootTest
class PayRefundServiceTest {

    @Resource
    private PayRefundService refundService;

    @Test
    public void testCreateRefund() {
        // 1. 创建退款单
        PayRefundCreateReqDTO reqDTO = new PayRefundCreateReqDTO()
                .setAppKey("test-app")
                .setMerchantOrderId("ORDER_001")
                .setMerchantRefundId("REFUND_001")
                .setPrice(50) // 退款 0.5 元
                .setReason("用户申请退款");
        Long refundId = refundService.createRefund(reqDTO);

        // 2. 验证退款单状态
        PayRefundDO refund = refundService.getRefund(refundId);
        assertEquals(PayRefundStatusEnum.SUCCESS.getStatus(), refund.getStatus());
    }
}
```
