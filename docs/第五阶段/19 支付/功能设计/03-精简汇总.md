# 03 - 精简汇总

## 支付流程速查

### 支付下单流程

```
业务系统 ──createOrder()──> PayOrderService ──> 创建 PayOrderDO (WAITING)
    │                                              │
    │                                              ▼
    │                               返回支付订单编号 (id)
    │
前端 ──submitOrder()──> PayOrderService
    │                      │
    │                      ├── 创建 PayOrderExtensionDO
    │                      ├── PayClient.unifiedOrder()
    │                      └── 返回支付参数
    │
    ▼
用户完成支付
    │
支付渠道 ──异步回调──> PayNotifyController
                        │
                        ├── PayClient.parseOrderNotify() (验签+解析)
                        ├── PayOrderService.notifyOrder() (更新状态)
                        └── PayNotifyService.createPayNotifyTask() (通知业务系统)
```

### 退款流程

```
业务系统 ──createRefund()──> PayRefundService
                                │
                                ├── 校验可退金额
                                ├── 创建 PayRefundDO (WAITING)
                                ├── PayClient.unifiedRefund()
                                └── PayNotifyService.createPayNotifyTask()

支付渠道 ──退款回调──> PayNotifyController
                        │
                        ├── PayClient.parseRefundNotify()
                        ├── PayRefundService.notifyRefund()
                        │     ├── 更新 PayRefundDO 状态
                        │     └── 更新 PayOrderDO.refundPrice
                        └── 通知业务系统
```

### 状态流转

```
支付订单状态：
  WAITING(0) ──支付成功──> SUCCESS(10) ──退款成功──> REFUND(20)
  WAITING(0) ──超时/关闭──> CLOSED(30)

退款订单状态：
  WAITING(0) ──退款成功──> SUCCESS(10)
  WAITING(0) ──退款失败──> FAILURE(10)

转账订单状态：
  WAITING(0) ──转账成功──> SUCCESS(10)
  WAITING(0) ──转账失败──> FAILURE(10)
```

## 核心 API 速查

### PayOrderApi（Feign 接口）

| 方法 | 说明 | 入参 | 出参 |
|------|------|------|------|
| `createOrder()` | 创建支付单 | `PayOrderCreateReqDTO` | 支付单编号 `Long` |
| `getOrder()` | 查询支付单 | `id` | `PayOrderRespDTO` |
| `updatePayOrderPrice()` | 更新支付单价格 | `id`, `payPrice` | `Boolean` |

### PayRefundApi（Feign 接口）

| 方法 | 说明 | 入参 | 出参 |
|------|------|------|------|
| `createRefund()` | 创建退款单 | `PayRefundCreateReqDTO` | 退款单编号 `Long` |
| `getRefund()` | 查询退款单 | `id` | `PayRefundRespDTO` |

### PayTransferApi（Feign 接口）

| 方法 | 说明 | 入参 | 出参 |
|------|------|------|------|
| `createTransfer()` | 创建转账单 | `PayTransferCreateReqDTO` | `PayTransferCreateRespDTO` |
| `getTransfer()` | 查询转账单 | `id` | `PayTransferRespDTO` |

### PayClient 接口

| 方法 | 说明 |
|------|------|
| `unifiedOrder()` | 统一下单 |
| `parseOrderNotify()` | 解析支付回调 |
| `getOrder()` | 查询订单状态 |
| `unifiedRefund()` | 统一退款 |
| `parseRefundNotify()` | 解析退款回调 |
| `getRefund()` | 查询退款状态 |
| `unifiedTransfer()` | 统一转账 |
| `parseTransferNotify()` | 解析转账回调 |
| `getTransfer()` | 查询转账状态 |

### 管理后台 API

| 接口 | 说明 |
|------|------|
| `POST /pay/order/submit` | 提交支付 |
| `GET /pay/order/get` | 查询支付订单 |
| `GET /pay/order/page` | 支付订单分页 |
| `POST /pay/notify/order/{channelId}` | 支付回调（给支付渠道调用） |
| `POST /pay/notify/refund/{channelId}` | 退款回调 |
| `POST /pay/notify/transfer/{channelId}` | 转账回调 |

## 核心实体速查

### PayAppDO（支付应用）

| 字段 | 说明 |
|------|------|
| `id` | 应用编号 |
| `appKey` | 应用标识（唯一） |
| `name` | 应用名 |
| `status` | 状态 |
| `orderNotifyUrl` | 支付结果回调地址 |
| `refundNotifyUrl` | 退款结果回调地址 |

### PayChannelDO（支付渠道）

| 字段 | 说明 |
|------|------|
| `id` | 渠道编号 |
| `code` | 渠道编码（如 `wx_lite`、`alipay_pc`） |
| `appId` | 所属应用编号 |
| `status` | 状态 |
| `feeRate` | 渠道费率（百分比） |
| `config` | 渠道配置（JSON，包含密钥等） |

### PayOrderDO（支付订单）

| 字段 | 说明 |
|------|------|
| `id` | 订单编号 |
| `appId` | 应用编号 |
| `channelId` | 渠道编号 |
| `channelCode` | 渠道编码 |
| `merchantOrderId` | 商户订单编号 |
| `subject` | 商品标题 |
| `price` | 支付金额（分） |
| `status` | 支付状态 |
| `expireTime` | 过期时间 |
| `successTime` | 支付成功时间 |
| `refundPrice` | 已退款金额（分） |
| `channelOrderNo` | 渠道订单号 |

### PayRefundDO（退款订单）

| 字段 | 说明 |
|------|------|
| `id` | 退款单编号 |
| `no` | 外部退款号 |
| `orderId` | 关联的支付订单编号 |
| `merchantRefundId` | 商户退款单号 |
| `payPrice` | 支付金额（分） |
| `refundPrice` | 退款金额（分） |
| `status` | 退款状态 |
| `reason` | 退款原因 |

### PayTransferDO（转账订单）

| 字段 | 说明 |
|------|------|
| `id` | 转账单编号 |
| `no` | 转账单号 |
| `appId` | 应用编号 |
| `merchantTransferId` | 商户转账单号 |
| `subject` | 转账标题 |
| `price` | 转账金额（分） |
| `userAccount` | 收款人账号 |
| `userName` | 收款人姓名 |
| `status` | 转账状态 |

### PayWalletDO（钱包）

| 字段 | 说明 |
|------|------|
| `id` | 钱包编号 |
| `userId` | 用户编号 |
| `userType` | 用户类型 |
| `balance` | 余额（分） |
| `freezePrice` | 冻结金额（分） |
| `totalExpense` | 累计支出（分） |
| `totalRecharge` | 累计充值（分） |

## 设计决策列表

| 决策 | 选择 | 放弃的方案 | 原因 |
|------|------|-----------|------|
| 支付渠道抽象 | 策略模式 + 工厂 | 直接 if-else 分支 | 易于扩展，新增渠道不改业务代码 |
| 模板方法 | `AbstractPayClient` | 每个 Client 独立实现 | 统一参数校验和异常处理 |
| 订单创建与提交分离 | 两步式 | 一步完成 | 关注点分离，支持切换支付方式 |
| 回调处理 | 统一入口 + PayClient 解析 | 每个渠道独立回调接口 | 统一验签逻辑，减少重复代码 |
| 通知机制 | 数据库任务 + 定时重试 | 直接 HTTP 调用 | 保证通知不丢失，支持重试 |
| 重试策略 | 指数退避（最多 9 次） | 固定间隔重试 | 避免频繁重试压垮业务系统 |
| 支付金额单位 | 分（Integer） | 元（BigDecimal） | 避免浮点数精度问题 |
| 渠道配置存储 | JSON 字段 | 独立配置表 | 简化查询，一个渠道一条记录 |
| 钱包支付 | 走 PayClient 接口 | 直接在业务层扣减 | 统一支付流程，便于对账 |
| 模拟支付 | MockPayClient | 不提供 | 开发测试环境不需要真实支付 |
