# 03 - 精简汇总

## ERP 流程速查

### 采购全流程

```
供应商管理 → 采购订单(草稿) → 提交审核 → 审核通过 → 采购入库 → 更新库存
                                                │
                                                └── 采购退货 → 更新库存
```

### 销售全流程

```
客户管理 → 销售订单(草稿) → 提交审核 → 审核通过 → 销售出库 → 更新库存
                                                │
                                                └── 销售退货 → 更新库存
```

### 库存变动汇总

| 业务类型 | 库存变动 | 关联实体 |
|---------|---------|---------|
| 采购入库 | +数量 | ErpPurchaseInDO |
| 采购退货 | -数量 | ErpPurchaseReturnDO |
| 销售出库 | -数量 | ErpSaleOutDO |
| 销售退货 | +数量 | ErpSaleReturnDO |
| 库存盘点 | 覆盖 | ErpStockCheckDO |
| 库存调拨 | 源-目标+ | ErpStockMoveDO |

### 审核状态流转

```
草稿(0) ──提交──> 审核中(10) ──通过──> 审核通过(20) ──完成──> 已完成(40)
                    │
                    └──驳回──> 审核驳回(30) ──修改──> 草稿(0)
```

### 金额计算链路

```
明细层：taxPrice = price * quantity * taxPercent / 100
明细层：totalPrice = price * quantity + taxPrice

订单头：totalProductPrice = SUM(item.price * item.quantity)
订单头：totalTaxPrice = SUM(item.taxPrice)
订单头：discountPrice = (totalProductPrice + totalTaxPrice) * discountPercent / 100
订单头：totalPrice = totalProductPrice + totalTaxPrice - discountPrice
```

## 核心设计决策

| 决策 | 选择 | 放弃的方案 | 原因 |
|------|------|-----------|------|
| 订单存储 | 主子表（订单头+明细表） | 单表 JSON 字段 | 支持明细聚合查询 |
| 审核状态 | 共享 ErpAuditStatus 枚举 | 各模块独立枚举 | 统一状态机，减少重复 |
| 库存更新 | `SET count = count + ?` | 先查后改 | 利用数据库行锁保证原子性 |
| 金额单位 | 元（BigDecimal） | 分（Integer） | ERP 场景金额较大，元更直观 |
| 入库/出库跟踪 | 订单头冗余 inCount/outCount | 每次聚合查询 | 减少查询开销，支持快速判断完成状态 |
| 产品信息 | 订单明细冗余存储 | 只存 productId | 防止产品变更影响历史订单 |
| 供应商/客户 | 独立管理 | 复用 CRM 客户 | ERP 的供应商/客户更偏向财务属性 |
| 账户余额 | 收付款审核通过后更新 | 实时更新 | 防止审核驳回时需要回滚余额 |

## 模块集成关系

```
ERP
  │
  ├── ◄── System（用户、部门）
  │
  ├── ◄── BPM（采购/销售订单审批）
  │
  ├── ◄── CRM（客户信息同步）
  │
  ├── ──> Trade（电商订单关联）
  │
  └── ──> 库存变动通知
```
