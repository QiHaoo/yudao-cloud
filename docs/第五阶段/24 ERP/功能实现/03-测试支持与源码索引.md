# 03-测试支持与源码索引

## 测试支持

### 现状

ERP 模块（yudao-module-erp）目前没有独立的单元测试目录。业务逻辑主要通过集成测试和手动测试验证。

### 测试策略建议

ERP 模块的测试挑战在于**主子表事务**和**库存并发安全**。建议按以下层次组织测试：

**1. 实体层测试**：
- 验证 DO 字段映射、BigDecimal 精度
- 验证 JSON 类型处理器

**2. Service 层测试**：
- 采购订单创建：Mock 供应商校验、产品校验，验证主子表联合写入和金额计算
- 审核状态变更：验证状态机流转的合法性（草稿→审核中→审核通过）
- 反审核校验：验证存在入库单时不允许反审核

**3. 库存层测试**：
- 采购入库：验证库存增加、订单 inCount 更新
- 销售出库：验证库存减少、订单 outCount 更新、库存不足校验
- 并发测试：模拟多个入库/出库同时执行，验证原子性

**4. 财务层测试**：
- 收款单：验证审核通过后账户余额增加
- 付款单：验证审核通过后账户余额减少、余额不足校验

### Mock 建议

```java
// 示例：测试采购订单创建时 Mock 供应商校验
@MockBean
private ErpSupplierService supplierService;

@Test
public void testCreatePurchaseOrder() {
    // given
    when(supplierService.validateSupplier(anyLong())).thenReturn(true);
    // when & then
    // ...
}
```

### 测试数据准备

- 产品数据：准备产品的测试数据（名称、单价、单位）
- 供应商/客户数据：准备基础数据
- 仓库数据：准备仓库的测试数据
- 库存数据：准备初始库存
