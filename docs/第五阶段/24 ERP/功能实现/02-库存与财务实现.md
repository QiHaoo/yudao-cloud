# 02-库存与财务实现

本章实现 ERP 的库存变动机制和财务收付款流程。

> **前置阅读**：本章对应功能设计中的「02-核心设计：进销存与财务」中库存变动和财务收付款部分。

## 库存实体：ErpStockDO

源码位置：`yudao-module-erp-server/.../dal/dataobject/stock/ErpStockDO.java`

```java
@TableName("erp_stock")
@KeySequence("erp_stock_seq")
@Data
@EqualsAndHashCode(callSuper = true)
@Builder
@NoArgsConstructor
@AllArgsConstructor
public class ErpStockDO extends BaseDO {

    @TableId
    private Long id;
    private Long productId;     // 产品编号，关联 ErpProductDO
    private Long warehouseId;   // 仓库编号，关联 ErpWarehouseDO
    private BigDecimal count;   // 库存数量
}
```

**关键设计：`productId + warehouseId` 联合唯一。**

同一个产品在不同仓库有独立的库存记录。比如"蓝牙耳机"在 A 仓库有 100 个，在 B 仓库有 200 个。

## 库存变动来源

```
库存变动来源
  │
  ├── 采购入库 (ErpPurchaseInDO)
  │     → 入库完成 → ErpStockDO.count += 入库数量
  │
  ├── 采购退货 (ErpPurchaseReturnDO)
  │     → 退货完成 → ErpStockDO.count -= 退货数量
  │
  ├── 销售出库 (ErpSaleOutDO)
  │     → 出库完成 → ErpStockDO.count -= 出库数量
  │
  ├── 销售退货 (ErpSaleReturnDO)
  │     → 退货完成 → ErpStockDO.count += 退货数量
  │
  ├── 库存盘点 (ErpStockCheckDO)
  │     → 盘点完成 → ErpStockDO.count = 盘点数量（直接覆盖）
  │
  └── 库存调拨 (ErpStockMoveDO)
        → 调拨完成 → 源仓库 count -= 数量，目标仓库 count += 数量
```

## 采购入库流程

### 入库单实体

源码位置：`yudao-module-erp-server/.../dal/dataobject/purchase/ErpPurchaseInDO.java`

```java
@TableName("erp_purchase_in")
@KeySequence("erp_purchase_in_seq")
@Data
@EqualsAndHashCode(callSuper = true)
@Builder
@NoArgsConstructor
@AllArgsConstructor
public class ErpPurchaseInDO extends BaseDO {

    @TableId
    private Long id;
    private String no;                // 入库单号
    private Integer status;           // 审核状态，枚举 ErpAuditStatus
    private Long orderId;             // 关联采购订单
    private Long warehouseId;         // 入库仓库
    private LocalDateTime inTime;     // 入库时间
    private BigDecimal totalCount;    // 合计数量
    private BigDecimal totalPrice;    // 合计金额
    private String remark;            // 备注
}
```

### 入库流程

```
采购入库流程
  │
  ├── 1. 选择已审核通过的采购订单
  │
  ├── 2. 创建采购入库单 (ErpPurchaseInDO)
  │     ├── 关联采购订单 (orderId)
  │     ├── 选择仓库 (warehouseId)
  │     └── 填写入库明细
  │           ├── 产品 1：入库 80 个（订单 100 个，分批入库）
  │           └── 产品 2：入库 200 个（全部入库）
  │
  ├── 3. 审核入库单
  │     ├── 通过 → 执行入库
  │     └── 驳回 → 修改后重新提交
  │
  └── 4. 入库完成
        ├── 更新库存：ErpStockDO.count += 入库数量
        ├── 更新订单：ErpPurchaseOrderDO.inCount += 入库数量
        └── 创建库存变动记录
```

### 分批入库

一个采购订单可以分多次入库：

```
采购订单：100 个鼠标
  │
  ├── 入库单 1：80 个 → inCount = 80
  │
  └── 入库单 2：20 个 → inCount = 100（全部完成）
```

入库数量不能超过订单剩余数量（`count - inCount - returnCount`）。

### 出库前的库存校验

```
创建出库单时
  │
  ▼
校验库存
  │
  ├── 查询 ErpStockDO（productId, warehouseId）
  ├── 当前库存 >= 出库数量？
  │     ├── 是 → 允许出库
  │     └── 否 → 提示"库存不足，当前库存 X，需要 Y"
  └── 注意：这里只是预校验，实际扣减在审核通过后
```

## 并发安全

多个操作同时修改库存时，使用数据库的原子更新保证安全：

```sql
UPDATE erp_stock SET count = count + #{incrCount}
WHERE id = #{id} AND product_id = #{productId} AND warehouse_id = #{warehouseId}
```

利用数据库的行锁保证原子性，避免超卖或负库存。

## 收款单（销售回款）

源码位置：`yudao-module-erp-server/.../dal/dataobject/finance/ErpFinanceReceiptDO.java`

```java
@TableName("erp_finance_receipt")

    private Long financeUserId;       // 财务人员
    private Long customerId;          // 客户编号，关联 ErpCustomerDO
    private Long accountId;           // 收款账户，关联 ErpAccountDO

    private BigDecimal totalPrice;    // 合计金额
    private BigDecimal discountPrice; // 优惠金额
    /** 实收金额 = totalPrice - discountPrice */
    private BigDecimal receiptPrice;  // 实收金额

}
```

收款单通过 `ErpFinanceReceiptItemDO` 关联具体的销售订单：

```
ErpFinanceReceiptDO (收款单)
  └── 1:N ── ErpFinanceReceiptItemDO (收款明细)
                ├── saleOrderId   - 销售订单编号
                ├── price         - 收款金额
                └── remark        - 备注
```

## 收付款与账户余额

```
收款单审核通过
  │
  ├── 1. 更新 ErpAccountDO.balance += receiptPrice
  └── 2. 记录账户变动明细

付款单审核通过
  │
  ├── 1. 校验账户余额 >= paymentPrice
  ├── 2. 更新 ErpAccountDO.balance -= paymentPrice
  └── 3. 记录账户变动明细
```

## 统计分析

ERP 模块提供了采购和销售的统计分析：

| Service | 说明 |
|---------|------|
| ErpPurchaseStatisticsServiceImpl | 采购统计（采购金额、采购趋势） |
| ErpSaleStatisticsServiceImpl | 销售统计（销售金额、销售趋势） |

## 小结

| 组件 | 职责 | 关键设计 |
|------|------|---------|
| ErpStockDO | 产品库存 | productId + warehouseId 联合唯一 |
| ErpStockRecordDO | 库存变动记录 | sourceType + sourceId 追溯来源 |
| ErpPurchaseInDO | 采购入库单 | 支持分批入库 |
| ErpSaleOutDO | 销售出库单 | 出库前库存预校验 |
| ErpStockCheckDO | 盘点单 | 直接覆盖库存数量 |
| ErpStockMoveDO | 调拨单 | 源仓库减、目标仓库加 |
| ErpAccountDO | 结算账户 | 余额管理 |
| ErpFinanceReceiptDO | 收款单 | 销售回款 |
| ErpFinancePaymentDO | 付款单 | 采购付款 |
