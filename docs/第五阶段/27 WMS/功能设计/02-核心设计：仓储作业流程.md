# 02 - 核心设计：仓储作业流程

## 场景引入

上一章我们看到了 WMS 的全景——入库、出库、库存、盘点。这一章深入几个核心设计问题：

1. **入库作业**：入库单怎么创建？入库明细怎么关联 SKU？
2. **出库作业**：出库前怎么校验库存？怎么支持分批出库？
3. **库存变动追踪**：每次库存变动都记录了什么？怎么追溯？
4. **盘点差异处理**：盘点结果和系统库存不一致怎么办？

---

## 入库作业流程

### 完整流程

```
入库作业流程
  │
  ├── 1. 创建入库单 (WmsReceiptOrderDO)
  │     ├── 选择入库类型（采购入库/退货入库/其他）
  │     ├── 选择仓库
  │     ├── 关联业务单号（采购单号/退货单号）
  │     └── 选择供应商
  │
  ├── 2. 填写入库明细 (WmsReceiptOrderDetailDO)
  │     ├── SKU 1：入库 100 个，单价 50 元
  │     ├── SKU 2：入库 200 个，单价 30 元
  │     └── ...
  │
  ├── 3. 审核入库单
  │     ├── 通过 → 执行入库
  │     └── 驳回 → 修改后重新提交
  │
  └── 4. 入库完成
        ├── 更新库存：WmsInventoryDO.quantity += 入库数量
        ├── 创建库存变动历史：WmsInventoryHistoryDO
        └── 更新入库单状态
```

### 入库明细实体

```java
// WmsReceiptOrderDetailDO - 入库明细
public class WmsReceiptOrderDetailDO extends BaseDO {
    private Long id;            // 明细编号
    private Long orderId;       // 入库单编号
    private Long skuId;         // 商品 SKU 编号
    private BigDecimal quantity; // 入库数量
    private BigDecimal price;   // 单价
    private BigDecimal totalPrice; // 小计金额
    private String remark;      // 备注
}
```

### 入库类型枚举

```java
// WmsReceiptOrderTypeEnum - 入库类型
public enum WmsReceiptOrderTypeEnum {
    PURCHASE(1, "采购入库"),      // 供应商送货
    SALE_RETURN(2, "销售退货"),   // 客户退货
    OTHER(3, "其他入库")          // 调拨入库、盘盈等
}
```

## 出库作业流程

### 完整流程

```
出库作业流程
  │
  ├── 1. 创建出库单 (WmsShipmentOrderDO)
  │     ├── 选择出库类型（销售出库/退货出库/其他）
  │     ├── 选择仓库
  │     ├── 关联业务单号（销售单号）
  │     └── 选择客户
  │
  ├── 2. 填写出库明细 (WmsShipmentOrderDetailDO)
  │     ├── SKU 1：出库 50 个
  │     ├── SKU 2：出库 100 个
  │     └── ...
  │
  ├── 3. 库存校验
  │     ├── SKU 1：库存 80 >= 50 → 通过
  │     ├── SKU 2：库存 60 < 100 → 不足，提示"库存不足"
  │     └── 全部通过才能审核
  │
  ├── 4. 审核出库单
  │     ├── 通过 → 执行出库
  │     └── 驳回 → 修改后重新提交
  │
  └── 5. 出库完成
        ├── 更新库存：WmsInventoryDO.quantity -= 出库数量
        ├── 创建库存变动历史：WmsInventoryHistoryDO
        └── 更新出库单状态
```

### 出库库存校验

```
创建出库单时
  │
  ▼
遍历出库明细
  │
  ├── 查询 WmsInventoryDO（skuId, warehouseId）
  ├── 当前库存 >= 出库数量？
  │     ├── 是 → 继续检查下一个 SKU
  │     └── 否 → 报错："SKU [xxx] 库存不足，当前库存 X，需要 Y"
  │
  └── 全部校验通过 → 允许创建出库单
```

### 出库类型枚举

```java
// WmsShipmentOrderTypeEnum - 出库类型
public enum WmsShipmentOrderTypeEnum {
    SALE(1, "销售出库"),        // 电商发货
    PURCHASE_RETURN(2, "采购退货"), // 退回供应商
    OTHER(3, "其他出库")         // 调拨出库、盘亏等
}
```

## 库存变动追踪

### 库存变动历史

每次库存变动都会记录一条历史，用于追溯：

```java
// WmsInventoryHistoryDO - 库存变动历史
public class WmsInventoryHistoryDO extends BaseDO {
    private Long id;            // 历史编号
    private Long skuId;         // 商品 SKU 编号
    private Long warehouseId;   // 仓库编号
    private Integer bizType;    // 业务类型（入库/出库/盘点/调拨）
    private String bizOrderId;  // 业务单据编号
    private BigDecimal quantity; // 变动数量（正数增加，负数减少）
    private BigDecimal totalQuantity; // 变动后库存
    private String remark;      // 备注
}
```

### 变动来源

| 业务类型 | 变动方向 | 关联单据 |
|---------|---------|---------|
| 采购入库 | +数量 | 入库单 |
| 销售退货 | +数量 | 入库单 |
| 销售出库 | -数量 | 出库单 |
| 采购退货 | -数量 | 出库单 |
| 盘盈 | +数量 | 盘点单 |
| 盘亏 | -数量 | 盘点单 |
| 调拨入库 | +数量 | 调拨单 |
| 调拨出库 | -数量 | 调拨单 |

### 变动记录的作用

```
库存变动历史的使用场景
  │
  ├── 库存对账
  │     → 核对某个 SKU 在某段时间内的所有变动
  │
  ├── 问题追溯
  │     → "为什么这个 SKU 库存少了 10 个？"
  │     → 查历史记录，找到对应的出库单
  │
  ├── 盘点辅助
  │     → 盘点时查看自上次盘点以来的所有变动
  │
  └── 数据分析
        → 库存周转率、出入库频率等
```

## 盘点差异处理

### 盘点流程

```
盘点流程
  │
  ├── 1. 创建盘点单 (WmsCheckOrderDO)
  │     └── 选择仓库、盘点范围
  │
  ├── 2. 生成盘点清单 (WmsCheckOrderDetailDO)
  │     └── 系统自动列出该仓库的所有 SKU 及账面数量
  │
  ├── 3. 实盘录入
  │     └── 仓库人员逐一录入实际数量
  │
  ├── 4. 差异计算
  │     └── 盈亏数量 = 实盘数量 - 账面数量
  │
  ├── 5. 审核盘点单
  │     ├── 通过 → 执行库存调整
  │     └── 驳回 → 重新盘点
  │
  └── 6. 库存调整
        ├── 盘盈：WmsInventoryDO.quantity += 盈亏数量
        ├── 盘亏：WmsInventoryDO.quantity -= |盈亏数量|
        └── 创建库存变动历史
```

### 盘点明细实体

```java
// WmsCheckOrderDetailDO - 盘点明细
public class WmsCheckOrderDetailDO extends BaseDO {
    private Long id;            // 明细编号
    private Long orderId;       // 盘点单编号
    private Long skuId;         // 商品 SKU 编号
    private BigDecimal bookQuantity;   // 账面数量
    private BigDecimal actualQuantity; // 实盘数量
    private BigDecimal diffQuantity;   // 差异数量
    private BigDecimal price;          // 单价
    private BigDecimal totalPrice;     // 账面金额
    private BigDecimal actualPrice;    // 实际金额
    private String remark;      // 备注
}
```

### 盘点差异分析

```
差异分析
  │
  ├── 盘盈（actualQuantity > bookQuantity）
  │     ├── 原因：多收了货、未入系统的退货、系统错误
  │     └── 处理：增加库存，记录盘盈
  │
  ├── 盘亏（actualQuantity < bookQuantity）
  │     ├── 原因：丢失、损坏、未入系统的出库、系统错误
  │     └── 处理：减少库存，记录盘亏
  │
  └── 盘平（actualQuantity = bookQuantity）
        └── 无差异
```

## 调拨作业流程

### 调拨流程

```
调拨流程
  │
  ├── 1. 创建调拨单 (WmsMovementOrderDO)
  │     ├── 源仓库：北京仓
  │     ├── 目标仓库：上海仓
  │     └── 调拨明细：SKU 1 调拨 200 个
  │
  ├── 2. 审核调拨单
  │     └── 确认调拨合理性
  │
  ├── 3. 源仓库出库
  │     ├── 北京仓：WmsInventoryDO.quantity -= 200
  │     └── 创建出库记录
  │
  ├── 4. 目标仓库入库
  │     ├── 上海仓：WmsInventoryDO.quantity += 200
  │     └── 创建入库记录
  │
  └── 5. 调拨完成
        └── 更新调拨单状态
```

### 调拨单状态

```
调拨状态
  │
  ├── 待调拨 (0)
  │     → 调拨单已创建
  │
  ├── 调拨中 (10)
  │     → 源仓库已出库，货物在途
  │
  └── 已调拨 (20)
        → 目标仓库已入库
```

## 思考题

1. 入库单和出库单的明细都关联了 SKU。如果一个 SKU 有多个批次（不同生产日期），怎么区分？
2. 库存变动历史记录了每次变动。如果历史记录太多（几百万条），查询会不会很慢？怎么优化？
3. 盘点时如果仓库很大（几千个 SKU），一次性盘点不现实。怎么支持"分区盘点"？
4. 调拨单的"在途库存"怎么管理？源仓库已出库但目标仓库还没入库时，这批货算谁的？
