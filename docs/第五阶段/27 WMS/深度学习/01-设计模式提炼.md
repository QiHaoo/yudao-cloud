## 一、库位管理的分区策略

### 1.1 什么是库位分区

仓库通常按功能划分为不同的区域：存储区、拣货区、暂存区等。库位分区策略是 WMS 系统的基础，决定了货物的存放位置和拣货路径。

### 1.4 可迁移经验

**设计要点**：

| 设计决策 | 理由 |
|---------|------|
| 地址编码 | 通过编码快速定位库位，支持批量操作 |
| SKU + 仓库唯一 | 同一 SKU 在同一仓库只有一个库存记录 |
| 分区管理 | 不同区域有不同的业务用途 |
| 批次管理 | 支持先进先出（FIFO）策略 |

**迁移建议**：任何需要"位置管理"的场景都可以使用此模式。例如：停车场管理、座位管理、设备位置管理。

### 3.2 设计模式：差异计算 + 审批流程

项目采用**差异计算**模式，自动计算系统库存与实盘库存的差异。

**盘点单数据模型**：

```java
// 盘点单主表
@TableName("wms_check_order")
public class WmsCheckOrderDO extends BaseDO {
    private Long id;              // 盘点单ID
    private String no;            // 盘点单号
    private Integer status;       // 状态
    private Long warehouseId;     // 仓库ID
    private Integer type;         // 盘点类型
    private LocalDateTime time;   // 盘点时间
    private String remark;        // 备注
}

// 盘点单明细
@TableName("wms_check_order_detail")
public class WmsCheckOrderDetailDO extends BaseDO {
    private Long id;              // 明细ID
    private Long orderId;         // 盘点单ID
    private Long skuId;           // 商品SKU ID
    private BigDecimal systemQty; // 系统库存
    private BigDecimal actualQty; // 实盘数量
    private BigDecimal diffQty;   // 差异数量（自动计算）
    private String remark;        // 备注
}
```

### 3.3 差异处理逻辑

```java
// 差异计算
public void calculateDifference(WmsCheckOrderDetailDO detail) {
    // 差异数量 = 实盘数量 - 系统库存
    BigDecimal diffQty = detail.getActualQty().subtract(detail.getSystemQty());
    detail.setDiffQty(diffQty);
}

// 审核后调整库存
@Override
@Transactional(rollbackFor = Exception.class)
public void approveCheckOrder(Long id) {
    // 1. 更新盘点单状态
    WmsCheckOrderDO checkOrder = validateCheckOrderExists(id);
    checkOrderMapper.updateStatus(id, WmsOrderStatusEnum.FINISHED.getStatus());

    // 2. 查询盘点明细
    List<WmsCheckOrderDetailDO> details = checkOrderDetailMapper.selectListByOrderId(id);

    // 3. 调整库存
    for (WmsCheckOrderDetailDO detail : details) {
        if (detail.getDiffQty().compareTo(BigDecimal.ZERO) != 0) {
            // 更新库存
            inventoryService.updateStockCountIncrement(
                detail.getSkuId(), checkOrder.getWarehouseId(), detail.getDiffQty());
            // 记录库存变动
            inventoryHistoryService.createHistory(detail, checkOrder);
        }
    }
}
```

### 3.4 可迁移经验

**设计要点**：

| 设计决策 | 理由 |
|---------|------|
| 差异自动计算 | 减少人工计算错误 |
| 审核流程 | 盘点结果需要审核确认 |
| 库存联动 | 审核后自动调整库存 |
| 历史记录 | 保留盘点历史，支持追溯 |

**迁移建议**：任何需要"核对差异"的场景都可以使用此模式。例如：账实核对、数据对账、版本对比。
