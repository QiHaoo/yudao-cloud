# 03 - 精简汇总

## 微信公众号流程速查

### 消息接收流程

```
用户消息 → 微信服务器 → /mp/message/{appId} → 签名验证 → 消息路由 → 处理器 → 回复
```

### 自动回复匹配

```
关注回复：用户关注时触发
关键词回复：消息内容匹配关键词
消息回复：消息类型匹配
```

### 模板消息推送

```
业务系统 → 公众号模块 → WxMpService.sendTemplateMsg() → 微信服务器 → 用户收到通知
```

### 菜单同步

```
管理后台编辑 → 保存数据库 → 同步到微信服务器
```

## 核心设计决策

| 决策 | 选择 | 放弃的方案 | 原因 |
|------|------|-----------|------|
| SDK 选择 | WxJava | 直接调用微信 API | 封装完善，社区活跃 |
| 多公众号 | 每个账号独立 WxMpService | 单例复用 | 隔离配置，互不影响 |
| 消息路由 | WxMpMessageRouter | 手动 if-else | 链式配置，易于扩展 |
| 自动回复 | 数据库配置 | 代码硬编码 | 运营可自行配置 |
| 关键词匹配 | 全匹配 + 半匹配 | 只支持全匹配 | 更灵活的匹配方式 |
| 菜单管理 | 数据库 + 同步微信 | 只在微信后台管理 | 统一管理，支持批量操作 |
| 素材管理 | 本地 + 微信双存储 | 只存微信 | 微信素材有数量限制 |
| 消息记录 | 持久化到数据库 | 不记录 | 支持消息追溯和分析 |

## 模块集成关系

```
微信公众号模块
  │
  ├── ◄── System（用户、租户）
  │
  ├── ◄── Trade（订单状态 → 模板消息推送）
  │
  ├── ◄── Member（关注绑定会员）
  │
  └── ──> 微信服务器（API 调用、消息回调）
```
