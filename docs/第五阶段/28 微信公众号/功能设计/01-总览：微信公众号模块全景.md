# 01 - 总览：微信公众号模块全景

## 场景引入

你做了一个电商系统，想通过微信公众号触达用户：

- 用户关注公众号后，自动发送欢迎消息
- 用户发送"查订单"，自动回复订单查询链接
- 电商大促时，给所有粉丝推送模板消息
- 管理公众号菜单（"商城入口"、"我的订单"、"客服中心"）
- 管理素材（图片、视频、图文消息）

这些功能都需要一个**微信公众号管理模块**来支撑。

### 为什么电商系统需要公众号管理？

微信公众号是电商的重要流量入口：

- **消息触达**：通过模板消息推送订单状态、促销活动
- **粉丝运营**：管理粉丝标签、分组，做精细化运营
- **自动回复**：关键词自动回复，提升客服效率
- **菜单导航**：自定义菜单，引导用户进入商城

核心价值是：**让公众号成为电商的"用户触达渠道"，而不是一个孤立的平台**。

## 核心能力

### 1. 账号管理

一个系统可以管理多个公众号：

```java
// MpAccountDO - 公众号账号
public class MpAccountDO extends TenantBaseDO {
    private Long id;            // 账号编号
    private String name;        // 公众号名称
    private String account;     // 公众号账号
    private String appId;       // App ID
    private String appSecret;   // App Secret
    private String token;       // Token（用于消息验证）
    private String aesKey;      // 消息加解密密钥
    private String qrCodeUrl;   // 二维码图片 URL
    private String remark;      // 备注
}
```

**为什么需要 token 和 aesKey？**

- **token**：微信服务器验证消息来源的合法性。微信会用 token 对消息签名，服务器验证签名确认消息来自微信
- **aesKey**：消息加解密密钥。微信支持"安全模式"，消息内容会用 aesKey 加密，服务器需要解密才能读取

### 2. 消息管理

记录公众号收到和发送的所有消息：

```java
// MpMessageDO - 消息记录
public class MpMessageDO extends BaseDO {
    private Long id;            // 消息编号
    private Long accountId;     // 公众号账号编号
    private String appId;       // App ID
    private String openId;      // 用户 OpenID
    private Integer type;       // 消息类型
    private String content;     // 消息内容
    private Boolean reply;      // 是否已回复
    // ...
}
```

### 3. 自动回复

支持三种自动回复类型：

```
自动回复类型 (MpAutoReplyTypeEnum)
  │
  ├── 关注回复 (FOLLOW)
  │     → 用户关注公众号时自动发送
  │     → 例："欢迎关注 XX 商城！"
  │
  ├── 关键词回复 (KEYWORD)
  │     → 用户发送的消息包含特定关键词时回复
  │     → 例：用户发送"查订单" → 回复订单查询链接
  │
  └── 消息回复 (MESSAGE)
        → 用户发送特定类型消息时回复
        → 例：用户发送图片 → 回复"已收到您的图片"
```

## WxJava SDK 集成

### WxJava 是什么

WxJava 是一个封装了微信各种 API 的 Java SDK。公众号模块用的是其中的 `weixin-java-mp` 组件。

```
WxJava SDK
  │
  ├── weixin-java-mp     - 微信公众号
  ├── weixin-java-miniapp - 微信小程序
  ├── weixin-java-pay    - 微信支付
  └── ...
```

### 为什么用 WxJava 而不是直接调用微信 API？

```
直接调用微信 API
  ├── 需要自己处理 HTTP 请求
  ├── 需要自己处理签名验证
  ├── 需要自己处理消息加解密
  ├── 需要自己处理 Token 刷新
  └── 需要自己处理各种错误码

使用 WxJava SDK
  ├── 封装了所有微信 API
  ├── 自动处理签名和加解密
  ├── 自动管理 Token
  └── 统一的异常处理
```

### WxJava 的核心类

```
WxMpService - 公众号服务接口
  │
  ├── getAccessToken()     - 获取访问令牌
  ├── getUserService()     - 粉丝管理
  ├── getMenuService()     - 菜单管理
  ├── getMaterialService() - 素材管理
  ├── getTemplateMsgService() - 模板消息
  └── getMassMessageService() - 群发消息
```

## 电商场景举例

1. **关注欢迎**：用户关注公众号 → 自动回复"欢迎关注 XX 商城，点击链接领取优惠券"
2. **订单查询**：用户发送"查订单" → 自动回复订单列表链接
3. **物流通知**：订单发货后 → 通过模板消息推送物流信息
4. **促销推送**：大促活动 → 通过群发消息通知所有粉丝
5. **菜单导航**：自定义菜单"商城入口" → 跳转到小程序商城

## 思考题

1. 一个系统管理多个公众号（MpAccountDO）。什么场景需要多个公众号？
2. 自动回复支持"关键词匹配"和"消息类型匹配"。如果用户发送的消息同时匹配多个规则，应该回复哪个？
3. 粉丝的 `openId` 是什么？为什么不能用 openId 作为用户的唯一标识？
4. 模板消息和群发消息有什么区别？各自的适用场景是什么？
