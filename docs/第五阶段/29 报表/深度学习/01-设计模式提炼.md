# 报表模块设计模式提炼

## 一、报表引擎的模板方法模式

### 1.1 什么是报表引擎

报表引擎是将数据转换为可视化报表的核心组件。不同类型的报表（表格、图表、仪表盘）有不同的渲染逻辑，但数据获取、处理、格式化的流程是相似的。

```
┌─────────────────────────────────────────────────────────────────┐
│                      报表引擎流程                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────┐                                              │
│   │  数据源配置 │                                              │
│   └──────┬──────┘                                              │
│          │                                                     │
│          ▼                                                     │
│   ┌─────────────┐                                              │
│   │  数据查询   │                                              │
│   └──────┬──────┘                                              │
│          │                                                     │
│          ▼                                                     │
│   ┌─────────────┐                                              │
│   │  数据处理   │                                              │
│   │  (聚合/过滤)│                                              │
│   └──────┬──────┘                                              │
│          │                                                     │
│          ▼                                                     │
│   ┌─────────────┐                                              │
│   │  数据格式化 │                                              │
│   └──────┬──────┘                                              │
│          │                                                     │
│          ▼                                                     │
│   ┌─────────────┐                                              │
│   │  报表渲染   │                                              │
│   └─────────────┘                                              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 设计模式：模板方法 + 策略模式

```java
@Service
public class GoViewDataServiceImpl implements GoViewDataService {

    @Resource

    @Override

        // 1. 获取数据源配置
        GoViewProjectDO project = goViewProjectMapper.selectById(reqVO.getProjectId());
        if (project == null) {
            return null;
        }

        // 2. 执行 SQL 查询
        // 3. 格式化数据
        // 4. 返回结果
        return new GoViewDataRespVO();
    }
}
```

### 1.4 可迁移经验

**设计要点**：

| 设计决策 | 理由 |
|---------|------|
| 多引擎支持 | 集成 GoView 和 JimuReport，满足不同需求 |

**迁移建议**：任何需要"报表生成"的场景都可以使用此模式。例如：数据大屏、BI 分析、Excel 导出。

### 2.2 设计模式：策略模式 + 工厂模式

项目通过**配置化**方式支持多种数据源，每个数据源有独立的连接配置。

**数据源配置示例**：

```yaml

      driver-class-name: com.mysql.cj.jdbc.Driver
    # 只读数据源
    readonly:
      url: jdbc:mysql://localhost:3307/yudao
      username: readonly
      password: readonly
      driver-class-name: com.mysql.cj.jdbc.Driver
```

### 2.3 SQL 执行策略

```java
// GoView 数据获取
@Override
public GoViewDataRespVO getSqlData(GoViewDataGetBySqlReqVO reqVO) {
    // 1. 获取数据源配置
    GoViewProjectDO project = goViewProjectMapper.selectById(reqVO.getProjectId());
    if (project == null) {
        return null;
    }

    // 2. 根据数据源类型选择执行策略
    DataSource dataSource = getDataSource(project.getDatasourceId());

    // 3. 执行 SQL 查询
    List<Map<String, Object>> data = dataSource.query(reqVO.getSql());

    // 4. 格式化数据
    return formatData(data);
}
```

### 2.4 可迁移经验

**设计要点**：

| 设计决策 | 理由 |
|---------|------|
| 数据源抽象 | 统一接口，支持多种数据库 |
| 配置化管理 | 数据源配置存储在数据库，动态切换 |
| 连接池管理 | 使用 HikariCP 管理连接池 |
| 只读分离 | 报表查询使用只读数据源，不影响主库 |

**迁移建议**：任何需要"多数据源"的场景都可以使用此模式。例如：多库查询、数据同步、ETL 流程。

### 3.2 设计模式：建造者模式 + 模板方法

项目通过**配置驱动**方式生成 SQL，用户通过界面配置查询条件，系统自动拼接 SQL。

**SQL 配置结构**：

```java
// SQL 配置 VO
@Data
public class SqlConfigVO {
    /**
     * 数据源 ID
     */
    private Long datasourceId;
    /**
     * 查询字段
     */
    private List<String> fields;
    /**
     * 表名
     */
    private String tableName;
    /**
     * 过滤条件
     */
    private List<FilterCondition> filters;
    /**
     * 排序字段
     */
    private List<SortField> sorts;
    /**
     * 分组字段
     */
    private List<String> groups;
    /**
     * 聚合函数
     */
    private List<AggregateFunction> aggregates;
}

// 过滤条件
@Data
public class FilterCondition {
    private String field;      // 字段名
    private String operator;   // 操作符（=, >, <, LIKE 等）
    private Object value;      // 值
}
```

### 3.3 SQL 生成示例

```java
// 动态 SQL 生成器
public class DynamicSqlBuilder {

    public String build(SqlConfigVO config) {
        StringBuilder sql = new StringBuilder();

        // 1. SELECT 子句
        sql.append("SELECT ");
        if (CollUtil.isEmpty(config.getFields())) {
            sql.append("*");
        } else {
            sql.append(String.join(", ", config.getFields()));
        }

        // 2. FROM 子句
        sql.append(" FROM ").append(config.getTableName());

        // 3. WHERE 子句
        if (CollUtil.isNotEmpty(config.getFilters())) {
            sql.append(" WHERE ");
            List<String> conditions = new ArrayList<>();
            for (FilterCondition filter : config.getFilters()) {
                conditions.add(buildCondition(filter));
            }
            sql.append(String.join(" AND ", conditions));
        }

        // 4. GROUP BY 子句
        if (CollUtil.isNotEmpty(config.getGroups())) {
            sql.append(" GROUP BY ");
            sql.append(String.join(", ", config.getGroups()));
        }

        // 5. ORDER BY 子句
        if (CollUtil.isNotEmpty(config.getSorts())) {
            sql.append(" ORDER BY ");
            List<String> sortFields = new ArrayList<>();
            for (SortField sort : config.getSorts()) {
                sortFields.add(sort.getField() + " " + sort.getDirection());
            }
            sql.append(String.join(", ", sortFields));
        }

        return sql.toString();
    }

    private String buildCondition(FilterCondition filter) {
        // 处理不同操作符
        switch (filter.getOperator()) {
            case "=":
                return filter.getField() + " = " + formatValue(filter.getValue());
            case "LIKE":
                return filter.getField() + " LIKE '%" + filter.getValue() + "%'";
            case ">":
                return filter.getField() + " > " + formatValue(filter.getValue());
            // ... 更多操作符
            default:
                throw new IllegalArgumentException("不支持的操作符: " + filter.getOperator());
        }
    }
}
```

### 3.4 可迁移经验

**设计要点**：

| 设计决策 | 理由 |
|---------|------|
| 配置驱动 | 用户通过界面配置，无需写 SQL |
| SQL 拼接 | 动态生成 SQL，灵活组合条件 |
| 参数化查询 | 防止 SQL 注入 |
| 条件组合 | 支持 AND/OR 逻辑组合 |

**迁移建议**：任何需要"动态查询"的场景都可以使用此模式。例如：高级搜索、数据导出、API 查询接口。

### 4.2 核心设计模式总结

| 模式 | 应用场景 | 核心类 |
|------|---------|--------|
| **模板方法** | 报表渲染流程 | GoViewDataService |
