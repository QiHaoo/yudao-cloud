# 01 - 总览：报表模块全景

## 场景引入

你做了一个电商系统，老板说"我要看数据"：

- "这个月销售额多少？比上个月增长了多少？"
- "哪些商品卖得最好？哪些滞销？"
- "用户从哪里来？转化率多少？"
- "仓库库存周转率多少？"

这些需求都需要**报表**来满足。

### 为什么电商系统需要报表模块？

报表是"让数据说话"的工具：

- **运营报表**：销售额、订单量、转化率
- **商品报表**：商品销量排行、库存周转
- **用户报表**：新增用户、活跃用户、留存率
- **财务报表**：收入、支出、利润

核心价值是：**把散落在各个模块的数据汇总成可视化的图表，辅助决策**。

## 两大报表工具

### 1. JimuReport（积木报表）

JimuReport 是一个开源的报表引擎，支持：

- **报表设计**：拖拽式设计器，所见即所得
- **数据集**：SQL 查询、API 接口、Java 对象
- **报表类型**：列表报表、分组报表、交叉报表、主子报表
- **导出格式**：Excel、PDF、Word、图片
- **打印**：直接打印、批量打印

### 2. GoView（数据大屏）

GoView 是一个数据可视化平台，支持：

- **可视化图表**：折线图、柱状图、饼图、地图、仪表盘
- **拖拽式布局**：拖拽组件到画布，调整大小和位置
- **数据绑定**：绑定 SQL 查询或 API 接口
- **大屏展示**：适配大屏幕，支持全屏展示
- **实时刷新**：定时刷新数据

---

## 两种报表的区别

| 维度 | JimuReport（积木报表） | GoView（数据大屏） |
|------|----------------------|-------------------|
| 定位 | 传统报表 | 数据可视化大屏 |
| 场景 | 财务报表、运营报表 | 监控大屏、展示大屏 |
| 格式 | 表格为主 | 图表为主 |
| 交互 | 查询、导出、打印 | 查看、刷新 |
| 使用者 | 运营、财务 | 管理层、展厅 |
| 部署方式 | 嵌入业务系统 | 独立大屏页面 |

## JimuReport 集成

### 集成方式

JimuReport 通过 JAR 包集成到 Spring Boot 项目中：

```
集成方式
  │
  ├── 方式 1：内嵌模式
  │     → JimuReport 作为依赖嵌入到业务系统
  │     → 共用业务系统的数据源
  │     → 通过 URL 访问报表设计器和报表页面
  │
  └── 方式 2：独立部署
        → JimuReport 独立部署为一个服务
        → 通过 API 获取报表数据
        → 适合微服务架构
```

### 核心概念

```
JimuReport 核心概念
  │
  ├── 数据源 (DataSource)
  │     → 报表数据来自哪个数据库
  │     → 支持 MySQL、PostgreSQL、Oracle 等
  │
  ├── 数据集 (DataSet)
  │     → 报表的数据查询
  │     ├── SQL 数据集：SELECT * FROM orders WHERE ...
  │     ├── API 数据集：调用 REST 接口获取数据
  │     └── Java 数据集：调用 Java 方法获取数据
  │
  ├── 报表模板 (Report)
  │     → 报表的布局和格式
  │     ├── 表头：列标题
  │     ├── 表体：数据行
  │     ├── 表尾：汇总行
  │     └── 参数区：查询条件
  │
  └── 报表实例 (Report Instance)
        → 运行时生成的报表
        → 根据参数查询数据，填充模板
```

### 报表设计流程

```
报表设计流程
  │
  ├── 1. 配置数据源
  │     └── 填写数据库连接信息
  │
  ├── 2. 创建数据集
  │     └── 编写 SQL 查询或配置 API 接口
  │
  ├── 3. 设计报表模板
  │     ├── 拖拽字段到报表区域
  │     ├── 设置列标题、列宽
  │     ├── 设置数据格式（数字、日期、金额）
  │     ├── 设置汇总公式（SUM、AVG、COUNT）
  │     └── 设置查询参数
  │
  ├── 4. 预览报表
  │     └── 查看报表效果
  │
  └── 5. 发布报表
        └── 生成报表访问 URL
```

## GoView 集成

### GoView 架构

```
GoView 架构
  │
  ├── 前端（Vue 3 + ECharts）
  │     ├── 可视化设计器
  │     ├── 大屏预览
  │     └── 组件库（图表、地图、文字、图片）
  │
  └── 后端（Spring Boot）
        ├── 项目管理 (GoViewProjectDO)
        ├── 数据源管理
        └── 数据接口
```

### GoView 项目实体

```java
// GoViewProjectDO - GoView 项目
public class GoViewProjectDO extends BaseDO {
    private Long id;            // 项目编号
    private String name;        // 项目名称
    private String picUrl;      // 预览图片 URL
    private String content;     // 报表内容（JSON 配置）
    private Integer status;     // 发布状态
    private String remark;      // 备注
}
```

### GoView 数据流

```
GoView 数据流
  │
  ├── 1. 设计师在 GoView 可视化设计器中拖拽组件
  │
  ├── 2. 每个组件绑定一个数据接口
  │     ├── 折线图 → GET /api/sales/trend
  │     ├── 饼图 → GET /api/sales/category
  │     └── 数字 → GET /api/sales/today
  │
  ├── 3. 保存项目配置（JSON 格式）到数据库
  │
  └── 4. 大屏页面加载时
        ├── 读取项目配置
        ├── 渲染组件布局
        └── 定时调用数据接口刷新数据
```

## 报表使用场景

### 场景 1：销售日报

```
销售日报（JimuReport）
  │
  ├── 数据集：SELECT date, SUM(amount) FROM orders GROUP BY date
  │
  ├── 报表格式：
  │     ┌──────────┬──────────┬──────────┐
  │     │ 日期      │ 订单数    │ 销售额    │
  │     ├──────────┼──────────┼──────────┤
  │     │ 2024-06-01│ 150      │ 45,000   │
  │     │ 2024-06-02│ 180      │ 52,000   │
  │     │ ...      │ ...      │ ...      │
  │     ├──────────┼──────────┼──────────┤
  │     │ 合计      │ 3,200    │ 980,000  │
  │     └──────────┴──────────┴──────────┘
  │
  └── 导出：Excel / PDF
```

### 场景 2：运营大屏

```
运营大屏（GoView）
  │
  ├── 左侧：销售趋势折线图

  │
  ├── 中间：地图热力图
  │     └── 各省份的订单分布
  │
  ├── 右侧：商品销量排行

  │
  └── 底部：实时数字
```

## 思考题

1. JimuReport 和 GoView 各自适合什么场景？能不能只用一个？
2. 报表的数据源直接连业务数据库。如果业务数据量很大（几千万行），报表查询会不会影响业务系统的性能？
3. GoView 的项目配置用 JSON 存储。如果 JSON 很大（几 MB），存储和读取会不会有性能问题？
4. 报表的权限怎么控制？不同角色的用户看到的报表应该一样吗？
