# 03-测试支持与源码索引

## 测试支持

### 现状

电商模块（yudao-module-mall）的测试资源位于各子服务的 `src/test/` 目录下，配置了 `application-unit-test.yaml` 用于单元测试。目前项目中电商模块的单元测试覆盖较少，主要依赖集成测试和手动测试。

### 测试策略建议

电商模块的测试挑战在于**跨服务调用**和**状态机流转**。建议按以下层次组织测试：

**1. 实体层测试**：验证 DO 字段映射、JSON 类型处理器（如 `ProductSkuDO.properties` 的 `JacksonTypeHandler`）

**2. Service 层测试**：
- SPU 创建：Mock 分类和品牌校验，验证 SPU+SKU 联合写入
- 订单创建：Mock 商品服务（获取 SKU 信息）、促销服务（计算优惠），验证价格计算逻辑
- 售后流程：验证状态机流转（申请→审核→退款）

**3. 跨服务测试**：
- 交易服务调用商品服务获取 SKU 信息
- 交易服务调用支付服务创建支付订单
- 统计服务聚合交易数据

### Mock 建议

```java
// 示例：测试 SPU 创建时 Mock 分类校验
@MockBean
private ProductCategoryService categoryService;

@Test
public void testCreateSpu() {
    // given
    when(categoryService.getCategoryLevel(anyLong())).thenReturn(2);
    // when & then
    // ...
}
```

### 测试数据准备

- 商品数据：准备 SPU、SKU、分类、品牌的测试数据
- 订单数据：准备购物车、订单、订单项的测试数据
- 使用 H2 内存数据库进行单元测试，避免依赖外部 MySQL
