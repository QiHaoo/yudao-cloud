# 01-起步：商品子模块实现

本章从零搭建商品子模块的核心实体和服务，让读者理解 SPU/SKU 模型的代码实现细节。

> **前置阅读**：本章对应功能设计中的「02-核心设计：商品与交易」中商品模型部分。如果对 SPU/SKU 的设计动机还不清楚，建议先阅读功能设计。

## 模块结构

商品子服务位于 `yudao-module-mall/yudao-module-product-server`，采用标准的分层架构：

```
yudao-module-product-server/src/main/java/cn/iocoder/yudao/module/product/
├── controller/admin/          ← 管理后台 API（/admin-api/product）
│   ├── spu/                   ← SPU 管理
│   ├── category/              ← 分类管理
│   ├── brand/                 ← 品牌管理
│   └── property/              ← 属性管理
├── controller/app/            ← 用户端 API（/app-api/product）
│   └── spu/                   ← 商品详情、列表
├── service/                   ← 业务逻辑层
├── dal/
│   ├── dataobject/            ← DO 实体
│   └── mysql/                 ← Mapper 接口
└── convert/                   ← MapStruct 对象转换器
```

## 核心实体：ProductSpuDO

源码位置：`yudao-module-product-server/.../dal/dataobject/spu/ProductSpuDO.java`

```java
@TableName(value = "product_spu", autoResultMap = true)
@KeySequence("product_spu_seq")
@Data
@EqualsAndHashCode(callSuper = true)
@Builder
@NoArgsConstructor
@AllArgsConstructor
public class ProductSpuDO extends BaseDO {

    @TableId
    private Long id;

    // ========== 基本信息 =========
    private String name;              // 商品名称
    private String keyword;           // 搜索关键字
    private String introduction;      // 商品简介（列表页展示）
    private String description;       // 商品详情（富文本，详情页展示）

    private Long categoryId;          // 商品分类编号，关联 ProductCategoryDO
    private Long brandId;             // 商品品牌编号，关联 ProductBrandDO
    private String picUrl;            // 商品封面图

    @TableField(typeHandler = JacksonTypeHandler.class)
    private List<String> sliderPicUrls; // 商品轮播图

    private Integer sort;             // 排序字段
    private Integer status;           // 商品状态：-1 回收站 / 0 下架 / 1 上架

    // ========== SKU 相关字段 =========
    private Boolean specType;         // 规格类型：false 单规格 / true 多规格
    private Integer price;            // 商品价格（取 SKU 最低价），单位：分
    private Integer marketPrice;      // 市场价（取 SKU 最低市场价），单位：分
    private Integer costPrice;        // 成本价（取 SKU 最低成本价），单位：分
    private Integer stock;            // 库存（所有 SKU 库存求和）

    // ========== 物流相关字段 =========
    @TableField(typeHandler = IntegerListTypeHandler.class)
    private List<Integer> deliveryTypes; // 配送方式数组
    private Long deliveryTemplateId;  // 物流配置模板编号

    // ========== 营销相关字段 =========
    private Integer giveIntegral;     // 赠送积分
    private Boolean subCommissionType; // 分销类型：false 默认 / true 自行设置

    // ========== 统计相关字段 =========
    private Integer salesCount;       // 商品销量
    private Integer virtualSalesCount;// 虚拟销量（运营设置）
    private Integer browseCount;      // 浏览量
}
```

**设计要点**：

- `price`、`marketPrice`、`costPrice`、`stock` 是从 SKU 聚合而来的冗余字段。SPU 列表页需要快速展示"价格从 xxx 元起"，如果每次都 JOIN SKU 表会很慢，这是典型的**读优化**设计
- `virtualSalesCount`（虚拟销量）是运营手段：新上架商品真实销量为 0，但可设置虚拟销量营造热销氛围
- `specType` 区分单规格和多规格。单规格商品无需用户选择属性，系统自动创建默认 SKU

## 核心实体：ProductSkuDO

源码位置：`yudao-module-product-server/.../dal/dataobject/sku/ProductSkuDO.java`

```java
@TableName(value = "product_sku", autoResultMap = true)
@KeySequence("product_sku_seq")
@Data
@EqualsAndHashCode(callSuper = true)
@Builder
@NoArgsConstructor
@AllArgsConstructor
public class ProductSkuDO extends BaseDO {

    @TableId
    private Long id;
    private Long spuId;               // SPU 编号，关联 ProductSpuDO

    @TableField(typeHandler = JacksonTypeHandler.class)
    private List<Property> properties; // 属性数组，JSON 格式

    private Integer price;            // 售价，单位：分
    private Integer marketPrice;      // 市场价，单位：分
    private Integer costPrice;        // 成本价，单位：分
    private String barCode;           // 商品条码
    private String picUrl;            // 图片地址
    private Integer stock;            // 库存
    private Double weight;            // 商品重量，单位：kg
    private Double volume;            // 商品体积，单位：m^3

    private Integer firstBrokeragePrice;  // 一级分销佣金，单位：分
    private Integer secondBrokeragePrice; // 二级分销佣金，单位：分
    private Integer salesCount;       // 商品销量

    /**
     * 商品属性
     */
    @Data
    @NoArgsConstructor
    @AllArgsConstructor
    public static class Property {
        private Long propertyId;      // 属性编号，关联 ProductPropertyDO
        private String propertyName;  // 属性名字（冗余）
        private Long valueId;         // 属性值编号，关联 ProductPropertyValueDO
        private String valueName;     // 属性值名字（冗余）
    }
}
```

**设计要点**：

- `properties` 用 JSON 数组存储属性组合，例如 `[{"propertyId":1,"propertyName":"颜色","valueId":10,"valueName":"黑色"}]`
- 属性名和属性值名被**冗余**存储。因为属性名可能被修改（如"颜色"改成"机身颜色"），但已生成的 SKU 需要保持历史快照。这和订单冗余商品信息是同一个设计思想：**关键时刻拍照，而非持续引用**
- 价格字段单位统一使用**分**，避免浮点数精度问题

## SPU 创建流程：ProductSpuServiceImpl

源码位置：`yudao-module-product-server/.../service/spu/ProductSpuServiceImpl.java`

SPU 创建是商品子模块最核心的业务方法，涉及 SPU 和 SKU 的联合写入：

```java
@Override
@Transactional(rollbackFor = Exception.class)
public Long createSpu(ProductSpuSaveReqVO createReqVO) {
    // 1. 校验分类、品牌
    validateCategory(createReqVO.getCategoryId());
    brandService.validateProductBrand(createReqVO.getBrandId());
    // 2. 校验 SKU
    List<ProductSkuSaveReqVO> skuSaveReqList = createReqVO.getSkus();
    productSkuService.validateSkuList(skuSaveReqList, createReqVO.getSpecType());

    // 3. 转换为 DO 并初始化聚合字段
    ProductSpuDO spu = BeanUtils.toBean(createReqVO, ProductSpuDO.class);
    initSpuFromSkus(spu, skuSaveReqList);

    // 4. 插入 SPU
    productSpuMapper.insert(spu);
    // 5. 插入 SKU
    productSkuService.createSkuList(spu.getId(), skuSaveReqList);
    return spu.getId();
}
```

### 聚合字段初始化

`initSpuFromSkus` 方法从 SKU 列表中计算 SPU 的聚合字段：

```java
private void initSpuFromSkus(ProductSpuDO spu, List<ProductSkuSaveReqVO> skus) {
    // sku 单价最低的商品的价格
    spu.setPrice(getMinValue(skus, ProductSkuSaveReqVO::getPrice));
    // sku 单价最低的商品的市场价格
    spu.setMarketPrice(getMinValue(skus, ProductSkuSaveReqVO::getMarketPrice));
    // sku 单价最低的商品的成本价格
    spu.setCostPrice(getMinValue(skus, ProductSkuSaveReqVO::getCostPrice));
    // skus 库存总数
    spu.setStock(getSumValue(skus, ProductSkuSaveReqVO::getStock, Math::addExact));
    // 默认状态为上架
    if (spu.getStatus() == null) {
        spu.setStatus(ProductSpuStatusEnum.ENABLE.getStatus());
        spu.setSalesCount(0);
        spu.setBrowseCount(0);
    }
}
```

这里用到了 `CollectionUtils` 工具类的 `getMinValue` 和 `getSumValue` 方法，是项目中常用的集合聚合工具。

### 分类层级校验

```java
private void validateCategory(Long id) {
    categoryService.validateCategory(id);
    // 校验层级：必须是二级分类
    if (categoryService.getCategoryLevel(id) < CATEGORY_LEVEL) {
        throw exception(SPU_SAVE_FAIL_CATEGORY_LEVEL_ERROR);
    }
}
```

商品必须挂在**二级分类**下，不允许挂在一级分类。这是业务约束——一级分类是"手机数码"，二级分类才是"智能手机"。

## SPU 更新流程

更新 SPU 时，需要同步更新关联的 SKU：

```java
@Override
@Transactional(rollbackFor = Exception.class)
public void updateSpu(ProductSpuSaveReqVO updateReqVO) {
    // 1. 校验 SPU 是否存在
    ProductSpuDO spu = validateSpuExists(updateReqVO.getId());
    // 2. 校验分类、品牌
    validateCategory(updateReqVO.getCategoryId());
    brandService.validateProductBrand(updateReqVO.getBrandId());
    // 3. 校验 SKU
    List<ProductSkuSaveReqVO> skuSaveReqList = updateReqVO.getSkus();
    productSkuService.validateSkuList(skuSaveReqList, updateReqVO.getSpecType());

    // 4. 更新 SPU（保留原状态）
    ProductSpuDO updateObj = BeanUtils.toBean(updateReqVO, ProductSpuDO.class).setStatus(spu.getStatus());
    initSpuFromSkus(updateObj, skuSaveReqList);
    productSpuMapper.updateById(updateObj);
    // 5. 批量更新 SKU
    productSkuService.updateSkuList(updateObj.getId(), updateReqVO.getSkus());
}
```

注意更新时**保留原状态**（`setStatus(spu.getStatus())`），避免编辑商品时意外改变上下架状态。

## SPU 删除流程

删除有前置条件——只有**回收站**状态的 SPU 才能删除：

```java
@Override
@Transactional(rollbackFor = Exception.class)
public void deleteSpu(Long id) {
    validateSpuExists(id);
    ProductSpuDO spuDO = productSpuMapper.selectById(id);
    // 只有回收站状态才能删除
    if (ObjectUtil.notEqual(spuDO.getStatus(), ProductSpuStatusEnum.RECYCLE.getStatus())) {
        throw exception(SPU_NOT_RECYCLE);
    }
    // 删除 SPU
    productSpuMapper.deleteById(id);
    // 删除关联的 SKU
    productSkuService.deleteSkuBySpuId(id);
}
```

这是一个**软删除 + 硬删除**的两阶段设计：先移到回收站（状态变更），确认不再需要后才真正删除。

## SPU 状态管理

SPU 有三种状态，通过 `ProductSpuStatusEnum` 枚举定义：

| 状态值 | 枚举 | 含义 |
|--------|------|------|
| 1 | ENABLE | 上架（销售中） |
| 0 | DISABLE | 下架（仓库中） |
| -1 | RECYCLE | 回收站 |

状态转换通过 `updateSpuStatus` 方法控制，支持上架、下架、移到回收站等操作。

管理后台的 Tab 页签统计也通过 `getTabsCount` 方法实现：

```java
@Override
public Map<Integer, Long> getTabsCount() {
    Map<Integer, Long> counts = Maps.newLinkedHashMapWithExpectedSize(5);
    counts.put(ProductSpuPageReqVO.FOR_SALE,
            productSpuMapper.selectCount(ProductSpuDO::getStatus, ProductSpuStatusEnum.ENABLE.getStatus()));
    counts.put(ProductSpuPageReqVO.IN_WAREHOUSE,
            productSpuMapper.selectCount(ProductSpuDO::getStatus, ProductSpuStatusEnum.DISABLE.getStatus()));
    counts.put(ProductSpuPageReqVO.SOLD_OUT,
            productSpuMapper.selectCount(ProductSpuDO::getStock, 0));
    counts.put(ProductSpuPageReqVO.ALERT_STOCK,
            productSpuMapper.selectCount());
    counts.put(ProductSpuPageReqVO.RECYCLE_BIN,
            productSpuMapper.selectCount(ProductSpuDO::getStatus, ProductSpuStatusEnum.RECYCLE.getStatus()));
    return counts;
}
```

## 库存扣减

交易下单时需要扣减 SPU 和 SKU 的库存。`updateSpuStock` 方法通过批量更新实现：

```java
@Override
@Transactional(rollbackFor = Exception.class)
public void updateSpuStock(Map<Long, Integer> stockIncrCounts) {
    stockIncrCounts.forEach((id, incCount) -> productSpuMapper.updateStock(id, incCount));
}
```

其中 `incCount` 是增量（负数表示扣减），SQL 层面用 `SET stock = stock + #{incrCount}` 保证原子性。

## 小结

| 组件 | 职责 | 关键设计 |
|------|------|---------|
| ProductSpuDO | 商品标准单元 | 聚合冗余 SKU 的最低价和总库存 |
| ProductSkuDO | 最小可售卖单元 | JSON 属性组合 + 冗余属性名 |
| ProductSpuServiceImpl | SPU 业务逻辑 | 事务保证 SPU+SKU 联合写入 |
| 单规格/多规格 | specType 字段区分 | 单规格自动创建默认属性 |
