# 02-交易子模块实现

本章实现交易子模块的核心流程：购物车、订单创建、支付回调、售后处理。

> **前置阅读**：本章对应功能设计中的「02-核心设计：商品与交易」中订单流程部分。

## 模块结构

交易子服务位于 `yudao-module-mall/yudao-module-trade-server`：

```
yudao-module-trade-server/src/main/java/cn/iocoder/yudao/module/trade/
├── controller/admin/          ← 管理后台 API（/admin-api/trade）
│   ├── order/                 ← 订单管理
│   ├── aftersale/             ← 售后管理
│   ├── delivery/              ← 物流管理
│   ├── brokerage/             ← 分销管理
│   └── config/                ← 交易配置
├── controller/app/            ← 用户端 API（/app-api/trade）
│   ├── order/                 ← 下单、支付
│   ├── cart/                  ← 购物车
│   └── aftersale/             ← 用户端售后
├── service/                   ← 业务逻辑层
├── dal/
│   ├── dataobject/            ← DO 实体
│   └── mysql/                 ← Mapper 接口
└── enums/                     ← 枚举定义
```

## 购物车实体：CartDO

源码位置：`yudao-module-trade-server/.../dal/dataobject/cart/CartDO.java`

```java
@TableName("trade_cart")
@Data
public class CartDO extends BaseDO {
    private Long id;
    private Long userId;      // 用户编号
    private Long spuId;       // 商品 SPU
    private Long skuId;       // 商品 SKU
    private Integer count;    // 购买数量
    private Boolean selected; // 是否选中（结算时只处理选中的商品）
}
```

`selected` 字段用于"部分结算"场景：用户购物车里有 5 件商品，但只想结算其中 3 件，通过 `selected = true` 标记要结算的商品。

## 订单主表：TradeOrderDO

源码位置：`yudao-module-trade-server/.../dal/dataobject/order/TradeOrderDO.java`

订单采用**主表 + 明细表**的经典设计。主表记录整体信息：

```java
@TableName(value = "trade_order", autoResultMap = true)
@KeySequence("trade_order_seq")
@Data
@EqualsAndHashCode(callSuper = true)
@Builder
@NoArgsConstructor
@AllArgsConstructor
public class TradeOrderDO extends BaseDO {

    // ========== 订单基本信息 ==========
    private Long id;
    private String no;                // 订单流水号
    private Integer type;             // 订单类型，枚举 TradeOrderTypeEnum
    private Integer terminal;         // 订单来源，枚举 TerminalEnum
    private Long userId;              // 用户编号
    private String userIp;            // 用户 IP
    private String userRemark;        // 用户备注
    private Integer status;           // 订单状态，枚举 TradeOrderStatusEnum
    private Integer productCount;     // 购买的商品数量
    private LocalDateTime finishTime; // 订单完成时间
    private LocalDateTime cancelTime; // 订单取消时间
    private Integer cancelType;       // 取消类型
    private String remark;            // 商家备注
    private Boolean commentStatus;    // 是否已评价
    private Long brokerageUserId;     // 推广人编号

    // ========== 价格 + 支付基本信息 ==========
    private Long payOrderId;          // 支付订单编号（关联 PayOrderDO）
    private Boolean payStatus;        // 是否已支付
    private LocalDateTime payTime;    // 付款时间
    private String payChannelCode;    // 支付渠道

    /**
     * 应付金额（总），单位：分
     * = totalPrice - couponPrice - pointPrice - discountPrice
     *   + deliveryPrice + adjustPrice - vipPrice
     */
    private Integer payPrice;
    private Integer totalPrice;       // 商品原价总和
    private Integer discountPrice;    // 优惠金额
    private Integer deliveryPrice;    // 运费金额
    private Integer adjustPrice;      // 订单调价（正数加价，负数减价）

    // ========== 收件 + 物流基本信息 ==========
    private Integer deliveryType;     // 配送方式，枚举 DeliveryTypeEnum
    private Long logisticsId;         // 发货物流公司编号
    private String logisticsNo;       // 发货物流单号
    private LocalDateTime deliveryTime; // 发货时间
    private LocalDateTime receiveTime;// 收货时间
    private String receiverName;      // 收件人名称
    private String receiverMobile;    // 收件人手机
    private Integer receiverAreaId;   // 收件人地区编号
    private String receiverDetailAddress; // 收件人详细地址
    private Long pickUpStoreId;       // 自提门店编号
    private String pickUpVerifyCode;  // 自提核销码

    // ========== 售后基本信息 ==========
    private Integer refundStatus;     // 售后状态，枚举 TradeOrderRefundStatusEnum
    private Integer refundPrice;      // 退款金额，单位：分

    // ========== 营销基本信息 ==========
    private Long couponId;            // 优惠券编号
    private Integer couponPrice;      // 优惠券减免金额
    private Integer usePoint;         // 使用的积分
    private Integer pointPrice;       // 积分抵扣的金额
    private Integer givePoint;        // 赠送的积分
    private Integer refundPoint;      // 退还的积分
    private Integer vipPrice;         // VIP 减免金额

    @TableField(typeHandler = JacksonTypeHandler.class)
    private Map<Long, Integer> giveCouponTemplateCounts; // 赠送的优惠券

    private Long seckillActivityId;   // 秒杀活动编号
    private Long combinationActivityId; // 拼团活动编号
    private Long combinationRecordId; // 拼团记录编号
}
```

**设计要点**：

- `payPrice` 的计算公式在注释中有完整说明，涉及 7 个价格维度的加减
- 物流相关字段中，`logisticsId = 0L` 和 `logisticsNo = ""` 表示无需发货（如虚拟商品），避免增加额外字段
- 营销字段支持优惠券、积分、VIP、秒杀、拼团等多种促销方式

## 订单明细：TradeOrderItemDO

源码位置：`yudao-module-trade-server/.../dal/dataobject/order/TradeOrderItemDO.java`

```java
@TableName(value = "trade_order_item", autoResultMap = true)
@KeySequence("trade_order_item_seq")
@Data
public class TradeOrderItemDO extends BaseDO {

    private Long id;
    private Long userId;              // 用户编号
    private Long orderId;             // 订单编号，关联 TradeOrderDO
    private Long cartId;              // 购物车项编号

    // ========== 商品基本信息（冗余快照） ==========
    private Long spuId;               // 商品 SPU 编号
    private String spuName;           // 商品 SPU 名称（冗余）
    private Long skuId;               // 商品 SKU 编号
    @TableField(typeHandler = JacksonTypeHandler.class)
    private List<Property> properties; // 属性数组（冗余）
    private String picUrl;            // 商品图片（冗余）
    private Integer count;            // 购买数量
    private Boolean commentStatus;    // 是否已评价

    // ========== 价格信息 ==========
    private Integer price;            // 商品原价（单），单位：分
    private Integer discountPrice;    // 优惠金额（总），单位：分
    private Integer deliveryPrice;    // 运费金额（总），单位：分
    private Integer adjustPrice;      // 订单调价（总），单位：分
    /** 应付金额（总）= price * count - couponPrice - pointPrice - discountPrice + deliveryPrice + adjustPrice - vipPrice */
    private Integer payPrice;

    // ========== 营销信息 ==========
    private Integer couponPrice;      // 优惠券减免金额
    private Integer pointPrice;       // 积分抵扣金额
    private Integer usePoint;         // 使用的积分
    private Integer givePoint;        // 赠送的积分
    private Integer vipPrice;         // VIP 减免金额

    // ========== 售后信息 ==========
    private Long afterSaleId;         // 售后单编号
    private Integer afterSaleStatus;  // 售后状态

    @Data
    public static class Property implements Serializable {
        private Long propertyId;
        private String propertyName;
        private Long valueId;
        private String valueName;
    }
}
```

**快照设计**：订单明细中**大量冗余**了商品信息（spuName、properties、picUrl）。下单时将商品信息"拍照"存入订单，即使后续商品信息变更，订单数据也不受影响。这是电商系统的经典模式。

## 订单状态机

订单状态通过 `TradeOrderStatusEnum` 枚举定义，状态流转如下：

```
                        用户取消                超时未支付
                    ┌───────────┐          ┌───────────┐
                    │           │          │           │
                    ▼           │          ▼           │
┌──────┐    ┌──────┐    ┌──────┐    ┌──────┐    ┌──────┐
│ 待支付 │───▶│ 已支付 │───▶│ 已发货 │───▶│ 已收货 │───▶│ 已完成 │
│  0    │    │  10   │    │  20   │    │  30   │    │  40   │
└──────┘    └──────┘    └──────┘    └──────┘    └──────┘
                │
                │  申请退款
                ▼
           ┌──────────┐
           │  退款中   │
           │   11     │
           └──────────┘
```

每个状态转换都会记录在 `TradeOrderLogDO` 中：

```java
// 来源：yudao-module-trade-server/.../order/TradeOrderLogDO.java
private Long orderId;         // 订单编号
private Integer beforeStatus; // 操作前状态
private Integer afterStatus;  // 操作后状态
private Integer operateType;  // 操作类型
private String content;       // 日志内容
private Long userId;          // 操作人
private Integer userType;     // 操作人类型（用户/管理员/系统）
```

这个日志表记录了订单的完整生命周期，便于排查问题和审计。

## 订单金额计算

订单的 `payPrice` 计算涉及多个优惠维度叠加：

```
应付金额 = 商品原价总和 (totalPrice)
         - 优惠券减免 (couponPrice)
         - 积分抵扣 (pointPrice)
         - 促销活动折扣 (discountPrice)
         + 运费 (deliveryPrice)
         + 手动调价 (adjustPrice)
         - VIP 减免 (vipPrice)
```

每个订单项也有独立的金额计算，保证退款时能精确到商品级别。

## 售后实体：AfterSaleDO

源码位置：`yudao-module-trade-server/.../dal/dataobject/aftersale/AfterSaleDO.java`

```java
@TableName(value = "trade_after_sale", autoResultMap = true)
@KeySequence("trade_after_sale_seq")
@Data
public class AfterSaleDO extends BaseDO {

    private Long id;
    private String no;                // 售后单号
    private Integer status;           // 退款状态，枚举 AfterSaleStatusEnum
    private Integer way;              // 售后方式：仅退款 / 退货退款
    private Integer type;             // 售后类型
    private Long userId;              // 用户编号
    private String applyReason;       // 申请原因
    private String applyDescription;  // 补充描述

    @TableField(typeHandler = JacksonTypeHandler.class)
    private List<String> applyPicUrls; // 补充凭证图片

    // ========== 交易订单相关（冗余） ==========
    private Long orderId;             // 交易订单编号
    private String orderNo;           // 订单流水号（冗余）
    private Long orderItemId;         // 订单项编号
    private Long spuId;               // 商品 SPU 编号（冗余）
    private String spuName;           // 商品 SPU 名称（冗余）
    private Long skuId;               // 商品 SKU 编号
    @TableField(typeHandler = JacksonTypeHandler.class)
    private List<TradeOrderItemDO.Property> properties; // 属性（冗余）
    private String picUrl;            // 商品图片（冗余）
    private Integer count;            // 退货商品数量

    // ========== 审批相关 ==========
    private LocalDateTime auditTime;  // 审批时间
    private Long auditUserId;         // 审批人
    private String auditReason;       // 审批备注

    // ========== 退款相关 ==========
    private Integer refundPrice;      // 退款金额，单位：分
    private Long payRefundId;         // 支付退款编号（关联 PayRefundDO）
    private LocalDateTime refundTime; // 退款时间

    // ========== 退货相关 ==========
    private Long logisticsId;         // 退货物流公司编号
    private String logisticsNo;       // 退货物流单号
    private LocalDateTime deliveryTime; // 退货时间
    private LocalDateTime receiveTime;// 收货时间
    private String receiveReason;     // 收货备注
}
```

**设计要点**：

- 售后单同样采用**快照设计**，冗余了商品信息
- 支持两种售后方式：`way` 字段区分仅退款和退货退款
- 退款通过 `payRefundId` 关联支付服务的退款订单，实现跨服务协作

## 售后流程

```
用户申请 → 商家审核 → [退货退款需要] 用户寄回商品 → 商家确认收货 → 退款
  │              │                                        │
  │              ▼                                        │
  │         拒绝申请                                      │
  │              │                                        │
  └──────────────┴────────────────────────────────────────┘
                    所有路径最终更新 AfterSaleDO.status
```

售后状态独立于订单状态，通过 `TradeOrderItemDO.afterSaleStatus` 跟踪每个订单项的售后情况。一个订单中的不同商品可以分别申请售后。

## 分销体系

### 核心实体

```
BrokerageUserDO（推广员）
├── id: Long              ← 用户编号
├── brokeragePrice: Integer  ← 可用佣金（已解冻，可提现），单位：分
├── frozenPrice: Integer     ← 冻结佣金（待确认收货后解冻），单位：分
├── parentId: Long        ← 上级推广员
└── bindUserTime: LocalDateTime ← 绑定时间

BrokerageRecordDO（佣金记录）
├── userId: Long          ← 佣金归属用户
├── price: Integer        ← 佣金金额
├── status: Integer       ← 状态
├── frozenDays: Integer   ← 冻结天数
├── sourceUserId: Long    ← 来源用户（谁下单产生的佣金）
└── sourceUserLevel: Integer ← 来源用户等级（一级/二级）

BrokerageWithdrawDO（提现记录）
├── price: Integer        ← 提现金额
├── feePrice: Integer     ← 手续费
├── type: Integer         ← 提现类型（微信/支付宝/银行卡）
└── status: Integer       ← 审核状态
```

### 佣金生命周期

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 佣金产生  │───▶│ 佣金冻结  │───▶│ 佣金解冻  │───▶│ 佣金提现  │
│          │    │          │    │          │    │          │
│ 用户下单  │    │ frozenDays│    │ 确认收货  │    │ 申请提现  │
│ 记录佣金  │    │ 冻结天数  │    │ 可用余额  │    │ 审核转账  │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
```

采用**二级分销**模型（中国电商法规允许的最深层级）：推广员 A 发展了用户 B，用户 B 又发展了用户 C。C 下单时，A 获得二级佣金，B 获得一级佣金。

## 运费模板

运费通过 `DeliveryExpressTemplateDO` 管理，支持按件/按重量/按体积计费，并可设置包邮条件：

```
DeliveryExpressTemplateDO（运费模板）
├── name: String           ← 模板名称
└── chargeMode: Integer    ← 计费方式

DeliveryExpressTemplateChargeDO（收费规则）
├── templateId: Long       ← 关联模板
├── startCount: Integer    ← 首件/首重/首体积
├── startPrice: Integer    ← 首费
├── extraCount: Integer    ← 续件/续重/续体积
└── extraPrice: Integer    ← 续费

DeliveryExpressTemplateFreeDO（包邮条件）
├── templateId: Long       ← 关联模板
├── areaId: Integer        ← 地区
├── freeCount: Integer     ← 满件包邮
└── freePrice: Integer     ← 满金额包邮
```

## 小结

| 组件 | 职责 | 关键设计 |
|------|------|---------|
| CartDO | 购物车 | selected 字段支持部分结算 |
| TradeOrderDO | 订单主表 | 7 维价格计算公式 |
| TradeOrderItemDO | 订单明细 | 快照模式冗余商品信息 |
| AfterSaleDO | 售后单 | 仅退款/退货退款两种方式 |
| BrokerageUserDO | 推广员 | 二级分销 + 冻结/解冻机制 |
| TradeOrderLogDO | 订单日志 | 完整状态变更审计 |
