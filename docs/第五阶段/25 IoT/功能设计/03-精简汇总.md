# 03 - 精简汇总

## IoT 流程速查

### 设备接入流程

```
设备上电 → 认证(productKey+deviceName+deviceSecret) → 建立连接 → 上报数据
                                                                    │
                                              规则引擎匹配 ◄────────┘
                                                    │
                                              ┌─────┴─────┐
                                              ▼           ▼
                                         数据流转     场景联动
```

### 设备认证三元组

```
productKey (产品标识) + deviceName (设备名称) + deviceSecret (设备密钥)
       │                      │                        │
       └── 同产品共享 ──────── 产品内唯一 ──────────── 设备唯一
```

### 消息流转链路

```
设备上报 → 协议层接收 → 消息解析 → 消息存储 → 规则引擎 → 数据流转
                           │
                           ├── IotDeviceMessageDO (消息记录)
                           ├── IotDevicePropertyDO (属性快照)
                           └── IotDeviceDO.state = ONLINE
```

### 物模型三要素

| 要素 | 含义 | 示例 |
|------|------|------|
| 属性 (Property) | 设备状态 | 温度=25.5, 开关=开 |
| 服务 (Service) | 设备能力 | 设置亮度(80) |
| 事件 (Event) | 设备通知 | 温度过高告警 |

### 规则引擎

```
数据规则：条件匹配 → 数据流转（写DB/推MQ/调HTTP）
场景规则：条件匹配 → 联动动作（控制其他设备）
```

## 核心实体速查

### IotProductDO（产品）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | Long | 产品 ID |
| `name` | String | 产品名称 |
| `productKey` | String | 产品标识（唯一） |
| `productSecret` | String | 产品密钥 |
| `registerEnabled` | Boolean | 是否开启动态注册 |
| `categoryId` | Long | 产品分类编号 |
| `status` | Integer | 产品状态 |
| `deviceType` | Integer | 设备类型（直连/网关/子设备） |
| `netType` | Integer | 联网方式 |
| `protocolType` | String | 协议类型 |
| `serializeType` | String | 序列化类型 |

### IotDeviceDO（设备）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | Long | 设备 ID |
| `deviceName` | String | 设备名称（产品内唯一） |
| `nickname` | String | 设备备注名称 |
| `serialNumber` | String | 设备序列号 |
| `productId` | Long | 产品编号 |
| `productKey` | String | 产品标识（冗余） |
| `deviceType` | Integer | 设备类型 |
| `gatewayId` | Long | 网关设备编号 |
| `state` | Integer | 设备状态（在线/离线/未激活） |
| `onlineTime` | LocalDateTime | 最后上线时间 |
| `offlineTime` | LocalDateTime | 最后离线时间 |
| `activeTime` | LocalDateTime | 激活时间 |
| `deviceSecret` | String | 设备密钥 |
| `latitude` | BigDecimal | 纬度 |

| `groupIds` | Set\<Long\> | 设备分组集合 |

## 核心设计决策

| 决策 | 选择 | 放弃的方案 | 原因 |
|------|------|-----------|------|
| 产品与设备分离 | 产品模板 + 设备实例 | 直接管理设备 | 减少重复配置，统一物模型 |
| 设备认证 | 三元组签名 | Token 认证 | 适合资源受限的嵌入式设备 |
| 属性存储 | 只存最新值 | 存历史记录 | 高频查询最新状态，历史走数据流转 |
| 消息存储 | 原始消息 + 属性快照 | 只存属性 | 保留原始数据用于调试 |
| 规则引擎 | 数据规则 + 场景规则 | 硬编码逻辑 | 灵活配置，无需改代码 |
| 数据流转 | 多目标配置化 | 只写数据库 | 适配不同业务场景 |
| 设备类型 | 直连/网关/子设备 | 统一处理 | 支持工业场景的网关拓扑 |
| 物模型 | JSON 定义 | 代码定义 | 非开发者可配置 |

## 设备类型与协议

| 设备类型 | 说明 | 协议支持 |
|---------|------|---------|
| 直连设备 (DEVICE) | 直接连接平台 | MQTT, TCP, WebSocket |
| 网关设备 (GATEWAY) | 代理子设备 | MQTT, TCP |
| 网关子设备 | 通过网关连接 | Modbus, Zigbee 等 |

| MySQL | `IotDataSinkDatabaseConfig` | 写入数据库表 |

| RabbitMQ | `IotDataSinkRabbitMQConfig` | 推送到 RabbitMQ |
| Redis | `IotDataSinkRedisConfig` | 写入 Redis |
| RocketMQ | `IotDataSinkRocketMQConfig` | 推送到 RocketMQ |
| TCP | `IotDataSinkTcpConfig` | 转发 TCP |
| WebSocket | `IotDataSinkWebSocketConfig` | 推送 WebSocket |
