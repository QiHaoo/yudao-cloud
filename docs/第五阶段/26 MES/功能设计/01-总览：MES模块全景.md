# 01 - 总览：MES 模块全景

## 场景引入

你做了一个电商系统，现在想管理自家工厂的生产过程。工厂里有这些问题：

- 车间主任问"今天要生产多少？哪些订单排产了？"
- 工人问"这批货的工艺路线是什么？先做哪道工序？"
- 质检员问"这批货合格率多少？有哪些缺陷？"
- 仓库管理员问"生产领了多少料？成品入库了多少？"
- 设备管理员问"这台设备上次保养是什么时候？需要检修了吗？"

**MES（Manufacturing Execution System，制造执行系统）** 就是为了解决这些问题而生的。它是 ERP（计划层）和车间设备（执行层）之间的桥梁。

### 为什么电商系统需要 MES？

如果你的电商业务涉及自建工厂或代工厂管理：

- **自有工厂**：需要管理生产工单、工序报工、质量检验
- **代工厂跟踪**：需要了解代工厂的生产进度和质量
- **产品溯源**：需要追溯某个产品的生产批次、质检记录

核心价值是：**让生产过程从"黑盒"变成"白盒"，每个环节都可追踪、可量化**。

## 核心子模块

### 1. 生产管理 (pro)

生产管理是 MES 的核心，覆盖从"下工单"到"完工入库"的全流程：

```
生产流程
  │
  ├── 生产工单 (MesProWorkOrderDO)
  │     └── "生产 1000 个蓝牙耳机，需求日期 6 月 15 日"
  │
  ├── 工艺路线 (MesProRouteDO)
  │     └── 定义产品经过哪些工序
  │         例：SMT 贴片 → 回流焊 → 组装 → 测试 → 包装
  │
  ├── 工序 (MesProProcessDO)
  │     └── 每道工序的具体操作和参数
  │
  ├── 生产任务 (MesProTaskDO)
  │     └── 工单拆分到具体工作站的任务
  │
  ├── 报工记录 (MesProWorkRecordDO)
  │     └── 工人完成某道工序后的记录
  │
  └── 流转卡 (MesProCardDO)
        └── 跟踪产品在各工序间的流转
```

### 2. 质量管理 (qc)

质量管理贯穿生产全过程：

```
质量检验
  │
  ├── IQC (Incoming Quality Control) - 来料检验
  │     └── 原材料入库前的检验
  │
  ├── IPQC (In-Process Quality Control) - 过程检验
  │     └── 生产过程中的抽检
  │
  ├── OQC (Outgoing Quality Control) - 出货检验
  │     └── 成品出库前的检验
  │
  ├── RQC (Return Quality Control) - 退货检验
  │     └── 客户退货后的检验
  │
  ├── 检验模板 (MesQcTemplateDO)
  │     └── 定义检验项目和标准
  │
  └── 缺陷管理 (MesQcDefectDO)
        └── 记录和分类缺陷
```

### 3. 设备管理 (dv)

```
设备管理
  │
  ├── 设备台账 (MesDvMachineryDO)
  │     └── 设备基本信息、状态、位置
  │
  ├── 点检计划 (MesDvCheckPlanDO)
  │     └── 定期检查设备状态
  │
  ├── 保养记录 (MesDvMaintenRecordDO)
  │     └── 设备保养历史
  │
  └── 维修记录 (MesDvRepairDO)
        └── 设备故障维修历史
```

### 4. 仓储管理 (wm)

MES 的仓储管理是**车间级**的，与 WMS（仓库级）互补：

```
车间仓储
  │
  ├── 物料库存 (MesWmMaterialStockDO)
  │     └── 车间线边仓的物料库存
  │
  ├── 生产领料 (MesWmProductIssueDO)
  │     └── 从仓库领取原材料到车间
  │
  ├── 生产入库 (MesWmProductReceiptDO)
  │     └── 成品从车间入库到仓库
  │
  ├── 条码管理 (MesWmBarcodeDO)
  │     └── 物料和产品的条码追溯
  │
  └── 批次管理 (MesWmBatchDO)
        └── 按批次追溯生产过程
```

## BOM（物料清单）

### BOM 的含义

BOM（Bill of Materials）定义"生产一个产品需要哪些原材料"。以蓝牙耳机为例：

```
蓝牙耳机 BOM
  │
  ├── 耳机壳体 × 2       (物料编码: M001)
  ├── 扬声器单元 × 2     (物料编码: M002)
  ├── PCB 电路板 × 1     (物料编码: M003)
  ├── 蓝牙模块 × 1       (物料编码: M004)
  ├── 电池 × 1           (物料编码: M005)
  ├── 充电接口 × 1       (物料编码: M006)
  ├── 包装盒 × 1         (物料编码: M007)
  └── 说明书 × 1         (物料编码: M008)
```

### BOM 与工单的关系

```
生产工单：生产 1000 个蓝牙耳机
  │
  ├── BOM 展开
  │     ├── 耳机壳体：1000 × 2 = 2000 个
  │     ├── 扬声器单元：1000 × 2 = 2000 个
  │     ├── PCB 电路板：1000 × 1 = 1000 个
  │     └── ...
  │
  └── 生成领料单
        └── 从仓库领取 2000 个壳体、2000 个扬声器、...
```

## 生产工单

### 工单实体

```java
// MesProWorkOrderDO - 生产工单
public class MesProWorkOrderDO extends BaseDO {
    private Long id;                // 工单编号
    private String code;            // 工单编码
    private String name;            // 工单名称
    private Integer type;           // 工单类型
    private Integer orderSourceType; // 来源类型（ERP/手动）
    private String orderSourceCode; // 来源单据编号

    private Long productId;         // 产品编号
    private BigDecimal quantity;    // 生产数量
    private BigDecimal quantityProduced;   // 已生产数量
    private BigDecimal quantityScheduled;  // 已排产数量

    private Long clientId;          // 客户编号
    private Long vendorId;          // 供应商编号
    private String batchCode;       // 批次号
    private LocalDateTime requestDate; // 需求日期
    private Long parentId;          // 父工单编号
    private LocalDateTime finishDate;  // 完成时间
    private Integer status;         // 工单状态
    private String remark;          // 备注
}
```

### 工单状态流转

```
工单状态
  │
  ├── 未排产 (0)
  │     → 工单刚创建，还没分配任务
  │
  ├── 排产中 (10)
  │     → 已经部分排产
  │
  ├── 生产中 (20)
  │     → 已开始生产，有报工记录
  │
  ├── 已完成 (30)
  │     → 生产完成
  │
  └── 已取消 (40)
        → 工单取消
```
