# 02 - 核心设计：生产执行流程

## 场景引入

上一章我们看到了 MES 的全景——生产、质量、设备、仓储。这一章深入几个核心设计问题：

1. **BOM 展开**：一个产品需要多少原材料？怎么自动计算？
2. **工单与任务**：工单怎么拆分成可执行的任务？任务怎么分配到工作站？
3. **报工机制**：工人怎么记录生产进度？报工数据怎么汇总？
4. **质量检验**：检验模板怎么定义？检验结果怎么判定？

## BOM 与物料需求

### BOM 结构

BOM 是"生产一个产品需要哪些物料"的定义。以蓝牙耳机为例：

```
产品：蓝牙耳机 (ITEM-001)
  │
  ├── 物料：耳机壳体 (M001) × 2
  ├── 物料：扬声器单元 (M002) × 2
  ├── 物料：PCB 电路板 (M003) × 1
  ├── 物料：蓝牙模块 (M004) × 1
  ├── 物料：电池 (M005) × 1
  ├── 物料：充电接口 (M006) × 1
  ├── 物料：包装盒 (M007) × 1
  └── 物料：说明书 (M008) × 1
```

### BOM 展开计算

当工单要求生产 1000 个蓝牙耳机时，需要展开 BOM 计算物料需求：

```
工单：生产 1000 个蓝牙耳机
  │
  ▼
BOM 展开
  │
  ├── 耳机壳体 (M001)：1000 × 2 = 2000 个
  ├── 扬声器单元 (M002)：1000 × 2 = 2000 个
  ├── PCB 电路板 (M003)：1000 × 1 = 1000 个
  ├── 蓝牙模块 (M004)：1000 × 1 = 1000 个
  ├── 电池 (M005)：1000 × 1 = 1000 个
  ├── 充电接口 (M006)：1000 × 1 = 1000 个
  ├── 包装盒 (M007)：1000 × 1 = 1000 个
  └── 说明书 (M008)：1000 × 1 = 1000 个
```

### 多层 BOM

有些产品的 BOM 是多层的——子组件也有自己的 BOM：

```
蓝牙耳机 BOM（多层）
  │
  ├── 耳机组件 (SUB-001)
  │     ├── 耳机壳体 (M001) × 1
  │     ├── 扬声器单元 (M002) × 1
  │     └── 防尘网 (M009) × 1
  │
  ├── 主板组件 (SUB-002)
  │     ├── PCB 电路板 (M003) × 1
  │     ├── 蓝牙模块 (M004) × 1
  │     └── 电容 (M010) × 4
  │
  ├── 电池 (M005) × 1
  ├── 充电接口 (M006) × 1
  ├── 包装盒 (M007) × 1
  └── 说明书 (M008) × 1
```

多层 BOM 展开时需要递归计算，最终得到所有叶子物料的需求量。

## 工单与任务

### 工单的含义

工单是"生产什么、生产多少、什么时候要"的计划。它是 MES 的起点。

```
工单信息
  │
  ├── 生产什么：蓝牙耳机 (productId)
  ├── 生产多少：1000 个 (quantity)
  ├── 什么时候要：6 月 15 日 (requestDate)
  ├── 来源：ERP 销售订单 SO-2024-001 (orderSourceCode)
  └── 批次号：BATCH-2024-0601 (batchCode)
```

### 工单拆分为任务

一个工单可能需要拆分成多个任务，分配到不同的工作站并行生产：

```
工单：生产 1000 个蓝牙耳机
  │
  ├── 任务 1：SMT 贴片（工作站 A）
  │     └── 数量：1000 个
  │
  ├── 任务 2：回流焊（工作站 B）
  │     └── 数量：1000 个
  │
  ├── 任务 3：组装（工作站 C、D 并行）
  │     ├── 任务 3a：500 个（工作站 C）
  │     └── 任务 3b：500 个（工作站 D）
  │
  ├── 任务 4：测试（工作站 E）
  │     └── 数量：1000 个
  │
  └── 任务 5：包装（工作站 F）
        └── 数量：1000 个
```

### 任务实体

```java
// MesProTaskDO - 生产任务
public class MesProTaskDO extends BaseDO {
    private Long id;            // 任务编号
    private Long workOrderId;   // 工单编号
    private Long processId;     // 工序编号
    private Long workstationId; // 工作站编号
    private BigDecimal quantity; // 任务数量
    private BigDecimal quantityProduced; // 已生产数量
    private Integer status;     // 任务状态
    private LocalDateTime startTime; // 开始时间
    private LocalDateTime endTime;   // 结束时间
}
```

### 任务状态流转

```
任务状态
  │
  ├── 未开始 (0)
  │     → 任务已创建，等待执行
  │
  ├── 进行中 (10)
  │     → 工人已开始作业
  │
  ├── 已完成 (20)
  │     → 数量达标
  │
  └── 已暂停 (30)
        → 临时暂停（设备故障、缺料等）
```

## 工艺路线与工序

### 工艺路线

工艺路线定义"产品经过哪些工序、按什么顺序"：

```
工艺路线：蓝牙耳机标准工艺
  │
  ├── 工序 1：SMT 贴片 (Process-001)
  │     ├── 标准工时：2 分钟/个
  │     └── 工作站：SMT 贴片机 A
  │
  ├── 工序 2：回流焊 (Process-002)
  │     ├── 标准工时：5 分钟/批（100 个/批）
  │     └── 工作站：回流焊炉 B
  │
  ├── 工序 3：组装 (Process-003)
  │     ├── 标准工时：10 分钟/个
  │     └── 工作站：组装台 C/D
  │
  ├── 工序 4：功能测试 (Process-004)
  │     ├── 标准工时：3 分钟/个
  │     └── 工作站：测试台 E
  │
  └── 工序 5：包装 (Process-005)
        ├── 标准工时：1 分钟/个
        └── 工作站：包装台 F
```

### 工序实体

```java
// MesProProcessDO - 工序
public class MesProProcessDO extends BaseDO {
    private Long id;            // 工序编号
    private String code;        // 工序编码
    private String name;        // 工序名称
    private BigDecimal standardTime; // 标准工时
    private Integer type;       // 工序类型
    private String description; // 工序描述
}
```

### 工序内容

每道工序可以有详细的操作指导：

```java
// MesProProcessContentDO - 工序内容
public class MesProProcessContentDO extends BaseDO {
    private Long processId;     // 工序编号
    private Integer type;       // 内容类型（文字/图片/视频）
    private String content;     // 操作内容
    private Integer sort;       // 排序
}
```

## 报工机制

### 报工的含义

工人完成某道工序后，记录完成数量和质量信息：

```
报工流程
  │
  ├── 1. 工人领取任务
  │     └── 选择工作站、工序、工单
  │
  ├── 2. 开始作业
  │     └── 记录开始时间
  │
  ├── 3. 完成作业
  │     ├── 记录完成数量
  │     ├── 记录合格数量
  │     ├── 记录不良数量和缺陷类型
  │     └── 记录结束时间
  │
  └── 4. 提交报工
        ├── 更新任务已生产数量
        ├── 更新工单已生产数量
        └── 触发质量检验（如果需要）
```

### 报工记录实体

```java
// MesProWorkRecordDO - 报工记录
public class MesProWorkRecordDO extends BaseDO {
    private Long id;            // 报工编号
    private Long workOrderId;   // 工单编号
    private Long taskId;        // 任务编号
    private Long processId;     // 工序编号
    private Long workstationId; // 工作站编号
    private Long workerId;      // 工人编号

    private BigDecimal quantity;        // 完成数量
    private BigDecimal qualifiedQuantity; // 合格数量
    private BigDecimal defectiveQuantity; // 不良数量

    private LocalDateTime startTime;    // 开始时间
    private LocalDateTime endTime;      // 结束时间
    private BigDecimal actualTime;      // 实际工时（分钟）

    private String remark;              // 备注
}
```

### 报工数据汇总

```
报工数据汇总
  │
  ├── 任务维度
  │     └── 任务已生产数量 = SUM(该任务的所有报工记录.完成数量)
  │
  ├── 工单维度
  │     └── 工单已生产数量 = SUM(该工单的所有报工记录.完成数量)
  │
  ├── 工人维度
  │     └── 工人产量统计 = 按工人分组汇总
  │
  └── 工序维度
        └── 工序效率统计 = 按工序分组汇总（标准工时 vs 实际工时）
```

## 质量检验

### 检验类型

```
质量检验类型
  │
  ├── IQC (来料检验)
  │     → 原材料入库前检验
  │     → 例：检查蓝牙模块的外观、功能
  │
  ├── IPQC (过程检验)
  │     → 生产过程中抽检
  │     → 例：组装完成后抽检 10%
  │
  ├── OQC (出货检验)
  │     → 成品出库前检验
  │     → 例：全检蓝牙耳机的音质、续航
  │
  └── RQC (退货检验)
        → 客户退货后检验
        → 例：确认退货原因、判定责任
```

### 检验模板

检验模板定义"检验哪些项目、标准是什么"：

```
检验模板：蓝牙耳机 OQC 检验
  │
  ├── 检验指标 1：外观检查
  │     ├── 类型：目视检查
  │     ├── 标准：无划痕、无变色
  │     └── 判定：合格/不合格
  │
  ├── 检验指标 2：音质测试
  │     ├── 类型：仪器测试
  │     ├── 标准：频率响应 20Hz-20kHz
  │     └── 判定：合格/不合格
  │
  ├── 检验指标 3：续航测试
  │     ├── 类型：计时测试
  │     ├── 标准：≥ 4 小时
  │     └── 判定：合格/不合格
  │
  └── 检验指标 4：充电测试
        ├── 类型：功能测试
        ├── 标准：充电正常、指示灯正常
        └── 判定：合格/不合格
```

### 检验记录

```java
// 检验记录包含：检验单 + 检验明细
// MesQcOqcDO - 出货检验单
public class MesQcOqcDO extends BaseDO {
    private Long id;            // 检验单编号
    private Long workOrderId;   // 关联工单
    private Long productId;     // 产品编号
    private BigDecimal quantity; // 检验数量
    private BigDecimal qualifiedQuantity; // 合格数量
    private BigDecimal defectiveQuantity; // 不良数量
    private Integer result;     // 检验结果（合格/不合格/让步接收）
    private Long inspectorId;   // 检验员
}
```

### 检验判定逻辑

```
检验判定
  │
  ├── 全部指标合格 → 整批合格
  │
  ├── 部分指标不合格
  │     ├── 不良率 < 阈值 → 让步接收（降级使用）
  │     └── 不良率 >= 阈值 → 整批不合格
  │
  └── 关键指标不合格 → 整批不合格（不论不良率）
```

## 设备管理

### 设备与工作站

```
工作站 (MesMdWorkstationDO)
  │
  ├── 工作站名称：组装台 C
  ├── 所属车间：组装车间
  │
  ├── 关联设备 (MesDvMachineryDO)
  │     ├── 电动螺丝刀 × 2
  │     └── 扭矩扳手 × 1
  │
  └── 关联工具 (MesTmToolDO)
        ├── 治具 A
        └── 治具 B
```

### 设备维护三件套

```
设备维护
  │
  ├── 点检 (Check)
  │     → 定期检查设备状态（日常巡检）
  │     → 例：每天检查螺丝是否松动
  │
  ├── 保养 (Maintenance)
  │     → 定期维护保养（预防性维护）
  │     → 例：每 500 小时更换润滑油
  │
  └── 维修 (Repair)
        → 故障后修复（纠正性维护）
        → 例：电机烧了，更换电机
```

## 排班管理

### 排班体系

```
排班管理
  │
  ├── 日历 (MesCalCalendarDO)
  │     └── 定义工作日和休息日
  │
  ├── 假期 (MesCalHolidayDO)
  │     └── 法定假日、公司假期
  │
  ├── 班组 (MesCalTeamDO)
  │     └── 工人分组（白班组、夜班组）
  │
  ├── 班次 (MesCalPlanShiftDO)
  │     └── 定义班次时间（白班 08:00-16:00，夜班 16:00-00:00）
  │
  └── 排班计划 (MesCalPlanDO)
        └── 为每个班组分配班次
```

## 思考题

1. BOM 展开时，如果子组件也有 BOM（多层 BOM），递归展开的终止条件是什么？

3. 报工时只记录"完成数量"和"合格数量"，不良品的详细缺陷信息在哪里记录？
