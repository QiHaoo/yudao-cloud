## BPMN 核心概念速查

### 事件（Event）

```
图形    名称          含义                    配置
──────────────────────────────────────────────────────────
○      开始事件      流程起点                 无特殊配置
◎      结束事件      流程终点                 可配置终止行为
○(中间) 中间捕获事件  等待某个条件             定时器/信号/消息
◎(中间) 中间抛出事件  触发某个事件             消息/错误/信号
```

### 任务（Task）

```
图形    名称          含义                    yudao-cloud 中的用途
──────────────────────────────────────────────────────────
□      用户任务      需要人工处理             审批节点
□(齿轮) 服务任务      自动执行代码             自动审批、调用外部系统
□(脚本) 脚本任务      执行脚本                 数据转换
□(消息) 发送任务      发送消息                 通知
□(接收) 接收任务      等待消息                 异步回调
```

### 网关（Gateway）

```
图形    名称          含义                    行为
──────────────────────────────────────────────────────────
◇      排他网关      条件分支                 只走第一条满足条件的路径
◇(+)   并行网关      并行分支                 同时走所有路径，等待汇聚
◇(○)   包含网关      条件并行                 走所有满足条件的路径
◇(事件) 事件网关      基于事件分支             等待第一个到达的事件
```

### 流程实例状态

```
状态码    状态名      说明              是否终态
──────────────────────────────────────────────────
  -1     未开始      流程未发起          否
   1     审批中      流程运行中          否
   2     审批通过    所有节点通过         是
   3     审批不通过  某节点拒绝          是
   4     已取消      发起人/管理员取消    是
```

### 任务状态

```
状态码    状态名      说明
──────────────────────────────────────────────────
   0     待审批      等待审批人处理
   1     已通过      审批人通过
   2     已拒绝      审批人拒绝
   3     已取消      流程取消
   4     已转办      转给其他人
```

## 核心设计决策

### 1. 引擎选型

| 决策 | 选择 | 原因 |
|------|------|------|
| 流程引擎 | Flowable | 性能好、Spring Boot 集成好、社区活跃 |
| 流程标准 | BPMN 2.0 | 业界标准，可移植，工具链丰富 |
| 表单方案 | form-generator JSON | 前端动态渲染，无需写代码 |

### 2. 扩展设计

| 决策 | 选择 | 原因 |
|------|------|------|
| 扩展信息 | 独立扩展表（bpm_process_definition_info） | 不侵入 Flowable 源码，升级无影响 |
| 表单存储 | 冗余到扩展表 | 避免每次查询都 JOIN 表单表 |
| 用户组 | 独立管理（BpmUserGroupDO） | 复用性好，流程变更不影响组定义 |

### 3. 业务集成

| 决策 | 选择 | 原因 |
|------|------|------|
| 关联方式 | processInstanceId 双向关联 | 从任一方都能查到另一方 |
| 表单模式 | 支持内置表单 + 自定义页面 | 简单场景用内置，复杂场景用自定义 |
| 状态同步 | 流程结束时回调更新业务表 | 保持业务表状态与流程状态一致 |

### 4. 权限控制

| 决策 | 选择 | 原因 |
|------|------|------|
| 发起权限 | 按用户 + 部门控制 | 精细化控制谁能发起什么流程 |
| 管理权限 | 独立的 managerUserIds | 区分发起人和管理人 |
| 可见性 | visible 字段控制 | 不可见的流程不展示在发起列表 |

## 与业务模块集成的检查清单

当需要为新的业务模块接入审批流程时，按以下步骤操作：

```
1. 创建流程分类（BpmCategoryDO）
2. 设计表单（BpmFormDO 或自定义页面）
3. 设计 BPMN 流程模型
4. 配置流程定义扩展信息（BpmProcessDefinitionInfoDO）
   ├── 表单类型和路径
   ├── 可发起人/部门
   └── 流程行为（是否允许取消等）
5. 业务表增加 processInstanceId 字段
6. 业务提交时调用流程发起 API
7. 流程结束时回调更新业务表状态
8. 配置通知（审批结果通知发起人）
```
