# 01 - 起步：流程模型与定义

## 本章目标

从零搭建工作流模块的流程建模层。完成后，你将拥有：

1. 流程模型（Model）的创建与管理
2. 流程定义（ProcessDefinition）的部署
3. 流程分类与表单管理
4. BPMN 可视化设计器集成
5. 仿钉钉简易设计器支持

---

## 一、核心概念关系

在开始实现之前，先理清几个核心概念：

```
流程分类 (BpmCategoryDO)
  │
  ├── 流程模型 (Flowable Model)
  │     │
  │     ├── 部署 → 流程定义 (ProcessDefinition)
  │     │           │
  │     │           ├── 版本 1
  │     │           ├── 版本 2
  │     │           └── 版本 3
  │     │
  │     └── 发起 → 流程实例 (ProcessInstance)
  │
  └── 表单定义 (BpmFormDO)
        │
        └── 关联到流程定义
```

- **流程模型（Model）**：设计阶段的概念，可编辑、可版本管理
- **流程定义（ProcessDefinition）**：部署后的产物，只读、有版本号
- **流程实例（ProcessInstance）**：正在运行的流程

## 三、表单管理

### 3.1 BpmFormDO

表单定义存储了审批表单的结构，支持动态表单和业务表单两种模式。

```java
// 来源：yudao-module-bpm-server/.../dal/dataobject/definition/BpmFormDO.java
@TableName("bpm_form")
public class BpmFormDO extends BaseDO {

    @TableId
    private Long id;
    private String name;     // 表单名称
    private Integer status;  // 状态
    private String conf;     // 表单配置（JSON）
    private Integer fields;  // 表单项数量
    private String remark;   // 备注
}
```

设计要点：
- `conf` 存储表单的 JSON 配置，基于 form-generator 的格式。
- 表单分为两种类型：
  - **内置动态表单**：表单配置存储在 `BpmFormDO` 中，前端根据配置动态渲染
  - **业务表单**：表单由业务模块自己实现，流程只负责审批流转

### 4.2 创建模型

```java
// 来源：yudao-module-bpm-server/.../service/definition/BpmModelServiceImpl.java
@Override
@Transactional(rollbackFor = Exception.class)
public String createModel(BpmModelSaveReqVO modelVO) {
    // 1. 创建 Flowable Model
    Model model = repositoryService.newModel();
    model.setTenantId(FlowableUtils.getTenantId());
    BpmModelConvert.INSTANCE.copyToModel(model, modelVO);
    repositoryService.saveModel(model);

    // 2. 初始化 BPMN XML
    String bpmnXml = BpmModelConvert.INSTANCE.buildBpmnXml(modelVO);
    repositoryService.addModelEditorSource(model.getId(), bpmnXml.getBytes());

    return model.getId();
}
```

设计要点：
- 使用 Flowable 的 `RepositoryService` 创建和管理模型。
- 模型创建时自动初始化一个基础的 BPMN XML 结构。

### 4.3 部署模型

```java
@Override
@Transactional(rollbackFor = Exception.class)
public void deployModel(Long userId, String id) {
    // 1. 获取模型
    Model model = getModel(id);

    // 2. 获取模型的 BPMN XML
    byte[] bpmnXml = getModelBpmnXML(id);

    // 3. 部署流程定义
    Deployment deployment = repositoryService.createDeployment()
            .addString(model.getKey() + ".bpmn20.xml", new String(bpmnXml))
            .name(model.getName())
            .tenantId(FlowableUtils.getTenantId())
            .deploy();

    // 4. 获取部署后的流程定义
    ProcessDefinition processDefinition = repositoryService.createProcessDefinitionQuery()
            .deploymentId(deployment.getId())
            .singleResult();

    // 5. 保存流程定义扩展信息
    processDefinitionService.createProcessDefinitionInfo(
            new BpmProcessDefinitionCreateReqDTO()
                    .setProcessDefinitionId(processDefinition.getId())
                    .setModelId(id)
                    .setModelType(BpmModelTypeEnum.BPMN.getType())
                    // ... 其他扩展信息
    );
}
```

部署流程：
1. 从模型中获取 BPMN XML
2. 调用 Flowable 的 `RepositoryService` 部署
3. 保存扩展信息到 `BpmProcessDefinitionInfoDO`

## 五、流程定义管理

### 5.1 BpmProcessDefinitionInfoDO

流程定义扩展信息，存储 Flowable 不支持的自定义字段。

```java
// 来源：yudao-module-bpm-server/.../dal/dataobject/definition/BpmProcessDefinitionInfoDO.java
@TableName("bpm_process_definition_info")
public class BpmProcessDefinitionInfoDO extends BaseDO {

    @TableId
    private Long id;
    private String processDefinitionId; // 关联 Flowable 的 ProcessDefinition
    private String modelId;             // 关联 Flowable 的 Model
    private Integer modelType;          // 模型类型（BPMN / 简易模式）
    private Integer formType;           // 表单类型（内置动态表单 / 业务表单）
    private Long formId;                // 关联的表单编号
    private String processDefinitionKey; // 流程定义 Key
    private String name;                // 流程名称
    private String description;         // 流程描述
    private String icon;                // 流程图标
    private Integer sort;               // 排序
    private Integer suspensionState;    // 挂起状态
    private List<Long> startUserIds;    // 可发起人（为空则所有人可发起）
    private List<Long> startDeptIds;    // 可发起部门
}
```

设计要点：
- `processDefinitionId` 与 Flowable 的 `ProcessDefinition` 通过 ID 关联。
- `formType` 区分表单类型：
  - `BpmModelFormTypeEnum.BPM_FORM`：使用内置动态表单
  - `BpmModelFormTypeEnum.CUSTOM`：使用业务自定义表单
- `startUserIds` 和 `startDeptIds` 控制流程的发起权限。

## 六、BPMN 可视化设计器

### 6.1 设计器集成

yudao-cloud 集成了 bpmn.js 作为 BPMN 可视化设计器，前端通过以下接口与后端交互：

```
获取模型 XML    GET  /admin-api/bpm/model/get?id=xxx
保存模型 XML    PUT  /admin-api/bpm/model/update-bpmn-xml?id=xxx&bpmnXml=xxx
部署模型        POST /admin-api/bpm/model/deploy?id=xxx
```

### 6.2 BPMN XML 示例

一个简单的请假审批流程的 BPMN XML：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<definitions xmlns="http://www.omg.org/spec/BPMN/20100524/MODEL"
             targetNamespace="http://www.flowable.org/processdef">

  <process id="leave" name="请假审批" isExecutable="true">
    <startEvent id="start" name="开始"/>

    <userTask id="approve" name="主管审批"
              flowable:assignee="${approver}"/>

    <endEvent id="end" name="结束"/>

    <sequenceFlow sourceRef="start" targetRef="approve"/>
    <sequenceFlow sourceRef="approve" targetRef="end"/>
  </process>

</definitions>
```

## 七、仿钉钉简易设计器

### 7.1 设计理念

仿钉钉简易设计器用简化的树形结构定义流程，降低使用门槛。用户不需要了解 BPMN 标准，只需要通过拖拽节点的方式设计流程。

### 7.2 简易模型结构

```java

@Data

    private Integer type;        // 节点类型
    private String name;         // 节点名称
    private BpmSimpleModelNodeVO childNode;       // 子节点（下一个节点）
    private List<BpmSimpleModelNodeVO> conditionNodes; // 条件分支节点
    // ... 其他配置
}
```

### 7.3 节点类型


### 7.4 简易模型转 BPMN

简易模型最终会被转换为标准的 BPMN XML，再交给 Flowable 引擎执行。

```java
// 来源：yudao-module-bpm-server/.../framework/flowable/core/util/SimpleModelUtils.java
public class SimpleModelUtils {

    // 将简易模型转换为 BPMN XML
    public static String convertToBpmn(BpmSimpleModelNodeVO simpleModel, String processKey, String processName) {
        // 1. 创建 BpmnModel
        BpmnModel bpmnModel = new BpmnModel();
        Process process = new Process();
        process.setId(processKey);
        process.setName(processName);
        bpmnModel.addProcess(process);

        // 2. 递归遍历简易模型，添加节点
        convertNode(process, simpleModel, null);

        // 3. 生成 BPMN XML
        return new BpmnXMLConverter().convertToXML(bpmnModel);
    }
}
```
