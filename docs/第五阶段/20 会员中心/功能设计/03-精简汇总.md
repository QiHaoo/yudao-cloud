## 核心设计决策

| 决策 | 选择 | 放弃的方案 | 原因 |
|------|------|-----------|------|
| 用户标识 | mobile（手机号） | username（用户名） | C 端用户自然标识，支持短信登录 |
| 登录体系 | 复用 system 的 OAuth2 | 自建令牌体系 | 减少重复代码，统一管理 |
| 令牌区分 | UserTypeEnum 字段 | 独立令牌表 | 共用一套令牌逻辑 |
| 升级依据 | 经验值 | 积分值 | 积分可消费，经验只增不减 |
| 标签存储 | tagIds 逗号分隔 | 关联表 | 简化查询，适合"按用户查标签"场景 |
| 分组关系 | 一对多（groupId） | 多对多 | 分组是管理概念，一个用户一个组 |
| 地址存储 | areaId + detailAddress | 全文存储 | 标准化、便于统计、节省空间 |
| 密码字段 | 可选 | 必填 | 支持无密码登录方式（验证码/小程序） |
| 签到规则 | 配置表 | 硬编码 | 运营可灵活调整 |
| 租户隔离 | TenantBaseDO | BaseDO | SaaS 模式下数据隔离 |

### 手机号 + 验证码登录

```
前端                     后端                     Redis          短信服务
 │                       │                        │               │
 │  sendSmsCode(mobile)  │                        │               │
 │──────────────────────>│                        │               │
 │                       │  SET sms:code:{mobile} │               │
 │                       │───────────────────────>│               │
 │                       │  sendSms(mobile, code) │               │
 │                       │───────────────────────────────────────>│
 │  success              │                        │               │
 │<──────────────────────│                        │               │
 │                       │                        │               │
 │  smsLogin(mobile,code)│                        │               │
 │──────────────────────>│                        │               │
 │                       │  GET sms:code:{mobile} │               │
 │                       │───────────────────────>│               │
 │                       │  code                  │               │
```

## 积分变动流程

```
业务系统(电商)             会员模块
 │                       │
 │  addPoint(userId,     │
 │    point, bizType)    │
 │──────────────────────>│
 │                       │
 │                       │  1. 校验用户存在
 │                       │  2. 更新 MemberUserDO.point
 │                       │  3. 创建 MemberPointRecordDO
 │                       │  4. 检查是否触发等级升级
 │                       │     ├── 计算新经验
 │                       │     └── 对比等级阈值
 │                       │         ├── 达到 → 升级
 │                       │         └── 未达到 → 不变
 │  success              │
 │<──────────────────────│
```
