### 1.2 积分变动实现

```java
// 来源：yudao-module-member-server/.../service/point/MemberPointRecordServiceImpl.java
@Override
@Transactional(rollbackFor = Exception.class)
public void createPointRecord(Long userId, Integer point, MemberPointBizTypeEnum bizType, String bizId) {
    if (point == 0) {
        return;
    }

    // 1. 校验用户积分余额
    MemberUserDO user = memberUserService.getUser(userId);
    Integer userPoint = ObjectUtil.defaultIfNull(user.getPoint(), 0);
    int totalPoint = userPoint + point; // 用户变动后的积分
    if (totalPoint < 0) {
        log.error("[createPointRecord][userId({}) point({}) bizType({}) bizId({}) {}]",
                userId, point, bizType, bizId, USER_POINT_NOT_ENOUGH);
        return;
    }

    // 2. 更新用户积分
    boolean success = memberUserService.updateUserPoint(userId, point);
    if (!success) {
        throw exception(USER_POINT_NOT_ENOUGH);
    }

    // 3. 增加积分记录
    MemberPointRecordDO record = new MemberPointRecordDO()
            .setUserId(userId).setBizId(bizId).setBizType(bizType.getType())
            .setTitle(bizType.getName())
            .setDescription(StrUtil.format(bizType.getDescription(), point))
            .setPoint(point).setTotalPoint(totalPoint);
    memberPointRecordMapper.insert(record);
}
```

设计要点：
- **正负数语义**：`point` 为正数表示增加，负数表示扣减。
- **余额校验**：扣减时校验余额是否充足，防止积分变为负数。
- **事务保证**：更新用户积分和插入积分记录在同一事务中，保证一致性。
- **业务类型**：`MemberPointBizTypeEnum` 枚举定义了积分变动的业务类型（签到、消费、退款等）。

### 2.2 经验增加与自动升级

```java
// 来源：yudao-module-member-server/.../service/level/MemberLevelServiceImpl.java
@Override
@Transactional(rollbackFor = Exception.class)
public void addExperience(Long userId, Integer experience, MemberExperienceBizTypeEnum bizType, String bizId) {
    if (experience == 0) {
        return;
    }

    // 1. 更新用户经验
    MemberUserDO user = memberUserService.getUser(userId);
    int newExperience = ObjectUtil.defaultIfNull(user.getExperience(), 0) + experience;
    memberUserService.updateUserExperience(userId, experience);

    // 2. 插入经验记录
    MemberExperienceRecordDO record = new MemberExperienceRecordDO()
            .setUserId(userId).setExperience(experience)
            .setTotalExperience(newExperience)
            .setBizType(bizType.getType()).setBizId(bizId);
    experienceRecordMapper.insert(record);

    // 3. 自动升级检查
    autoUpgradeLevel(userId, newExperience);
}

private void autoUpgradeLevel(Long userId, int newExperience) {
    // 1. 获取所有启用的等级（按经验升序）
    List<MemberLevelDO> levels = getEnableLevelList();
    if (CollUtil.isEmpty(levels)) {
        return;
    }

    // 2. 获取用户当前等级
    MemberUserDO user = memberUserService.getUser(userId);
    MemberLevelDO currentLevel = memberLevelMapper.selectById(user.getLevelId());

    // 3. 找到符合经验的最高等级
    MemberLevelDO targetLevel = null;
    for (MemberLevelDO level : levels) {
        if (newExperience >= level.getExperience()) {
            targetLevel = level;
        }
    }

    // 4. 如果需要升级
    if (targetLevel != null && (currentLevel == null || targetLevel.getLevel() > currentLevel.getLevel())) {
        memberUserService.updateUserLevel(userId, targetLevel.getId());

        // 5. 插入等级变动记录
        MemberLevelRecordDO record = new MemberLevelRecordDO()
                .setUserId(userId)
                .setLevelId(targetLevel.getId())
                .setOldLevelId(currentLevel != null ? currentLevel.getId() : null);
        levelRecordMapper.insert(record);
    }
}
```

设计要点：
- **自动升级**：经验增加后自动检查是否满足升级条件，无需用户手动操作。
- **只升不降**：等级体系通常只支持升级，不支持降级（除非运营手动调整）。
- **等级记录**：每次升级都记录变动历史，方便追溯。

```java
// 来源：yudao-module-member-server/.../service/signin/MemberSignInRecordServiceImpl.java
@Override
@Transactional(rollbackFor = Exception.class)
public MemberSignInRecordDO createSignRecord(Long userId) {
    // 1. 获取当前用户最近的签到记录
    MemberSignInRecordDO lastRecord = signInRecordMapper.selectLastRecordByUserId(userId);

    // 1.1 判断是否重复签到
    validateSigned(lastRecord);

    // 2. 获取所有的签到规则
    List<MemberSignInConfigDO> signInConfigs = signInConfigService.getSignInConfigList(
            CommonStatusEnum.ENABLE.getStatus());

    // 3. 组合数据（计算连续天数、积分、经验）
    MemberSignInRecordDO record = MemberSignInRecordConvert.INSTANCE.convert(
            userId, lastRecord, signInConfigs);

    // 4. 插入签到记录
    signInRecordMapper.insert(record);

    // 5. 增加积分
    if (!ObjectUtils.equalsAny(record.getPoint(), null, 0)) {
        pointRecordService.createPointRecord(userId, record.getPoint(),
                MemberPointBizTypeEnum.SIGN, String.valueOf(record.getId()));
    }

    // 6. 增加经验
    if (!ObjectUtils.equalsAny(record.getExperience(), null, 0)) {
        memberLevelService.addExperience(userId, record.getExperience(),
                MemberExperienceBizTypeEnum.SIGN_IN, String.valueOf(record.getId()));
    }

    return record;
}

private void validateSigned(MemberSignInRecordDO signInRecordDO) {
    if (signInRecordDO == null) {
        return;
    }
    if (DateUtils.isToday(signInRecordDO.getCreateTime())) {
        throw exception(SIGN_IN_RECORD_TODAY_EXISTS); // 今日已签到
    }
}
```

### 3.3 连续天数计算

```java
// MemberSignInRecordConvert 中的逻辑
public MemberSignInRecordDO convert(Long userId, MemberSignInRecordDO lastRecord,
                                    List<MemberSignInConfigDO> configs) {
    MemberSignInRecordDO record = new MemberSignInRecordDO()
            .setUserId(userId);

    // 计算连续天数
    int day;
    if (lastRecord == null || !DateUtils.isYesterday(lastRecord.getCreateTime())) {
        day = 1; // 首次签到或中断，从第 1 天开始
    } else {
        day = lastRecord.getDay() + 1; // 连续签到
    }
    record.setDay(day);

    // 根据签到规则获取奖励
    MemberSignInConfigDO config = getConfigByDay(configs, day);
    if (config != null) {
        record.setPoint(config.getPoint());
        record.setExperience(config.getExperience());
    }

    return record;
}

private MemberSignInConfigDO getConfigByDay(List<MemberSignInConfigDO> configs, int day) {
    // 找到匹配的签到规则（按天数匹配）
    for (MemberSignInConfigDO config : configs) {
        if (config.getDay() == day) {
            return config;
        }
    }
    // 如果没有匹配的规则，使用默认规则（day = 0 的规则）
    return configs.stream()
            .filter(c -> c.getDay() == 0)
            .findFirst().orElse(null);
}
```

设计要点：
- **连续签到判断**：如果最后一条签到记录是昨天，则连续天数 +1；否则重置为 1。
- **签到规则**：`MemberSignInConfigDO` 定义了不同天数的奖励，如第 1 天 10 积分、第 7 天 30 积分。
- **默认规则**：`day = 0` 的规则作为默认规则，当没有匹配的天数规则时使用。

### 3.4 签到汇总查询

```java
@Override
public AppMemberSignInRecordSummaryRespVO getSignInRecordSummary(Long userId) {
    AppMemberSignInRecordSummaryRespVO summary = new AppMemberSignInRecordSummaryRespVO();
    summary.setTotalDay(0);
    summary.setContinuousDay(0);
    summary.setTodaySignIn(false);

    // 获取签到总天数
    Long signCount = signInRecordMapper.selectCountByUserId(userId);
    if (ObjUtil.equal(signCount, 0L)) {
        return summary;
    }
    summary.setTotalDay(signCount.intValue());

    // 获取最后一条签到记录
    MemberSignInRecordDO lastRecord = signInRecordMapper.selectLastRecordByUserId(userId);
    if (lastRecord == null) {
        return summary;
    }

    // 判断今日是否已签到
    summary.setTodaySignIn(DateUtils.isToday(lastRecord.getCreateTime()));

    // 如果今日未签到且最后签到不是昨天，连续天数归零
    if (!summary.getTodaySignIn() && !DateUtils.isYesterday(lastRecord.getCreateTime())) {
        return summary;
    }

    // 设置连续签到天数
    summary.setContinuousDay(lastRecord.getDay());
    return summary;
}
```

### 4.3 标签 vs 分组

| 维度 | 标签 (Tag) | 分组 (Group) |
|------|-----------|-------------|
| 关系 | 多对多（一个用户可有多个标签） | 多对一（一个用户属于一个分组） |
| 存储 | `MemberUserDO.tagIds` (JSON 数组) | `MemberUserDO.groupId` |
| 用途 | 打标记（如"高价值"、"活跃"） | 归类（如"VIP 群"、"新人群"） |
| 灵活性 | 高（可随时添加/删除标签） | 低（用户只能在一个分组） |
