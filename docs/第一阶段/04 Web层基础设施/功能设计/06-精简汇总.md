# 精简汇总

## 第一部分：能力与使用速查

### 子模块清单

| 子模块 | 配置前缀 | 核心能力 | 默认状态 |
|--------|---------|---------|---------|
| web | `yudao.web` | URL 前缀注入、全局异常处理、CORS、RestTemplate | 启用 |
| xss | `yudao.xss` | XSS 过滤（Filter + Jackson 反序列化两层） | 启用 |
| encrypt | `yudao.api-encrypt` | API 请求/响应加解密 | 禁用 |
| swagger | `yudao.swagger` + `springdoc.*` | OpenAPI 3 文档 + Knife4j 增强 | 启用 |
| apilog | `yudao.access-log` | 访问日志自动采集 | 启用 |
| jackson | — | Jackson 全局配置（Long→String 等） | 启用 |
| banner | — | 自定义启动 Banner | 启用 |
| desensitize | — | 字段脱敏（手机号、邮箱、身份证等） | 启用 |

### URL 前缀注入

```yaml
yudao:
  web:
    admin-api:
      prefix: /admin-api
      controller: "**.controller.admin.**"
    app-api:
      prefix: /app-api
      controller: "**.controller.app.**"
```

- Controller 只写业务路径，框架根据包路径自动注入前缀
- `controller.admin` 包 → `/admin-api` 前缀
- `controller.app` 包 → `/app-api` 前缀
- 实现原理：`WebMvcRegistrations` + `setPathPrefixes()` + `AntPathMatcher(".")`

### 全局异常处理

| 异常类型 | 错误码 | 日志级别 | 持久化 |
|---------|--------|---------|--------|
| 参数校验异常 | 400 | WARN | 否 |
| 业务异常 ServiceException | 自定义 | WARN(第一层) | 否 |
| 权限异常 AccessDeniedException | 403 | WARN | 否 |
| 系统异常（兜底） | 500 | ERROR | 是 |

- Filter 层异常通过 `allExceptionHandler()` 手动调用
- Controller 层异常通过 `@ExceptionHandler` 自动拦截
- 系统异常持久化到数据库，包含完整的诊断信息（userId、traceId、请求参数、堆栈等）
- "表不存在"异常自动识别未启用模块，返回友好提示

### XSS 防护

```yaml
yudao:
  xss:
    enable: true
    exclude-urls:
      - /admin-api/tool/swagger/**
```

- 第一层：`XssFilter` + `XssRequestWrapper` 清理 Query 参数
- 第二层：`XssStringJsonDeserializer` 清理 JSON Body
- 清理引擎：`JsoupXssCleaner`，基于 Jsoup Safelist（白名单）

### API 加解密

```yaml
yudao:
  api-encrypt:
    enable: true
    algorithm: AES
    request-key: 1234567890abcdef
    response-key: 1234567890abcdef
```

- 通过 `@ApiEncrypt` 注解标记需要加密的接口
- 支持 AES（对称）和 RSA（非对称）两种算法
- 请求解密在 Filter 层完成，早于日志记录和 XSS 清理

### 过滤器执行顺序

| 顺序 | Filter | 作用 |
|------|--------|------|
| MIN_VALUE | CorsFilter | 跨域处理 |
| MIN_VALUE + 500 | CacheRequestBodyFilter | Body 可重复读取 |
| MIN_VALUE + 501 | ApiEncryptFilter | 请求解密 |
| -104 | TenantContextFilter | 租户上下文 |
| -103 | ApiAccessLogFilter | 访问日志 |
| -102 | XssFilter | XSS 清理 |
| -100 | Spring Security | 认证授权 |
| -99 | TenantSecurityFilter | 租户安全 |
| MAX_VALUE | DemoFilter | 演示模式拦截 |

### 访问日志

```java
// 记录日志，标记为导出操作
@ApiAccessLog(operateType = OperateTypeEnum.EXPORT)
@GetMapping("/export") ...

// 不记录日志
@ApiAccessLog(enable = false)
@GetMapping("/list") ...

// 记录日志，脱敏 newPassword 字段
@ApiAccessLog(sanitizeKeys = "newPassword")
@PutMapping("/update-password") ...

// 记录日志，同时记录响应结果
@ApiAccessLog(responseEnable = true)
@GetMapping("/get") ...
```

- 自动脱敏字段：`password`、`token`、`accessToken`、`refreshToken`
- 操作信息优先级：`@ApiAccessLog` > `@Tag` + `@Operation` > HTTP 方法推断
- 异步持久化，不阻塞请求

### API 文档

```yaml
yudao:
  swagger:
    title: 电商后台管理系统 API
    description: 提供商品、订单、库存等管理接口
springdoc:
  api-docs:
    enabled: true  # 生产环境设为 false
```

- 自动注入 `tenant-id` 和 `Authorization` 请求头参数
- operationId = `{类名去掉Controller}_{方法名}`
- 分组：`YudaoSwaggerAutoConfiguration.buildGroupedOpenApi("product")`

## 第二部分：设计与概念精要

### 核心设计决策

1. **横切关注点与业务逻辑分离**
   - 异常处理、安全防护、日志采集、文档生成都是 Filter/Advice 层完成
   - Controller 只关心业务逻辑，返回 `CommonResult`
   - 这是 AOP（面向切面编程）思想在 Web 层的体现

2. **约定优于配置**
   - URL 前缀通过包路径自动推断（`controller.admin` → `/admin-api`）
   - 用户类型通过 URL 前缀自动推断（`/admin-api` → ADMIN，`/app-api` → MEMBER）
   - 操作类型通过 HTTP 方法自动推断（GET → 查询，POST → 新增）
   - XSS 过滤默认开启，敏感字段默认脱敏

3. **两层异常处理架构**
   - `@ExceptionHandler` 处理 Controller 层异常（Spring MVC 自动触发）
   - `allExceptionHandler()` 处理 Filter 层异常（手动调用）
   - 两层共享相同的处理逻辑，保证结果一致

4. **两层 XSS 防护**
   - Filter 层清理 Query 参数（`XssRequestWrapper`）
   - Jackson 反序列化层清理 JSON Body（`XssStringJsonDeserializer`）
   - 因为 JSON Body 不经过 `getParameter()` 方法

5. **Filter + Interceptor 协作**
   - Filter 负责全局拦截（CORS、日志、XSS）
   - Interceptor 负责传递 Spring MVC 的元信息（HandlerMethod）
   - Filter 先于 Interceptor 执行，但 Filter 无法获取 HandlerMethod

6. **Wrapper 模式增强请求/响应**
   - `CacheRequestBodyWrapper`：Body 可重复读取
   - `XssRequestWrapper`：参数 XSS 清理
   - `ApiDecryptRequestWrapper`：请求 Body 解密
   - `ApiEncryptResponseWrapper`：响应 Body 加密

### 关键源码文件

| 文件 | 职责 |
|------|------|
| `YudaoWebAutoConfiguration` | 主配置类：URL 前缀、异常处理、CORS、Filter 注册 |
| `WebProperties` | 配置属性：adminApi/appApi 的前缀和包路径 |
| `GlobalExceptionHandler` | 全局异常处理：所有异常类型 → CommonResult |
| `GlobalResponseBodyHandler` | 响应记录：将 CommonResult 存入 Request 属性 |
| `WebFrameworkUtils` | 工具类：获取租户 ID、用户 ID、用户类型等 |
| `XssFilter` + `XssRequestWrapper` | XSS 第一层：清理 Query 参数 |
| `XssStringJsonDeserializer` | XSS 第二层：清理 JSON Body |
| `JsoupXssCleaner` | XSS 清理引擎：基于 Jsoup Safelist |
| `ApiEncryptFilter` | API 加解密：请求解密 + 响应加密 |
| `ApiAccessLogFilter` | 访问日志：记录请求、响应、耗时 |
| `ApiAccessLogInterceptor` | 辅助：传递 HandlerMethod，开发环境打印日志 |
| `CacheRequestBodyFilter` | Body 缓存：使请求 Body 可重复读取 |
| `DemoFilter` | 演示模式：拦截登录用户的写操作 |
| `WebFilterOrderEnum` | Filter 顺序定义 |
