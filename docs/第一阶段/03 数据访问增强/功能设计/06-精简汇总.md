# 数据访问增强：精简汇总

---

## 第一部分：能力与使用速查

### 实体基类

| 类 | 所在模块 | 字段 | 作用 |
|-----|---------|------|------|
| `BaseDO` | mybatis starter | createTime, updateTime, creator, updater, deleted | 所有 DO 的基类，提供审计字段和软删除 |
| `TenantBaseDO` | tenant starter | + tenantId | 多租户场景的基类 |

### 自动填充（DefaultDBFieldHandler）

| 时机 | 填充字段 | 填充逻辑 |
|------|---------|---------|
| INSERT | createTime | 当前时间（为空时） |
| INSERT | updateTime | 当前时间（为空时） |
| INSERT | creator | 当前登录用户 ID（为空时） |
| INSERT | updater | 当前登录用户 ID（为空时） |
| UPDATE | updateTime | 当前时间（为空时） |
| UPDATE | updater | 当前登录用户 ID（为空时） |

### BaseMapperX 核心方法

| 方法 | 作用 | 示例 |
|------|------|------|
| `selectPage(pageParam, wrapper)` | 分页查询，返回 `PageResult` | `selectPage(reqVO, wrapper)` |
| `selectOne(field, value)` | 单字段精确查询 | `selectOne(AdminUserDO::getUsername, "admin")` |
| `selectOne(f1, v1, f2, v2)` | 双字段精确查询 | `selectOne(AdminUserDO::getDeptId, 1L, AdminUserDO::getStatus, 0)` |
| `selectOneForUpdate(field, value)` | 悲观锁查询（FOR UPDATE） | `selectOneForUpdate(OrderDO::getId, orderId)` |
| `selectFirstOne(field, value)` | 并发安全查询（多条取第一条） | `selectFirstOne(AdminUserDO::getUsername, name)` |
| `selectList(field, values)` | IN 查询 | `selectList(AdminUserDO::getDeptId, deptIds)` |
| `insertBatch(entities)` | 批量插入（兼容 SQL Server） | `insertBatch(productList)` |
| `updateBatch(entities)` | 按 ID 批量更新 | `updateBatch(skuList)` |
| `deleteBatch(field, values)` | 按字段 IN 批量删除 | `deleteBatch(OrderDO::getStatus, statuses)` |

### LambdaQueryWrapperX 的 *IfPresent 方法

| 方法 | 空值判断 | 典型场景 |
|------|---------|---------|
| `eqIfPresent` | `val != null` | 状态筛选：`eqIfPresent(DO::getStatus, reqVO.getStatus())` |
| `likeIfPresent` | `hasText(val)` | 名称模糊搜索：`likeIfPresent(DO::getName, reqVO.getName())` |
| `inIfPresent` | 集合非空 | 部门过滤：`inIfPresent(DO::getDeptId, deptIds)` |
| `betweenIfPresent` | 智能处理 | 时间范围：`betweenIfPresent(DO::getCreateTime, begin, end)` |
| `neIfPresent` | `val != null` | 排除某值 |
| `gtIfPresent` / `geIfPresent` | `val != null` | 大于 / 大于等于 |
| `ltIfPresent` / `leIfPresent` | `val != null` | 小于 / 小于等于 |

### 数据库兼容

| 组件 | 作用 |
|------|------|
| `IdTypeEnvironmentPostProcessor` | 启动时根据数据库类型自动切换 ID 策略（AUTO / INPUT） |
| `DbTypeEnum` | 数据库类型枚举，包含 `findInSet` SQL 模板 |
| `JdbcUtils` | 从 JDBC URL 或连接获取数据库类型 |
| `MyBatisUtils.findInSet()` | 跨数据库的 FIND_IN_SET 实现 |
| `QueryWrapperX.limitN()` | 跨数据库的 LIMIT N 实现 |

### 多数据源

| 组件 | 作用 |
|------|------|
| `dynamic-datasource` | 第三方库，提供多数据源路由 |
| `@DS("slave")` | 方法级数据源切换注解 |
| `DataSourceEnum` | 数据源常量（MASTER / SLAVE） |

### 典型 Mapper 示例

```java
@Mapper
public interface AdminUserMapper extends BaseMapperX<AdminUserDO> {

    default AdminUserDO selectByUsername(String username) {
        return selectOne(AdminUserDO::getUsername, username);
    }

    default PageResult<AdminUserDO> selectPage(UserPageReqVO reqVO) {
        return selectPage(reqVO, new LambdaQueryWrapperX<AdminUserDO>()
            .likeIfPresent(AdminUserDO::getUsername, reqVO.getUsername())
            .eqIfPresent(AdminUserDO::getStatus, reqVO.getStatus())
            .betweenIfPresent(AdminUserDO::getCreateTime, reqVO.getCreateTime())
            .orderByDesc(AdminUserDO::getId));
    }
}
```

## 第二部分：设计与概念精要

### 实体基类设计

- **BaseDO 是所有 DO 的根基类**，包含 5 个字段：createTime、updateTime、creator、updater、deleted
- **creator/updater 用 String 类型**而非 Long，预留非数值 ID 的扩展性（如 "SYSTEM"、"WECHAT_SYNC"）
- **deleted 使用 @TableLogic 注解**，MyBatis-Plus 自动将 DELETE 转为 UPDATE，所有查询自动追加 `WHERE deleted = false`
- **clean() 方法清空审计字段**，用于更新操作前防止前端传入的值覆盖自动填充
- **TenantBaseDO 在 tenant starter 中**而非 mybatis starter，不使用多租户的项目不会有多余的 tenantId 字段

### 自动填充设计

- **DefaultDBFieldHandler 实现 MetaObjectHandler**，在 INSERT 和 UPDATE 时自动填充审计字段
- **"为空才填充"原则**：如果业务代码已显式赋值，自动填充不覆盖。这保留了数据迁移等场景的灵活性
- **用户 ID 来自 SecurityFrameworkUtils**：依赖 Spring Security 上下文。定时任务等无用户上下文场景，creator/updater 保持 null
- **insertFill 只处理 BaseDO 类型**：通过 `instanceof` 判断，不会影响非 BaseDO 的实体

### 查询增强设计

- **BaseMapperX 继承 MPJBaseMapper**：同时具备 MyBatis-Plus 的 CRUD 和 mybatis-plus-join 的连表能力
- **selectPage 直接返回 PageResult**：消除每个 Mapper 中 `Page -> PageResult` 的重复转换
- **PAGE_SIZE_NONE 自动处理**：`pageSize = -1` 时跳过分页插件，统一"分页"和"不分页"的调用方式
- **selectOne/selectFirstOne 分工明确**：`selectOne` 在多条时抛异常（适合有唯一约束的场景），`selectFirstOne` 多条时取第一条（适合并发场景）
- **selectOneForUpdate 追加 FOR UPDATE**：悲观锁查询，必须在 @Transactional 中使用
- **insertBatch 兼容 SQL Server**：SQL Server 批量插入后获取自增 ID 会报错，降级为逐条插入

### *IfPresent 模式

- **核心思想**：条件方法内部判断值是否为空，为空则跳过该条件
- **字符串用 hasText 判断**：null、空串、纯空白都跳过
- **数值只判 null**：0 是合法值，不能跳过
- **集合判 isEmpty**：空集合表示"不限制"
- **betweenIfPresent 智能降级**：只有一个值时自动转为 >= 或 <=
- **重写父类方法返回子类型**：确保链式调用时始终能使用 *IfPresent 方法

### 多数据库兼容设计

- **启动时检测，运行时不变**：IdTypeEnvironmentPostProcessor 在 Spring 启动阶段读取 JDBC URL，确定数据库类型，设置 ID 策略。运行时不再切换
- **INPUT 策略配合 @KeySequence**：Oracle/PostgreSQL 使用序列生成 ID，@KeySequence 指定序列名。对 MySQL 无副作用
- **DbTypeEnum 封装 SQL 差异**：每种数据库的 FIND_IN_SET 等价写法定义在枚举中，业务代码通过 MyBatisUtils.findInSet() 统一调用
- **JdbcUtils.getDbType() 兼容多数据源**：优先从 DynamicRoutingDataSource 获取，回退到普通 DataSource
