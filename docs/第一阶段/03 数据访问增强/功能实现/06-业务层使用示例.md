# 业务层使用示例

---

本章通过完整的业务代码示例，展示前面实现的各个组件如何协同工作。以系统模块的"部门管理"为主线，涵盖 Mapper 定义、Service 实现、分页查询、批量操作等典型场景。

> 设计参考：[06-精简汇总](../功能设计/06-精简汇总.md)

## 1. 实体定义 -- 继承 BaseDO

> 源文件：`cn.iocoder.yudao.module.system.dal.dataobject.dept.DeptDO`

```java
@TableName("system_dept")
@KeySequence("system_dept_seq") // 用于 Oracle、PostgreSQL、Kingbase、DB2、H2 数据库的主键自增
@Data
@EqualsAndHashCode(callSuper = true)
public class DeptDO extends TenantBaseDO {

    public static final Long PARENT_ID_ROOT = 0L;

    /**
     * 部门ID
     */
    @TableId
    private Long id;
    /**
     * 部门名称
     */
    private String name;
    /**
     * 父部门ID
     */
    private Long parentId;
    /**
     * 显示顺序
     */
    private Integer sort;
    /**
     * 负责人
     */
    private Long leaderUserId;
    /**
     * 联系电话
     */
    private String phone;
    /**
     * 邮箱
     */
    private String email;
    /**
     * 部门状态
     */
    private Integer status;

}
```

### 关键点

- 继承 `TenantBaseDO`（在 `BaseDO` 基础上增加了 `tenantId` 字段），自动获得 5 个审计字段 + 租户 ID
- `@KeySequence("system_dept_seq")` 指定序列名，仅对 Oracle/PostgreSQL 等使用序列的数据库生效，MySQL 不受影响
- `@TableId` 标记主键字段

## 2. Mapper 定义 -- 继承 BaseMapperX

> 源文件：`cn.iocoder.yudao.module.system.dal.mysql.dept.DeptMapper`

```java
@Mapper
public interface DeptMapper extends BaseMapperX<DeptDO> {

    default List<DeptDO> selectList(DeptListReqVO reqVO) {
        return selectList(new LambdaQueryWrapperX<DeptDO>()
                .likeIfPresent(DeptDO::getName, reqVO.getName())
                .eqIfPresent(DeptDO::getStatus, reqVO.getStatus()));
    }

    default DeptDO selectByParentIdAndName(Long parentId, String name) {
        return selectOne(DeptDO::getParentId, parentId, DeptDO::getName, name);
    }

    default Long selectCountByParentId(Long parentId) {
        return selectCount(DeptDO::getParentId, parentId);
    }

    default List<DeptDO> selectListByParentId(Collection<Long> parentIds) {
        return selectList(DeptDO::getParentId, parentIds);
    }

    default List<DeptDO> selectListByLeaderUserId(Long id) {
        return selectList(DeptDO::getLeaderUserId, id);
    }

}
```

### Mapper 方法解读

| 方法 | 使用的 BaseMapperX 能力 | 说明 |
|------|----------------------|------|
| `selectList(DeptListReqVO)` | `selectList(Wrapper)` + `LambdaQueryWrapperX` | 条件查询，用 `*IfPresent` 跳过空值 |
| `selectByParentIdAndName` | `selectOne(field1, value1, field2, value2)` | 双字段精确查询 |
| `selectCountByParentId` | `selectCount(field, value)` | 单字段计数 |
| `selectListByParentId` | `selectList(field, values)` | IN 查询，自动处理空集合 |
| `selectListByLeaderUserId` | `selectList(field, value)` | 单字段精确查询 |

所有方法都是 `default` 方法，不需要写 XML 或 `@Select` 注解。`BaseMapperX` 提供的快捷方法 + `LambdaQueryWrapperX` 的条件拼接，覆盖了 90% 以上的查询场景。

## 3. Service 实现 -- 完整 CRUD

> 源文件：`cn.iocoder.yudao.module.system.service.dept.DeptServiceImpl`

```java
@Service
@Validated
@Slf4j
public class DeptServiceImpl implements DeptService {

    @Resource
    private DeptMapper deptMapper;

    // ========== 新增 ==========

    @Override
    @CacheEvict(cacheNames = RedisKeyConstants.DEPT_CHILDREN_ID_LIST, allEntries = true)
    public Long createDept(DeptSaveReqVO createReqVO) {
        if (createReqVO.getParentId() == null) {
            createReqVO.setParentId(DeptDO.PARENT_ID_ROOT);
        }
        // 校验父部门的有效性
        validateParentDept(null, createReqVO.getParentId());
        // 校验部门名的唯一性
        validateDeptNameUnique(null, createReqVO.getParentId(), createReqVO.getName());

        // 插入部门
        DeptDO dept = BeanUtils.toBean(createReqVO, DeptDO.class);
        deptMapper.insert(dept);
        return dept.getId();
    }

    // ========== 修改 ==========

    @Override
    @CacheEvict(cacheNames = RedisKeyConstants.DEPT_CHILDREN_ID_LIST, allEntries = true)
    public void updateDept(DeptSaveReqVO updateReqVO) {
        if (updateReqVO.getParentId() == null) {
            updateReqVO.setParentId(DeptDO.PARENT_ID_ROOT);
        }
        // 校验自己存在
        validateDeptExists(updateReqVO.getId());
        // 校验父部门的有效性
        validateParentDept(updateReqVO.getId(), updateReqVO.getParentId());
        // 校验部门名的唯一性
        validateDeptNameUnique(updateReqVO.getId(), updateReqVO.getParentId(), updateReqVO.getName());

        // 更新部门
        DeptDO updateObj = BeanUtils.toBean(updateReqVO, DeptDO.class);
        deptMapper.updateById(updateObj);
    }

    // ========== 删除 ==========

    @Override
    @CacheEvict(cacheNames = RedisKeyConstants.DEPT_CHILDREN_ID_LIST, allEntries = true)
    public void deleteDept(Long id) {
        // 校验是否存在
        validateDeptExists(id);
        // 校验是否有子部门
        if (deptMapper.selectCountByParentId(id) > 0) {
            throw exception(DEPT_EXITS_CHILDREN);
        }
        // 删除部门
        deptMapper.deleteById(id);
    }

    // ========== 查询 ==========

    @Override
    public DeptDO getDept(Long id) {
        return deptMapper.selectById(id);
    }

    @Override
    public List<DeptDO> getDeptList(DeptListReqVO reqVO) {
        List<DeptDO> list = deptMapper.selectList(reqVO);
        list.sort(Comparator.comparing(DeptDO::getSort));
        return list;
    }

    @Override
    public List<DeptDO> getDeptList(Collection<Long> ids) {
        if (CollUtil.isEmpty(ids)) {
            return Collections.emptyList();
        }
        return deptMapper.selectByIds(ids);
    }

    // ========== 校验方法 ==========

    @VisibleForTesting
    void validateDeptExists(Long id) {
        if (id == null) {
            return;
        }
        DeptDO dept = deptMapper.selectById(id);
        if (dept == null) {
            throw exception(DEPT_NOT_FOUND);
        }
    }

    @VisibleForTesting
    void validateDeptNameUnique(Long id, Long parentId, String name) {
        DeptDO dept = deptMapper.selectByParentIdAndName(parentId, name);
        if (dept == null) {
            return;
        }
        if (id == null) {
            throw exception(DEPT_NAME_DUPLICATE);
        }
        if (ObjectUtil.notEqual(dept.getId(), id)) {
            throw exception(DEPT_NAME_DUPLICATE);
        }
    }

}
```

### CRUD 使用的 Mapper 方法

| 操作 | Mapper 方法 | 说明 |
|------|------------|------|
| 新增 | `deptMapper.insert(dept)` | BaseMapper 的 insert，自动填充审计字段 |
| 修改 | `deptMapper.updateById(updateObj)` | BaseMapper 的 updateById |
| 删除 | `deptMapper.deleteById(id)` | 逻辑删除，自动转为 UPDATE SET deleted = true |
| 按 ID 查 | `deptMapper.selectById(id)` | BaseMapper 的 selectById |
| 批量查 | `deptMapper.selectByIds(ids)` | BaseMapperX 的 selectByIds |
| 条件查 | `deptMapper.selectList(reqVO)` | 自定义的 default 方法，使用 LambdaQueryWrapperX |
| 唯一查 | `deptMapper.selectByParentIdAndName(parentId, name)` | 使用 selectOne(field1, value1, field2, value2) |
| 计数 | `deptMapper.selectCountByParentId(id)` | 使用 selectCount(field, value) |

## 4. 分页查询示例

以用户管理为例，展示完整的分页查询流程：

> 源文件：`cn.iocoder.yudao.module.system.dal.mysql.user.AdminUserMapper`

```java
@Mapper
public interface AdminUserMapper extends BaseMapperX<AdminUserDO> {

    default AdminUserDO selectByUsername(String username) {
        return selectOne(AdminUserDO::getUsername, username);
    }

    default PageResult<AdminUserDO> selectPage(UserPageReqVO reqVO, Collection<Long> deptIds,
                                                 Collection<Long> userIds) {
        return selectPage(reqVO, new LambdaQueryWrapperX<AdminUserDO>()
                .likeIfPresent(AdminUserDO::getUsername, reqVO.getUsername())
                .likeIfPresent(AdminUserDO::getMobile, reqVO.getMobile())
                .eqIfPresent(AdminUserDO::getStatus, reqVO.getStatus())
                .betweenIfPresent(AdminUserDO::getCreateTime, reqVO.getCreateTime())
                .inIfPresent(AdminUserDO::getDeptId, deptIds)
                .inIfPresent(AdminUserDO::getId, userIds)
                .orderByDesc(AdminUserDO::getId));
    }

    default List<AdminUserDO> selectListByStatus(Integer status) {
        return selectList(AdminUserDO::getStatus, status);
    }

    default List<AdminUserDO> selectListByDeptIds(Collection<Long> deptIds) {
        return selectList(AdminUserDO::getDeptId, deptIds);
    }

}
```

分页查询的调用链：

```
Controller 接收 UserPageReqVO（包含 pageNo、pageSize、查询条件）
    │
    ├── 调用 Service.getUserPage(reqVO)
    │
    ├── Service 调用 Mapper.selectPage(reqVO, deptIds, userIds)
    │
    ├── Mapper 内部：
    │   ├── 构建 LambdaQueryWrapperX，拼接条件（跳过空值）
    │   ├── 调用 BaseMapperX.selectPage(pageParam, wrapper)
    │   │
    │   ├── BaseMapperX 内部：
    │   │   ├── MyBatisUtils.buildPage() 构建 MyBatis-Plus 的 Page 对象
    │   │   ├── 调用 MyBatis-Plus 的 selectPage(mpPage, wrapper) 执行查询
    │   │   └── 将 IPage 转换为 PageResult 返回
    │   │
    │   └── 返回 PageResult<AdminUserDO>
    │
    └── 返回给 Controller
```

public void importProductSkus(List<ProductSkuDO> skuList) {

### 5.3 diffList -- 对比新老数据批量保存

在主子表场景中，编辑时需要对比新老数据，分别执行新增、修改、删除：

> 源文件参考：`cn.iocoder.yudao.module.wms.service.order.shipment.WmsShipmentOrderDetailServiceImpl`

```java
public void updateOrderDetailList(Long orderId, OrderSaveReqVO reqVO) {
    // 第一步，对比新老数据，获得添加、修改、删除的列表
    List<OrderDetailDO> oldList = orderDetailMapper.selectListByOrderId(orderId);
    List<OrderDetailDO> newList = buildOrderDetailList(reqVO);

    // diffList 返回 [新增列表, 修改列表, 删除列表]
    List<List<OrderDetailDO>> diffList = diffList(oldList, newList,
            (oldVal, newVal) -> ObjectUtil.equal(oldVal.getId(), newVal.getId()));

    // 第二步，批量添加、修改、删除
    // 新增：newList 中存在但 oldList 中不存在的
    if (CollUtil.isNotEmpty(diffList.get(0))) {
        diffList.get(0).forEach(detail -> detail.setOrderId(orderId));
        orderDetailMapper.insertBatch(diffList.get(0));
    }
    // 修改：newList 和 oldList 中都存在的
    if (CollUtil.isNotEmpty(diffList.get(1))) {
        diffList.get(1).forEach(detail -> detail.setOrderId(orderId));
        orderDetailMapper.updateBatch(diffList.get(1));
    }
    // 删除：oldList 中存在但 newList 中不存在的
    if (CollUtil.isNotEmpty(diffList.get(2))) {
        orderDetailMapper.deleteByIds(convertList(diffList.get(2), OrderDetailDO::getId));
    }
}
```

`diffList` 的返回值是一个长度为 3 的列表：

| 索引 | 含义 | 操作 |
|------|------|------|
| 0 | 新增列表（newList 中有，oldList 中没有） | insertBatch |
| 1 | 修改列表（newList 和 oldList 中都有） | updateBatch |
| 2 | 删除列表（oldList 中有，newList 中没有） | deleteByIds |

匹配规则通过第二个参数 `BiFunction` 定义，这里是按 `id` 是否相等判断。

## 6. 悲观锁示例

电商下单扣减库存的典型场景：

```java
@Service
public class ProductSkuServiceImpl implements ProductSkuService {

    @Resource
    private ProductSkuMapper productSkuMapper;

    @Override
    @Transactional(rollbackFor = Exception.class) // 必须在事务中，否则锁立即释放
    public void deductStock(Long skuId, Integer quantity) {
        // SELECT * FROM product_sku WHERE id = ? FOR UPDATE
        ProductSkuDO sku = productSkuMapper.selectOneForUpdate(ProductSkuDO::getId, skuId);
        if (sku.getStock() < quantity) {
            throw exception(STOCK_NOT_ENOUGH);
        }
        sku.setStock(sku.getStock() - quantity);
        productSkuMapper.updateById(sku);
    }

}
```

`selectOneForUpdate` 在 SQL 末尾追加 `FOR UPDATE`，锁定该行直到事务提交。其他事务对同一行的 `FOR UPDATE` 查询会阻塞等待。

## 7. 总结：数据访问增强的完整能力

经过前面各章的实现，我们现在拥有了一个强大的数据访问层：

```
业务代码
    │
    ├── Entity（继承 BaseDO）
    │   └── 自动获得：createTime, updateTime, creator, updater, deleted
    │
    ├── Mapper（继承 BaseMapperX）
    │   ├── 分页查询：selectPage → PageResult
    │   ├── 快捷查询：selectOne, selectFirstOne, selectCount, selectList
    │   ├── 悲观锁：selectOneForUpdate
    │   ├── 批量操作：insertBatch, updateBatch, deleteBatch
    │   └── 联表查询：selectJoinPage（通过 MPJBaseMapper）
    │
    ├── QueryWrapper（使用 LambdaQueryWrapperX）
    │   └── 条件拼接：eqIfPresent, likeIfPresent, inIfPresent, betweenIfPresent...
    │
    └── 自动化
        ├── 自动填充：DefaultDBFieldHandler 自动设置审计字段
        ├── 逻辑删除：@TableLogic 自动处理 DELETE/SELECT
        ├── ID 策略：IdTypeEnvironmentPostProcessor 根据数据库自动选择
        └── 多数据源：@DS 注解切换数据源
```

这些能力在后续的每个业务模块中都会用到，形成统一的数据访问风格。
