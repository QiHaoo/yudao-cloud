# 起步：骨架搭建

---

本章是数据访问增强组件的实现起点。我们将创建 `yudao-spring-boot-starter-mybatis` 模块的骨架，包括 Maven 依赖、目录结构和自动配置入口。

## 1. 模块 pom.xml

首先创建模块的 `pom.xml`，声明所有需要的依赖。

> 源文件：`yudao-framework/yudao-spring-boot-starter-mybatis/pom.xml`


### 依赖说明

| 依赖 | 作用 |
|------|------|
| `yudao-common` | 提供 `PageParam`、`PageResult`、`CommonResult` 等基础类型 |
| `yudao-spring-boot-starter-security` | `provided` 作用域，仅 `DefaultDBFieldHandler` 用到 `SecurityFrameworkUtils` |
| `druid-spring-boot-starter` | Druid 连接池，提供监控页面 |
| `mybatis-plus-boot-starter` | MyBatis-Plus 核心，提供 CRUD、分页、代码生成等 |
| `mybatis-plus-jsqlparser-4.9` | SQL 解析器，用于多租户、数据权限等 SQL 改写 |
| `dynamic-datasource-spring-boot-starter` | 多数据源路由，支持 `@DS` 注解切换数据源 |
| `mybatis-plus-join-boot-starter` | MyBatis-Plus Join，支持 Java 代码写联表查询 |
| `easy-trans-*` | VO 数据翻译，自动填充关联字段名称 |

数据库驱动采用 `optional` 模式：MySQL 默认引入，其他数据库（Oracle、PostgreSQL、SQL Server、达梦、人大金仓、openGauss）标记为 `optional`，业务模块按需引入。

## 3. spring.factories 自动配置

Spring Boot 通过 `spring.factories` 文件发现自动配置类。本模块注册了一个 `EnvironmentPostProcessor`：

> 源文件：`yudao-spring-boot-starter-mybatis/src/main/resources/META-INF/spring.factories`

```properties
org.springframework.boot.env.EnvironmentPostProcessor=\
  cn.iocoder.yudao.framework.mybatis.config.IdTypeEnvironmentPostProcessor
```

`EnvironmentPostProcessor` 是 Spring Boot 提供的扩展点，在 **所有 Bean 创建之前** 执行，用于修改 Environment 中的属性。这里用它来根据数据库类型自动设置 ID 策略。

两个自动配置类通过 `@AutoConfiguration` 注解自动注册（Spring Boot 2.7+ 的新机制，不再需要在 spring.factories 中声明 `EnableAutoConfiguration`）：

- `YudaoMybatisAutoConfiguration` -- MyBatis-Plus 配置（分页插件、自动填充、KeyGenerator）
- `YudaoDataSourceAutoConfiguration` -- 数据源配置（事务管理、Druid 广告过滤）

## 4. 数据源自动配置

> 源文件：`cn.iocoder.yudao.framework.datasource.config.YudaoDataSourceAutoConfiguration`

```java
@AutoConfiguration
@EnableTransactionManagement(proxyTargetClass = true) // 启动事务管理
@EnableConfigurationProperties(DruidStatProperties.class)
public class YudaoDataSourceAutoConfiguration {

    /**
     * 创建 DruidAdRemoveFilter 过滤器，过滤 common.js 的广告
     */
    @Bean
    @ConditionalOnProperty(name = "spring.datasource.druid.stat-view-servlet.enabled", havingValue = "true")
    public FilterRegistrationBean<DruidAdRemoveFilter> druidAdRemoveFilterFilter(DruidStatProperties properties) {
        // 获取 druid web 监控页面的参数
        DruidStatProperties.StatViewServlet config = properties.getStatViewServlet();
        // 提取 common.js 的配置路径
        String pattern = config.getUrlPattern() != null ? config.getUrlPattern() : "/druid/*";
        String commonJsPattern = pattern.replaceAll("\\*", "js/common.js");
        // 创建 DruidAdRemoveFilter Bean
        FilterRegistrationBean<DruidAdRemoveFilter> registrationBean = new FilterRegistrationBean<>();
        registrationBean.setFilter(new DruidAdRemoveFilter());
        registrationBean.addUrlPatterns(commonJsPattern);
        return registrationBean;
    }

}
```

这个配置类做了两件事：

1. **启动事务管理** -- `@EnableTransactionManagement` 使 `@Transactional` 生效
2. **Druid 监控页广告过滤** -- Druid 免费版的监控页面底部有广告，通过 Filter 替换掉 `common.js` 中的广告 HTML

### 数据源常量

> 源文件：`cn.iocoder.yudao.framework.datasource.core.enums.DataSourceEnum`

```java
public interface DataSourceEnum {

    /**
     * 主库，推荐使用 @Master 注解
     */
    String MASTER = "master";
    /**
     * 从库，推荐使用 @Slave 注解
     */
    String SLAVE = "slave";

}
```

在业务代码中使用 `@DS("slave")` 注解切换到从库。数据源的名称与配置文件中的 `spring.datasource.dynamic.datasource.slave` 对应。
