# Maven 多模块项目结构：精简汇总

---

## 第一部分：能力与使用速查

### 三层模块体系

| 层级 | 模块 | 职责 | 产出物 |
|------|------|------|--------|
| 第一层：BOM | `yudao-dependencies` | 集中管理所有依赖版本（70+ 个版本属性） | 被根 POM `<scope>import</scope>` 导入 |
| 第二层：框架层 | `yudao-framework/` 下 17 个 Starter | 封装通用技术能力（Web、数据库、缓存、安全等） | Spring Boot Starter JAR |
| 第三层：业务模块 | `yudao-module-*/` | 实现业务逻辑，每个模块拆分为 `-api` + `-server` | 可独立部署的 Spring Boot 应用 |

### 版本管理

| 机制 | 位置 | 说明 |
|------|------|------|
| `${revision}` 属性 | 根 POM + BOM 的 `<properties>` | 所有模块和内部依赖共用一个版本号 |
| `flatten-maven-plugin` | 根 POM（`oss` 模式）+ BOM（`bom` 模式） | 构建时将 `${revision}` 解析为实际版本字符串 |
| BOM `<scope>import</scope>` | 根 POM 的 `<dependencyManagement>` | 子模块无需写 `<version>`，版本全由 BOM 控制 |

### 模块依赖规则

| 规则 | 示例 |
|------|------|
| 所有模块可依赖 `yudao-common` 和框架 Starter | `mall-server → yudao-spring-boot-starter-web` |
| 业务模块可依赖其他模块的 `-api` | `mall-server → system-api` |
| 业务模块不可依赖其他模块的 `-server` | `mall-server → system-server` **禁止** |
| 框架 Starter 不可依赖业务模块 | `starter-web → module-system` **禁止** |

### 单体/微服务切换

| 操作 | 单体模式 | 微服务模式 |
|------|---------|-----------|
| 启用模块 | 取消 `yudao-server/pom.xml` 中的注释 | 每个模块独立启动 |
| Nacos | `spring.cloud.nacos.discovery.enabled=false` | `enabled=true` |
| OpenFeign | 从 `yudao-server` 依赖中排除 | 正常引入 |
| 模块间调用 | 同 JVM 内直接方法调用 | Feign HTTP 远程调用 |

## 第二部分：设计与概念精要

### 分层与隔离

- **三层模块体系**：BOM 管版本 → 框架层管能力 → 业务层管逻辑。每层有明确职责，层间单向依赖
- **api/server 拆分**：每个业务模块的 `-api` 是轻量级契约（接口 + DTO），`-server` 是完整实现。其他模块只依赖 `-api`，实现可替换
- **框架不依赖业务**：通过 `yudao-common` 中的 `CommonApi` 接口打破框架→业务的反向依赖

### 版本管理

- **${revision} 统一版本**：一个属性控制所有模块版本，发版只改一处
- **flatten-maven-plugin**：BOM 用 `bom` 模式（保留 `<dependencyManagement>`），普通模块用 `oss` 模式（生成消费级 POM）
- **BOM 无 parent**：独立的 BOM 可被外部项目干净导入，不带入项目特有的依赖管理

### 双模式部署

- **组件扫描是桥梁**：`scanBasePackages=cn.iocoder.yudao.module` 让单体模式自动加载所有模块的 Bean
- **Feign 排除机制**：单体模式排除 `spring-cloud-starter-openfeign`，`@FeignClient` 接口由本地 `@RestController` 实现类直接提供服务
- **优雅降级**：禁用模块的请求由 `DefaultController` 返回明确的 `NOT_IMPLEMENTED` 提示
- **Profile 隔离**：`application.yaml` 存通用配置，`application-local.yaml` / `application-dev.yaml` 存环境相关配置
