# 起步：从零搭建多模块骨架

本章从一个空目录开始，一步步搭建 yudao-cloud 的 Maven 多模块结构。完成后你将拥有一个可编译的项目骨架。

> 本章对应的功能设计章节：[01-问题域与设计目标](../功能设计/01-问题域与设计目标.md)、[02-三层模块体系设计](../功能设计/02-三层模块体系设计.md)

---

## 1. 创建根 POM

根 POM 是整个项目的聚合器，定义了所有子模块和全局构建配置。

```xml
<!-- pom.xml（项目根目录） -->
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>

    <groupId>cn.iocoder.cloud</groupId>
    <artifactId>yudao</artifactId>
    <version>${revision}</version>
    <packaging>pom</packaging>
    <name>yudao-cloud</name>

    <modules>
        <module>yudao-dependencies</module>
        <module>yudao-gateway</module>
        <module>yudao-framework</module>
        <module>yudao-server</module>
        <module>yudao-module-system</module>
        <module>yudao-module-infra</module>
        <!-- 更多业务模块按需添加 -->
    </modules>

    <properties>
        <revision>2026.04-jdk8-SNAPSHOT</revision>
        <java.version>1.8</java.version>
    </properties>

    <!-- 导入 BOM，所有子模块自动继承版本管理 -->
    <dependencyManagement>
        <dependencies>
            <dependency>
                <groupId>cn.iocoder.cloud</groupId>
                <artifactId>yudao-dependencies</artifactId>
                <version>${revision}</version>
                <type>pom</type>
                <scope>import</scope>
            </dependency>
        </dependencies>
    </dependencyManagement>

    <build>
        <pluginManagement>
            <plugins>
                <plugin>
                    <groupId>org.apache.maven.plugins</groupId>
                    <artifactId>maven-compiler-plugin</artifactId>
                    <version>3.14.0</version>
                    <configuration>
                        <source>${java.version}</source>
                        <target>${java.version}</target>
                    </configuration>
                </plugin>
            </plugins>
        </pluginManagement>

        <!-- flatten-maven-plugin：将 ${revision} 解析为实际版本 -->
        <plugins>
            <plugin>
                <groupId>org.codehaus.mojo</groupId>
                <artifactId>flatten-maven-plugin</artifactId>
                <version>1.7.2</version>
                <configuration>
                    <flattenMode>oss</flattenMode>
                    <updatePomFile>true</updatePomFile>
                </configuration>
                <executions>
                    <execution>
                        <id>flatten</id>
                        <phase>process-resources</phase>
                        <goals><goal>flatten</goal></goals>
                    </execution>
                    <execution>
                        <id>flatten.clean</id>
                        <phase>clean</phase>
                        <goals><goal>clean</goal></goals>
                    </execution>
                </executions>
            </plugin>
        </plugins>
    </build>
</project>
```

关键点：
- `<version>${revision}</version>` — 所有模块共用一个版本号
- `<scope>import</scope>` — 导入 BOM，子模块无需写 `<version>`
- `flatten-maven-plugin` 的 `oss` 模式 — 生成消费级 POM

## 2. 创建 BOM

BOM 是一个**独立的 POM**（没有 parent），集中管理所有依赖版本。

```xml
<!-- yudao-dependencies/pom.xml -->
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>

    <!-- 注意：没有 parent -->
    <groupId>cn.iocoder.cloud</groupId>
    <artifactId>yudao-dependencies</artifactId>
    <version>${revision}</version>
    <packaging>pom</packaging>

    <properties>
        <revision>2026.04-jdk8-SNAPSHOT</revision>

        <!-- Spring 生态 -->
        <spring.boot.version>2.7.18</spring.boot.version>
        <spring.cloud.version>2021.0.9</spring.cloud.version>
        <spring.cloud.alibaba.version>2021.0.6.2</spring.cloud.alibaba.version>

        <!-- 数据层 -->
        <mybatis-plus.version>3.5.16</mybatis-plus.version>
        <druid.version>1.2.28</druid.version>
        <redisson.version>4.4.0</redisson.version>

        <!-- 工具库 -->
        <lombok.version>1.18.46</lombok.version>
        <mapstruct.version>1.6.3</mapstruct.version>
        <hutool-5.version>5.8.44</hutool-5.version>
        <guava.version>33.6.0-jre</guava.version>

        <!-- 更多版本属性... -->
    </properties>

    <dependencyManagement>
        <!-- 第一步：导入上游 BOM -->
        <dependencies>
            <dependency>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-dependencies</artifactId>
                <version>${spring.boot.version}</version>
                <type>pom</type>
                <scope>import</scope>
            </dependency>
            <dependency>
                <groupId>org.springframework.cloud</groupId>
                <artifactId>spring-cloud-dependencies</artifactId>
                <version>${spring.cloud.version}</version>
                <type>pom</type>
                <scope>import</scope>
            </dependency>
            <dependency>
                <groupId>com.alibaba.cloud</groupId>
                <artifactId>spring-cloud-alibaba-dependencies</artifactId>
                <version>${spring.cloud.alibaba.version}</version>
                <type>pom</type>
                <scope>import</scope>
            </dependency>

            <!-- 第二步：声明框架模块 -->
            <dependency>
                <groupId>cn.iocoder.cloud</groupId>
                <artifactId>yudao-common</artifactId>
                <version>${revision}</version>
            </dependency>
            <dependency>
                <groupId>cn.iocoder.cloud</groupId>
                <artifactId>yudao-spring-boot-starter-web</artifactId>
                <version>${revision}</version>
            </dependency>
            <dependency>
                <groupId>cn.iocoder.cloud</groupId>
                <artifactId>yudao-spring-boot-starter-mybatis</artifactId>
                <version>${revision}</version>
            </dependency>
            <!-- 更多框架 Starter... -->

            <!-- 第三步：声明第三方库 -->
            <dependency>
                <groupId>com.baomidou</groupId>
                <artifactId>mybatis-plus-boot-starter</artifactId>
                <version>${mybatis-plus.version}</version>
            </dependency>
            <dependency>
                <groupId>com.alibaba</groupId>
                <artifactId>druid-spring-boot-starter</artifactId>
                <version>${druid.version}</version>
            </dependency>
            <!-- 更多第三方库... -->
        </dependencies>
    </dependencyManagement>

    <!-- BOM 使用 bom 模式的 flatten 插件 -->
    <build>
        <plugins>
            <plugin>
                <groupId>org.codehaus.mojo</groupId>
                <artifactId>flatten-maven-plugin</artifactId>
                <version>1.7.2</version>
                <configuration>
                    <flattenMode>bom</flattenMode>
                    <updatePomFile>true</updatePomFile>
                </configuration>
                <executions>
                    <execution>
                        <id>flatten</id>
                        <phase>process-resources</phase>
                        <goals><goal>flatten</goal></goals>
                    </execution>
                    <execution>
                        <id>flatten.clean</id>
                        <phase>clean</phase>
                        <goals><goal>clean</goal></goals>
                    </execution>
                </executions>
            </plugin>
        </plugins>
    </build>
</project>
```

## 4. 创建一个业务模块的骨架

以 `yudao-module-system` 为例，展示 api/server 拆分：

```xml
<!-- yudao-module-system/pom.xml（聚合 POM） -->
<project ...>
    <parent>
        <groupId>cn.iocoder.cloud</groupId>
        <artifactId>yudao</artifactId>
        <version>${revision}</version>
    </parent>

    <artifactId>yudao-module-system</artifactId>
    <packaging>pom</packaging>

    <modules>
        <module>yudao-module-system-api</module>
        <module>yudao-module-system-server</module>
    </modules>
</project>
```

```xml
<!-- yudao-module-system/yudao-module-system-api/pom.xml -->
<project ...>
    <parent>
        <groupId>cn.iocoder.cloud</groupId>
        <artifactId>yudao-module-system</artifactId>
        <version>${revision}</version>
    </parent>

    <artifactId>yudao-module-system-api</artifactId>
    <packaging>jar</packaging>

    <dependencies>
        <!-- 共享基础类型 -->
        <dependency>
            <groupId>cn.iocoder.cloud</groupId>
            <artifactId>yudao-common</artifactId>
        </dependency>

        <!-- OpenFeign：optional，不强制传递给消费者 -->
        <dependency>
            <groupId>org.springframework.cloud</groupId>
            <artifactId>spring-cloud-starter-openfeign</artifactId>
            <optional>true</optional>
        </dependency>

        <!-- Validation：optional -->
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-validation</artifactId>
            <optional>true</optional>
        </dependency>
    </dependencies>
</project>
```

注意 API 模块的依赖都标记为 `optional`——这意味着当其他模块依赖 `yudao-module-system-api` 时，这些传递依赖不会自动引入。这是刻意的设计：API 模块应该尽可能轻量。

```xml
<!-- yudao-module-system/yudao-module-system-server/pom.xml -->
<project ...>
    <parent>
        <groupId>cn.iocoder.cloud</groupId>
        <artifactId>yudao-module-system</artifactId>
        <version>${revision}</version>
    </parent>

    <artifactId>yudao-module-system-server</artifactId>
    <packaging>jar</packaging>

    <dependencies>
        <!-- 自己的 API -->
        <dependency>
            <groupId>cn.iocoder.cloud</groupId>
            <artifactId>yudao-module-system-api</artifactId>
        </dependency>

        <!-- 框架 Starter（注意：无需写 version，由 BOM 管理） -->
        <dependency>
            <groupId>cn.iocoder.cloud</groupId>
            <artifactId>yudao-spring-boot-starter-env</artifactId>
        </dependency>
        <dependency>
            <groupId>cn.iocoder.cloud</groupId>
            <artifactId>yudao-spring-boot-starter-security</artifactId>
        </dependency>
        <dependency>
            <groupId>cn.iocoder.cloud</groupId>
            <artifactId>yudao-spring-boot-starter-mybatis</artifactId>
        </dependency>
        <dependency>
            <groupId>cn.iocoder.cloud</groupId>
            <artifactId>yudao-spring-boot-starter-redis</artifactId>
        </dependency>
        <!-- 更多 Starter... -->

        <!-- 其他模块的 API（用于 RPC 调用） -->
        <dependency>
            <groupId>cn.iocoder.cloud</groupId>
            <artifactId>yudao-module-infra-api</artifactId>
        </dependency>
    </dependencies>

    <build>
        <plugins>
            <!-- 打包为可执行 JAR -->
            <plugin>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-maven-plugin</artifactId>
                <executions>
                    <execution>
                        <goals><goal>repackage</goal></goals>
                    </execution>
                </executions>
            </plugin>
        </plugins>
    </build>
</project>
```

## 5. 创建单体启动壳

```xml
<!-- yudao-server/pom.xml -->
<project ...>
    <parent>
        <groupId>cn.iocoder.cloud</groupId>
        <artifactId>yudao</artifactId>
        <version>${revision}</version>
    </parent>

    <artifactId>yudao-server</artifactId>
    <packaging>jar</packaging>

    <dependencies>
        <!-- 必选模块 -->
        <dependency>
            <groupId>cn.iocoder.cloud</groupId>
            <artifactId>yudao-module-system-server</artifactId>
        </dependency>
        <dependency>
            <groupId>cn.iocoder.cloud</groupId>
            <artifactId>yudao-module-infra-server</artifactId>
        </dependency>

        <!-- 可选模块：按需取消注释 -->
        <!--
        <dependency>
            <groupId>cn.iocoder.cloud</groupId>
            <artifactId>yudao-module-mall-product-server</artifactId>
        </dependency>
        -->

        <!-- 服务保护 -->
        <dependency>
            <groupId>cn.iocoder.cloud</groupId>
            <artifactId>yudao-spring-boot-starter-protection</artifactId>
        </dependency>

        <!-- RPC（排除 OpenFeign，单体模式下不需要） -->
        <dependency>
            <groupId>cn.iocoder.cloud</groupId>
            <artifactId>yudao-spring-boot-starter-rpc</artifactId>
            <exclusions>
                <exclusion>
                    <groupId>org.springframework.cloud</groupId>
                    <artifactId>spring-cloud-starter-openfeign</artifactId>
                </exclusion>
            </exclusions>
        </dependency>

        <!-- Nacos（即使单体模式也引入，但配置中关闭） -->
        <dependency>
            <groupId>com.alibaba.cloud</groupId>
            <artifactId>spring-cloud-starter-alibaba-nacos-discovery</artifactId>
        </dependency>
        <dependency>
            <groupId>com.alibaba.cloud</groupId>
            <artifactId>spring-cloud-starter-alibaba-nacos-config</artifactId>
        </dependency>
    </dependencies>

    <build>
        <plugins>
            <plugin>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-maven-plugin</artifactId>
                <executions>
                    <execution>
                        <goals><goal>repackage</goal></goals>
                    </execution>
                </executions>
            </plugin>
        </plugins>
    </build>
</project>
```

## 6. 验证骨架

执行以下命令验证项目结构是否正确：

```bash
# 在项目根目录
mvn clean install -DskipTests
```

如果构建成功，你会看到每个模块都被安装到本地 Maven 仓库，且 `${revision}` 已被替换为实际版本号。

```bash
# 检查安装结果
ls ~/.m2/repository/cn/iocoder/cloud/yudao-common/2026.04-jdk8-SNAPSHOT/
# 应该看到 yudao-common-2026.04-jdk8-SNAPSHOT.jar 和 .pom
```
