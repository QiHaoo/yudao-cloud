# 配置体系实现

> 本章是"单体应用启动与配置"功能实现的第二篇。在功能设计中，我们分析了为什么需要分层配置——200+ 配置项不能全写在一个文件里，不同环境（本地、开发、生产）的配置需要隔离。
>
> 本章的目标是：**完整拆解三个 YAML 文件的每一个关键配置区段**，理解它们各自的职责和协作方式。

## 1. 配置文件全景

yudao-server 模块下有三个配置文件：

```
yudao-server/src/main/resources/
├── application.yaml          ← 公共配置（所有环境共享）
├── application-local.yaml    ← 本地开发环境
└── application-dev.yaml      ← 开发服务器环境
```

它们的加载顺序和覆盖关系：

```
1. 先加载 application.yaml（公共配置）
       │
2. 再加载 application-{profile}.yaml（环境配置，覆盖同名属性）
       │
3. 最终合并成一个完整的配置集
```

切换环境只需修改 `application.yaml` 中的一行：

```yaml
spring:
  profiles:
    active: local   # 改成 dev 即可切换到开发服务器环境
```

或者通过命令行覆盖（不改文件）：

```bash
java -jar yudao-server.jar --spring.profiles.active=dev
```

## 2. application.yaml 详解

这个文件存放所有环境共享的配置。使用 YAML 的 `---` 分隔符按关注点分区。

### 2.1 Spring 基础配置区

```yaml
# 来源：yudao-server/src/main/resources/application.yaml

spring:
  application:
    name: yudao-server

  profiles:
    active: local

  main:
    allow-circular-references: true # 允许循环依赖，因为项目是三层架构，无法避免这个情况。

  # Servlet 配置
  servlet:
    # 文件上传相关配置项
    multipart:
      max-file-size: 16MB # 单个文件大小
      max-request-size: 32MB # 设置总上传的文件大小

  # Jackson 配置项
  jackson:
    serialization:
      write-dates-as-timestamps: true # 设置 Date 的格式，使用时间戳
      write-date-timestamps-as-nanoseconds: false # 设置不使用 nanoseconds 的格式
      write-durations-as-timestamps: true # 设置 Duration 的格式，使用时间戳
      fail-on-empty-beans: false # 允许序列化无属性的 Bean

  # Cache 配置项
  cache:
    type: REDIS
    redis:
      time-to-live: 1h # 设置过期时间为 1 小时

server:
  servlet:
    encoding:
      enabled: true
      charset: UTF-8 # 必须设置 UTF-8，避免 WebFlux 流式返回（AI 场景）会乱码问题
      force: true
```

逐项说明：

| 配置项 | 值 | 为什么 |
|--------|-----|--------|
| `allow-circular-references` | `true` | 三层架构中，模块间存在循环依赖（如 system 和 infra 互相引用），必须允许 |
| `write-dates-as-timestamps` | `true` | 前后端约定用时间戳传递日期，避免时区问题 |
| `cache.type` | `REDIS` | 统一使用 Redis 作为缓存后端 |
| `charset` | `UTF-8` | WebFlux 流式返回（如 AI 场景）必须指定 UTF-8，否则中文乱码 |

### 2.2 Spring Cloud 禁用配置区

```yaml
--- #################### Spring Cloud 禁用配置 ####################

spring:
  cloud:
    nacos:
      discovery:
        enabled: false # 禁用 Nacos 的注册发现
      config:
        enabled: false # 禁用 Nacos 的配置中心
```

**为什么放在公共配置（application.yaml）而不是环境配置（application-local.yaml）中？**

因为默认就应该禁用 Nacos。只有在微服务模式下才需要启用，那时通过 Nacos 配置中心或环境变量来覆盖。这样做更安全——万一忘记配置 Profile，不会因为尝试连接不存在的 Nacos 而启动失败。

### 2.3 接口文档配置区

```yaml
--- #################### 接口文档配置 ####################

springdoc:
  api-docs:
    enabled: true
    path: /v3/api-docs
  swagger-ui:
    enabled: true
    path: /swagger-ui
  default-flat-param-object: true # 参见 Knife4j 文档

knife4j:
  enable: true
  setting:
    language: zh_cn
```

启动后访问 `http://127.0.0.1:48080/doc.html` 即可看到 Knife4j 增强版 Swagger UI。所有 Controller 中用 `@Operation`、`@Tag` 等注解标注的接口都会自动出现在文档中。

### 2.4 MyBatis-Plus 配置区

```yaml
--- #################### MyBatis Plus 的配置项 ####################

mybatis-plus:
  configuration:
    map-underscore-to-camel-case: true # 下划线转驼峰
  global-config:
    db-config:
      id-type: NONE # "智能"模式，基于 IdTypeEnvironmentPostProcessor 自动适配
      logic-delete-value: 1 # 逻辑已删除值
      logic-not-delete-value: 0 # 逻辑未删除值
    banner: false # 关闭控制台的 Banner 打印
  type-aliases-package: ${yudao.info.base-package}.module.*.dal.dataobject
  encryptor:
    password: XDV71a+xqStEA3WH # 加解密的秘钥
```

#### id-type: NONE 的智能切换机制

`id-type: NONE` 是一个哨兵值，不是"不设置 ID"。`IdTypeEnvironmentPostProcessor` 在启动时根据 JDBC URL 自动识别数据库类型：

```java
// 来源：yudao-framework/yudao-spring-boot-starter-mybatis/src/main/java/cn/iocoder/yudao/framework/mybatis/config/IdTypeEnvironmentPostProcessor.java
public class IdTypeEnvironmentPostProcessor implements EnvironmentPostProcessor {

    private static final Set<DbType> INPUT_ID_TYPES = SetUtils.asSet(
        DbType.ORACLE, DbType.ORACLE_12C, DbType.POSTGRE_SQL,
        DbType.KINGBASE_ES, DbType.DB2, DbType.H2);

    @Override
    public void postProcessEnvironment(ConfigurableEnvironment environment, SpringApplication application) {
        DbType dbType = getDbType(environment);  // 从 JDBC URL 解析数据库类型
        if (dbType == null) {
            return;
        }
        IdType idType = getIdType(environment);
        if (idType != IdType.NONE) {
            return;  // 用户已显式配置，不覆盖
        }
        // 情况一，用户输入 ID，适合 Oracle、PostgreSQL、Kingbase、DB2、H2 数据库
        if (INPUT_ID_TYPES.contains(dbType)) {
            setIdType(environment, IdType.INPUT);
            return;
        }
        // 情况二，自增 ID，适合 MySQL、DM 达梦等直接自增的数据库
        setIdType(environment, IdType.AUTO);
    }
}
```

这个处理器通过 `spring.factories` 注册：

```properties
# 来源：yudao-framework/yudao-spring-boot-starter-mybatis/src/main/resources/META-INF/spring.factories
org.springframework.boot.env.EnvironmentPostProcessor=\
  cn.iocoder.yudao.framework.mybatis.config.IdTypeEnvironmentPostProcessor
```

这意味着：**同一份 `application.yaml` 不需要为不同数据库做任何修改**。从 MySQL 切换到 PostgreSQL，只需要改 JDBC URL，ID 策略自动适配。

#### type-aliases-package 的通配符

```yaml
type-aliases-package: ${yudao.info.base-package}.module.*.dal.dataobject
```

通配符 `module.*` 覆盖所有业务模块的 `dal.dataobject` 包。这样 MyBatis-Plus 在解析 Mapper XML 时，可以直接用类名（如 `DeptDO`）而不是全限定名。

### 2.5 芋道自定义配置区

```yaml
--- #################### 芋道相关配置 ####################

yudao:
  info:
    version: 1.0.0
    base-package: cn.iocoder.yudao
  web:
    admin-ui:
      url: http://dashboard.yudao.iocoder.cn # Admin 管理后台 UI 的地址
  xss:
    enable: false
  security:
    permit-all_urls:
      - /admin-api/mp/open/** # 微信公众号开放平台，不需要登录
  api-encrypt:
    enable: true # 是否开启 API 加密
    algorithm: AES
    request-key: 52549111389893486934626385991395
    response-key: 96103715984234343991809655248883
  tenant:
    enable: true
    ignore-urls:
      - /jmreport/* # 积木报表，无法携带租户编号
    ignore-visit-urls:
      - /admin-api/system/user/profile/**
      - /admin-api/system/auth/**
    ignore-caches:
      - user_role_ids
      - permission_menu_ids
      - oauth_client
  swagger:
    title: 芋道快速开发平台
    description: 提供管理后台、用户 App 的所有功能
    version: ${yudao.info.version}
    url: ${yudao.web.admin-ui.url}
```

几个关键配置：

| 配置键 | 默认值 | 作用 |
|--------|--------|------|
| `yudao.info.base-package` | `cn.iocoder.yudao` | 组件扫描基准包，被 `scanBasePackages` 和 `type-aliases-package` 引用 |
| `yudao.tenant.enable` | `true` | 多租户开关，开启后 MyBatis-Plus 自动在 SQL 中追加 `WHERE tenant_id = ?` |
| `yudao.api-encrypt.enable` | `true` | API 加密开关，前后端传输数据加密 |

## 3. application-local.yaml 详解

本地开发环境的配置，遵循一个原则：**降低一切阻碍开发效率的门槛**。

### 3.1 服务端口

```yaml
server:
  port: 48080
```

### 3.2 数据源配置

```yaml
--- #################### 数据库相关配置 ####################

spring:
  autoconfigure:
    exclude:
      - com.alibaba.druid.spring.boot.autoconfigure.DruidDataSourceAutoConfigure # 排除 Druid 自动配置

  datasource:
    druid: # Druid 监控相关
      web-stat-filter:
        enabled: true
      stat-view-servlet:
        enabled: true
        url-pattern: /druid/*
      filter:
        stat:
          enabled: true
          log-slow-sql: true # 慢 SQL 记录
          slow-sql-millis: 100

    dynamic: # 多数据源配置
      druid: # Druid 连接池相关
        initial-size: 1 # 初始连接数（本地环境用小值）
        min-idle: 1
        max-active: 20
        max-wait: 60000
      primary: master
      datasource:
        master:
          url: jdbc:mysql://127.0.0.1:3306/ruoyi-vue-pro?useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true&nullCatalogMeansCurrent=true&rewriteBatchedStatements=true
          username: root
          password: 123456
        slave: # 模拟从库
          lazy: true # 开启懒加载，保证启动速度
          url: jdbc:mysql://127.0.0.1:3306/ruoyi-vue-pro?...
          username: root
          password: 123456

  redis:
    host: 127.0.0.1
    port: 6379
    database: 0
```

几个设计要点：

**为什么排除 DruidDataSourceAutoConfigure？** 因为使用了 `dynamic-datasource-spring-boot-starter` 来管理多数据源，它有自己的 Druid 配置方式。如果两个自动配置同时加载，会冲突。

**slave.lazy: true 是什么？** 本地开发时，slave 数据源在第一次使用时才初始化连接，而不是启动时就建立。这加快了启动速度。

### 3.3 定时任务配置

```yaml
--- #################### 定时任务相关配置 ####################

xxl:
  job:
    enabled: false # 本地环境默认关闭
    admin:
      addresses: http://127.0.0.1:9090/xxl-job-admin
```

### 3.4 本地开发特殊配置

```yaml
--- #################### 芋道相关配置 ####################

yudao:
  captcha:
    enable: false # 关闭验证码，方便登录等接口的测试
  security:
    mock-enable: true # 开启 mock 登录
  demo: false # 关闭演示模式
```

| 配置项 | 生产值 | 本地值 | 原因 |
|--------|--------|--------|------|
| `yudao.captcha.enable` | true | false | 每次测试登录都要输验证码太烦 |
| `yudao.security.mock-enable` | false | true | 可以用 mock 用户快速测试 |
| `yudao.demo` | false | false | 本地可以执行写操作 |

## 4. application-dev.yaml 详解

开发服务器环境的配置，与 local 的关键区别：

```yaml
# 来源：yudao-server/src/main/resources/application-dev.yaml

server:
  port: 48080

spring:
  datasource:
    dynamic:
      primary: master
      datasource:
        master:
          url: jdbc:mysql://127.0.0.1:3306/ruoyi-vue-pro?... # 可以替换成远程地址
          username: root
          password: 123456
        slave:
          lazy: true
          url: jdbc:mysql://127.0.0.1:3306/ruoyi-vue-pro?...
          username: root
          password: 123456

  redis:
    host: 400-infra.server.iocoder.cn # 远程 Redis
    port: 6379
    database: 1 # 使用不同的数据库索引，避免和本地冲突
```

### local 与 dev 的对比

| 维度 | local | dev |
|------|-------|-----|
| 数据库 | 本机 127.0.0.1 | 可以是远程服务器 |
| Redis | 本机 127.0.0.1, database=0 | 远程服务器, database=1 |
| Nacos | 禁用 | 可启用 |
| 验证码 | 关闭 | 开启 |
| mock 登录 | 开启 | 关闭 |
| 使用场景 | 开发者自己的电脑 | 团队共享的开发/测试服务器 |

## 5. 配置的优先级与覆盖规则

Spring Boot 的配置优先级从高到低：

```
1. 命令行参数          --server.port=9090
2. 环境变量            SERVER_PORT=9090
3. application-{profile}.yaml   ← 环境配置
4. application.yaml              ← 公共配置
5. @PropertySource 默认值
```

实际效果举例：

```yaml
# application.yaml 中
yudao:
  captcha:
    enable: true       # 默认开启验证码

# application-local.yaml 中
yudao:
  captcha:
    enable: false      # 本地开发关闭验证码
```

当 `spring.profiles.active=local` 时，最终值为 `false`（环境配置覆盖公共配置）。

## 6. 如何新增一个环境

假设要新增"生产环境"（prod），只需两步：

**第一步**：创建 `application-prod.yaml`

```yaml
# application-prod.yaml

server:
  port: 48080

spring:
  datasource:
    dynamic:
      primary: master
      datasource:
        master:
          url: jdbc:mysql://生产数据库地址:3306/ruoyi-vue-pro?...
          username: ${DB_USERNAME}    # 从环境变量读取
          password: ${DB_PASSWORD}    # 从环境变量读取
  redis:
    host: ${REDIS_HOST}              # 从环境变量读取
    port: 6379
    password: ${REDIS_PASSWORD}      # 生产环境必须设密码

yudao:
  captcha:
    enable: true                     # 生产环境开启验证码
  security:
    mock-enable: false               # 生产环境禁止 mock
  demo: false
```

**第二步**：修改 Profile

```yaml
# application.yaml
spring:
  profiles:
    active: prod
```

或通过命令行：

```bash
java -jar yudao-server.jar --spring.profiles.active=prod
```

生产环境配置的关键原则：**敏感信息（数据库密码、Redis 密码等）不写在配置文件中**，而是通过环境变量或配置中心注入。

## 本章小结

| 文件 | 职责 | 典型内容 |
|------|------|---------|
| `application.yaml` | 公共配置 | 应用名、Nacos 开关、MyBatis-Plus、Swagger、yudao.* 自定义配置 |
| `application-local.yaml` | 本地开发 | 本机数据库/Redis、关闭验证码、开启 mock |
| `application-dev.yaml` | 开发服务器 | 远程数据库/Redis、开启验证码 |

配置体系的核心设计思想是：**公共配置只写一次，环境差异通过 Profile 隔离，敏感信息通过环境变量注入**。
