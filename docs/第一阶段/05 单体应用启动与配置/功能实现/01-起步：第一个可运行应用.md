# 起步：第一个可运行应用

> 本章是"单体应用启动与配置"功能实现的第一篇。在功能设计中，我们分析了 yudao-server 作为"空壳容器"的设计动机——它本身不包含业务逻辑，而是通过组件扫描、Maven 依赖和配置体系，把散落在各模块中的组件组装成一个可运行的应用。
>
> 本章的目标是：**从零搭建一个最小可运行的 yudao-server**，跑起来后能访问 Swagger 文档。

## 1. 创建 yudao-server 模块

yudao-server 是整个项目的启动入口模块。它遵循"空壳容器"模式——自己不写业务代码，只做三件事：

1. 提供 `main` 方法启动 Spring Boot
2. 通过 `pom.xml` 声明要包含哪些业务模块
3. 提供配置文件

### 1.1 模块的 pom.xml

先看完整的 `pom.xml`，它是模块组装的核心：

```xml
<!-- 来源：yudao-server/pom.xml -->
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <parent>
        <groupId>cn.iocoder.cloud</groupId>
        <artifactId>yudao</artifactId>
        <version>${revision}</version>
    </parent>
    <modelVersion>4.0.0</modelVersion>

    <artifactId>yudao-server</artifactId>
    <packaging>jar</packaging>

    <name>${project.artifactId}</name>
    <description>
        后端 Server 的主项目，通过引入需要 yudao-module-xxx 的依赖，
        从而实现提供 RESTful API 给 yudao-ui-admin、yudao-ui-user 等前端项目。
        本质上来说，它就是个空壳（容器）！
    </description>

    <dependencies>
        <!-- 系统管理模块（默认启用） -->
        <dependency>
            <groupId>cn.iocoder.cloud</groupId>
            <artifactId>yudao-module-system-server</artifactId>
            <version>${revision}</version>
        </dependency>

        <!-- 基础设施模块（默认启用） -->
        <dependency>
            <groupId>cn.iocoder.cloud</groupId>
            <artifactId>yudao-module-infra-server</artifactId>
            <version>${revision}</version>
        </dependency>

        <!-- 其他业务模块默认注释，按需开启 -->
        <!-- 会员中心。默认注释，保证编译速度 -->
        <!-- <dependency>
            <groupId>cn.iocoder.cloud</groupId>
            <artifactId>yudao-module-member-server</artifactId>
            <version>${revision}</version>
        </dependency> -->

        <!-- 商城相关模块。默认注释，保证编译速度 -->
        <!-- <dependency>
            <groupId>cn.iocoder.cloud</groupId>
            <artifactId>yudao-module-product-server</artifactId>
            <version>${revision}</version>
        </dependency> -->

        <!-- spring boot 配置所需依赖 -->
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-configuration-processor</artifactId>
            <optional>true</optional>
        </dependency>

        <!-- 服务保障相关 -->
        <dependency>
            <groupId>cn.iocoder.cloud</groupId>
            <artifactId>yudao-spring-boot-starter-protection</artifactId>
        </dependency>

        <!-- Registry 注册中心相关 -->
        <dependency>
            <groupId>com.alibaba.cloud</groupId>
            <artifactId>spring-cloud-starter-alibaba-nacos-discovery</artifactId>
        </dependency>

        <!-- Config 配置中心相关 -->
        <dependency>
            <groupId>com.alibaba.cloud</groupId>
            <artifactId>spring-cloud-starter-alibaba-nacos-config</artifactId>
        </dependency>

        <!-- RPC 远程调用相关 -->
        <dependency>
            <groupId>cn.iocoder.cloud</groupId>
            <artifactId>yudao-spring-boot-starter-rpc</artifactId>
            <!-- 目的：yudao-server 单体启动，禁用 openfeign -->
            <exclusions>
                <exclusion>
                    <groupId>org.springframework.cloud</groupId>
                    <artifactId>spring-cloud-starter-openfeign</artifactId>
                </exclusion>
            </exclusions>
        </dependency>
    </dependencies>

    <build>
        <finalName>${project.artifactId}</finalName>
        <plugins>
            <plugin>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-maven-plugin</artifactId>
                <version>${spring.boot.version}</version>
                <executions>
                    <execution>
                        <goals>
                            <goal>repackage</goal>
                        </goals>
                    </execution>
                </executions>
            </plugin>
        </plugins>
    </build>
</project>
```

这段配置中有三个关键设计，我们在后面的章节会逐一讲解，这里先有个印象：

- **业务模块依赖**：通过声明 `yudao-module-system-server` 和 `yudao-module-infra-server` 两个依赖，把系统管理和基础设施模块引入
- **OpenFeign 排除**：`spring-cloud-starter-openfeign` 被排除，确保单体模式下不会尝试连接 Nacos
- **注释的依赖**：其他业务模块被注释掉，需要时取消注释即可开启

## 2. 编写启动类

### 2.1 最小启动类

```java
// 来源：yudao-server/src/main/java/cn/iocoder/yudao/server/YudaoServerApplication.java
package cn.iocoder.yudao.server;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SuppressWarnings("SpringComponentScan") // 忽略 IDEA 无法识别 ${yudao.info.base-package}
@SpringBootApplication(scanBasePackages = {"${yudao.info.base-package}.server", "${yudao.info.base-package}.module"})
public class YudaoServerApplication {

    public static void main(String[] args) {
        SpringApplication.run(YudaoServerApplication.class, args);
    }

}
```

看起来和普通的 Spring Boot 启动类没什么区别，但有一个关键细节：

```java
scanBasePackages = {"${yudao.info.base-package}.server", "${yudao.info.base-package}.module"}
```

### 2.2 scanBasePackages 解析

`${yudao.info.base-package}` 是一个配置属性占位符，它在 `application.yaml` 中定义：

```yaml
yudao:
  info:
    base-package: cn.iocoder.yudao
```

Spring Boot 启动时会解析这个占位符，最终生效的扫描路径是：

```
cn.iocoder.yudao.server   → 扫描 yudao-server 自身的 Bean（如 DefaultController）
cn.iocoder.yudao.module   → 扫描所有业务模块的 Bean（system、infra、product...）
```

**为什么用配置属性而不是硬编码？** 当有人需要把包名从 `cn.iocoder.yudao` 改成 `com.mycompany.myproject` 时，只需要修改 `yudao.info.base-package` 这一处配置，不需要改启动类代码。

### 2.3 @SuppressWarnings("SpringComponentScan")

IDEA 无法识别 `${yudao.info.base-package}` 这种属性占位符（它在编译期无法解析），会报一个"无法解析包名"的警告。这个注解只是告诉 IDEA "我知道我在做什么，别报警告了"，不影响实际运行。

## 3. 编写基础配置文件

yudao-server 的配置采用分层设计，先写公共配置。

### 3.1 application.yaml（公共配置）

这是所有环境共享的配置文件。我们先只保留最核心的部分：

```yaml
# 来源：yudao-server/src/main/resources/application.yaml

spring:
  application:
    name: yudao-server          # 应用名称

  profiles:
    active: local               # 默认激活 local Profile

  main:
    allow-circular-references: true  # 允许循环依赖（三层架构无法避免）

  # Jackson 配置项
  jackson:
    serialization:
      write-dates-as-timestamps: true  # Date 使用时间戳格式
      write-date-timestamps-as-nanoseconds: false
      write-durations-as-timestamps: true
      fail-on-empty-beans: false

--- #################### Spring Cloud 禁用配置 ####################

spring:
  cloud:
    nacos:
      discovery:
        enabled: false    # 单体模式下禁用 Nacos 注册发现
      config:
        enabled: false    # 单体模式下禁用 Nacos 配置中心

--- #################### 接口文档配置 ####################

springdoc:
  api-docs:
    enabled: true
    path: /v3/api-docs
  swagger-ui:
    enabled: true
    path: /swagger-ui
  default-flat-param-object: true

knife4j:
  enable: true
  setting:
    language: zh_cn

--- #################### MyBatis Plus 配置 ####################

mybatis-plus:
  configuration:
    map-underscore-to-camel-case: true  # 下划线转驼峰
  global-config:
    db-config:
      id-type: NONE   # "智能"模式，由 IdTypeEnvironmentPostProcessor 自动适配
      logic-delete-value: 1
      logic-not-delete-value: 0
    banner: false
  type-aliases-package: ${yudao.info.base-package}.module.*.dal.dataobject

--- #################### 芋道相关配置 ####################

yudao:
  info:
    version: 1.0.0
    base-package: cn.iocoder.yudao   # 组件扫描基准包
```

几个关键配置的含义：

| 配置项 | 值 | 作用 |
|--------|-----|------|
| `spring.profiles.active` | `local` | 默认激活本地开发环境 |
| `spring.main.allow-circular-references` | `true` | 允许循环依赖（三层架构的模块间依赖无法完全避免） |
| `spring.cloud.nacos.discovery.enabled` | `false` | 单体模式下禁用 Nacos，避免启动时尝试连接 |
| `mybatis-plus.global-config.db-config.id-type` | `NONE` | 哨兵值，由 `IdTypeEnvironmentPostProcessor` 根据数据库类型自动设置 |
| `yudao.info.base-package` | `cn.iocoder.yudao` | 被 `scanBasePackages` 和 `type-aliases-package` 引用 |

### 3.2 application-local.yaml（本地开发环境）

```yaml
# 来源：yudao-server/src/main/resources/application-local.yaml

server:
  port: 48080    # 本地开发端口

--- #################### 数据库相关配置 ####################

spring:
  autoconfigure:
    exclude:
      - com.alibaba.druid.spring.boot.autoconfigure.DruidDataSourceAutoConfigure  # 排除 Druid 自动配置，使用 dynamic-datasource
  datasource:
    dynamic:
      primary: master
      datasource:
        master:
          url: jdbc:mysql://127.0.0.1:3306/ruoyi-vue-pro?useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true&nullCatalogMeansCurrent=true&rewriteBatchedStatements=true
          username: root
          password: 123456
  redis:
    host: 127.0.0.1
    port: 6379
    database: 0

--- #################### 芋道相关配置 ####################

yudao:
  captcha:
    enable: false      # 本地环境关闭验证码，方便测试登录
  security:
    mock-enable: true   # 开启 mock 登录
  demo: false           # 关闭演示模式
```

这个配置文件的特点是：**一切为了开发效率**。关闭验证码、开启 mock 登录、使用本机数据库。

## 4. 启动应用

### 4.1 前置条件

在启动之前，确保以下基础设施已经就绪：

1. **MySQL 数据库**：本地运行，端口 3306，已创建 `ruoyi-vue-pro` 数据库并导入 SQL
2. **Redis**：本地运行，端口 6379

### 4.2 启动步骤

1. 在 IDEA 中找到 `YudaoServerApplication.java`
2. 右键 → Run
3. 等待控制台出现类似以下日志：

```
Tomcat started on port(s): 48080 (http) with context path ''
Started YudaoServerApplication in 15.234 seconds
```

### 4.3 验证启动成功

浏览器访问 Knife4j（增强版 Swagger UI）：

```
http://127.0.0.1:48080/doc.html
```

如果能看到接口文档页面，说明应用启动成功。你可以在这里看到所有已启用模块的 API 接口。

## 5. 启动过程中的关键事件

让我们回顾一下从 `main` 方法到应用就绪，中间发生了什么：

```
1. YudaoServerApplication.main()
   │
   ▼
2. SpringApplication.run() 启动 Spring Boot
   │
   ▼
3. 加载 application.yaml（公共配置）
   │
   ▼
4. 激活 spring.profiles.active=local
   │
   ▼
5. 加载 application-local.yaml（本地配置，覆盖公共配置）
   │
   ▼
6. 解析 yudao.info.base-package = cn.iocoder.yudao
   │
   ▼
7. @SpringBootApplication 开始组件扫描
   ├─ 扫描 cn.iocoder.yudao.server.**
   │   └─ 发现 DefaultController（兜底路由）
   └─ 扫描 cn.iocoder.yudao.module.**
       ├─ 发现 system 模块的所有 Bean
       │   └─ DeptController, DeptService, DeptMapper, ...
       └─ 发现 infra 模块的所有 Bean
           └─ ConfigController, FileController, ...
   │
   ▼
8. MyBatis-Plus 初始化
   ├─ IdTypeEnvironmentPostProcessor 检测 JDBC URL
   │   └─ MySQL → 设置 id-type = AUTO
   └─ 创建 SqlSessionFactory
   │
   ▼
9. Spring 容器初始化完成
   │
   ▼
10. Tomcat 启动在 48080 端口
```

## 6. 常见启动问题

### 问题一：数据库连接失败

```
Communications link failure
```

**原因**：MySQL 未启动，或 `application-local.yaml` 中的数据库地址/端口不正确。

**解决**：确认 MySQL 运行在 127.0.0.1:3306，且 `ruoyi-vue-pro` 数据库已创建。

### 问题二：Redis 连接失败

```
Unable to connect to Redis
```

**原因**：Redis 未启动。

**解决**：启动本地 Redis 服务。

### 问题三：Bean 找不到

```
No qualifying bean of type 'DeptService'
```

**原因**：业务模块的代码不在 `scanBasePackages` 的扫描范围内。

**解决**：检查业务模块的包名是否以 `cn.iocoder.yudao.module` 开头。
