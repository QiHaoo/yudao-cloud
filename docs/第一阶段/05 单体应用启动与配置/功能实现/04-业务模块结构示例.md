# 业务模块结构示例

> 本章是"单体应用启动与配置"功能实现的第四篇。在功能设计中，我们分析了业务模块的 api/server 双子结构、各层的职责分工、以及 VO 命名规范。
>
> 本章的目标是：**以部门管理（Dept）为例，完整展示一个业务模块从 DO 到 Controller 的每一层代码**，让读者对"一个请求是如何经过各层"有直观的认识。

## 1. 部门管理模块的代码全景

部门管理的代码位于 `yudao-module-system-server` 模块中，结构如下：

```
yudao-module-system-server/src/main/java/cn/iocoder/yudao/module/system/
├── controller/admin/dept/
│   ├── DeptController.java                ← Controller 层
│   └── vo/dept/
│       ├── DeptSaveReqVO.java             ← 创建/修改请求 VO
│       ├── DeptListReqVO.java             ← 列表查询请求 VO
│       ├── DeptRespVO.java                ← 完整响应 VO
│       └── DeptSimpleRespVO.java          ← 精简响应 VO
├── service/dept/
│   ├── DeptService.java                   ← Service 接口
│   └── DeptServiceImpl.java               ← Service 实现
└── dal/
    ├── dataobject/dept/
    │   └── DeptDO.java                    ← DO 实体
    └── mysql/dept/
        └── DeptMapper.java                ← Mapper 接口
```

从底向上，我们逐层分析。

## 2. 数据层：DeptDO 实体

### 2.1 DO 类代码

```java
// 来源：yudao-module-system-server/src/main/java/cn/iocoder/yudao/module/system/dal/dataobject/dept/DeptDO.java
package cn.iocoder.yudao.module.system.dal.dataobject.dept;

import cn.iocoder.yudao.framework.common.enums.CommonStatusEnum;
import cn.iocoder.yudao.framework.tenant.core.db.TenantBaseDO;
import cn.iocoder.yudao.module.system.dal.dataobject.user.AdminUserDO;
import com.baomidou.mybatisplus.annotation.KeySequence;
import com.baomidou.mybatisplus.annotation.TableId;
import com.baomidou.mybatisplus.annotation.TableName;
import lombok.Data;
import lombok.EqualsAndHashCode;

/**
 * 部门表
 */
@TableName("system_dept")
@KeySequence("system_dept_seq") // 用于 Oracle、PostgreSQL 等数据库的主键自增
@Data
@EqualsAndHashCode(callSuper = true)
public class DeptDO extends TenantBaseDO {

    public static final Long PARENT_ID_ROOT = 0L;

    /**
     * 部门ID
     */
    @TableId
    private Long id;
    /**
     * 部门名称
     */
    private String name;
    /**
     * 父部门ID
     */
    private Long parentId;
    /**
     * 显示顺序
     */
    private Integer sort;
    /**
     * 负责人
     */
    private Long leaderUserId;
    /**
     * 联系电话
     */
    private String phone;
    /**
     * 邮箱
     */
    private String email;
    /**
     * 部门状态
     */
    private Integer status;

}
```

### 2.2 继承体系解析

`DeptDO` 的继承链：

```
DeptDO
  └── TenantBaseDO（添加 tenantId 字段，支持多租户）
        └── BaseDO（添加 createTime、updateTime、creator、updater、deleted 字段）
              └── Serializable + TransPojo
```

`BaseDO` 提供的公共字段：

```java
// 来源：yudao-framework/yudao-spring-boot-starter-mybatis/src/main/java/cn/iocoder/yudao/framework/mybatis/core/dataobject/BaseDO.java
@Data
public abstract class BaseDO implements Serializable, TransPojo {

    @TableField(fill = FieldFill.INSERT)
    private LocalDateTime createTime;      // 创建时间（自动填充）

    @TableField(fill = FieldFill.INSERT_UPDATE)
    private LocalDateTime updateTime;      // 更新时间（自动填充）

    @TableField(fill = FieldFill.INSERT, jdbcType = JdbcType.VARCHAR)
    private String creator;                // 创建者（自动填充）

    @TableField(fill = FieldFill.INSERT_UPDATE, jdbcType = JdbcType.VARCHAR)
    private String updater;                // 更新者（自动填充）

    @TableLogic
    private Boolean deleted;               // 逻辑删除标记
}
```

`TenantBaseDO` 在此基础上增加：

```java
// 来源：yudao-framework/yudao-spring-boot-starter-biz-tenant/src/main/java/cn/iocoder/yudao/framework/tenant/core/db/TenantBaseDO.java
@Data
@EqualsAndHashCode(callSuper = true)
public abstract class TenantBaseDO extends BaseDO {
    private Long tenantId;  // 多租户编号
}
```

所以 `DeptDO` 最终对应 `system_dept` 表，包含以下字段：

| 字段 | 类型 | 来源 | 说明 |
|------|------|------|------|
| `id` | Long | DeptDO | 部门ID（主键） |
| `name` | String | DeptDO | 部门名称 |
| `parent_id` | Long | DeptDO | 父部门ID |
| `sort` | Integer | DeptDO | 显示顺序 |
| `leader_user_id` | Long | DeptDO | 负责人用户ID |
| `phone` | String | DeptDO | 联系电话 |
| `email` | String | DeptDO | 邮箱 |
| `status` | Integer | DeptDO | 状态（启用/禁用） |
| `tenant_id` | Long | TenantBaseDO | 租户编号 |
| `create_time` | LocalDateTime | BaseDO | 创建时间 |
| `update_time` | LocalDateTime | BaseDO | 更新时间 |
| `creator` | String | BaseDO | 创建者 |
| `updater` | String | BaseDO | 更新者 |
| `deleted` | Boolean | BaseDO | 逻辑删除标记 |

### 2.3 关键注解说明

- `@TableName("system_dept")`：映射到 `system_dept` 表
- `@TableId`：标记主键字段
- `@KeySequence("system_dept_seq")`：用于 Oracle、PostgreSQL 等数据库的主键序列。MySQL 不需要这个，但加上也无害
- `@TableLogic`（在 BaseDO 中）：逻辑删除标记，删除时不会真正 DELETE，而是 UPDATE deleted=1

## 3. 数据访问层：DeptMapper

### 3.1 Mapper 接口代码

```java
// 来源：yudao-module-system-server/src/main/java/cn/iocoder/yudao/module/system/dal/mysql/dept/DeptMapper.java
package cn.iocoder.yudao.module.system.dal.mysql.dept;

import cn.iocoder.yudao.framework.mybatis.core.mapper.BaseMapperX;
import cn.iocoder.yudao.framework.mybatis.core.query.LambdaQueryWrapperX;
import cn.iocoder.yudao.module.system.controller.admin.dept.vo.dept.DeptListReqVO;
import cn.iocoder.yudao.module.system.dal.dataobject.dept.DeptDO;
import org.apache.ibatis.annotations.Mapper;

import java.util.Collection;
import java.util.List;

@Mapper
public interface DeptMapper extends BaseMapperX<DeptDO> {

    default List<DeptDO> selectList(DeptListReqVO reqVO) {
        return selectList(new LambdaQueryWrapperX<DeptDO>()
                .likeIfPresent(DeptDO::getName, reqVO.getName())
                .eqIfPresent(DeptDO::getStatus, reqVO.getStatus()));
    }

    default DeptDO selectByParentIdAndName(Long parentId, String name) {
        return selectOne(DeptDO::getParentId, parentId, DeptDO::getName, name);
    }

    default Long selectCountByParentId(Long parentId) {
        return selectCount(DeptDO::getParentId, parentId);
    }

    default List<DeptDO> selectListByParentId(Collection<Long> parentIds) {
        return selectList(DeptDO::getParentId, parentIds);
    }

    default List<DeptDO> selectListByLeaderUserId(Long id) {
        return selectList(DeptDO::getLeaderUserId, id);
    }

}
```

### 3.2 关键设计解析

**继承 `BaseMapperX` 而非原生 `BaseMapper`**

`BaseMapperX` 是项目对 MyBatis-Plus 的增强封装，提供了：

- `selectPage()`：分页查询，返回 `PageResult<T>`
- `selectOne(field, value)`：按单字段查询
- `selectOne(field1, value1, field2, value2)`：按多字段查询
- `selectList(field, value)`：按字段查询列表
- `insertBatch()`：批量插入
- `updateBatch()`：批量更新

这些方法避免了每次都要手动创建 `LambdaQueryWrapper` 的繁琐。

**`default` 方法直接实现查询逻辑**

```java
default List<DeptDO> selectList(DeptListReqVO reqVO) {
    return selectList(new LambdaQueryWrapperX<DeptDO>()
            .likeIfPresent(DeptDO::getName, reqVO.getName())
            .eqIfPresent(DeptDO::getStatus, reqVO.getStatus()));
}
```

这个方法直接在接口中用 `default` 方法实现，不需要 XML Mapper 文件。关键点是使用了 `LambdaQueryWrapperX` 的 `*IfPresent` 方法：

- `likeIfPresent(DeptDO::getName, reqVO.getName())`：如果 `name` 不为空，才添加 LIKE 条件
- `eqIfPresent(DeptDO::getStatus, reqVO.getStatus())`：如果 `status` 不为空，才添加等值条件

这消除了传统 MyBatis 中大量的 `<if test="name != null">` XML 判断。

### 4.2 Service 实现

```java
// 来源：yudao-module-system-server/src/main/java/cn/iocoder/yudao/module/system/service/dept/DeptServiceImpl.java
package cn.iocoder.yudao.module.system.service.dept;

import cn.hutool.core.collection.CollUtil;
import cn.hutool.core.util.ObjectUtil;
import cn.iocoder.yudao.framework.common.enums.CommonStatusEnum;
import cn.iocoder.yudao.framework.common.util.object.BeanUtils;
import cn.iocoder.yudao.framework.datapermission.core.annotation.DataPermission;
import cn.iocoder.yudao.module.system.controller.admin.dept.vo.dept.DeptListReqVO;
import cn.iocoder.yudao.module.system.controller.admin.dept.vo.dept.DeptSaveReqVO;
import cn.iocoder.yudao.module.system.dal.dataobject.dept.DeptDO;
import cn.iocoder.yudao.module.system.dal.mysql.dept.DeptMapper;
import cn.iocoder.yudao.module.system.dal.redis.RedisKeyConstants;
import com.google.common.annotations.VisibleForTesting;
import lombok.extern.slf4j.Slf4j;
import org.springframework.cache.annotation.CacheEvict;
import org.springframework.cache.annotation.Cacheable;
import org.springframework.stereotype.Service;
import org.springframework.validation.annotation.Validated;

import javax.annotation.Resource;
import java.util.*;

import static cn.iocoder.yudao.framework.common.exception.util.ServiceExceptionUtil.exception;
import static cn.iocoder.yudao.module.system.enums.ErrorCodeConstants.*;

@Service
@Validated
@Slf4j
public class DeptServiceImpl implements DeptService {

    @Resource
    private DeptMapper deptMapper;

    @Override
    @CacheEvict(cacheNames = RedisKeyConstants.DEPT_CHILDREN_ID_LIST, allEntries = true)
    public Long createDept(DeptSaveReqVO createReqVO) {
        if (createReqVO.getParentId() == null) {
            createReqVO.setParentId(DeptDO.PARENT_ID_ROOT);
        }
        // 校验父部门的有效性
        validateParentDept(null, createReqVO.getParentId());
        // 校验部门名的唯一性
        validateDeptNameUnique(null, createReqVO.getParentId(), createReqVO.getName());

        // 插入部门
        DeptDO dept = BeanUtils.toBean(createReqVO, DeptDO.class);  // VO → DO
        deptMapper.insert(dept);
        return dept.getId();
    }

    @Override
    @CacheEvict(cacheNames = RedisKeyConstants.DEPT_CHILDREN_ID_LIST, allEntries = true)
    public void updateDept(DeptSaveReqVO updateReqVO) {
        if (updateReqVO.getParentId() == null) {
            updateReqVO.setParentId(DeptDO.PARENT_ID_ROOT);
        }
        // 校验自己存在
        validateDeptExists(updateReqVO.getId());
        // 校验父部门的有效性
        validateParentDept(updateReqVO.getId(), updateReqVO.getParentId());
        // 校验部门名的唯一性
        validateDeptNameUnique(updateReqVO.getId(), updateReqVO.getParentId(), updateReqVO.getName());

        // 更新部门
        DeptDO updateObj = BeanUtils.toBean(updateReqVO, DeptDO.class);
        deptMapper.updateById(updateObj);
    }

    @Override
    @CacheEvict(cacheNames = RedisKeyConstants.DEPT_CHILDREN_ID_LIST, allEntries = true)
    public void deleteDept(Long id) {
        // 校验是否存在
        validateDeptExists(id);
        // 校验是否有子部门
        if (deptMapper.selectCountByParentId(id) > 0) {
            throw exception(DEPT_EXITS_CHILDREN);
        }
        // 删除部门
        deptMapper.deleteById(id);
    }

    @VisibleForTesting
    void validateDeptExists(Long id) {
        if (id == null) {
            return;
        }
        DeptDO dept = deptMapper.selectById(id);
        if (dept == null) {
            throw exception(DEPT_NOT_FOUND);
        }
    }

    @VisibleForTesting
    void validateParentDept(Long id, Long parentId) {
        if (parentId == null || DeptDO.PARENT_ID_ROOT.equals(parentId)) {
            return;
        }
        // 1. 不能设置自己为父部门
        if (Objects.equals(id, parentId)) {
            throw exception(DEPT_PARENT_ERROR);
        }
        // 2. 父部门不存在
        DeptDO parentDept = deptMapper.selectById(parentId);
        if (parentDept == null) {
            throw exception(DEPT_PARENT_NOT_EXITS);
        }
        // 3. 递归校验父部门，避免形成环路
        if (id == null) { // id 为空，说明新增，不需要考虑环路
            return;
        }
        for (int i = 0; i < Short.MAX_VALUE; i++) {
            parentId = parentDept.getParentId();
            if (Objects.equals(id, parentId)) {
                throw exception(DEPT_PARENT_IS_CHILD);
            }
            if (parentId == null || DeptDO.PARENT_ID_ROOT.equals(parentId)) {
                break;
            }
            parentDept = deptMapper.selectById(parentId);
            if (parentDept == null) {
                break;
            }
        }
    }

    @VisibleForTesting
    void validateDeptNameUnique(Long id, Long parentId, String name) {
        DeptDO dept = deptMapper.selectByParentIdAndName(parentId, name);
        if (dept == null) {
            return;
        }
        if (id == null) {
            throw exception(DEPT_NAME_DUPLICATE);
        }
        if (ObjectUtil.notEqual(dept.getId(), id)) {
            throw exception(DEPT_NAME_DUPLICATE);
        }
    }

    @Override
    public DeptDO getDept(Long id) {
        return deptMapper.selectById(id);
    }

    @Override
    public List<DeptDO> getDeptList(DeptListReqVO reqVO) {
        List<DeptDO> list = deptMapper.selectList(reqVO);
        list.sort(Comparator.comparing(DeptDO::getSort));
        return list;
    }

    @Override
    public List<DeptDO> getChildDeptList(Collection<Long> ids) {
        List<DeptDO> children = new LinkedList<>();
        Collection<Long> parentIds = ids;
        for (int i = 0; i < Short.MAX_VALUE; i++) {
            List<DeptDO> depts = deptMapper.selectListByParentId(parentIds);
            if (CollUtil.isEmpty(depts)) {
                break;
            }
            children.addAll(depts);
            parentIds = CollectionUtils.convertSet(depts, DeptDO::getId);
        }
        return children;
    }

    @Override
    @DataPermission(enable = false) // 禁用数据权限，避免建立不正确的缓存
    @Cacheable(cacheNames = RedisKeyConstants.DEPT_CHILDREN_ID_LIST, key = "#id")
    public Set<Long> getChildDeptIdListFromCache(Long id) {
        List<DeptDO> children = getChildDeptList(id);
        return CollectionUtils.convertSet(children, DeptDO::getId);
    }

}
```

### 4.3 Service 层的关键设计

**1. VO → DO 转换在 Service 层完成**

```java
DeptDO dept = BeanUtils.toBean(createReqVO, DeptDO.class);
```

使用 `BeanUtils.toBean()` 将请求 VO 转换为 DO，然后交给 Mapper 插入数据库。

**2. 校验逻辑使用 `ServiceExceptionUtil.exception()` 抛异常**

```java
if (dept == null) {
    throw exception(DEPT_NOT_FOUND);
}
```

`exception()` 方法创建 `ServiceException`，由全局异常处理器（`GlobalExceptionHandler`）统一转换为 `CommonResult` 返回给前端。

**3. 缓存管理**

```java
@CacheEvict(cacheNames = RedisKeyConstants.DEPT_CHILDREN_ID_LIST, allEntries = true)
public Long createDept(DeptSaveReqVO createReqVO) { ... }
```

增删改操作会清空部门缓存。查询操作使用 `@Cacheable` 缓存结果。

**4. 数据权限注解**

```java
@DataPermission(enable = false) // 禁用数据权限，避免建立不正确的缓存
@Cacheable(cacheNames = RedisKeyConstants.DEPT_CHILDREN_ID_LIST, key = "#id")
public Set<Long> getChildDeptIdListFromCache(Long id) { ... }
```

缓存方法禁用数据权限，避免不同用户查询到不同的缓存结果。

### 5.2 Controller 层的关键设计

**1. 注解分工**

| 注解 | 作用 |
|------|------|
| `@Tag(name = "管理后台 - 部门")` | Swagger 文档分组 |
| `@RestController` | 标记为 REST 控制器，返回 JSON |
| `@RequestMapping("/system/dept")` | URL 路径前缀（注意：最终路径会加上 `/admin-api` 前缀） |
| `@Validated` | 启用参数校验 |
| `@Operation(summary = "...")` | Swagger 接口描述 |
| `@PreAuthorize("@ss.hasPermission('system:dept:create')")` | 权限校验 |
| `@Valid @RequestBody` | 校验请求体参数 |

**2. 权限校验机制**

```java
@PreAuthorize("@ss.hasPermission('system:dept:create')")
```

`@ss` 是项目自定义的 SpEL Bean，`hasPermission` 方法检查当前登录用户是否拥有指定权限。这是 Spring Security 的 `@PreAuthorize` 注解的扩展用法。

**3. DO → VO 转换在 Controller 层完成**

```java
List<DeptDO> list = deptService.getDeptList(reqVO);
return success(BeanUtils.toBean(list, DeptRespVO.class));
```

Service 返回 DO，Controller 使用 `BeanUtils.toBean()` 转换为 VO 后返回给前端。

## 7. 数据在各层的形态变化

以"创建部门"为例，追踪数据从请求到响应的完整形态变化：

```
前端请求 JSON:
    {"name": "电商运营部", "parentId": 100, "sort": 1, "status": 1}
        │
        ▼ Spring MVC 反序列化 + @Valid 校验
    DeptSaveReqVO {name="电商运营部", parentId=100, sort=1, status=1}
        │
        ▼ Service 层: BeanUtils.toBean(createReqVO, DeptDO.class)
    DeptDO {name="电商运营部", parentId=100, sort=1, status=1, tenantId=...}
        │
        ▼ Mapper 层: deptMapper.insert(dept)
    SQL: INSERT INTO system_dept (name, parent_id, sort, status, tenant_id, ...)
         VALUES ('电商运营部', 100, 1, 1, 1, ...)
        │
        ▼ 数据库返回自增 ID
    dept.getId() = 1024
        │
        ▼ Controller 层: return success(deptId)
    响应 JSON: {"code": 0, "data": 1024, "msg": "success"}
```

以"查询部门列表"为例：

```
前端请求: GET /admin-api/system/dept/list?name=电商
        │
        ▼ Controller 层: deptService.getDeptList(reqVO)
    Service 返回: List<DeptDO>
        │
        ▼ Controller 层: BeanUtils.toBean(list, DeptRespVO.class)
    转换为: List<DeptRespVO>
        │
        ▼ return success(voList)
    响应 JSON: {"code": 0, "data": [{...}, {...}], "msg": "success"}
```
