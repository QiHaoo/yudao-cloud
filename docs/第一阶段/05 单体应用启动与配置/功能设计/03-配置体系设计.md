# 配置体系设计

## 从"配置地狱"说起

一个真实的电商后台系统需要配置什么？让我们粗略列举：

- 数据库连接（MySQL 地址、账号、密码、连接池参数）
- Redis 连接
- 消息队列（RocketMQ / RabbitMQ / Kafka）
- 定时任务（XXL-Job 调度中心地址）
- OAuth2 客户端信息
- 微信公众号/小程序密钥
- 支付回调地址
- 多租户开关
- API 文档（Swagger）
- 验证码配置
- ...

粗略估计有 **200+ 个配置项**。如果全部写在一个文件里，不仅难以维护，而且不同环境（本地、开发、生产）的配置混在一起，切换环境时需要手动注释/取消注释——这是事故的温床。

yudao-cloud 的配置体系通过**分层 + Profile 机制**来解决这个问题。

## 配置文件全景

yudao-server 模块下有三个配置文件：

```
yudao-server/src/main/resources/
├── application.yaml          ← 公共配置（所有环境共享）
├── application-local.yaml    ← 本地开发环境
└── application-dev.yaml      ← 开发服务器环境
```

它们的加载顺序和覆盖关系是：

```
1. 先加载 application.yaml（公共配置）
       │
2. 再加载 application-{profile}.yaml（环境配置，覆盖同名属性）
       │
3. 最终合并成一个完整的配置集

以 spring.profiles.active=local 为例：

  application.yaml        →  提供 MyBatis-Plus、Swagger、Flowable、yudao.* 等配置
  application-local.yaml  →  提供数据源、Redis、消息队列等本地连接信息
                           →  覆盖公共配置中的部分值（如 yudao.captcha.enable=false）
```

## application.yaml：公共配置层

这一层放置**所有环境共享的配置**——即不管在本地、开发还是生产环境，行为都一样的配置。

### 配置分区

文件使用 YAML 的 `---` 分隔符将配置按关注点分成多个区块：

```yaml
# 来源：yudao-server/src/main/resources/application.yaml

# ===== 区块 1：Spring 基础配置 =====
spring:
  application:
    name: yudao-server
  profiles:
    active: local
  main:
    allow-circular-references: true  # 允许循环依赖
  servlet:
    multipart:
      max-file-size: 16MB
  jackson:
    serialization:
      write-dates-as-timestamps: true
  cache:
    type: REDIS
    redis:
      time-to-live: 1h

--- # ===== 区块 2：Spring Cloud 禁用配置 =====
spring:
  cloud:
    nacos:
      discovery:
        enabled: false   # 默认禁用 Nacos 注册发现
      config:
        enabled: false   # 默认禁用 Nacos 配置中心

--- # ===== 区块 3：接口文档配置 =====
springdoc:
  api-docs:
    enabled: true
  swagger-ui:
    enabled: true
knife4j:
  enable: true
  setting:
    language: zh_cn

--- # ===== 区块 4：MyBatis Plus 配置 =====
mybatis-plus:
  configuration:
    map-underscore-to-camel-case: true
  global-config:
    db-config:
      id-type: NONE          # 智能模式，后面详解
      logic-delete-value: 1
      logic-not-delete-value: 0
  type-aliases-package: ${yudao.info.base-package}.module.*.dal.dataobject

--- # ===== 区块 5：芋道自定义配置 =====
yudao:
  info:
    version: 1.0.0
    base-package: cn.iocoder.yudao
  tenant:
    enable: true             # 多租户开关
  web:
    admin-ui:
      url: http://dashboard.yudao.iocoder.cn
```

### 为什么用 `---` 分隔而不是写多个文件？

YAML 的 `---` 分隔符允许在一个文件中定义多个"文档"，它们会被 Spring Boot 合并加载。这比拆成多个文件（如 `application-mybatis.yaml`、`application-swagger.yaml`）更好，因为：

- 所有公共配置**一目了然**，打开一个文件就能看到全局
- 不需要额外的 `spring.profiles.include` 来引入子配置
- 减少文件数量，降低配置管理的心智负担

## application-local.yaml：本地开发环境

这一层放置**只有本地开发时才需要的配置**。

### 核心配置项

```yaml
# 来源：yudao-server/src/main/resources/application-local.yaml

server:
  port: 48080    # 本地开发端口

# ===== 数据库配置 =====
spring:
  datasource:
    dynamic:
      primary: master
      datasource:
        master:
          url: jdbc:mysql://127.0.0.1:3306/ruoyi-vue-pro?...
          username: root
          password: 123456
        slave:                 # 模拟从库
          lazy: true           # 懒加载，保证启动速度
          url: jdbc:mysql://127.0.0.1:3306/ruoyi-vue-pro?...
          username: root
          password: 123456
  redis:
    host: 127.0.0.1
    port: 6379
    database: 0

# ===== 定时任务（本地默认关闭） =====
xxl:
  job:
    enabled: false

# ===== 本地开发特殊配置 =====
yudao:
  captcha:
    enable: false            # 关闭验证码，方便测试登录
  security:
    mock-enable: true        # 开启 mock 登录
  demo: false                # 关闭演示模式
```

### 本地环境的设计哲学

本地开发环境的配置遵循一个原则：**降低一切阻碍开发效率的门槛**。

| 配置项 | 生产值 | 本地值 | 原因 |
|--------|--------|--------|------|
| `yudao.captcha.enable` | true | false | 每次测试登录都要输验证码太烦 |
| `yudao.security.mock-enable` | false | true | 可以用 mock 用户快速测试 |
| `yudao.demo` | false | false | 本地可以执行写操作 |
| `xxl.job.enabled` | true | false | 本地不一定要启动 XXL-Job |
| `spring.datasource.dynamic.datasource.master.url` | 远程地址 | 127.0.0.1 | 本地 MySQL |

## application-dev.yaml：开发服务器环境

这一层放置**开发服务器（团队共享的开发环境）的配置**。

```yaml
# 来源：yudao-server/src/main/resources/application-dev.yaml

server:
  port: 48080

spring:
  datasource:
    dynamic:
      primary: master
      datasource:
        master:
          url: jdbc:mysql://127.0.0.1:3306/ruoyi-vue-pro?...  # 这里可以替换成远程地址
          username: root
          password: 123456
  redis:
    host: 400-infra.server.iocoder.cn   # 远程 Redis
    port: 6379
    database: 1

# Nacos 在 dev 环境通常启用（此处未显式覆盖，需要时可添加）
```

### local 与 dev 的关键区别

| 维度 | local | dev |
|------|-------|-----|
| 数据库 | 本机 127.0.0.1 | 可以是远程服务器 |
| Redis | 本机 127.0.0.1 | 远程服务器 |
| Nacos | 禁用 | 可启用 |
| 验证码 | 关闭 | 开启 |
| mock 登录 | 开启 | 关闭 |
| 使用场景 | 开发者自己的电脑 | 团队共享的开发/测试服务器 |

## 关键配置项深度解析

### yudao.info.base-package：组件扫描的基石

```yaml
yudao:
  info:
    base-package: cn.iocoder.yudao
```

这个配置项被启动类的 `scanBasePackages` 引用（详见第二章），也被 MyBatis-Plus 的 `type-aliases-package` 引用：

```yaml
mybatis-plus:
  type-aliases-package: ${yudao.info.base-package}.module.*.dal.dataobject
```

这行配置告诉 MyBatis-Plus 去哪里扫描 DO 实体类。使用通配符 `module.*` 可以覆盖所有业务模块的 dataobject 包。

### yudao.tenant.enable：多租户开关

```yaml
yudao:
  tenant:
    enable: true
    ignore-urls:             # 不需要租户过滤的 URL
      - /jmreport/*
    ignore-tables:           # 不需要租户过滤的表
    ignore-caches:           # 不需要租户过滤的缓存
      - user_role_ids
      - permission_menu_ids
      - oauth_client
```

当 `enable=true` 时，MyBatis-Plus 的 `TenantLineInnerInterceptor` 会自动在 SQL 中追加 `WHERE tenant_id = ?` 条件。这是一个**运行时开关**，不需要改代码就能开启/关闭多租户。

### mybatis-plus.global-config.db-config.id-type：智能 ID 策略

```yaml
mybatis-plus:
  global-config:
    db-config:
      id-type: NONE   # "智能"模式
```

这个 `NONE` 不是"不设置 ID"，而是一个**哨兵值**。实际的 ID 策略由 `IdTypeEnvironmentPostProcessor` 在启动时根据数据库类型自动决定：

```
启动时检测主数据源的 JDBC URL
    │
    ├─ MySQL / DM（达梦）     → 自动设置为 AUTO（自增 ID）
    │
    ├─ PostgreSQL / Oracle    → 自动设置为 INPUT（用户输入 ID）
    │   / Kingbase / DB2 / H2
    │
    └─ 其他数据库              → 保持 NONE
```

源码实现（简化）：

```java
// 来源：yudao-framework/yudao-spring-boot-starter-mybatis/.../IdTypeEnvironmentPostProcessor.java
public class IdTypeEnvironmentPostProcessor implements EnvironmentPostProcessor {

    private static final Set<DbType> INPUT_ID_TYPES = SetUtils.asSet(
        DbType.ORACLE, DbType.ORACLE_12C, DbType.POSTGRE_SQL,
        DbType.KINGBASE_ES, DbType.DB2, DbType.H2);

    @Override
    public void postProcessEnvironment(ConfigurableEnvironment environment, SpringApplication application) {
        DbType dbType = getDbType(environment);  // 从 JDBC URL 解析数据库类型
        IdType idType = getIdType(environment);
        if (idType != IdType.NONE) {
            return;  // 用户已显式配置，不覆盖
        }
        if (INPUT_ID_TYPES.contains(dbType)) {
            setIdType(environment, IdType.INPUT);   // Oracle/PG 等用 INPUT
        } else {
            setIdType(environment, IdType.AUTO);     // MySQL/DM 等用 AUTO
        }
    }
}
```

这个设计的巧妙之处在于：**同一份 `application.yaml` 不需要为不同数据库做任何修改**。从 MySQL 切换到 PostgreSQL，只需要改 JDBC URL，ID 策略自动适配。

### spring.datasource.dynamic：多数据源配置

```yaml
spring:
  datasource:
    dynamic:
      primary: master       # 主数据源
      datasource:
        master:             # 主库（读写）
          url: jdbc:mysql://127.0.0.1:3306/ruoyi-vue-pro?...
          username: root
          password: 123456
        slave:              # 从库（只读，模拟）
          lazy: true        # 懒加载
          url: jdbc:mysql://127.0.0.1:3306/ruoyi-vue-pro?...
          username: root
          password: 123456
```

这里使用了 `dynamic-datasource-spring-boot-starter` 实现多数据源。`primary: master` 指定默认数据源，未标注 `@DS` 注解的 Mapper 默认使用 master。

`slave.lazy: true` 是一个本地开发的优化：slave 数据源在第一次使用时才初始化，而不是启动时就建立连接，这样可以加快启动速度。

## 配置的优先级与覆盖规则

Spring Boot 的配置优先级从高到低是：

```
1. 命令行参数          --server.port=9090
2. 环境变量            SERVER_PORT=9090
3. application-{profile}.yaml   ← 环境配置
4. application.yaml              ← 公共配置
5. @PropertySource 默认值
```

这意味着 `application-local.yaml` 中的同名配置会**覆盖** `application.yaml` 中的值。例如：

```yaml
# application.yaml 中
yudao:
  captcha:
    enable: true       # 默认开启验证码

# application-local.yaml 中
yudao:
  captcha:
    enable: false      # 本地开发关闭验证码
```

最终 `local` 环境下 `yudao.captcha.enable=false`。

## 环境切换的完整流程

```
开发者想从"本地开发"切换到"连开发服务器"：
    │
    ▼
修改 application.yaml 中的 spring.profiles.active
    │
    ├─ local → 连本机 MySQL/Redis，关闭验证码，开启 mock
    │
    └─ dev   → 连远程 MySQL/Redis，开启验证码，关闭 mock

或者通过命令行覆盖（不改文件）：
    │
    ▼
java -jar yudao-server.jar --spring.profiles.active=dev
```

在微服务架构中，还可以通过 Nacos 配置中心动态修改配置，不需要重启应用。但在单体开发阶段，Profile 机制已经足够。
