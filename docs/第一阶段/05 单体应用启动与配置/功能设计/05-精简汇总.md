# 精简汇总

## 第一部分：能力与使用速查

### yudao-server 模块的核心能力

| 能力 | 实现方式 | 关键配置/代码 |
|------|---------|--------------|
| 应用启动入口 | `YudaoServerApplication.main()` | `SpringApplication.run()` |
| 组件扫描 | `scanBasePackages` 动态占位符 | `${yudao.info.base-package}.server` + `${yudao.info.base-package}.module` |
| 模块组装 | Maven pom.xml 依赖声明 | 注释/取消注释 `<dependency>` |
| 未启用模块兜底 | `DefaultController` 通配符映射 | 返回 `NOT_IMPLEMENTED` (501) |
| OpenFeign 单体禁用 | pom.xml 中排除 `spring-cloud-starter-openfeign` | `<exclusions>` |
| 环境隔离 | `spring.profiles.active` + 多 YAML 文件 | `application.yaml` / `application-local.yaml` / `application-dev.yaml` |
| 智能 ID 策略 | `IdTypeEnvironmentPostProcessor` | `id-type: NONE` → 根据数据库类型自动选择 AUTO/INPUT |
| 多数据源 | `dynamic-datasource-spring-boot-starter` | `spring.datasource.dynamic` |
| 多租户开关 | `yudao.tenant.enable` | 运行时开关，无需改代码 |
| URL 前缀注入 | `YudaoWebAutoConfiguration` | `/admin-api` → `controller.admin.**`，`/app-api` → `controller.app.**` |

### 业务模块包结构速查

```
yudao-module-xxx/
├── yudao-module-xxx-api/            ← API 模块（轻量，跨模块依赖）
│   └── cn.iocoder.yudao.module.xxx/
│       ├── api/                     ← Feign 接口 + DTO
│       │   └── {domain}/
│       │       ├── XxxApi.java      ← @FeignClient 接口
│       │       └── dto/             ← 数据传输对象
│       └── enums/                   ← 共享枚举
│           ├── ApiConstants.java    ← NAME / PREFIX / VERSION
│           └── ErrorCodeConstants.java
│
└── yudao-module-xxx-server/         ← Server 模块（完整实现）
    └── cn.iocoder.yudao.module.xxx/
        ├── controller/
        │   ├── admin/{domain}/      ← 管理后台 API
        │   │   ├── XxxController.java
        │   │   └── vo/
        │   │       ├── XxxSaveReqVO.java     ← 创建/修改请求
        │   │       ├── XxxPageReqVO.java     ← 分页查询请求
        │   │       ├── XxxRespVO.java        ← 完整响应
        │   │       └── XxxSimpleRespVO.java  ← 精简响应
        │   └── app/{domain}/        ← 用户端 API
        ├── service/{domain}/
        │   ├── XxxService.java      ← 接口
        │   └── XxxServiceImpl.java  ← 实现
        ├── dal/
        │   ├── dataobject/{domain}/ ← DO 实体
        │   ├── mysql/{domain}/      ← Mapper 接口
        │   └── redis/               ← Redis DAO
        └── convert/{domain}/        ← MapStruct 转换器
```

### VO 命名速查

| VO 类型 | 命名模式 | 使用场景 |
|---------|---------|---------|
| 保存请求 | `*SaveReqVO` | POST create / PUT update 共用 |
| 分页查询 | `*PageReqVO` | GET page（带 pageNo、pageSize） |
| 列表查询 | `*ListReqVO` | GET list（不带分页） |
| 完整响应 | `*RespVO` | 详情查询，包含所有字段 |
| 精简响应 | `*SimpleRespVO` | 下拉选项，只含 id + name |

## 第二部分：设计与概念精要

### 核心设计决策

**1. "空壳容器"模式**

yudao-server 本身不包含任何业务逻辑，它是一个纯粹的组装层。所有功能来自被引入的 Maven 依赖。这让同一个代码库可以通过不同的依赖组合，产出不同的应用。

**2. 包扫描的动态化**

`scanBasePackages = {"${yudao.info.base-package}.server", "${yudao.info.base-package}.module"}` 使用配置属性占位符而非硬编码包名，支持"一键改包"。两个扫描路径的分工：
- `.server`：扫描 yudao-server 自身的 Bean（如 DefaultController）
- `.module`：扫描所有业务模块的 Bean

**3. Maven 依赖即模块开关**

通过注释/取消注释 pom.xml 中的 `<dependency>` 来控制模块包含。这比 Maven Profile 更直观，适合学习项目。代价是需要手动编辑 XML 文件。

**4. DefaultController 的优雅降级**

为未启用的模块提供通配符路由兜底，返回 501 而非 404。当模块启用后，Controller 中的精确路径自动覆盖通配符路径（Spring MVC 的匹配规则）。

**5. OpenFeign 的单体排除**

在 yudao-server 的 pom.xml 中排除 `spring-cloud-starter-openfeign`，让 `@FeignClient` 注解变成无害标记。单体模式下，API 接口由本地 `*ApiImpl` Bean 提供实现。

**6. 配置分层**

```
application.yaml        → 公共配置（所有环境共享）
application-local.yaml  → 本地开发（本机数据库、关闭验证码）
application-dev.yaml    → 开发服务器（远程数据库、开启 Nacos）
```

Profile 机制保证环境隔离，公共配置只写一次。

**7. id-type: NONE 的智能切换**

`IdTypeEnvironmentPostProcessor` 在启动早期根据 JDBC URL 自动识别数据库类型，将 `NONE` 转换为 `AUTO`（MySQL）或 `INPUT`（PostgreSQL/Oracle）。同一份配置适配多种数据库。

**8. api/server 双子模块**

每个业务模块拆分为 api（轻量契约）和 server（完整实现）。其他模块只依赖 api，避免传递依赖膨胀和循环依赖。

### 关键配置属性一览

| 配置键 | 默认值 | 作用 |
|--------|--------|------|
| `yudao.info.base-package` | `cn.iocoder.yudao` | 组件扫描基准包 |
| `yudao.tenant.enable` | `true` | 多租户开关 |
| `yudao.captcha.enable` | `true` | 验证码开关 |
| `yudao.security.mock-enable` | `false` | Mock 登录开关 |
| `yudao.demo` | `false` | 演示模式（禁止写操作） |
| `mybatis-plus.global-config.db-config.id-type` | `NONE` | ID 策略（智能切换） |
| `spring.datasource.dynamic.primary` | `master` | 默认数据源 |
| `spring.profiles.active` | `local` | 激活的 Profile |
| `spring.cloud.nacos.discovery.enabled` | `false` | Nacos 注册发现开关 |

### 与前序组件的关系

```
01 Maven 多模块结构 ──→ yudao-server 的 pom.xml 依赖结构
02 共享基础类型与工具 ──→ CommonResult、ErrorCode、BeanUtils 在各层使用
03 数据访问增强 ──────→ BaseMapperX、LambdaQueryWrapperX 在 Mapper 层使用
04 Web 层基础设施 ────→ GlobalExceptionHandler、URL 前缀注入在启动时加载
05 单体应用启动与配置 ← 汇集以上所有组件，启动为可运行的应用
```

### 关键源码文件

| 文件 | 路径 | 核心作用 |
|------|------|---------|
| 启动类 | `yudao-server/.../YudaoServerApplication.java` | 组件扫描 + 应用启动 |
| 兜底控制器 | `yudao-server/.../controller/DefaultController.java` | 未启用模块的友好提示 |
| 公共配置 | `yudao-server/src/main/resources/application.yaml` | 所有环境共享的配置 |
| 本地配置 | `yudao-server/src/main/resources/application-local.yaml` | 本地开发环境配置 |
| 开发配置 | `yudao-server/src/main/resources/application-dev.yaml` | 开发服务器环境配置 |
| 模块组装 | `yudao-server/pom.xml` | 声明要包含的业务模块 |
| ID 策略切换 | `yudao-framework/.../IdTypeEnvironmentPostProcessor.java` | 根据数据库类型自动设置 ID 策略 |
| URL 前缀注入 | `yudao-framework/.../YudaoWebAutoConfiguration.java` | /admin-api、/app-api 前缀 |
| 前缀配置 | `yudao-framework/.../WebProperties.java` | 前缀与包路径的映射规则 |
