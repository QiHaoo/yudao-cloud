# 共享基础类型与工具：精简汇总

---

## 第一部分：能力与使用速查

### 核心类型

| 类型 | 用途 | 关键方法 |
|------|------|---------|
| `CommonResult<T>` | 统一 API 响应封装 | `success(data)`, `error(errorCode)`, `checkError()`, `getCheckedData()` |
| `PageParam` | 分页请求参数 | `pageNo`(默认1), `pageSize`(默认10,上限200), `PAGE_SIZE_NONE=-1` |
| `PageResult<T>` | 分页响应结果 | `total`, `list` |
| `SortablePageParam` | 带排序的分页请求 | 继承 `PageParam` + `List<SortingField>` |
| `ErrorCode` | 错误码值对象 | `code`(整数), `msg`(消息模板) |
| `ServiceException` | 业务异常 | 携带 `ErrorCode`，被 `GlobalExceptionHandler` 捕获 |
| `ServerException` | 系统异常 | 携带 `ErrorCode`，记录完整堆栈 |

### 错误码分配

| 区间 | 归属 | 示例 |
|------|------|------|
| `[0, 999]` | 全局错误码（`GlobalErrorCodeConstants`） | 0=成功, 401=未登录, 500=系统异常 |
| `[1_000_000_000, ...)` | 业务错误码，按模块分配 | `100200001`=系统模块-用户子模块-用户不存在 |

### 工具类

| 工具类 | 核心能力 |
|--------|---------|
| `BeanUtils` | 对象转换（单对象/列表/PageResult），基于 Hutool |
| `CollectionUtils` | 集合转换（convertList/convertSet/convertMap）、diffList 新旧比较 |
| `LocalDateTimeUtils` | 日期解析、范围判断、时间桶生成 |
| `JsonUtils` | Jackson JSON 序列化，LocalDateTime→时间戳，Long→String 安全序列化 |
| `MoneyUtils` | 分转元、百分比计算、金额乘法 |
| `HttpUtils` | URL 编解码、HTTP 请求、Basic Auth |
| `ServletUtils` | 获取客户端 IP/UserAgent、读请求体、写 JSON 响应 |
| `CacheUtils` | Guava LoadingCache 构建 |

### 自定义校验注解

| 注解 | 用途 |
|------|------|
| `@InEnum(CommonStatusEnum.class)` | 校验值在枚举合法范围内 |
| `@Mobile` | 校验中国手机号 |
| `@Telephone` | 校验电话号码 |

### CommonApi 接口（框架级 RPC 契约）

| 接口 | 服务提供方 | 用途 |
|------|-----------|------|
| `OAuth2TokenCommonApi` | system-server | Token 创建/校验/刷新/删除 |
| `PermissionCommonApi` | system-server | 权限/角色校验 |
| `TenantCommonApi` | system-server | 租户查询 |
| `DictDataCommonApi` | system-server | 字典数据查询 |
| `OperateLogCommonApi` | system-server | 操作日志创建 |
| `ApiAccessLogCommonApi` | infra-server | 访问日志创建 |
| `ApiErrorLogCommonApi` | infra-server | 错误日志创建 |

## 第二部分：设计与概念精要

### 统一响应与异常

- **CommonResult 是唯一出口**：所有 API 返回 `CommonResult<T>`，`code=0` 表示成功，非 0 表示失败
- **ErrorCode 是错误的载体**：10 位数字编码（`T-SSS-MMM-EEE`），全局码 `[0,999]`，业务码按模块分配
- **ServiceException vs ServerException**：业务异常（用户操作导致）不打堆栈返回明确消息，系统异常（代码/基础设施故障）打完整堆栈返回"系统异常"
- **checkError/getCheckedData 桥接模式**：在 Feign 调用后，一行代码完成"检查远程错误 + 取值"

### 分页

- **PageParam 有防呆设计**：pageNo 从 1 开始，pageSize 上限 200，`PAGE_SIZE_NONE=-1` 表示不分页
- **SortablePageParam 支持多字段排序**：`sortingFields` 列表，每个元素包含 `field` 和 `order`

### 工具类哲学

- **薄封装**：基于 Hutool 做项目级封装，增加 null 安全、PageResult 感知、lambda 字段解析
- **provided scope**：大部分运行时依赖（Spring、Jackson、Guava）使用 `provided` scope，不强制传递
- **BeanUtils 适合简单场景**：同名属性直接映射。复杂映射用 MapStruct + `default` 方法

### 跨模块 RPC 契约

- **两级 API 体系**：CommonApi（框架级，放在 yudao-common）+ 业务 Api（业务级，放在模块 api 子模块）
- **CommonApi 打破循环依赖**：框架 Starter 通过 CommonApi 接口调用业务功能，依赖方向始终单向
- **API 模块轻量化**：依赖 `optional`，不传递 OpenFeign 等消费者不需要的依赖
- **/rpc-api/ 前缀**：所有 RPC 接口统一使用 `/rpc-api/{module}/...` 路径
