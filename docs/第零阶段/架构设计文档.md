## 1. 项目概览

yudao-cloud（ruoyi-vue-pro）是一个基于 Spring Boot + Spring Cloud 的 Java 微服务快速开发平台。

| 属性 | 值 |
|------|-----|
| groupId | `cn.iocoder.cloud` |
| 版本 | `2026.04-jdk8-SNAPSHOT`（通过 `${revision}` + flatten-maven-plugin 统一管理） |
| Java 版本 | JDK 8（同时维护 `master-jdk17` 分支） |
| 构建工具 | Maven（多模块，72 个 pom.xml） |
| 许可证 | MIT |

### 版本分支矩阵

| | JDK 8 + Spring Boot 2.7 | JDK 17/21 + Spring Boot 3.2 |
|---|---|---|
| 完整版（yudao-cloud） | `master` | `master-jdk17` |
| 精简版（yudao-cloud-mini） | `master` | `master-jdk17` |

## 5. 部署模式

### 单体模式

```
yudao-server (一个 JVM)
├── YudaoServerApplication.java         ← 启动类
│   scanBasePackages = {"cn.iocoder.yudao.server", "cn.iocoder.yudao.module"}
│
├── application.yaml                    ← 通用配置
└── application-local.yaml              ← 本地环境（MySQL、Redis 直连）
    ├── spring.cloud.nacos.discovery.enabled = false
    └── spring.cloud.nacos.config.enabled = false
```

单体模式下，所有模块在同一个 JVM 中运行。Feign 调用通过 Spring Cloud 的本地解析机制直接路由到同 JVM 内的 Controller，无网络开销。

### 微服务模式

```
Gateway (48080) ──→ system-server (48081)
              ──→ infra-server (48082)
              ──→ pay-server (48083)
              ──→ ...

每个 server 是独立的 Spring Boot 应用：
├── XxxServerApplication.java           ← 独立启动类
├── application.yaml
└── application-dev.yaml                ← Nacos 注册 + 配置
    ├── spring.cloud.nacos.discovery.enabled = true
    └── spring.cloud.nacos.config.enabled = true
```

### 配置切换

| 配置项 | 单体模式 | 微服务模式 |
|--------|---------|-----------|
| `spring.cloud.nacos.discovery.enabled` | `false` | `true` |
| `spring.cloud.nacos.config.enabled` | `false` | `true` |
| Feign 调用 | 本地解析 | HTTP 远程调用 |
| 服务注册 | 不需要 | Nacos |

## 6. 请求生命周期

### 完整流程

```
浏览器
  │  GET /admin-api/system/dept/list?name=技术部
  │  Authorization: Bearer <token>
  ▼
Gateway (port 48080)
  │  ① 路由匹配: Path=/admin-api/system/** → system-server
  │  ② TokenAuthenticationFilter (GlobalFilter, order -100):
  │     a. 清除伪造的 login-user 头
  │     b. 提取 Bearer Token
  │     c. 查 Guava 缓存（1 分钟 TTL）或调用 system-server 校验
  │     d. 有效 → 序列化 LoginUser 到 login-user 头
  │  ③ 转发: grayLb://system-server（负载均衡）
  ▼
system-server (Spring Boot)
  │  ④ TokenAuthenticationFilter (OncePerRequestFilter):
  │     a. 读取 login-user 头，反序列化为 LoginUser
  │     b. 校验 userType 匹配（admin-api → admin 用户）
  │     c. 存入 Spring SecurityContext
  │  ⑤ Spring Security 过滤器链:
  │     a. URL 权限检查
  │     b. anyRequest().authenticated() → 要求登录
  ▼
DeptController
  │  ⑥ @PreAuthorize("@ss.hasPermission('system:dept:query')") → 权限校验
  │  ⑦ @Validated DeptListReqVO → 参数校验
  │  ⑧ deptService.getDeptList(reqVO) → 委托 Service
  ▼
DeptServiceImpl
  │  ⑨ deptMapper.selectList(reqVO) → 查询数据库
  ▼
DeptMapper (MyBatis-Plus)
  │  ⑩ LambdaQueryWrapperX
  │     .likeIfPresent(DeptDO::getName, reqVO.getName())
  │     .eqIfPresent(DeptDO::getStatus, reqVO.getStatus())
  │  ⑪ SQL: SELECT * FROM system_dept WHERE name LIKE ? AND status = ? AND deleted = 0
  │  ⑫ 返回 List<DeptDO>
  ▼
返回路径:
  DeptMapper → DeptService → DeptController
  ⑬ BeanUtils.toBean(list, DeptRespVO.class)    ← DO → VO
  ⑭ CommonResult.success(voList)                 ← 统一包装
  ⑮ GlobalResponseBodyHandler 记录响应（访问日志）
  ⑯ Jackson 序列化 → JSON
  ▼
Gateway → 转发响应 → 浏览器
  ⑰ {"code": 0, "msg": "", "data": [...]}
```

### 三种 URL 前缀

| 前缀 | 用途 | 来源 |
|------|------|------|
| `/admin-api/` | 管理端接口 | Controller 在 `*.controller.admin.*` 包下 |
| `/app-api/` | 用户端接口 | Controller 在 `*.controller.app.*` 包下 |
| `/rpc-api/` | 模块间 RPC | Feign 接口中定义 |

前缀由 `YudaoWebAutoConfiguration` 通过 `WebMvcRegistrations.setPathPrefixes()` 自动注入，Controller 代码中不写前缀。

## 8. 认证鉴权架构

### 令牌机制

```
登录:
  用户名+密码 → AdminAuthService.login()
    → 校验密码 → 生成 Access Token + Refresh Token
    → Token 信息存入 Redis（key: oauth2_access_token:{token}）
    → 返回 Token 给前端

请求校验:
  Authorization: Bearer <token>
    → Gateway TokenFilter 从 Redis 校验
    → 缓存 LoginUser（Guava, 1 分钟 TTL）
    → 设置 login-user 头转发给下游

权限校验:
  @PreAuthorize("@ss.hasPermission('system:dept:query')")
    → SecurityFrameworkServiceImpl.hasPermission()
    → 从 Redis 加载用户权限列表（Guava 本地缓存 1 分钟）
    → 检查是否包含指定权限码
```

### 权限模型

```
AdminUser ──(N:N)──→ Role ──(N:N)──→ Menu（按钮=权限标识）
                                       │
                                       ├── system:dept:create
                                       ├── system:dept:update
                                       ├── system:dept:delete
                                       └── system:dept:query
```

### 数据权限

```
Role.dataScope
  ├── ALL          → 不过滤
  ├── DEPT         → WHERE dept_id = ?
  ├── DEPT_AND_CHILD → WHERE dept_id IN (本部门及下级)
  └── CUSTOM       → WHERE dept_id IN (自定义部门列表)
```

通过 `@DataPermission` + `DeptDataPermissionRule` 在 SQL 层面自动注入 WHERE 条件。

## 9. 多租户架构

### 方案选择

| 方案 | 优点 | 缺点 | yudao-cloud 是否采用 |
|------|------|------|---------------------|
| 独立数据库 | 隔离性最强 | 成本高、运维复杂 | 否 |
| 共享数据库、独立 Schema | 中等隔离 | 跨租户查询困难 | 否 |
| **共享数据库、共享 Schema** | **成本最低、运维简单** | **需要行级过滤** | **是** |

### 实现机制

```
实体层:
  TenantBaseDO 加 tenantId 字段
    → 继承 TenantBaseDO 的实体自动参与租户过滤

SQL 层:
  TenantDatabaseInterceptor implements TenantLineHandler
    → 拦截所有 SQL
    → 检查表是否需要租户过滤（基于实体继承关系自动判断）
    → 需要 → 追加 WHERE tenant_id = ?
    → 不需要 → 跳过

上下文传播:
  HTTP → TenantContextWebFilter（从 Header 提取 tenantId）
  RPC  → Feign RequestInterceptor（传递 tenantId Header）
  MQ   → MQ Interceptor（传递 tenantId）
  Job  → @TenantJob AOP（设置 tenantId）
```

### 忽略租户过滤的条件

以下情况不追加 `WHERE tenant_id`：
1. 表对应的实体不继承 `TenantBaseDO`
2. 实体标注了 `@TenantIgnore`
3. 在配置文件中将表加入 `ignore-tables`
4. 代码中设置了 `TenantContextHolder.setIgnore(true)`

## 10. 跨模块 RPC

### 两级 API 体系

| 级别 | 位置 | 用途 | 示例 |
|------|------|------|------|
| 业务 API | `yudao-module-xxx-api` | 模块间业务调用 | `DeptApi`, `PayOrderApi` |
| 通用 API | `yudao-common` | 框架级跨模块调用 | `OAuth2TokenCommonApi`, `PermissionCommonApi` |

### Feign 接口定义

```java
@FeignClient(name = "system-server")  // 服务名，匹配 spring.application.name
public interface DeptApi {

    @GetMapping("/rpc-api/system/dept/list-all-simple")
    @Operation(summary = "获取精简部门列表")
    CommonResult<List<DeptRespDTO>> getDeptList(
            @RequestParam("name") String name,
            @RequestParam("status") Integer status);
}
```

### 调用链路

```
单体模式:
  mall-server → DeptApi.getDeptList()
    → Spring Cloud 本地解析 → 直接调用 system-server 的 DeptController
    → 零网络开销

微服务模式:
  mall-server → DeptApi.getDeptList()
    → OpenFeign → Nacos 服务发现 → HTTP 请求到 system-server
    → DeptController 处理并返回
```

## 11. 关键设计模式速览

| 模式 | 应用位置 | 一句话说明 |
|------|---------|-----------|
| 模板方法 | `BaseMapperX` | 封装分页/批量操作的通用逻辑 |
| 策略模式 | `DataPermissionRule` | 数据权限规则可插拔 |
| 装饰器模式 | `LambdaQueryWrapperX` | 扩展 MyBatis-Plus 查询构建器 |
| 观察者模式 | Spring Event / MQ | 模块间解耦的事件通知 |
| 工厂模式 | `ServiceExceptionUtil.exception()` | 统一创建业务异常 |
| 代理模式 | Feign 客户端 | 接口自动生成 HTTP 调用代码 |
| 责任链模式 | Spring Security Filter Chain | 认证/授权过滤器链 |
| 缓存旁路 | Gateway Token 缓存 + Service Redis 缓存 | 先查缓存，未命中再查数据源 |
| 消费者驱动契约 | `-api` 子模块 | 模块间通过接口契约通信 |
| SQL 改写 | 多租户 + 数据权限 | 拦截 SQL AST 注入 WHERE 条件 |

## 12. 配置体系

### Profile 结构

```
yudao-server/src/main/resources/
├── application.yaml              ← 通用配置（MyBatis-Plus、Flowable、Swagger 等）
├── application-local.yaml        ← 本地开发（直连 MySQL/Redis，Nacos 关闭）
└── application-dev.yaml          ← 开发环境（Nacos 注册 + 配置）
```

### 关键配置项

```yaml
# 单体模式开关
spring.cloud.nacos.discovery.enabled: false   # true = 微服务模式
spring.cloud.nacos.config.enabled: false

# 多租户
yudao.tenant.enable: true
yudao.tenant.ignore-tables: [ ... ]           # 不做租户过滤的表

# 数据源（dynamic-datasource）
spring.datasource.dynamic:
  primary: master
  datasource:
    master: { url, username, password }
    slave: { lazy: true, url, username, password }  # 读写分离

# MyBatis-Plus
mybatis-plus:
  id-type: NONE                  # 由 IdTypeEnvironmentPostProcessor 根据数据库类型自动切换
  logic-delete-field: deleted
  logic-delete-value: 1
  logic-not-delete-value: 0
```
