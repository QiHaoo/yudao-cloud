## Feign 测试策略

### 单元测试：Mock Feign 客户端

在单元测试中，不希望真正发起 HTTP 请求，需要 Mock Feign 客户端：

```java
@SpringBootTest
@ActiveProfiles("unit-test")
public class OrderServiceTest {

    @MockBean
    private ProductSpuApi productSpuApi;  // Mock Feign 客户端

    @Resource
    private OrderService orderService;

    @Test
    void testCreateOrder() {
        // 准备 Mock 数据
        ProductSpuRespDTO mockSpu = new ProductSpuRespDTO();
        mockSpu.setId(1L);
        mockSpu.setName("iPhone 15");
        mockSpu.setPrice(99900);

        // 设置 Mock 行为
        when(productSpuApi.getSpu(1L))
            .thenReturn(CommonResult.success(mockSpu));

        // 执行测试
        orderService.createOrder(1L, 2);

        // 验证调用
        verify(productSpuApi).getSpu(1L);
    }
}
```

### 集成测试：使用 WireMock

如果需要测试完整的 Feign 调用链路，可以使用 WireMock 模拟远程服务：

```java
@SpringBootTest
@ActiveProfiles("integration-test")
public class OrderServiceIntegrationTest {

    @RegisterExtension
    static WireMockExtension wireMock = WireMockExtension.newInstance()
            .options(wireMockConfig().port(8089))
            .build();

    @Test
    void testCreateOrderWithWireMock() {
        // 模拟 product-server 的响应
        wireMock.stubFor(get(urlEqualTo("/rpc-api/product/spu/get?id=1"))
            .willReturn(aResponse()
                .withHeader("Content-Type", "application/json")
                .withBody("{\"code\":0,\"msg\":\"success\",\"data\":{\"id\":1,\"name\":\"iPhone 15\",\"price\":99900}}")));

        // 执行测试
        orderService.createOrder(1L, 2);

        // 验证请求
        wireMock.verify(getRequestedFor(urlEqualTo("/rpc-api/product/spu/get?id=1")));
    }
}
```

### 测试 Feign 拦截器

如果需要测试自定义的 Feign 拦截器：

```java
public class TenantRequestInterceptorTest {

    @Test
    void testApply() {
        // 准备
        TenantRequestInterceptor interceptor = new TenantRequestInterceptor();
        RequestTemplate template = new RequestTemplate();

        // 设置租户上下文
        TenantContextHolder.setTenantId(1L);

        // 执行
        interceptor.apply(template);

        // 验证
        assertThat(template.headers().get("tenant-id"))
            .containsExactly("1");

        // 清理
        TenantContextHolder.clear();
    }

    @Test
    void testApplyWithoutTenant() {
        // 准备
        TenantRequestInterceptor interceptor = new TenantRequestInterceptor();
        RequestTemplate template = new RequestTemplate();

        // 不设置租户上下文

        // 执行
        interceptor.apply(template);

        // 验证：不应该添加 tenant-id 头
        assertThat(template.headers()).doesNotContainKey("tenant-id");
    }
}
```

## 常见问题排查速查表

| 问题现象 | 可能原因 | 排查步骤 |
|----------|----------|----------|
| Feign 调用 404 | URL 路径不匹配 | 检查 Controller 的 `@RequestMapping` 是否与 Feign 接口一致 |
| 找不到服务 | 服务未注册 | 检查 Nacos 控制台，确认服务已注册 |
| 序列化错误 | DTO 字段不匹配 | 检查 *-api 模块版本是否一致 |
| 租户 ID 丢失 | 拦截器未生效 | 检查拦截器配置，检查 TenantContextHolder |
| 调用超时 | 网络不通或服务负载高 | 检查网络连通性，调整超时配置 |
| 错误码不正确 | ErrorDecoder 未注册 | 检查 FeignConfiguration 配置 |
| Sentinel 降级未生效 | 未开启 Sentinel | 检查 `spring.cloud.openfeign.sentinel.enabled` |

## Feign 调试技巧

### 1. 开启 Feign 日志

```yaml
logging:
  level:
    cn.iocoder.yudao.module.product.api: DEBUG
```

### 2. 使用 curl 测试

Feign 接口本质是 HTTP 接口，可以直接用 curl 测试：

```bash
# 测试 product-server 的 RPC 接口
curl "http://localhost:48082/rpc-api/product/spu/get?id=1"

# 测试带租户 ID 的请求
curl -H "tenant-id: 1" "http://localhost:48082/rpc-api/product/spu/get?id=1"
```

### 3. 查看 Feign 代理

在调试时，可以查看 Feign 代理对象：

```java
@Resource
private ProductSpuApi productSpuApi;

@GetMapping("/test/feign")
public String testFeign() {
    // 查看代理对象
    log.info("Feign 代理类：{}", productSpuApi.getClass().getName());
    // 输出：cn.iocoder.yudao.module.product.api.spu.$Proxy123
    return productSpuApi.getClass().getName();
}
```

### 4. 断点调试

在以下位置打断点，可以观察 Feign 调用的完整流程：

- `TenantRequestInterceptor.apply()`：观察租户 ID 是否正确传递
- `FeignErrorDecoder.decode()`：观察错误响应的详细信息
- `ProductSpuController.getSpu()`：观察 Controller 是否正确实现
