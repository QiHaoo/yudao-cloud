# 04 - 精简汇总

## Feign 客户端定义速查

### 标准模板

```java
@FeignClient(name = ApiConstants.NAME)
@Tag(name = "RPC 服务 - {业务描述}")
public interface {业务}Api {

    String PREFIX = ApiConstants.PREFIX + "/{资源}";

    @GetMapping(PREFIX + "/get")
    @Operation(summary = "获得{资源}")
    CommonResult<{业务}RespVO> get(@RequestParam("id") Long id);

    @GetMapping(PREFIX + "/list")
    @Operation(summary = "获得{资源}列表")
    CommonResult<List<{业务}RespVO>> getList(@RequestParam("ids") Collection<Long> ids);
}
```

### URL 路径规则

| 类型 | 路径格式 | 示例 |
|------|----------|------|
| 管理后台 API | `/admin-api/{module}/{resource}/...` | `/admin-api/product/spu/get` |
| 用户端 API | `/app-api/{module}/{resource}/...` | `/app-api/product/spu/get` |
| RPC 调用 API | `/rpc-api/{module}/{resource}/...` | `/rpc-api/product/spu/get` |

### 返回值规范

| 场景 | 返回类型 | 说明 |
|------|----------|------|
| 查询单个 | `CommonResult<T>` | 包装为统一响应 |
| 查询列表 | `CommonResult<List<T>>` | 包装为统一响应 |
| 校验操作 | `CommonResult<Boolean>` | 返回校验结果 |
| 操作成功 | `CommonResult<Void>` | 无返回数据 |

## *-api 模块规范速查

### 目录结构

```
yudao-module-{module}-api/
├── src/main/java/
│   └── cn/iocoder/yudao/module/{module}/
│       ├── api/                    ← Feign 接口
│       │   ├── {resource}Api.java
│       │   └── dto/
│       │       ├── {Resource}RespVO.java

│       └── enums/                  ← 枚举常量
│           ├── ApiConstants.java
│           ├── ErrorCodeConstants.java
│           └── DictTypeConstants.java
└── pom.xml                         ← 依赖 api 模块
```

### pom.xml 依赖

```xml
<dependencies>
    <!-- 公共模块 -->
    <dependency>
        <groupId>cn.iocoder.cloud</groupId>

    </dependency>
</dependencies>
```

### ApiConstants 定义

```java
public class ApiConstants {
    public static final String NAME = "{module}-server";  // 服务名
    public static final String PREFIX = RpcConstants.RPC_API_PREFIX + "/{module}";  // URL 前缀
}
```

## 核心设计决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| RPC 框架 | OpenFeign | Spring Cloud 原生，HTTP 协议通用性好 |
| API 模块 | 独立 *-api 模块 | 接口契约共享，编译时类型检查 |
| URL 前缀 | `/rpc-api/{module}` | 避免与业务 API 路径冲突 |
| 返回值 | `CommonResult<T>` | 统一响应格式，便于错误处理 |
| 租户传递 | 请求拦截器 | 统一处理，无需每个调用点手动添加 |
| 熔断降级 | Sentinel | 与 Spring Cloud 生态集成好 |

## 关键配置

```yaml
# 开启 Sentinel 熔断降级
spring:
  cloud:
    openfeign:
      sentinel:
        enabled: true

# Feign 日志级别
logging:
  level:
    cn.iocoder.yudao.module.{module}.api: DEBUG
```

## Feign 调用流程

```
调用方代码
    │
    ▼
Feign 代理（JDK 动态代理）
    │
    ├── 1. 解析方法注解，构建 HTTP 请求
    ├── 2. 执行请求拦截器（租户、数据权限）
    ├── 3. 通过 LoadBalancer 选择实例
    ├── 4. 发起 HTTP 请求
    ├── 5. 处理响应（成功/失败）
    └── 6. 反序列化返回值
    │
    ▼
返回 CommonResult<T>
```

## 常见问题排查

| 问题 | 可能原因 | 排查方法 |
|------|----------|----------|
| 调用超时 | 网络不通、服务负载高 | 检查网络连通性、调整超时配置 |
| 找不到服务 | 服务未注册、服务名错误 | 检查 Nacos 控制台、检查 ApiConstants.NAME |
| 序列化错误 | DTO 字段不匹配 | 检查 *-api 模块版本是否一致 |
| 租户丢失 | 拦截器未生效 | 检查拦截器配置、检查 TenantContextHolder |
