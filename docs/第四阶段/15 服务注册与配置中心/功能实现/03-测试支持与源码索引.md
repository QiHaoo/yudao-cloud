## 本地开发 Nacos 配置

### 方案一：使用本地 Nacos Server（推荐）

最简单的方式是在本地启动一个 Nacos Server：

```bash
# Docker 方式（推荐）
docker run -d \
  --name nacos-standalone \
  -p 8848:8848 \
  -p 9848:9848 \
  -e MODE=standalone \
  -e NACOS_AUTH_ENABLE=false \
  nacos/nacos-server:v2.2.3
```

启动后访问 `http://127.0.0.1:8848/nacos`，默认账号密码 `nacos/nacos`。

**本地 Nacos 配置初始化脚本：**

为了快速初始化本地 Nacos 的配置，可以创建一个初始化脚本：

```bash
#!/bin/bash
# init-nacos-config.sh - 初始化本地 Nacos 配置

NACOS_ADDR="127.0.0.1:8848"

# 创建共享配置
curl -X POST "http://${NACOS_ADDR}/nacos/v1/cs/configs" \
  -d "dataId=application-common.yaml&group=SHARED_GROUP&content=$(cat application-common.yaml)&type=yaml"

# 创建 system-server 配置
curl -X POST "http://${NACOS_ADDR}/nacos/v1/cs/configs" \
  -d "dataId=system-server-local.yaml&group=DEFAULT_GROUP&content=$(cat system-server-local.yaml)&type=yaml"

echo "Nacos 配置初始化完成"
```

### 方案二：禁用 Nacos（本地调试）

如果只是调试业务逻辑，不需要 Nacos，可以禁用：

```yaml
# application-local.yaml
spring:
  cloud:
    nacos:
      discovery:
        enabled: false
      config:
        enabled: false
```

这样服务启动时不会连接 Nacos，所有配置使用本地文件。

### 方案三：使用 Testcontainers（自动化测试）

对于集成测试，可以使用 Testcontainers 自动启动 Nacos：

```java
@TestContainer
public class NacosIntegrationTest {

    @Container
    static GenericContainer<?> nacos = new GenericContainer<>("nacos/nacos-server:v2.2.3")
            .withExposedPorts(8848)
            .withEnv("MODE", "standalone");

    @DynamicPropertySource
    static void nacosProperties(DynamicPropertyRegistry registry) {
        registry.add("spring.cloud.nacos.server-addr",
            () -> nacos.getHost() + ":" + nacos.getMappedPort(8848));
    }

    @Test
    void testNacosConfig() {
        // 测试配置拉取
    }
}
```

## 单元测试策略

### 测试本地配置

单元测试时不依赖 Nacos，使用本地配置文件：

```yaml
# src/test/resources/application-unit-test.yaml
spring:
  cloud:

        enabled: false
      config:
        enabled: false
```

## 配置文件速查

### Nacos 连接配置模板

```yaml
spring:
  cloud:
    nacos:
      server-addr: ${NACOS_ADDR:127.0.0.1:8848}
      username: ${NACOS_USERNAME:nacos}
      password: ${NACOS_PASSWORD:nacos-admin}
      discovery:
        namespace: ${NACOS_NAMESPACE:dev}
        group: ${NACOS_GROUP:DEFAULT_GROUP}
        metadata:
          version: ${SERVICE_VERSION:1.0.0}
      config:
        namespace: ${NACOS_NAMESPACE:dev}
```

## 常见问题排查速查表

| 问题现象 | 可能原因 | 排查步骤 |
|----------|----------|----------|
| 服务注册失败 | Nacos 地址错误 | 检查 `server-addr`，telnet 测试连通性 |
| 配置未生效 | Data ID 不匹配 | 检查 `${spring.application.name}-${profile}.yaml` |
| 配置未热更新 | 未使用 @RefreshScope | 在 Bean 上添加注解 |
| 启动报错连接失败 | Nacos 不可用且未使用 optional | 使用 `optional:nacos:` 前缀 |
| Namespace 不匹配 | 控制台配置与代码不一致 | 检查 namespace ID 是否正确 |
| 共享配置未加载 | Group 不匹配 | 检查 shared-configs 中的 group |
