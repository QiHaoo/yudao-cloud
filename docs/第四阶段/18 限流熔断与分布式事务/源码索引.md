## 业务层使用示例

### Sentinel 限流配置

```yaml
spring:
  cloud:
    sentinel:
      transport:
        dashboard: localhost:8080
        port: 8719
      datasource:
        flow:
          nacos:
            server-addr: ${spring.cloud.nacos.server-addr}
            data-id: ${spring.application.name}-flow-rules
            group-id: SENTINEL_GROUP
            data-type: json
            rule-type: flow
        degrade:
          nacos:
            server-addr: ${spring.cloud.nacos.server-addr}
            data-id: ${spring.application.name}-degrade-rules
            group-id: SENTINEL_GROUP
            data-type: json
            rule-type: degrade
    openfeign:
      sentinel:
        enabled: true
```

### FallbackFactory 定义

```java
@Slf4j
@Component
public class ProductSpuApiFallbackFactory implements FallbackFactory<ProductSpuApi> {

    @Override
    public ProductSpuApi create(Throwable cause) {
        return new ProductSpuApi() {
            @Override
            public CommonResult<ProductSpuRespVO> getSpu(Long id) {
                log.error("[getSpu][商品服务调用失败，商品ID：{}]", id, cause);
                return CommonResult.error(SERVICE_UNAVAILABLE.getCode(),
                        "商品服务暂时不可用，请稍后重试");
            }
        };
    }
}
```

### @GlobalTransactional 使用

```java
@Service
public class TradeOrderServiceImpl implements TradeOrderService {

    @Resource
    private ProductSpuApi productSpuApi;

    @Resource
    private MemberPointApi memberPointApi;

    @Override
    @GlobalTransactional  // 开启全局事务
    public Long createOrder(TradeOrderCreateReqVO reqVO) {
        // 1. 创建订单（本地事务）
        TradeOrderDO order = new TradeOrderDO();
        order.setSpuId(reqVO.getSpuId());
        order.setUserId(reqVO.getUserId());
        tradeOrderMapper.insert(order);

        // 2. 扣减库存（远程调用，分支事务）
        productSpuApi.deductStock(reqVO.getSpuId(), 1);

        // 3. 扣减积分（远程调用，分支事务）
        memberPointApi.deductPoints(reqVO.getUserId(), 100);

        // 如果步骤 3 失败，步骤 1 和 2 会自动回滚
        return order.getId();
    }
}
```

### Seata 配置

```yaml
seata:
  enabled: true
  application-id: ${spring.application.name}
  tx-service-group: yudao-cloud-tx-group
  service:
    vgroup-mapping:
      yudao-cloud-tx-group: default
  registry:
    type: nacos
    nacos:
      server-addr: ${spring.cloud.nacos.server-addr}
      namespace: ${spring.cloud.nacos.discovery.namespace}
      group: DEFAULT_GROUP
      application: seata-server
  config:
    type: nacos
    nacos:
      server-addr: ${spring.cloud.nacos.server-addr}
      namespace: ${spring.cloud.nacos.config.namespace}
      group: DEFAULT_GROUP
      data-id: seataServer.properties
```

### undo_log 表结构

```sql
CREATE TABLE `undo_log` (
    `id` bigint(20) NOT NULL AUTO_INCREMENT,
    `branch_id` bigint(20) NOT NULL,
    `xid` varchar(100) NOT NULL,
    `context` varchar(128) NOT NULL,
    `rollback_info` longblob NOT NULL,
    `log_status` int(11) NOT NULL,
    `log_created` datetime NOT NULL,
    `log_modified` datetime NOT NULL,
    `ext` varchar(100) DEFAULT NULL,
    PRIMARY KEY (`id`),
    UNIQUE KEY `ux_undo_log` (`xid`, `branch_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8;
```

### Sentinel 规则参数速查

#### 流量控制规则（FlowRule）

| 参数 | 说明 | 示例值 |
|------|------|--------|
| `resource` | 资源名 | `getProduct` |
| `grade` | 限流模式（0=线程数，1=QPS） | `1` |
| `count` | 限流阈值 | `1000` |
| `strategy` | 流控策略（0=直接，1=关联，2=链路） | `0` |
| `controlBehavior` | 流控效果（0=快速失败，1=Warm Up，2=排队等待） | `0` |

#### 熔断降级规则（DegradeRule）

| 参数 | 说明 | 示例值 |
|------|------|--------|
| `resource` | 资源名 | `getProduct` |
| `grade` | 熔断策略（0=慢调用比例，1=异常比例，2=异常数） | `0` |
| `count` | 熔断阈值 | `0.5`（50%） |
| `slowRatioThreshold` | 慢调用阈值（毫秒） | `1000` |
| `timeWindow` | 熔断时长（秒） | `10` |
| `minRequestAmount` | 最小请求数 | `5` |
| `statIntervalMs` | 统计时长（毫秒） | `10000` |

| 模式 | 实现方式 | 侵入性 | 性能 | 适用场景 |
|------|----------|--------|------|----------|
| AT | 自动补偿（undo_log） | 低 | 中 | 大多数场景 |
| TCC | 手动编写 Try/Confirm/Cancel | 高 | 高 | 高性能场景 |

## 常见问题排查速查

| 问题现象 | 可能原因 | 排查步骤 |
|----------|----------|----------|
| 限流不生效 | 规则未配置、资源名不匹配 | 检查 Dashboard 规则、检查资源名 |
| 熔断不恢复 | 熔断时长过长、探测请求一直失败 | 检查熔断规则、检查服务健康状态 |
| FallbackFactory 不调用 | 未启用 Sentinel、未指定 fallbackFactory | 检查配置、检查注解 |
| Dashboard 看不到资源 | Sentinel 懒加载、网络问题 | 发送请求触发加载、检查网络 |
| 事务未回滚 | @GlobalTransactional 未生效、XID 未传播 | 检查注解位置、检查请求头 |
| undo_log 冲突 | 并发事务冲突 | 增加重试机制、优化事务粒度 |
| Seata Server 不可达 | 网络问题、配置错误 | 检查网络连通性、检查配置 |
| 全局事务超时 | 业务逻辑耗时过长 | 优化业务逻辑、增加超时时间 |
| XID 未传播 | OpenFeign 未集成 Seata | 检查 seata-spring-boot-starter 依赖 |

## Sentinel + Seata 协作流程

```
请求 → Gateway
         │
         ├── Sentinel 网关限流
         │   └── 超限 → 返回 429
         │
         ▼
    trade-server
         │
         ├── Sentinel 服务限流
         │   └── 超限 → 返回降级响应
         │
         ├── @GlobalTransactional 开启全局事务
         │
         ├── 调用 product-server
         │   │
         │   ├── Sentinel 熔断检查
         │   │   └── 熔断 → 调用 FallbackFactory
         │   │
         │   └── 执行分支事务（扣减库存）
         │
         ├── 调用 member-server
         │   │
         │   ├── Sentinel 熔断检查
         │   │   └── 熔断 → 调用 FallbackFactory
         │   │
         │   └── 执行分支事务（扣减积分）
         │
         └── 全部成功 → Seata 提交全局事务
             任一失败 → Seata 回滚全局事务
```
