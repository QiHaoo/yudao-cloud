# 05 - 精简汇总

## Sentinel 规则配置速查

### 流量控制规则

| 参数 | 说明 | 示例 |
|------|------|------|
| `resource` | 资源名 | `getProduct` |
| `grade` | 限流模式（0=线程数，1=QPS） | `1` |
| `count` | 限流阈值 | `1000` |
| `strategy` | 流控策略（0=直接，1=关联，2=链路） | `0` |
| `controlBehavior` | 流控效果（0=快速失败，1=Warm Up，2=排队等待） | `0` |

### 熔断降级规则

| 参数 | 说明 | 示例 |
|------|------|------|
| `resource` | 资源名 | `getProduct` |
| `grade` | 熔断策略（0=慢调用比例，1=异常比例，2=异常数） | `0` |
| `count` | 熔断阈值 | `0.5`（50%） |
| `slowRatioThreshold` | 慢调用阈值（毫秒） | `1000` |
| `timeWindow` | 熔断时长（秒） | `10` |
| `minRequestAmount` | 最小请求数 | `5` |
| `statIntervalMs` | 统计时长（毫秒） | `10000` |

### 熔断器状态

| 状态 | 说明 | 行为 |
|------|------|------|
| CLOSED | 关闭 | 正常放行 |
| OPEN | 打开 | 拒绝所有请求 |
| HALF-OPEN | 半开 | 放行部分探测请求 |

### 配置示例

```yaml
# application.yaml
spring:
  cloud:
    sentinel:
      transport:
        dashboard: localhost:8080
    openfeign:
      sentinel:
        enabled: true
```

## Seata 使用速查

### 依赖和配置

```xml
<!-- pom.xml -->
<dependency>
    <groupId>io.seata</groupId>
    <artifactId>seata-spring-boot-starter</artifactId>
</dependency>
```

```yaml
# application.yaml
seata:
  enabled: true
  tx-service-group: yudao-cloud-tx-group
  service:
    vgroup-mapping:
      yudao-cloud-tx-group: default
  registry:
    type: nacos
```

### 注解使用

```java
// 开启全局事务
@GlobalTransactional
public void createOrder() {
    // 业务逻辑
    // 远程调用会自动作为分支事务
}
```

### 三种模式对比

| 模式 | 实现方式 | 侵入性 | 性能 | 适用场景 |
|------|----------|--------|------|----------|
| AT | 自动补偿（undo_log） | 低 | 中 | 大多数场景 |
| TCC | 手动编写 Try/Confirm/Cancel | 高 | 高 | 高性能场景 |
| Saga | 补偿操作 | 中 | 中 | 长事务 |

### AT 模式 undo_log 表

```sql
CREATE TABLE undo_log (
    id bigint(20) NOT NULL AUTO_INCREMENT,
    branch_id bigint(20) NOT NULL,
    xid varchar(100) NOT NULL,
    context varchar(128) NOT NULL,
    rollback_info longblob NOT NULL,
    log_status int(11) NOT NULL,
    log_created datetime NOT NULL,
    log_modified datetime NOT NULL,
    ext varchar(100) DEFAULT NULL,
    PRIMARY KEY (id),
    UNIQUE KEY ux_undo_log (xid, branch_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8;
```

## 核心设计决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 限流熔断 | Sentinel | 功能丰富，社区活跃，与 Spring Cloud 集成好 |
| 分布式事务 | Seata AT | 低侵入，自动补偿，开发效率高 |
| 熔断策略 | 慢调用比例 + 异常比例 | 覆盖大多数场景 |
| 限流模式 | QPS 限流 | 直观易理解 |
| 事务传播 | OpenFeign 自动传播 XID | 无需手动处理 |
| Dashboard | Sentinel Dashboard | 实时监控，动态规则 |

## Sentinel + Seata 协作流程

```
请求 → Gateway
         │
         ├── Sentinel 网关限流
         │   └── 超限 → 返回 429
         │
         ▼
    trade-server
         │
         ├── Sentinel 服务限流
         │   └── 超限 → 返回降级响应
         │
         ├── @GlobalTransactional 开启全局事务
         │
         ├── 调用 product-server
         │   │
         │   ├── Sentinel 熔断检查
         │   │   └── 熔断 → 调用 FallbackFactory
         │   │
         │   └── 执行分支事务（扣减库存）
         │
         ├── 调用 member-server
         │   │
         │   ├── Sentinel 熔断检查
         │   │   └── 熔断 → 调用 FallbackFactory
         │   │
         │   └── 执行分支事务（扣减积分）
         │
         └── 全部成功 → Seata 提交全局事务
             任一失败 → Seata 回滚全局事务
```

## 常见问题排查

| 问题 | 可能原因 | 排查方法 |
|------|----------|----------|
| 限流不生效 | 规则未配置、资源名不匹配 | 检查 Dashboard 规则、检查资源名 |
| 熔断不恢复 | 熔断时长过长、探测请求一直失败 | 检查熔断规则、检查服务健康状态 |
| 事务未回滚 | @GlobalTransactional 未生效、XID 未传播 | 检查注解位置、检查请求头 |
| undo_log 冲突 | 并发事务冲突 | 增加重试机制、优化事务粒度 |
| Seata Server 不可达 | 网络问题、配置错误 | 检查网络连通性、检查配置 |
