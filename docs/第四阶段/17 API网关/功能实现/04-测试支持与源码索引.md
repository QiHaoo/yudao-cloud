# 04 - 测试支持与源码索引

> 本章介绍网关的测试策略，并提供完整的源码索引，方便查阅和对照。

## 一、网关测试策略

### 1.1 网关测试的特殊性

Gateway 基于 WebFlux（响应式），测试方式与传统的 WebMvc 有所不同：

| 维度 | WebMvc 测试 | WebFlux 测试 |
|------|------------|-------------|
| 测试框架 | MockMvc | WebTestClient |
| 请求模拟 | `MockHttpServletRequest` | `MockServerHttpRequest` |
| 响应验证 | `MvcResult` | `EntityExchangeResult` |
| 异步支持 | 需要 `@Async` | 原生支持 Reactor |

### 1.2 单元测试：过滤器

过滤器是网关的核心组件，需要重点测试。

**测试 TokenAuthenticationFilter：**

```java
// yudao-gateway/src/test/java/cn/iocoder/yudao/gateway/filter/security/TokenAuthenticationFilterTest.java
package cn.iocoder.yudao.gateway.filter.security;

import cn.iocoder.yudao.framework.common.pojo.CommonResult;
import cn.iocoder.yudao.gateway.util.SecurityFrameworkUtils;
import org.junit.jupiter.api.BeforeEach;
import org.junit.jupiter.api.Test;
import org.springframework.cloud.gateway.filter.GatewayFilterChain;
import org.springframework.http.HttpHeaders;
import org.springframework.mock.http.server.reactive.MockServerHttpRequest;
import org.springframework.mock.web.server.MockServerWebExchange;
import reactor.core.publisher.Mono;
import reactor.test.StepVerifier;

import static org.mockito.Mockito.*;

class TokenAuthenticationFilterTest {

    private TokenAuthenticationFilter filter;
    private GatewayFilterChain chain;

    @BeforeEach
    void setUp() {
        // 模拟依赖
        filter = new TokenAuthenticationFilter(lbFunction);
        chain = mock(GatewayFilterChain.class);
    }

    @Test
    void testFilter_NoToken_ShouldPassThrough() {
        // Given：无 Token 的请求
        MockServerHttpRequest request = MockServerHttpRequest
                .get("/admin-api/system/user/get")
                .build();
        MockServerWebExchange exchange = MockServerWebExchange.from(request);

        when(chain.filter(any())).thenReturn(Mono.empty());

        // When
        Mono<Void> result = filter.filter(exchange, chain);

        // Then：应该直接放行
        StepVerifier.create(result).verifyComplete();
        verify(chain).filter(exchange);
    }

    @Test
    void testFilter_InvalidToken_ShouldPassThrough() {
        // Given：无效 Token 的请求
        MockServerHttpRequest request = MockServerHttpRequest
                .get("/admin-api/system/user/get")
                .header(HttpHeaders.AUTHORIZATION, "Bearer invalid-token")
                .build();
        MockServerWebExchange exchange = MockServerWebExchange.from(request);

        when(chain.filter(any())).thenReturn(Mono.empty());

        // When
        Mono<Void> result = filter.filter(exchange, chain);

        // Then：应该放行（网关不拒绝，交给服务端判断）
        StepVerifier.create(result).verifyComplete();
    }

    @Test
    void testFilter_ForgedLoginUserHeader_ShouldBeRemoved() {
        // Given：伪造的 login-user 请求头
        MockServerHttpRequest request = MockServerHttpRequest
                .get("/admin-api/system/user/get")
                .header("login-user", "forged-user-info")
                .build();
        MockServerWebExchange exchange = MockServerWebExchange.from(request);

        when(chain.filter(any())).thenReturn(Mono.empty());

        // When
        filter.filter(exchange, chain).block();

        // Then：login-user 头应该被移除
        verify(chain).filter(argThat(ex ->
                !ex.getRequest().getHeaders().containsKey("login-user")));
    }

}
```

**测试 CorsFilter：**

```java
// yudao-gateway/src/test/java/cn/iocoder/yudao/gateway/filter/cors/CorsFilterTest.java
package cn.iocoder.yudao.gateway.filter.cors;

import org.junit.jupiter.api.Test;
import org.springframework.http.HttpHeaders;
import org.springframework.http.HttpMethod;
import org.springframework.mock.http.server.reactive.MockServerHttpRequest;
import org.springframework.mock.web.server.MockServerWebExchange;
import org.springframework.web.server.WebFilterChain;
import reactor.core.publisher.Mono;
import reactor.test.StepVerifier;

import static org.junit.jupiter.api.Assertions.assertEquals;
import static org.mockito.Mockito.*;

class CorsFilterTest {

    private CorsFilter filter = new CorsFilter();

    @Test
    void testFilter_CorsRequest_ShouldSetHeaders() {
        // Given：跨域请求
        MockServerHttpRequest request = MockServerHttpRequest
                .options("/admin-api/system/user/get")
                .header(HttpHeaders.ORIGIN, "http://localhost:8080")
                .header("Access-Control-Request-Method", "GET")
                .build();
        MockServerWebExchange exchange = MockServerWebExchange.from(request);

        WebFilterChain chain = mock(WebFilterChain.class);
        when(chain.filter(any())).thenReturn(Mono.empty());

        // When
        Mono<Void> result = filter.filter(exchange, chain);

        // Then：应该设置 CORS 头
        StepVerifier.create(result).verifyComplete();
        assertEquals("*", exchange.getResponse().getHeaders()
                .getFirst("Access-Control-Allow-Origin"));
    }

    @Test
    void testFilter_NonCorsRequest_ShouldPassThrough() {
        // Given：非跨域请求
        MockServerHttpRequest request = MockServerHttpRequest
                .get("/admin-api/system/user/get")
                .build();
        MockServerWebExchange exchange = MockServerWebExchange.from(request);

        WebFilterChain chain = mock(WebFilterChain.class);
        when(chain.filter(any())).thenReturn(Mono.empty());

        // When
        Mono<Void> result = filter.filter(exchange, chain);

        // Then：应该直接放行
        StepVerifier.create(result).verifyComplete();
        verify(chain).filter(exchange);
    }

}
```

### 1.3 集成测试：路由转发

```java
// yudao-gateway/src/test/java/cn/iocoder/yudao/gateway/GatewayIntegrationTest.java

                .uri("/admin-api/product/spu/get?id=1")

    }

}
```

### 1.5 测试检查清单

| 测试场景 | 测试方法 | 预期结果 |
|---------|---------|---------|
| 无 Token 请求 | 发送无 Authorization 头的请求 | 直接放行 |
| 有效 Token | 发送有效 Bearer Token | 注入 login-user 头，转发 |
| 无效 Token | 发送无效 Bearer Token | 放行（交给服务端判断） |
| 伪造 login-user | 请求头包含 login-user | 头被移除，由网关重新设置 |
| 跨域请求 | 发送 OPTIONS 预检请求 | 返回 CORS 头，200 |
| 路由匹配 | 访问 /admin-api/system/** | 转发到 system-server |
| 路由不匹配 | 访问不存在的路径 | 返回 404 |
| 限流 | 发送超过阈值的请求 | 返回 429 |

### 2.2 业务层使用示例

**前端请求 → Gateway → 下游服务的完整流程：**

```bash
# 1. 前端登录获取 Token
curl -X POST http://localhost:48080/admin-api/system/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}'

# 返回：{"code":0,"data":{"accessToken":"xxx","refreshToken":"yyy"}}

# 2. 前端使用 Token 访问业务接口
curl http://localhost:48080/admin-api/system/user/get?id=1 \
  -H "Authorization: Bearer xxx" \
  -H "tenant-id: 1"

# Gateway 处理流程：
#   CorsFilter → 设置 CORS 头
#   TokenAuthenticationFilter → 校验 Token，注入 login-user 头
#   GrayLoadBalancer → 选择 system-server 实例
#   转发到 system-server

# 3. system-server 收到请求
#   从 login-user 头解析用户信息
#   执行业务逻辑
#   返回响应
```

## 三、组件设计精要回顾

### 3.1 核心设计决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 网关框架 | Spring Cloud Gateway | Spring Cloud 原生，响应式架构 |
| 路由协议 | grayLb:// | 支持灰度路由 |
| Token 校验 | WebClient + Guava Cache | Gateway 不能用 OpenFeign |
| 跨域处理 | WebFilter | 在 GlobalFilter 之前执行 |
| 用户信息传递 | login-user 请求头 | 下游服务统一解析 |
| 异常处理 | ErrorWebExceptionHandler | WebFlux 专用 |

### 3.2 过滤器链全景

```
请求进入
  │
  ▼
CorsFilter (WebFilter)
  └── 处理跨域，设置 CORS 头
  │
  ▼
AccessLogFilter (GlobalFilter, order=MIN_VALUE)
  └── 记录请求信息，装饰 Response
  │
  ▼
TokenAuthenticationFilter (GlobalFilter, order=-100)
  ├── 移除伪造的 login-user 头
  ├── 提取 Token
  ├── 校验 Token（缓存 → 远程）
  └── 注入 login-user 头
  │
  ▼
GrayReactiveLoadBalancerClientFilter (GlobalFilter, order=10100)
  ├── 解析 grayLb:// 协议
  ├── GrayLoadBalancer 选择实例
  └── 替换为实际 URL
  │
  ▼
路由过滤器（RewritePath 等）
  └── 路径重写
  │
  ▼
转发到下游服务
  │
  ▼
AccessLogFilter 记录响应日志
```

### 3.3 与其他组件的关系

```
Gateway 依赖的组件：
  ├── yudao-module-system-api：提供 OAuth2TokenCommonApi 接口
  ├── Nacos：服务发现，获取实例列表
  ├── Sentinel：限流保护
  └── Knife4j：API 文档聚合

Gateway 为下游服务提供的能力：
  ├── Token 校验：下游服务不需要重复校验
  ├── 用户信息：通过 login-user 头传递
  ├── 租户信息：通过 tenant-id 头传递
  └── 跨域处理：下游服务不需要配置 CORS
```

## 四、常见问题排查

| 问题 | 可能原因 | 排查方法 |
|------|----------|----------|
| 路由 404 | 路径配置错误 | 检查 `predicates` 中的 Path 配置 |
| 跨域失败 | CORS 头未设置 | 检查 CorsFilter 是否生效 |
| Token 校验失败 | 缓存未过期 | 等待 1 分钟或重启 Gateway |
| 服务不可用 | 服务未注册 | 检查 Nacos 控制台 |
| 响应慢 | 网络延迟、服务负载高 | 检查 AccessLog 中的 duration |
| 限流不生效 | 规则未配置 | 检查 Sentinel Dashboard |
| 灰度不生效 | version 头未传递 | 检查请求头和实例 metadata |
