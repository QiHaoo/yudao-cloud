# 04 - 精简汇总

## 路由配置速查

### 路由规则格式

```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: {module}-{api-type}        # 路由编号
          uri: grayLb://{service-name}    # 目标服务
          predicates:                     # 匹配条件
            - Path=/{api-type}/{module}/**
          filters:                        # 过滤器
            - RewritePath=/{api-type}/{module}/v3/api-docs, /v3/api-docs
```

### 路由命名规范

| 路由 ID | 说明 | 示例 |
|---------|------|------|
| `{module}-admin-api` | 管理后台 API | `system-admin-api` |
| `{module}-app-api` | 用户端 API | `product-app-api` |

### URL 路径对照

| 前端路径 | 转发到 | 实际路径 |
|----------|--------|----------|
| `/admin-api/system/**` | system-server | `/admin-api/system/**` |
| `/admin-api/product/**` | product-server | `/admin-api/product/**` |
| `/app-api/trade/**` | trade-server | `/app-api/trade/**` |

## 过滤器链顺序

```
过滤器执行顺序（order 越小，优先级越高）：

1. AccessLogFilter           (order = HIGHEST_PRECEDENCE)
   └── 记录访问日志

2. CorsFilter                (WebFilter，最外层)
   └── 处理跨域

3. TokenAuthenticationFilter  (order = -100)
   └── Token 校验 + 用户信息注入

4. GrayLoadBalancer           (LoadBalancer)
   └── 灰度负载均衡选择实例

5. 路由过滤器                 (GatewayFilter)
   └── RewritePath 等
```

### 请求处理流程

```
请求 → AccessLogFilter → CorsFilter → TokenAuthenticationFilter
                                          │
                                          ├── 白名单？→ 直接放行
                                          ├── 无 Token？→ 直接放行
                                          ├── Token 有效？→ 注入用户信息，放行
                                          └── Token 无效？→ 放行（交给服务端判断）
                                          │
                                          ▼
                                    GrayLoadBalancer
                                          │
                                          ├── 获取服务实例列表
                                          ├── 按 version/tag 过滤
                                          └── 随机+权重选择
                                          │
                                          ▼
                                    转发到目标服务
```

## 核心设计决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 网关框架 | Spring Cloud Gateway | Spring Cloud 原生，响应式架构 |
| 路由方式 | grayLb:// 协议 | 支持灰度路由 |
| Token 校验 | 网关统一校验 | 集中管理，减少服务间重复 |
| Token 缓存 | Guava LoadingCache | 1 分钟缓存，减少远程调用 |
| 跨域处理 | WebFilter | 统一处理，无需每个服务配置 |
| 限流方案 | Sentinel | 与 Spring Cloud 生态集成 |

## 常见问题排查

| 问题 | 可能原因 | 排查方法 |
|------|----------|----------|
| 路由 404 | 路径配置错误 | 检查 predicates 中的 Path 配置 |
| 跨域失败 | CORS 头未设置 | 检查 CorsFilter 是否生效 |
| Token 校验失败 | 缓存未过期 | 等待 1 分钟或清除缓存 |
| 服务不可用 | 服务未注册 | 检查 Nacos 控制台 |
| 响应慢 | 网络延迟、服务负载高 | 检查 AccessLog 中的 duration |

## 网关核心组件

| 组件 | 职责 | 优先级 |
|------|------|--------|
| AccessLogFilter | 记录请求日志 | 最高 |
| CorsFilter | 处理跨域 | 高 |
| TokenAuthenticationFilter | Token 校验 | 高 (-100) |
| GrayLoadBalancer | 灰度负载均衡 | 中 |
| GlobalExceptionHandler | 异常处理 | 高 (-1) |
